# Maxim Collector Skill

## このSkillを使うタイミング

以下のいずれかに該当する場合に読み込む:

- 「格言収集」「格言を集めて」「maxim collect」等の指示があった場合
- 「続けて」「前回の続き」と言われ、文脈が格言収集である場合
- `/loop` でデイリー自動実行する場合

---

## 禁止事項

- ユーザーのツール許可以外の確認ダイアログを出さない
- **ログにPR情報（URL・番号等）を記載しない**
- **`## Pull Request` / `## PR情報` ブロックをログに追記しない**（PR URL・番号・ラベル等を含む）
- **コミット/PR作成後にログファイルへの追記は一切行わない**
  - 理由: PRマージ後のブランチへの追記はmainに反映されず、不要な差分を生む
- **コンテキスト制限を理由に途中で中断しない**。本当に制限に近い場合は、現時点までの作業をコミットした上でユーザーに状況を伝える。勝手な早期中断は禁止

---

## 収集ワークフロー（全8ステップ）

### Step 1: カテゴリの決定

**1回の収集は1カテゴリのみ**。以下の優先順で決定する:

1. ユーザーが指定した場合 → そのカテゴリ
2. 指定がない場合 → `log/` の直近ログから前回カテゴリを確認し、下記ローテーション順で次を選ぶ

**ローテーション順**:
`western-philosophy` → `eastern-thought` → `japanese` → `chinese-idioms` → `world-proverbs` → `notable-figures` → `latin` → `literature` → `western-philosophy` …

### Step 2: 該当カテゴリの収集済み確認

**`references/{category}.md` のみ** を読み込み、収集済み格言一覧を取得する。
（REFERENCES.md 全体は読まない → コンテキスト節約）

### Step 3: Web検索＆出典検証（サブエージェント委譲）

サブエージェントに以下を委譲する:

- 該当カテゴリの検索キーワードでWeb検索
- 検索結果から未収集の格言候補を抽出
- 各候補の出典（著者・作品・章節）を検証
- **誤帰属に最大限注意**。著名な誤帰属（例: アリストテレスとされる "We are what we repeatedly do…" は実際は Will Durant の言い換え）を排除する

サブエージェントへの指示に含めること:
- 収集済み格言一覧（重複回避用）
- 検索キーワード
- 抽出すべき情報（日本語訳/原語原文/著者/出典作品・章節/解説/タグ候補/検証URL）

**検索キーワード例**:

| カテゴリ | キーワード例 |
|---------|------------|
| western-philosophy | 「西洋哲学 名言 出典 原文」 |
| eastern-thought | 「東洋思想 仏教 道教 名言 出典」 |
| japanese | 「日本 格言 名言 出典 ことわざ」 |
| chinese-idioms | 「中国 故事成語 由来 出典」 |
| world-proverbs | 「世界のことわざ 各国 意味」 |
| notable-figures | 「著名人 名言 出典 verified quote」 |
| latin | 「ラテン語 格言 名言 出典」 |
| literature | 「文学 名言 名セリフ 出典 作品」 |

### Step 4: 格言の選別

サブエージェントの結果から以下の基準で選別する:

- 収集済み格言と重複しないこと
- **出典（著者・作品・章節）が確認できること**。確認できないものは採用しない／作者不詳扱い
- 誤帰属でないこと

**1回の収集件数: 10件**

### Step 5: 格言ファイルの作成

`_template.md` のフォーマットに従い、`maxims/{category}/` に格言ファイルを作成する。

ファイル命名規則: `{category}-{3桁連番}-{english-slug}.md`

- 連番は同一カテゴリの既存最大番号 + 1
- english-slug は内容を表す英語ケバブケース

### Step 6: index の更新

`index/{category}.md` の該当カテゴリテーブルに新規格言を追加する。
`INDEX.md` の収録状況テーブル（合計数・カテゴリ別数）も更新する。

### Step 7: references の更新

`references/{category}.md` の出典テーブルに新規格言を追加する。参照元URLと追加日を記載。
`REFERENCES.md` の合計数も更新する。

### Step 8: デイリーログの作成

`log/{YYYY}/{MM}/{YYYY-MM-DD}.md` を作成（既存なら追記）する。

```markdown
# {YYYY-MM-DD} 収集ログ

## Round {N}（{category}）— 通算 {X}〜{Y} 件目

- 収集数: {N}件
- カテゴリ: {対象カテゴリ}

| id | 格言 | 著者 | 出典 | 参照元URL |
|----|------|------|------|----------|
| ... | ... | ... | ... | ... |

## メモ

（特記事項・次回カテゴリ）
```

### 失敗時の挙動

- 10件揃わない場合や検証に失敗した場合は、その旨をログに記録し、確認できた分だけ保存する（中途半端な状態で放置しない）
- ユーザーから「次のラウンド」を依頼された場合は、Step 1 から次のカテゴリで再実行する

---

## 出力フォーマット

実行完了時、以下を表示する:
- 今回収集したカテゴリ
- 追加した格言タイトル
- 累計収録数（カテゴリ別・全体）
- 次回ローテーション予定カテゴリ

---

## 品質基準

- 日本語訳は **原文に忠実** に
- 解説は格言の **意味・文脈** を説明する
- 出典は可能な限り **原典（作品名・章節）まで遡る**
- 誤帰属を排除し、典拠が確認できないものは「作者不詳」として扱うか採用しない

---

## デイリー自動実行

ユーザーが希望した場合、`/loop 24h` でデイリー実行を設定する（セッション内のみ有効）。
