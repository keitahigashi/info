# 使いうる最も危険な言葉は「これまでずっとこのやり方でやってきた」だ

> The most dangerous phrase a DP manager can use is "We've always done it that way."

## 著者

グレース・ホッパー（1906–1992）。米国の計算機科学者、米海軍少将。ハーバード大学のMark Iの初期プログラマの一人で、コンパイラの先駆的概念を打ち立て、プログラミング言語COBOLの礎を築いた。「アメイジング・グレース」の愛称で知られる。

## 解説

前例踏襲・現状維持の思考停止こそが、技術と組織にとって最も危険だという変化を促す警句。原文は当時のデータ処理（DP）管理者に向けて語られたものだが、ソフトウェア開発から組織変革まで幅広い文脈で引用される。なお、広く流布する洗練形「The most dangerous phrase in the language is: 'We've always done it this way.'」は後年の言い換えであり、上記が記録に残る最古の実文言である。
