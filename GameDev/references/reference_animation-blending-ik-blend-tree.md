## ブレンドIK：アニメーションブレンドで逆運動学を代替する

### 問題：数学的IKの限界

- 数学ベースのIK（逆運動学: Inverse Kinematics）は正確だが「重さ・タイミング感」が欠ける
- チェーンが180度回転するなど予測不可能な結果が生じやすい
- アーティストがコントロールを保ちにくい

### 解決策：アニメーションブレンドによるIK

- 事前にアーティストが制作したアニメーションを複数用意し、実行時にブレンド比率で補間
- アーティストが重量感・タイミングを完全にコントロール可能
- 予測可能で一貫したキャラクター動作を実現

### ブレンドシステムのアーキテクチャ

#### アニメーションプレイヤー
- クォータニオン (quaternion) による回転処理
- キーフレーム補間対応

#### ブレンドマネージャー（Blend Manager）
- 階層的ブレンディングを管理するルートノード
- 複数軸ブレンディング（多次元補間）に対応
- プライオリティブレンドツリーで上位レイヤーが下位を上書き

#### ブレンダーノード（Blender Nodes）
- 単一軸ブレンダー：1パラメータで複数アニメーションを補間
- 多軸ブレンダー：2D平面上でアニメーションをブレンド（Blend Space）
- トランジションブレンダー：状態遷移時のスムーズな切り替え

### パフォーマンス上の課題と対策

| 課題 | 対策 |
|------|------|
| メモリ（元：数MB/メック） | 圧縮最適化で700KBに削減 |
| CPU（クォータニオン補間） | SIMDベクトル演算で最適化 |
| 未使用ブランチのコスト | 非活性ブランチのカリング実装 |

### アーティスト支援ツール

- **Mirror Tool**：左右対称アニメーションを自動生成
- **Reverse Tool**：時間方向を反転したアニメーションを生成
- **Hill Tool**：傾斜地に対応したアニメーションを自動生成

### 成果

- 数学的IKより自然で重量感のあるキャラクター動作を実現
- アーティストのイテレーション速度が大幅に向上
- ゲームプレイ中の予測不可能な動作バグが激減

## 汎用化ポイント
