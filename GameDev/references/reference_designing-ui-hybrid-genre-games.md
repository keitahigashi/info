## ハイブリッドジャンルゲームのUI設計

シティビルディング・RTS・RPGを組み合わせた Folk Tale の開発を通じて見えてきた、複数ジャンル混在ゲームのUI設計原則。「情報過負荷のリスク」と「プレイスタイルの多様性」への対処が中心課題。

### 設計プロセスの基本方針

- ベクトル図形を用いた初期プロトタイプで、画面占有率と相互作用フローを素早くテスト
- 「下部配置設計」は空間浪費が多く廃棄 → 反復と検証が前提
- シンプルから始め、反復し、最後に仕上げる（simple → iterate → polish）

### データ段階化（Data Tiering）戦略

プレイヤーの意思決定コンテキストに応じて表示情報を段階的に切り替える。

| コンテキスト | 視点 | 表示情報 |
|------------|------|---------|
| マクロ管理 | 村全体の俯瞰 | 資源状況・建設状態など全体指標 |
| ミクロ操作 | 個別キャラクター選択 | キャラクター固有ステータス・アクション |

不要な情報を排除し、視覚的合図（色変化・アイコン状態変化）を優先。情報はプレイヤーが必要とした瞬間にのみ表示する設計。

### レイヤーベースのアイコン設計

| レイヤー | 役割 |
|--------|------|
| 背景（Background） | ベースカラー・形状 |
| 背景エフェクト | グラデーション・影 |
| 中景（Midground） | メインシンボル |
| 前景（Foreground） | アクセントや状態インジケーター |
| ハイライト | 選択・ホバー状態の強調 |

最終表示解像度（47×47px）より高解像度で制作し、スケール耐性を確保。色変更や組み合わせの変更に対応できる構造にする。

### アクセシビリティの基本実装

複数ジャンルをまたぐ複雑なゲームほど多様なプレイヤーニーズが生じる。

- ポーズボタン（一時停止）の確保
- カットシーン中の字幕（Subtitles）
- 操作リマッピング（Key Remapping）

### 商業的現実との折り合い

予算・スケジュール制約の中でUI品質を維持するために：
- 早期段階でのUXプロトタイピングによる手戻りの最小化
- 視覚的複雑さを段階化する設計を採用し、後付けのUI追加を減らす

## 汎用化ポイント

Folk Tale 固有の仕様（シティビルディング×RTS×RPG）は特定エンジンに依存しないが、「データ段階化」の手法はあらゆる情報量の多いジャンル（ストラテジー・シミュレーション・MMO等）に応用可能。プレイヤーの視点（マクロ/ミクロ）に連動した情報フィルタリングは汎用的な設計パターン。
