## コアメカニズム

ロールバックネットコードはリモートプレイヤーデータを待たずにローカル入力で即座にシミュレーションを進行。「入力への即座の応答を一貫性のコストで提供」するクライアント予測方式。

## 予測と修正プロセス

リモート入力が遅延して到着した場合:
1. 最後に確認されたフレームからゲーム状態を**復元**
2. 実際のリモート入力を使用して**再シミュレーション**
3. 後続の予測フレームを**再計算**
4. 修正結果を**レンダリング**

未知のリモート入力には「最後に既知の入力を繰り越す」— プレイヤーは60Hzでもそのレートで入力を変更することはまれなため有効。

## Input Decay 最適化

一部のゲームは古い予測入力の強度を段階的に低減。Rocket League の例: 修正時に後方にラバーバンドするのではなく、動きが「アンダーシュートしてキャッチアップ」。

## 遅延ベースとの比較

ロックステップアーキテクチャ（全プレイヤーの入力を待って進行）と比較して知覚レイテンシーを劇的に削減。

## 重大な欠点

- **不一貫性**: 予測ミスで視覚的不一致。「100-150msを超える予測は急速にプレイ不能に」
- **パフォーマンス要求**: 1レンダーウィンドウ内に多数のフレームを再シミュレーション（60Hz+300msで16.66ms内に15フレーム → 1フレーム約1.1ms）

## 実装要件

- **決定論的シミュレーション**と**固定ティックレート**（ロックステップから継承）
- **ゲーム状態のシリアライゼーション**: オンデマンドで完全な状態を保存/復元
- オーディオ・ビジュアルエフェクトは容易に保存/復元できないため特別な処理が必要

## 適したジャンル

格闘ゲーム・スポーツタイトルなど、入力応答性が最重要でシミュレーション複雑性が管理可能なゲーム。
