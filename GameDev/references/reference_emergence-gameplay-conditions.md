## 出典

Machinations.io: https://machinations.io/glossary/emergence-gameplay

## 定義

ゲームのルール・要素・AIシステムが予期しない方法で組み合わさることで、ユニークで脚本化されていない体験が生まれる現象。比較的単純なメカニクスやシステムから複雑で予測不可能なパターンが発生する。

## 創発の4発生条件

| 条件 | 説明 | 例 |
|------|------|-----|
| **複数の相互作用システム** | ゲーム要素が互いに影響し合う環境 | 水/溶岩/素材の相互作用 |
| **プレイヤーの自由度** | デザイナーの意図から逸脱した選択肢 | 意図されていない戦略の発見 |
| **単純なルールセット** | 複雑さは相互作用から自然に生まれる | Minecraftのブロック操作 |
| **予測困難性** | 各プレイスルーで異なる結果 | 動的要素・ランダム性 |

## システム相互作用メカニズム

異なるゲーム要素が組み合わさる際、デザイナーが直接プログラムしていない**相乗効果**が発生。Minecraftでは基本的なブロック操作ルールから複雑な自動化機械や建築が誕生する。

## 創発がもたらす4つの効果

| 効果 | 説明 |
|------|------|
| **リプレイ価値の向上** | 各プレイスルーで異なるシナリオが展開 |
| **プレイヤー主導のナラティブ** | 個別の物語や戦略の創造 |
| **創造性と適応力の促進** | 進化する状況への対応能力向上 |
| **エンゲージメント向上** | 発見と驚きの感覚 |

## 設計上の制約バランス

無制限の相互作用はゲームバランスを損なう可能性がある。**意図的な制約と自由のバランス**が重要:
- 過度な創発 → ゲームバランス崩壊
- 不足する創発 → 予測可能で退屈
- 最適な創発 → 驚きがありつつコントロール可能
