## バークの定義

プレイヤーが直接反応できない、特定のトリガーで発動するNPCまたはプレイヤーキャラクターの台詞。警備兵の警告、敵の挑発、NPC同士の会話など。

### 主な役割
- **システム情報伝達**: プレイヤーの状態（HP低下、弾薬切れ等）を通知
- **世界の自律性演出**: NPCが独自に生活している感覚
- **刺激-応答フレームワーク**: プレイヤーの行動（刺激）にNPCが反応（応答）

## 執筆の3層活用

### 1. ロア（世界観）への統合
- バークに世界固有の神話用語や派閥特有の表現を組み込む
- 言語体系そのものがゲーム世界を特徴づける
- 短い台詞の中に世界設定の断片を埋め込む

### 2. キャラクター性の表現

同じ挨拶シチュエーションでも性格で異なる:

| 性格タイプ | バーク例 |
|-----------|---------|
| 内向的 | 「え？」（曖昧・控えめ） |
| 敵対的 | 「ここから出ていけ」 |
| 親密的 | 「来てくれるのを待っていた！」 |

グリーティングバークだけでキャラクターの気質・対プレイヤー関係が露呈する。

### 3. 物語的潜在性 (Narrative Potentiality)
- **語られない情報**をバークに含ませ、プレイヤーの想像力を刺激
- 例: 「彼女は絶対に許さない」と亡霊が叫ぶ → 過去の人間関係を暗示
- プレイヤーが空白を補完する「氷山の一角」手法

## バリエーション設計

### 刺激-応答の多重化
- 同一トリガーに対して2〜10のバリエーションを用意
- 例: プレイヤーHP低下 → 敵の反応バリエーション複数
- 反復感の排除と自然さの維持

### 段階的反応
- 『Arcanum』の死者の霊: 複数バリエーション（「これが死か」→「痛い！解放してくれ」）
- プレイヤーが段階的に状況を理解する設計

## 具体事例

### 『A Plague Tale: Innocence』
- 重傷バーク: 敵が「少し錆びついてるだけ」→ プレイヤーの優位性を婉曲的に伝達
- システム情報（敵の弱体化）をキャラクター性で包む

## 制約条件

| 制約 | 対応 |
|------|------|
| 文字数制限 | 情報密度を高く、簡潔に |
| 音声録音 | テキストで表現のニュアンスを補完 |
| 反復頻度 | テンプレート化を避け、バリエーション確保 |
| トリガー精度 | 不適切なタイミングの発動は没入感破壊 |

## ベストプラクティス

- バーク執筆は**ミニマリスト的シナリオ技法** — 限定的な言語空間で最大の情報と心理的効果
- 映画・演劇の対話技法をゲーム執筆に応用
- 頻出バークは長期記憶に残るため、品質管理を厳格に
- トリガー設計の精密化が没入感の鍵
