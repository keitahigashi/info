## Holy Trinityの歴史的発展

| 時代 | 変化 |
|------|------|
| D&D起源 | クラスに明示的ロールなし、プレイヤーが有機的に役割を採用 |
| EverQuestの固定化 | 「各クラスが1つの役割のみ」— Warrior=Tank, Cleric=Healer |
| WoWの進化 | タレントツリーで1クラス複数専門化、ただしエンカウンター中は1ロール |

## Trinityの4つの利点

| 利点 | 説明 |
|------|------|
| **エンカウンター設計** | 予測可能なグループ構成で対象を絞った機械的チャレンジが可能 |
| **ロール明確性** | 親しみのある用語でプレイヤーが迅速に責任を理解 |
| **リプレイ性** | 異なるロールで同コンテンツに新しい体験を提供 |
| **パフォーマンス測定** | DPS=数値測定可能、Tank/Healer=成功/失敗のバイナリ判定 |

## 3つの重大な問題

### 1. 人気の不均衡
- DPSがクエスト報酬で優位（攻撃完了型の報酬設計）
- Healing/Tankingはグループ内でのみ真価を発揮
- 結果: DPS過多、Tank/Healer不足の慢性的な募集問題

### 2. パフォーマンス評価の不透明性
- DPS: ダメージメーターで定量評価可能
- Tank/Healer: 「うまくいった/いかなかった」のバイナリ評価のみ
- バランス調整とスキル評価が困難

### 3. ソロコンテンツでのペナルティ
- 特化型ロールはグループ外での進行が遅い
- ヒーラーのソロプレイ: 「ヒーラーはソロで何をするのか？」
- タンクのPvP: 専用の敵がいない環境での役割不明確

## 代替モデル

### Guild Wars 2
- 従来のTrinityを排除
- 柔軟なクラスビルドとロールフレキシビリティ
- すべてのクラスが自己回復能力を保有
- **副作用**: ヒーラーロールが冗長化

### ハイブリッドアプローチ
- 動的なトレイト切替でセッション中にロール変更
- 恒久的なキャラクター投資なしで適応可能

## 核心的ジレンマ

「Holy Trinityは各プレイヤーにロールを定義する点で優れているが、そのロールに終始固定されることがプレイヤーを制限する」
