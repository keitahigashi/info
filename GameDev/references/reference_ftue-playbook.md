## FTUEの定義

プレイヤーが初回セッションでゲームと接する最初のインタラクション。通常10〜60分の範囲で、オンボーディングシーケンス、チュートリアル、導入ゲームプレイ、基礎メカニクスのオリエンテーションを含む。

「FTUEはおそらく最も重要なバリデーション」。混乱する指示、過度なロード時間、不明確な目標に遭遇したプレイヤーは、後半コンテンツの質に関係なくゲームを放棄する。

## 効果的なFTUE設計の要素

### 障壁の除去 (Remove Stoppers)
プレイヤーが「何が起きているかわからない」と感じる摩擦ポイントを特定。これらはエンゲージメント終了の潜在的離脱点。

### プレイヤー期待との一致
オンボーディング体験はマーケティング素材やトレーラーと整合しなければならない。高速アクションを約束しながら長いテキストチュートリアルで始まると離脱を引き起こす。

### ジャンル別コンテキスト
- **モバイルマッチ3**: 数秒でインタラクティブなゲームプレイに到達すべき
- **複雑なRPG** (Baldur's Gate等): より包括的なシステム説明が必要
- 原則: ゲームプレイ開始前に**最低限の必須事項のみ**教える

### 「何を」ではなく「なぜ」を教える
効果的なオンボーディングはプレイヤーのアクションの背後にある目的を説明する。ハイライトされたUI要素を受動的にタップするだけのチュートリアルは真の理解を確立しない。

## 5つの計測指標

| 指標 | 計測内容 |
|------|---------|
| **完了率とドロップオフポイント** | チュートリアル/初期ミッション内でプレイヤーが行き詰まる箇所 |
| **明瞭性と理解度** | オンボーディング後にプレイヤーがゲーム目標・コアメカニクス・操作を説明できるか |
| **満足度 vs 期待値** | 提供体験とマーケティングに基づくプレイヤーの予想の整合性 |
| **エンゲージメントシグナル** | 能動的探索と受動的クリックの区別。プレイ継続意欲の監視 |
| **楽しさまでの時間** | 起動から楽しいゲームプレイまでの所要時間（DL・サインアップ・パッチ含む） |

## 5つの一般的FTUE問題

1. **情報過負荷 (Information Overload)**: 複数システムの同時説明、過度なテキスト
2. **ゲームプレイ前の摩擦**: アカウント作成、DL、パッチ、マッチメイキング待機
3. **チュートリアルの断絶**: ゲームプレイと隔離された環境でのオンボーディング
4. **メカニカルな不理解**: ハイライトシーケンスを完了しても基礎システムを未理解
5. **マーケティングと体験の不一致**: プロモーションと実際のゲーム開始体験の矛盾

## テストセッション設計

| 要素 | 推奨 |
|------|------|
| **セッション時間** | 45-60分（オンボーディング + ポストチュートリアル + 継続判断） |
| **サンプルサイズ** | 10名で十分。カジュアル/ハードコア区分時は16-20名（各8-10名） |
| **モデレーション** | 非モデレートセッションが自然なプレイヤー行動を明らかにする |
| **リクルーティング** | ターゲットデモグラフィックを代表するプレイヤーを選定 |

## 実績事例: Ankama北米ローンチ

10名の地域プレイヤーでFTUEプレイテストを実施 → 北米プレイヤーの期待に特化したオンボーディング最適化 → **新規プレイヤーリテンション25%向上** + **20万DL超**達成。
