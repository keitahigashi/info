## 出典

Emily Short's Interactive Storytelling: https://emshort.blog/2016/04/12/beyond-branching-quality-based-and-salience-based-narrative-structures/

## 概要

従来の分岐ナラティブ以外のインタラクティブストーリー構造を体系的に分析。分岐が唯一の非線形ストーリーテリング手法ではないことを示す。

## 主要なナラティブ構造

### 1. Quality-Based Narrative (QBN)

「クオリティ（数値変数）によってアンロックされるストーリーレット（短いナラティブ断片）」で構成される物語。

| 項目 | 内容 |
|------|------|
| **強み** | モジュラーコンテンツが自然に結合。非線形情報発見に最適（3つの手がかりを任意順序で発見 → 2^3分岐ではなく個別ストーリーレット） |
| **弱み** | 品質追跡の手動管理が大量に必要。選択肢過多のリスク |
| **代表例** | Fallen London (Failbetter Games), StoryNexus |
| **適したケース** | エピソード的・モジュラーな物語、複数著者による分担執筆 |

### 2. Salience-Based Narrative

現在のワールド状態に最も「適合する (fits)」コンテンツをシステムが選択。プレイヤーの選択や作者の指定ではなく、関連度スコアリングで決定。

| 項目 | 内容 |
|------|------|
| **強み** | 段階的なコンテンツ拡張が可能。環境文脈に応じたダイアログシステムに特に効果的 |
| **弱み** | ドラマティックアークを考慮しないと物語的に不整合。数値ウェイトのチューニングに大量テスト必要 |
| **代表例** | Left 4 Dead のダイナミックダイアログ、Firewatch のロケーションベース会話 |
| **適したケース** | 探索可能な環境上のダイアログレイヤー |

### 3. Waypoint Narrative

会話トピックがウェイポイントとして機能し、トピック間の遷移が物語を進行させる。NPCはゴールトピックへパスファインドし、プレイヤーは代替トピックを導入して会話をリダイレクト可能。

| 項目 | 内容 |
|------|------|
| **強み** | 真のNPCエージェンシー。コンテンツ追加が組み合わせ爆発なしにパスを増やす |
| **弱み** | 実装が複雑。ダイアログ重視のナラティブ向き |
| **代表例** | Emily Short の Glass |
| **適したケース** | ダイアログ駆動のナラティブ |

## その他の構造

- **マップベース (Map-based)**: 80 Days
- **データベースリサーチ型**: Her Story, Analogue: A Hate Story
- **ホログラフィックナラティブ**: テキストをその場で展開して深掘り
- **カードデッキ型**: プレイヤーが制御するシーケンシング

## 設計上の示唆

「Twine は素晴らしいが、これらの構造の一部には設計されていない」— 唯一のプロトタイピングツールとして教えることへの警告。異なるシステムは異なるナラティブ目標に適合し、すべてのインタラクティブストーリーを従来の分岐構造に押し込むべきではない。
