## MBTIとプレイヤー理論の統合フレームワーク

### 概要

既存の8つのプレイヤー理論（バートル・ラザーロ・MDAなど）を「どれを使えばよいか」の問いに答えるため、MBTI（マイヤーズ・ブリッグス型指標）の4つの心理軸をフィルターとして使う手法を提案。プリプロダクション段階でペルソナを設定し、そのMBTI結果から適用すべき理論を絞り込む。

### MBTI 4軸とゲーム設計への対応

| MBTI 軸 | 問い | 設計への示唆 |
|---------|------|------------|
| 外向 (E) ／ 内向 (I) | 行動への動機は外部刺激か内部思考か | 協力・競争 vs. 探索・内省 |
| 感覚 (S) ／ 直観 (N) | 情報収集は具体的事実か抽象パターンか | チュートリアル詳細度・メカニクス直感性 |
| 感情 (F) ／ 思考 (T) | 意思決定は感情優先か論理優先か | ナラティブ重視 vs. システム最適化 |
| 判断 (J) ／ 知覚 (P) | 行動スタイルは計画的か柔軟対応か | 目標明示 vs. オープンエンド探索 |

### 対応する8つのプレイヤー理論

| 理論 | 対応する MBTI 軸 | ポイント |
|------|---------------|---------|
| カイヨワ (Caillois) のプレイ4分類 | 意思決定・行動モード | アゴン・アレア・ミミクリ・イリンクス |
| バートル (Bartle) のプレイヤータイプ | 情報収集・意思決定 | キラー・アチーバー・ソーシャライザー・エクスプローラー |
| ラザーロ (Lazzaro) の4つの楽しみ | 全4軸 | Hard Fun・Easy Fun・People Fun・Serious Fun |
| GNS+モデル | 意思決定・行動モード | ゲーミズム・ナラティビズム・シミュレーショニズム |
| MDA+フレームワーク | 全4軸 | Mechanics・Dynamics・Aesthetics・Kinetics |
| ルブラン (LeBlanc) の楽しみ分類 | 情報収集・動機 | 8種の楽しみカテゴリー |
| 周 (Chou) のオクタリシス (Octalysis) | 全4軸 | 8つの動機コア |
| Sid Meierの「意味ある選択」 | 意思決定 | 選択肢の重さとトレードオフ設計 |

### ペルソナ駆動型設計プロセス

1. **ペルソナの作成**: 年齢・職業・プレイ習慣・好みを持つ理想的プレイヤー像を設定
2. **MBTI評価**: ペルソナのMBTI4軸を決定
3. **理論マッピング**: MBTI結果から関連性の高い2〜3理論を選択
4. **要件リスト化**: 各理論からゲームに必要な要素を抽出
5. **メカニクス設計**: 要件に沿ったシステム・コンテンツを設計

### 実践上の注意

- ペルソナは「理想的だが信憑性のある架空プレイヤー」であり、既存プレイヤーデータの代替ではない
- 全8理論を同等に適用する必要はなく、ペルソナに対応した2〜3の重点理論に絞る
- プレイヤーはゲーム内で成長・変化するため、「初回プレイ時」と「熟練後」で異なるメカニクスのアンロックが必要
- フレームワークはプリプロダクション段階の出発点であり、実際の設計判断はデザイナーの裁量による
