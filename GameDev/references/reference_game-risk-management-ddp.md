## 概要

ゲーム開発プロジェクトにおける失敗の根本原因は「リスクを正確に測定・管理できていないこと」。本記事は Discovery-Driven Planning (DDP) というフレームワークを用いて、「仮定 (assumption) を知識 (knowledge) に変える」プロセスを7段階で解説する。

## ゲーム開発における主要リスク分類

| リスク分類 | 具体例 |
|-----------|--------|
| **技術的リスク** | プラットフォーム選択の失誤、技術的債務蓄積、バグ対応遅延 |
| **クリエイティブリスク** | 人材採用失敗、ビジョンの欠落、後期段階での大幅変更 |
| **マーケティングリスク** | 市場分析不十分、顧客ニーズの誤認識 |
| **組織リスク** | 士気低下と離職率上昇、チームワーク機能不全 |

## Discovery-Driven Planning（DDP）の3つの特徴

### 1. 逆向きの収益性計画 (Reverse Income Statement)
```
従来型：コスト推定 → 収益予測 → 利益算出
DDP：必要利益を先に設定 → その達成可能性を検証
```
先に「どれだけ利益が必要か」を明確にし、それを達成できるかを検証する。

### 2. 仮定と知識の分離
- **知識**：確実な事実（例：プラットフォーム手数料30%）
- **仮定**：不確実な要素（例：ユーザー獲得コスト、開発期間、リテンション率）
- 仮定は「点推定」でなく「上限・下限の範囲」で表現

### 3. 仮定から知識への変換
プロジェクト進行に伴い、仮定の範囲を段階的に狭め、知識に転換していく。**「仮定-知識比率」の継続的な低減が目標**。

## DDPの7ステップ実装プロセス

### ステップ1：必要な収益性を明記
```
例：最低利益 $1.3M、営業利益率 20%
　　→ 必要売上 $6.5M、許容コスト $5.2M 以下
```

### ステップ2：収益を推定
- 月間新規ユーザー数、リテンション率、コンバージョン率を設定
- 課金ユーザーをセグメント分類：

| セグメント | 割合 | ARPPU |
|-----------|------|-------|
| ホエール（大口） | 10% | $29 |
| ドルフィン（中口） | 35% | $5.50 |
| ミノー（小口） | 55% | $0.90 |

### ステップ3：コストを推定
- ユーザー獲得コスト (CPI)
- 開発コスト（職種別給与 × 予定工数）
- 運用オーバーヘッド、市場調査費用

### ステップ4：逆向き損益計算書の作成
```
コスト余裕度 = 許容コスト − 推定コスト
利益余裕度 = 推定利益 − 必要利益
```
いずれかがゼロ以下の場合、ステップ5に進まず再計画する。

### ステップ5：重要変数と仮定を抽出
- すべての仮定を一覧化（スプレッドシート等で可視化）
- 各仮定に上限・下限値を設定
- 確率分布タイプを指定（正規分布、三角分布など）

### ステップ6：感度分析 (Sensitivity Analysis) を実施
**使用ツール**：Oracle Crystal Ball など

**成果物**：トルネード図 (Tornado Chart)
```
最大影響度（例）：
1. リテンション率（最大の不確実性）
2. コンバージョン率
3. 新規ユーザー数/月
```
影響度の高い仮定から優先的に早期検証する。

### ステップ7：チェックポイント (Checkpoint) 一覧を作成
各チェックポイントに以下を記載：
- 検証対象の仮定
- 検証コスト
- 合否判断基準

**20×30ルール**：
- チェックポイント数は20以内
- 仮定数は30以内（超過時は実施率が低下する傾向）

## チェックポイントでの意思決定

| 判定 | アクション |
|------|----------|
| 仮定が必要利益ハードルを満たす | 次の段階へ進む |
| 満たさない（軽微） | 計画再調整（スコープ縮小・コスト削減・収益モデル変更） |
| 満たさない（致命的） | プロジェクト中止（損失最小化） |

## 導入時のベストプラクティス

- **仮定の管理者を設置**：専任者が各チェックポイントで仮定を更新・記録。チーム共有（Wiki/SharePoint）
- **学習の文書化**：各チェックポイントで得た知見と仮定範囲の変化を記録
- **最重要仮定の多重検証**：感度が高い仮定は複数チェックポイントで繰り返し検証

## 核心的メッセージ

> 「変化は避けられない。しかし、管理可能なリスクの理解・分析・管理能力は向上できる」

DDPにより、**最短・最低コストで最大学習**を実現し、プロジェクト失敗の予防、意思決定の根拠づけ、チーム全体の学習文化構築を同時に達成できる。
