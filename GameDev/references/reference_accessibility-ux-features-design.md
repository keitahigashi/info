## アクセシビリティ機能のUX設計フレームワーク

### アクセシビリティの広義な定義

障害者だけでなく、**状況的な障害**（明るい屋外でのプレイ、音の出せない環境、利き手のけが等）を抱えるすべてのプレイヤーに対してアクセシビリティは適用される。

- 世界人口の約15%が何らかの障害を経験（WHO）
- 優れたアクセシビリティ機能はすべてのプレイヤーの体験を改善する

### アクセシビリティ機能の6カテゴリ

| カテゴリ | 対象 | 機能例 |
|---------|------|--------|
| **運動機能 (Motor)** | 肢体不自由、運動障害 | ボタンリマップ、連打→長押し変換、オートエイム |
| **認知機能 (Cognitive)** | 注意・記憶・処理速度 | ゲーム速度調整、チュートリアル再表示、マーカー強化 |
| **視覚 (Vision)** | 視力低下、色覚異常 | カラーブラインドモード、テキストサイズ、高コントラスト |
| **聴覚 (Hearing)** | 聴覚障害、難聴 | 字幕・クローズドキャプション、視覚的サウンドインジケータ |
| **音声 (Speech)** | 言語障害（音声入力ゲーム向け） | テキスト入力での代替、言語認識精度調整 |
| **統合型 (Integrated)** | 複合的・文脈的なアクセシビリティ | カスタマイズ可能なHUD、難易度の細粒度設定 |

### 3段階の優先度レベル

```
基本 (Basic)
  └─ 最低限のアクセシビリティ要件（テキストサイズ、カラーブラインドモード）

習熟 (Proficient)
  └─ 広範なプレイヤーをカバーする推奨機能（リマップ、字幕カスタマイズ）

高度 (Advanced)
  └─ 包括的なアクセシビリティ（細粒度の難易度、感覚的な代替出力）
```

### 業界標準：The Last of Us Part II の実装

60以上のアクセシビリティ機能を実装した先行事例：

- **HUDのカスタマイズ**: 個別要素のオン/オフ、サイズ調整
- **高コントラスト表示**: 敵・味方・インタラクタブルを色別に強調
- **認識インジケータ**: 音の方向・種類を視覚的に表示
- **テキスト読み上げ**: UI要素のすべてを音声で案内
- **ゲーム速度調整**: コンバットタイマーを最大25%に減速

### 設計プロセスへの統合

#### 初期段階での検討がコストを下げる

```
プリプロダクション段階での検討
  → アーキテクチャへの統合が容易
  → 後付け実装より大幅にコスト削減

リリース後の後付け
  → システム再設計が必要
  → 開発コストが数倍に増加
```

#### 共感ツールの活用

- シミュレーションツールで視覚障害・色覚異常を体験
- アクセシビリティユーザーによる早期プレイテスト
- 障害者コミュニティとの継続的なフィードバックループ

### 法的背景

- 2019年1月以降のビデオゲームは**CVAA（通信・映像アクセシビリティ法）**への準拠が必要（米国）
- IGNは2020年よりアクセシビリティ革新賞を創設
- Xbox Adaptive Controller等のハードウェアアクセシビリティも業界課題
