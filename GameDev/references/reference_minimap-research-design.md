## ミニマップが必要な場合

- 大規模なプレイフィールド
- 複数のルート選択肢（非線形ゲーム）
- 多数の目標地点
- 味方・敵ユニット位置の把握
- マクロ管理が必要なRTS

## ミニマップの2種類

### 1. ワールド型 (World Map)
- スポーツ、レーシング、MOBA、ストラテジーで使用
- ミニチュア化された世界地図として機能
- マップ全体を俯瞰

### 2. プレイヤー中心型 (Player-Centered)
- アクション、シューター、RPGで採用
- プレイヤーの周辺視野のみを限定的に表示
- レーダー形式が多い

## ジャンル別の配置位置

| ジャンル | 配置位置 | 備考 |
|---------|---------|------|
| MOBA | 下部 | 統一的 |
| ストラテジー | 左下/右上 | RTSは右上が多い |
| MMORPG | 右側（上部） | 排他的 |
| レーシング | 下部 | |
| FPS | 左側 | 反応時間短縮 |

**心理学的根拠**: 西洋文化圏では画面左側への注視時間が長く、高速ゲームでの反応時間短縮に効果的。

## 表示要素

- 地形・バリア
- プレイヤー・敵・味方の位置
- POI（チェックポイント、支配地点、建物等）にアイコン付き
- 範囲外POIへの方向矢印
- 戦争の霧 (Fog of War)
- ローテーション機能
- オートマップ機能

## インタラクション機能

| 機能 | 用途 |
|------|------|
| クリック移動 | RTS/MOBAでのナビゲーション |
| ピング/アラート | チーム通信（標的、撤退指示） |
| ズーム | 詳細確認 |
| マップ回転/固定切替 | 方向感の調整 |
| ツールチップ | 追加情報表示 |

## 設計上の注意

- 誤配置は反応時間の延長、誤クリック、知覚ミスにつながる
- 理想的にはプレイヤーが**位置・サイズをカスタマイズ可能**
- 色盲対応の配色検討
- 要素間の高コントラスト設定
