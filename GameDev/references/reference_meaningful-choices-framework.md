## 意味ある選択 (Meaningful Choices) の定義

ゲームの核心的定義は「意味ある選択の集合体 (a collection of meaningful choices)」である。移動、リソース配分、キャラクターカスタマイズなど、プレイヤーがゲーム体験にどう関わるかを決定する選択を指す。

## 4つの必須要素

| 要素 | 説明 |
|------|------|
| **認識 (Awareness)** | プレイヤーが選択肢の存在を認識できること |
| **帰結 (Consequences)** | ゲームがプレイヤーの決定をゲームプレイやナラティブの変化として正確に反映すること |
| **想起 (Reminders)** | 選択後も、その決定がプレイ中に想起されること |
| **永続性 (Permanence)** | 結果を見た後に決定を覆せないこと。責任感を生む |

## 設計原則

### エージェンシー (Agency) と権限付与
- プレイヤーは意味ある選択を通じてエージェンシー（自分の道を形作る能力）を行使する
- 美的選択（ゲームピースの色など）もエージェンシーを提供し、より重大なナラティブ/ゲームプレイ決定へと発展しうる

### 接続的共感 (Connected Empathy)
- RPGのキャラクター作成は「接続的共感」を育む
- ゼロからキャラクターを構築する感情的投資が、その後の決定に個人的な重みを持たせる

### バランスと制限 (Balance & Restriction)
- 戦略的に選択肢を制限し、プレイヤーを圧倒しない
- 制限が創造的問題解決と記憶に残る意思決定の瞬間を促進
- 競技的実行可能性を維持する

## 選択構築フレームワーク (Choice Construction)

効果的な選択が提示すべき要素:

1. **明確な結果予測** — 起こりうるアウトカムの可視性
2. **定義された報酬** — 何が得られるかの明示
3. **潜在的リスク** — 何を失う可能性があるかの明示

この透明性が情報に基づく意思決定を可能にする。帰結はビジュアル面とインタラクティブなゲームプレイ要素の両方に影響すべきである。

## 戦略的考慮事項

### ドミナント戦略 (Dominant Strategy)
- プレイヤーが同じ成功アプローチを繰り返し使用すると、選択のインパクトが減衰する
- 複数の勝利パスをサポートするようオプションをバランスさせるべき

### エクスプロイト (Exploits)
- プレイヤーが発見する意図しない優位性がドミナント戦略化する可能性
- 早期プレイテストと反復的改善で新興戦術をバランスする

## 設計上の要点

意味ある選択は、デザイナーが導く制約を維持しつつ、プレイヤーに体験への真の重みを与えることで、「良いゲーム」と「優れたゲーム」を分ける。
