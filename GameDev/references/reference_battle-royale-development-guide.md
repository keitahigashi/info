## コアメカニクスの基盤

「サバイバル、マップデザイン、バランスの取れた戦闘システム」が機械的骨格。

### 必須要素

| 要素 | 説明 |
|------|------|
| リソース収集 | 最小装備で開始、武器/弾薬/防具をスカベンジ |
| 動的セーフゾーン | 縮小するプレイアブルエリアがプレイヤーを接近戦闘に強制 |
| 戦略的ポジショニング | 地形バリエーションで多様な戦闘戦術を支援 |

## マップ設計原則

| 原則 | 説明 |
|------|------|
| 多様な地形 | オープンフィールド/森林/都市/高台の混合 |
| 戦略的アイテム配置 | 高活動ゾーン生成、過度の競争集中を回避 |
| 環境要素 | 植生/構造物/動的要素が戦術カバーと探索を促進 |

## セーフゾーン設計

- 「ストーム」/「毒サークル」/「ゾーン制限」とも呼ばれる
- 目的: 過度に保守的な戦略（隠れ続ける/戦闘回避）を防止
- **ゲームペースと観戦価値**への影響
- 通常15-20分で勝敗決定
- 各試合でゾーン中心がランダム選択 → マップ学習のインセンティブ

## ルート分布

- 全プレイヤーが無装備で開始（通常50人以上）
- 基本装備がマップ全体にランダムスポーン
- 高品質アイテムは特定エリアに集中
- シーズンごとにルートテーブルが変動
- 武器/装備/弾薬を戦略的に配分 → バランスの取れた競争と探索の促進

## プレイヤーエンゲージメント＆リテンション

- 定期的マップ更新＋新武器/ゲームモード
- バトルパス＋デイリーチャレンジの段階的報酬システム
- コミュニティイベント＋期間限定コラボレーション

## エンジン比較

| エンジン | 強み |
|---------|------|
| Unity | クロスプラットフォームモバイルスケーリング、ラピッドプロトタイピング |
| Unreal Engine | 大規模マップ＋複雑ビジュアル、50人以上の高プレイヤーカウント処理 |

## マネタイゼーション

- コスメティック限定購入（Pay-to-Win回避）
- ローテーション式ゲーム内ストア
- プレイヤー満足度との均衡を重視
