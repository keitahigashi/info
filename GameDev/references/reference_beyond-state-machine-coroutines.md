## コルーチンベースの振る舞い記述：FSMを超える設計パターン

### FSMの限界とその先へ

- 有限状態機械（FSM: Finite State Machine）はシンプルだが、状態数が増えると「スパゲッティ状態遷移」に陥りやすい
- コルーチン (coroutine)・クロージャ (closure)・継続 (continuation) を活用することで、振る舞いを手続き的に記述しながら実行を一時停止・再開できる

### コルーチンの基礎実装（C#）

- `IEnumerable<Instruction>`型のイテレータを使用
- `yield return`で任意フレームに実行を停止・再開
- `ExecuteWhile`・`Call`などの制御フロー命令を組み合わせて合成

### グラフベースのランタイム構造

- コルーチンノードを中心とした階層的ノードグラフを実行時に動的に構築
- 命令の種類に応じてサブノードを動的生成・廃棄
- ノードプーリングシステムによるメモリ効率化が可能

### 並行処理と同期の仕組み

- `Concurrent`ノード：複数の子コルーチンを並列実行・管理
- `While`ノード：述語条件が成立する間だけ子を実行し続ける
- 任意の述語関数による柔軟な実行継続/中断ルール

### リソース管理と安全な割り込み

- `IDisposable`インターフェースを活用
- `try/finally`ブロックで確実なクリーンアップを保証
- 例外を使わない安全な割り込みメカニズム

### 実装例：タレットAI

- ターゲット検索と追跡を並列コルーチンで記述
- ターゲット消失時にクロージャを使って状態をリセット
- FSMで書いた場合より大幅に可読性・保守性が向上

### 設計上の利点

- 振る舞いをフラットな状態遷移図ではなく「手続きの流れ」として記述できる
- 複雑な条件付き並行処理もネストしたコルーチンで直感的に表現可能
