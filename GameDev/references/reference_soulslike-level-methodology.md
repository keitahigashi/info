## コア概念

### 垂直構造
「目に見える全空間が実際に到達可能な立体構造」— プレイヤーが視認できる場所はすべて行ける場所。

### ループパス
「直線的なルートを意図的に円形化し、ショートカット発見時の驚嘆を生成」。

## 3つの設計レイヤー

### 1. 形状レイヤー
| 要素 | 役割 |
|------|------|
| 点 | 展望台、プラットフォーム |
| 線 | 通路、橋、分断壁 |
| 面 | 室内空間、広場 |
| 体積 | 多層構造体 |

**収束点**: 複数ルートから同一地点に導かれる設計 → チェックポイント効率化とループ体験強化。

### 2. 空間的言語

| 空間タイプ | 効果 |
|-----------|------|
| 狭小空間 | コントロール感と制限の両義性 |
| 展望空間 | 自由感と脆弱性 |
| 親密空間 | 安全と機能性（Firelink Shrineモデル） |
| 抑圧空間 | 上方からの脅威感（Smouldering Lakeの弩砲） |
| 高所空間 | 戦略的優位性と全体把握 |

### 3. 環境類型

## 5つの環境タイプ

### 町型
- 構造: 複数建造物を循環ルートで接続
- 特性: 高密度の分岐、豊かな探索体験
- モデル: Undead Burg（教会を核とする高密度設計）
- 派生: 城型（Stormveil）、都市型（Leyndell）

### 地下迷宮型
- 特性: 方向感喪失、最高密度
- 対策: レベル指向性（上昇vs下降の心理的差異）
- 例: Blighttown → Ash Lake

### 平野型
- 構造: 広大な地形に小規模遺跡を配置
- 形状言語: 三角(危険/選択肢) / 四角(遮蔽/驚き) / 円形(安全性)

### 聖堂型
- 垂直分層（複数階層）+ 外周ループ（複数入口）
- モデル: Cathedral of the Deep

## 密度勾配の原則

「レベル進行は複雑から合理化へ、最終的にボス前で線形に収束」:
- 初期: 複雑な選択肢（探索フェーズ）
- 中盤: 統合（ルート収束）
- ボス直前: 線形化（認知負荷管理）

## チェックポイント設計

Cleansing Chapel例: 単一セーブポイントから複数ショートカット分岐
- 教会正面: 第1階
- 左側扉: 第2階への直接アクセス
- 右側扉: 第2階上部
→ ループ発見の感情的高揚を複数回生成

## 非線形進行の構造

### マクロ: Firelink Shrine中核の複数ループ
- Undead Burg(教会) → Firelink
- Undead Burg(Havel) → Darkroot Garden → 鍛冶屋
- Lower Undead Burg → Blighttown → Firelink

### ミクロ: 大ループ内の小ループ構造
- 主要ルートが大圏、垂直探索が小圏

## 実装7段階ワークフロー

1. ムードボード（参考資料蒐集）
2. フローチャート（ルート構造）
3. ビートシート（感情的弧）
4. メトリクス（スケール基準）
5. ホワイトボックス（エンジン内構築）
6. 敵配置・ナラティブ検証
7. 美術スタイル確定
