## 反復プロトタイピング (Rapid Iterative Prototyping) の原則

> プロトタイピングは「ゲーム設計プロセスにおいて本質的かつ効率的な部分」であり、開発初期段階で問題を特定することで時間とコストを大幅に削減する。

### 主なメリット

- 機能的に成立しないゲームコンセプトを早期に排除できる
- 失敗を低コストで実験できる（後工程ほど修正コストが高くなる）
- 開発チーム全員がゲーム仕様を深く理解し、ステークホルダーへの説明が容易になる

## 4段階のプロトタイピングプロセス

### 段階1: テーブルトップ・プロトタイピング (Tabletop Prototyping)

- 紙・カード・ボードゲーム形式で初期コンセプトを検証
- **投資ゼロ**で「コアループが面白いか」を確認できる
- デジタル実装前に致命的な設計欠陥を発見できる

### 段階2: デジタル化 (Digitization)

- Pygame・GameMaker などの軽量ツールを活用
- 借用アセット（temp art）を使い、**1〜2週間でプレイアブル**な状態を目指す
- ビジュアルより「ゲームプレイの感触」の検証に集中する

### 段階3: テストと改善 (Test & Iteration)

- 社外の外部テスター（moddb.com 等のコミュニティ）からフィードバックを収集する
- 内部テストバイアスを排除し、ゲーム性の実現可能性を客観的に判断する
- フィードバックを分析し、次のイテレーションに反映するサイクルを繰り返す

### 段階4: ビジュアルスライス作成 (Visual Slice)

- パブリッシャーへのピッチ・投資家向けデモ用に、完成度の高い単一レベルを構築する
- 「ゲーム全体の品質を一断面で体現」するスライスで、ゲームの可能性を証明する

## イテレーション・プロセスの本質

```
コンセプト → プロトタイプ → プレイテスト → 発見 → 修正 → プロトタイプ → ...
```

各イテレーションで「何が機能し、何が機能しないか」を発見→調整を繰り返す。このサイクルを早く・安く回すことが、最終的な品質向上の最短経路となる。

## 発見コストの法則

| 発見タイミング | 修正コスト（相対値） |
|--------------|------------------|
| コンセプト段階（紙プロト） | ×1 |
| デジタルプロトタイプ段階 | ×10 |
| プロダクション中盤 | ×100 |
| リリース後 | ×1000 |

問題の発見が遅れるほど、修正コストが指数関数的に増大する。反復プロトタイピングは「発見を早める投資」として位置づけられる。
