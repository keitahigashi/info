## レイテンシーの構成

ネットワークインフラがトータルレイテンシーの約**31%**を占める（サーバーハードウェアやゲームコードより大きい）。

## 入力遅延 (Input Delay / Lockstep)

クライアントがボタン入力をサーバーに送信し、シミュレーション結果を待ってから画面表示。

| 項目 | 内容 |
|------|------|
| **利点** | 全プレイヤーの視覚的一貫性。グラフィカルな不一致なし |
| **欠点** | 入力から応答までの遅延 = ラウンドトリップping。100msのpingで0.1秒の遅延 |
| **適したケース** | RTS、一部のスポーツタイトル、旧世代格闘ゲーム |

## ロールバック (Rollback / Client-Side Prediction)

クライアントがサーバー確認を待たずに即座にアクションをシミュレーション。サーバーの権威的応答到着時に「ロールバック」して調整。

| 項目 | 内容 |
|------|------|
| **利点** | 即座の応答性。反射が勝敗を分けるゲームに不可欠 |
| **欠点** | 予測とサーバー結果の乖離で視覚的アーティファクト。再シミュレーションのCPUオーバーヘッド |
| **適したケース** | 競技シューター、現代格闘ゲーム |

## ハイブリッドアプローチ（推奨）

3つの設定パラメータで両技法をバランス:

1. **最小入力遅延** (Minimum Input Delay): レイテンシーに関係ない基準遅延（通常0）
2. **最大入力遅延** (Maximum Input Delay): 予測が有効化される前にカバーする遅延（推奨50ms）
3. **最大予測時間** (Maximum Predicted Time): クライアントの先行シミュレーション上限（通常100ms）

### Ping別パフォーマンス

| Ping | 体験 |
|------|------|
| **50ms未満** | 純粋な入力遅延でクリーンな体験 |
| **50-150ms** | ロールバックが追加レイテンシーを処理し応答的 |
| **150ms超** | 拡張予測より入力遅延を増加（ゲームプレイ感覚を維持） |

## インフラの重要性

サーバー配置がネットコード選択単独より影響が大きい。615+ロケーションでの最新エッジオーケストレーションで平均レイテンシー**58%削減**。地理的マッチメイキングが不均等なレイテンシー条件を防止。
