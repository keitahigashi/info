## 問題提起

現代のゲームストーリーテリングは映画的手法（カットシーン・固定シナリオ）に依存しすぎており、ゲーム本来の「対話性」を十分に活かせていない。著者は「エマージェント・ナラティブ (emergent narrative)」という代替アプローチを提唱する。

## エマージェント・ナラティブとは

> 「個々のコンポーネントには見られない特性が、相互作用によって全体から現れる現象」

鳥の群れの飛行（各鳥の単純ルールから複雑な群れ運動が生まれる）と同様、NPCの単純なルール群から予め設計されていないストーリーが動的に生成される仕組み。

## 実装事例: Fabula Persona プロトタイプ

著者が開発したプロトタイプの構造:

### NPC の内部モデル

**性格システム** — ビッグファイブ性格特性 (Big Five) をランダム割り当て:

| 特性 | 内容 |
|------|------|
| 外向性 (Extraversion) | 他 NPC への積極的関与の傾向 |
| 協調性 (Agreeableness) | 協力・親切な行動の傾向 |
| 誠実性 (Conscientiousness) | 目標への一貫した行動傾向 |
| 神経症傾向 (Neuroticism) | ストレス・感情の不安定性 |
| 開放性 (Openness) | 新しい経験への好奇心 |

**感情システム** — 4つの基本感情: 喜び・悲しみ・怒り・恐怖

### 相互作用メカニクス

- 個人的距離（近づきすぎると不快感が発生）
- 目標の競合・協調
- 話題の好み（特定のNPCが特定トピックに強く反応）

これらの単純ルールが連鎖することで、設計者が意図しなかったドラマが自然発生する。

## ナラティブの分類フレームワーク

著者が提案する2軸4象限のナラティブ分類:

|  | **明示的 (Explicit)** | **暗示的 (Implicit)** |
|--|---------------------|---------------------|
| **外部 (External)** | 会話・明確なイベント | 環境や文脈から読み取る意味 |
| **内部 (Internal)** | プレイヤーの直接的体験 | 感情・潜在意識の反応 |

エマージェント・ナラティブは「内部×暗示的」象限（プレイヤー自身が感情的に解釈するストーリー）を強化する手法。

## 設計上の注意点

- **相互依存システムのデバッグ**: 一つのバグが他コンポーネントに波及しやすいため、開発初期から丁寧なデバッグが必須
- **均衡の維持**: 感情・性格パラメータのバランスが崩れると意図しない行動ループが発生する
- **プレイヤーの解釈余地**: エマージェントなストーリーはプレイヤーが「意味を見出す」余地があってこそ機能する

## 将来の拡張方向

- 機械学習との組み合わせ（行動パターンの学習・最適化）
- NPC が物理的行動（武器所持・環境操作）を取れるシステムへの拡張
- コミュニティ形成・派閥形成のシミュレーション

## ナレッジベース活用メモ

- 映画的ナラティブとエマージェント・ナラティブは対立するのではなく補完的。重要シーンには映画的手法、日常的インタラクションにはエマージェントを組み合わせると効果的
- ビッグファイブ性格モデルは心理学的根拠のある汎用フレームワーク。NPC 設計の性格パラメータとして活用しやすい
- プレイヤーが「物語を見つける」体験はルドナラティブ調和の向上にも寄与する
