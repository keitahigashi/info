## 概要

Project: HasteのプログラマーConnor Robertsが、壁走り・スライド・ダッシュ等を組み合わせたFPS高速移動システムの技術設計を解説（2024年11月）。ステートマシンによるモジュール設計と視覚・音響フィードバックの統合で、4ヶ月の開発期間内に爽快感のある移動体験を実現した手法。

## 移動システムアーキテクチャ

```
入力 ─→ ステートマシン
           ├─ ウォールラン（衝突ベクトル計算）
           ├─ スライド
           ├─ ダッシュ
           └─ ダブルジャンプ
              ↓
        各アビリティの速度値を加算蓄積
              ↓
        複合高速移動（slide-jump → boost → wall-run 連鎖）
```

## 設計の核心原則

- 各移動アビリティを**「入力/出力パラメータ付きクローズドシステム」**として独立設計
- アビリティ間の**加算速度蓄積**によりスキル連携で複合的な高速移動を実現
- 壁走りはプレイヤー速度ベクトル＋重力ベクトルの合成でコリジョンベクトルを算出し曲面壁のトラバースを可能にする

## ビジュアルフィードバック設計

| フィードバック | 実装手法 |
|-------------|----------|
| FOVスケーリング | 速度値に比例した動的FOV変化で速度感を演出 |
| ダッシュライン | 速度連動スケールで加速を可視化 |
| ハーフトーンシェーダー | アビリティ別の方向性マスクによるスクリーンエッジエフェクト |
| カメラチルト | 壁接触・ジャンプ時の物理的衝撃感を表現 |
| 方向別アニメーション | 移動方向ごとに異なるキャラクターアニメーションで状態を明示 |

## デザイナーフレンドリー設計

- データ駆動設計でパラメータを外出しし、プログラマー介入なしで数百回のプレイテストを実施
- 具体的なデザインドキュメントが存在しない段階から高速イテレーション可能
- 移動システムを「ゲームの基盤」として最初期から設計することで後の調整コストを削減

## 汎用化ポイント

スクリプタブルオブジェクト（ScriptableObject）はUnity固有の実装だが、以下のパターンはエンジン非依存で適用可能：

- **ステートマシンによる移動アビリティのモジュール分割設計**：各アビリティをクローズドシステムとして独立させることで複合的な高速移動連鎖を実現
- **速度値に連動したFOV・エフェクトのビジュアルフィードバック階層**：速度パラメータを軸とした多層フィードバック設計
- **データ駆動パラメータ管理によるデザイナーフレンドリーなイテレーション設計**：Unreal・Godot・カスタムエンジンいずれにも適用可能
