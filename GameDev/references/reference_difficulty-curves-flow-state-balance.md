## 難易度カーブとフロー状態の設計論

### なぜ難易度カーブが重要か

適切でない難易度はプレイヤーの離脱（チャーン / churn）を直接引き起こし、収益にも影響する。設計目標は以下の両極を避けること：

- **簡単すぎる**: 退屈（boredom）→ プレイヤーが興味を失う
- **難しすぎる**: フラストレーション（frustration）→ プレイヤーが諦める

### フロー状態（Flow State）の実現

Csikszentmihalyiのフロー理論をゲーム設計に直結させたフレームワーク：

- 最適な難易度カーブ：プレイヤースキルをわずかに上回る挑戦を常に提供する
- フロー（flow）はスキルと挑戦度が均衡した「集中没入状態」
- この均衡が崩れるとチャーンまたは退屈が発生する

### 問題の特定方法（データ指標）

| 指標 | 意味 |
|------|------|
| リテンション率（Retention Rate） | 特定ステージ・セッションでの離脱ポイントを把握 |
| クリア率（Completion Rate） | 難易度カーブが機能しているか/失敗しているかの定量的な判断材料 |

### 調整戦略

- リソース（パワーアップ・報酬）の配置変更でバランス調整
- 複雑さの段階的導入（complexity ramp-up）
- チェックポイント間隔の見直し

**ケーススタディ（Tall Man Run）**: 障害物の間隔とパワーアップ頻度の微調整で離脱率を改善。プレイヤーデータが設計判断の根拠になることを実証した事例。

### 難易度カーブの形状パターン

良い難易度カーブはノコギリ波状：新しいチャレンジ→習熟→次のチャレンジの繰り返し。各「山」はフローゾーン内に収まる高さに設定する。
