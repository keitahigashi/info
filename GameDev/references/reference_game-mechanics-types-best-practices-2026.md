## メカニクスの4分類フレームワーク

ゲームメカニクスを「ゲーム世界との相互作用を支配するルール・システム・インタラクティブ要素」と定義し、以下4種に分類する。

| 種類 | 定義 | 例 |
|------|------|----|
| コアメカニクス | ゲームプレイの根幹 | 移動・戦闘・資源管理 |
| 進行メカニクス | 成長・習熟の表現 | レベルアップ・スキルツリー・アンロック |
| ソーシャルメカニクス | 他プレイヤーとの相互作用 | 協力・取引・競争 |
| メタメカニクス | ゲームシステム自体との相互作用 | セーブ・NG+・メタゲーム |

### 事例による設計の深み

- **Celeste** — 段位システム（Assist Mode）で幅広いスキルレベルに対応し、コアの「移動」メカニクスの深みを損なわない
- **Hades** — 永続的進行（Boon・Keepsake）でローグライクの失敗を次の試行への投資に変換
- **Stardew Valley** — リソース管理が農業・関係・探索ループを有機的に接続する

### 設計ベストプラクティス

1. **コアループ優先** — 最も繰り返されるメカニクスを最初に磨く
2. **早期プロトタイプ** — メカニクスの感触 (game feel) はテキストでは検証不能
3. **反復的テスト** — プレイヤーの誤解・詰まりポイントを継続的に収集する
4. **段階的複雑化** — 基礎習熟後に上位メカニクスを積み重ねる
