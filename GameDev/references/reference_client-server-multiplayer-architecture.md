## 核心原則: プレイヤーを信頼しない

チート防止の基盤哲学。クライアントは入力を送信し結果を受信する「特権的観客 (privileged spectators)」に過ぎない。

## 権威的サーバーモデル (Authoritative Server)

| 役割 | 責務 |
|------|------|
| **サーバー** | ゲーム状態の排他的制御。全変更を検証 |
| **クライアント** | 入力送信＋状態更新受信 |

例: クライアントが「右に移動」を要求 → サーバーが新位置の妥当性を検証 → 壁貫通/速度チートを防止。

## ネットワークレイテンシの課題

| 条件 | 遅延 |
|------|------|
| 理論最小（SF↔NY） | ~50ms片道 |
| ラウンドトリップ | ~100ms |
| 実環境（混雑/ルーティング） | 100-500ms |

→ この応答性問題が以下4技法を動機付け。

## 4つの核心技法

### 1. Client-Side Prediction（クライアント側予測）

サーバー確認を**待たずに**ユーザー入力にローカルで即座に反応。ゲーム状態をローカルで予測し即時フィードバックを提供。

### 2. Server Reconciliation（サーバー照合）

更新されたサーバー状態が到着したら、予測クライアント状態を**更新状態＋未承認入力から再計算**。予測と権威状態の不一致を修正。

### 3. Entity Interpolation（エンティティ補間）

他プレイヤーを**過去の時点で**表示。サーバーが提供する実データのみ使用し、離散的ジャンプではなくスムーズな動きを実現。

### 4. Lag Compensation（ラグ補償）

サーバーがタイムスタンプ付き入力を受信 → **任意の過去時点の世界を権威的に再構築**。プレイヤーが発射時に照準にいたものを正確に判定。
