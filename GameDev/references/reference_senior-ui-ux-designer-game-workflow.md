## ゲームUI/UXデザイナーの実務知識

### デザインワークフロー: Lo-Fi → Mid-Fi → Hi-Fiの段階化

Warhammer Age of Sigmar: Realms of Ruin での設計プロセス:

| フェーズ | 担当 | 内容 |
|---------|------|------|
| Low Fidelity | デザイナー複数名 | 画面フロー確認、レイアウトスケッチ |
| Mid Fidelity | デザイナー複数名 | インタラクション設計、情報設計の精緻化 |
| High Fidelity | UI アーティスト | 実際にリリースするビジュアルの完成 |

このアプローチにより:
- 早期に設計の問題を発見・修正できる
- アウトソーシングパートナーへの指示が明確になる
- アートディレクターの意図を全チームで共有できる

### FUI (Fictional UI / Film UI) の実務的課題

Jurassic World Evolution 2 のフロントエンド設計から:
- FUI は「実用的でなくても映像として映える」ために設計されている
- ゲームUIはFUIの美的魅力と実際の操作性を両立する必要がある
- 参照ソースがない場合は複数の関連作品からモチーフを抽出・統合する

> 「FUIの魅力と、ゲームUIとしての機能性・実用性を融合させることは本当の挑戦だった」

### クロスプラットフォームUI設計の考え方 (Witcher リメイクコンセプト)

キーボード+マウス / ゲームパッドの両方に対応したUI設計アプローチ:
- 既存シリーズの視覚的言語（タイプフェース・カラースキーム・テクスチャーモチーフ）を継承
- 特定の世界観モチーフ（mutation）を維持しつつ過剰表現を削除
- ソフトウェア: Figma / Adobe XD でワイヤーフレーム → Photoshop で高品質ビジュアル

### 分析的観察習慣の重要性

> 「常に様々なデザインを見て、それがなぜそのように決定されたかを分析的に考えること。ハードウェア制限・ターゲット層の期待・マネタイズ要件など、その決定を形作った状況を考える」

実践法:
- Dribbble・Behance で視覚的インプット
- 異なるジャンルのゲームをプレイして比較・分析
- 「このUIがなぜこうなっているのか」を常に問い続ける

### 推薦参考文献

- **"The Design of Everyday Things"** (Don Norman): UI/UX の古典
- **"Don't Make Me Think"** (Steve Krug): 使いやすさの原則
- **"The Gamer's Brain"** (Celia Hodent): ゲーム向け認知心理学
- UX Collective (Medium): 実務向けUI/UX記事

### 初心者へのアドバイス

**ポートフォリオ構築**:
- NDA 下の作業が多くてもコンセプト作品を制作・公開する
- UX デザイナー → プロセスと問題解決の思考を見せる
- UI アーティスト → 2Dアート基礎・レイアウト可読性・アートディレクションの一貫性を示す

**学習すべきスキル領域**:
- Technical UI Design（UIアーキテクチャ・実装）
- Motion Design（アニメーション・トランジション）
- User Research and Testing（プレイテスト・ユーザビリティ評価）

### UIデザインの本質

> 「多くの意味でシステムデザインはUIデザイン。すべての複雑なシステムは最終的にプレイヤーに説明・提示される必要がある。ワイヤーフレームを何十枚も作り、レイアウト・フォント・アイコンを調整する作業はとても満足感があった」
