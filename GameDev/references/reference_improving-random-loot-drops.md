## 従来の固定確率ドロップの問題

ドロップ率0.2%のような希少アイテムは、運次第でプレイヤーを消耗させゲーム離脱につながる。固定確率は「平均的なプレイヤー」を想定するが、実際の体験は大きく分散する。

## 提案: 失敗蓄積型確率上昇システム (Escalating Pity System)

アイテムを入手できるたびに失敗回数をリセット、失敗するたびにドロップ率を段階的に引き上げる。

```
試行 1回目: 基本ドロップ率（例: 0.2%）
試行 2回目: 前回の2倍
試行 N回目: 最終的に100%（確定入手）に収束
```

### 実装上の3つの工夫

1. **リセット機能** — アイテム獲得後は確率を初期値に戻す
2. **追跡対象の制限** — ストレージコスト削減のため、追跡可能アイテムを5個程度に限定する
3. **透明性 (Transparency)** — UIで進捗（現在のドロップ率・失敗回数）をプレイヤーが確認できるようにする

## 期待される効果

- プレイヤーの満足度向上とゲーム継続率 (retention) の改善
- 「頑張れば必ず報われる」という信頼感の構築
- 極端な不運（最悪ケース）の上限制御

## 現代への発展

このアプローチは現在「天井 (pity system / pity timer)」として業界標準化しており、ガチャゲームや主要ARPGに広く採用されている。透明性を高めた実装（確率開示・進捗表示）はアクセシビリティとプレイヤー信頼の向上にも貢献する。
