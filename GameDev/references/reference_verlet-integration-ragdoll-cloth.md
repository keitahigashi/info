## Verlet積分とは

速度ではなく**粒子位置を追跡**する数値的物理シミュレーション手法。速度は明示的に格納せず、「現在位置と前回位置の差」から暗黙的に表現。

### 基本方程式

```
x(t+dt) = 2x(t) - x(t-dt) + a(t)dt²
```

## Euler法との比較

| 特性 | Euler法 | Verlet積分 |
|------|---------|-----------|
| 安定性 | 低い（制約付きで不安定） | **高い** |
| コスト | 低い | Runge-Kuttaより低い |
| 精度 | 低い | Runge-Kuttaより低いがEulerより高い |
| 速度格納 | 明示的 | 暗黙的（位置差から導出） |
| 制約対応 | 困難 | **最適**（相互接続粒子に適合） |

## システムアーキテクチャ

### Particle（粒子/ポイント）構造

| プロパティ | 説明 |
|-----------|------|
| position (x, y) | 現在位置 |
| prevPosition (prevx, prevy) | 前回位置 |
| mass | 質量値 |
| constraints[] | 制約への接続参照 |

### Constraint/Stick（制約/スティック）システム

2つの粒子間の固定距離を維持:
1. 2点間の距離を計算
2. 望ましい長さに戻すオフセットを算出
3. 各粒子をオフセットの**半分ずつ**調整

→ 明示的な力計算なしに複雑なインタラクションを処理。

## 数値積分ループ

1. 力を適用し加速度を計算（F=ma）
2. Verlet式で新位置を積分
3. ドラッグ減衰適用: `newPos = pos + (pos-prevPos)(1-drag) + accel(1-drag)dt²`
4. **制約を反復的に解決**（粒子位置を調整）
5. 境界条件を処理

## 高度な機能

| 機能 | 実装 |
|------|------|
| **マウスインタラクション** | 前回位置を変更→速度効果を生成。高速移動→大きな力 |
| **制約破断（布の引き裂き）** | 制約をinactiveに設定→布シミュレーションの破れ |
| **ピン留めポイント** | 特定粒子を初期位置にロック→吊り下げアンカー |

## ゲーム開発での応用

- **ラグドール物理**: キャラクター骨格を粒子＋制約でモデル化
- **布シミュレーション**: メッシュグリッドの各頂点を粒子として処理
- **ロープ/チェーン**: 連鎖した粒子＋距離制約
- Thomas Jakobsen（Hitman: Codename 47）: 精度より信頼性とパフォーマンスを優先
