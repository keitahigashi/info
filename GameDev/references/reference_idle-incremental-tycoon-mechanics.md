## 3サブジャンルの比較

### Incremental Games — 「進行の力」

「数字が上がる」の純粋なエッセンス。

| メカニクス | 説明 |
|-----------|------|
| アクティブエンゲージメント | 直接アクション（クリック/タップ）で収入生成 |
| 指数的スケーリング | 各アップグレードが前回より顕著に強力 |
| 即時フィードバック | サウンド/アニメーション/数字表示で全アクションを報酬 |
| マイルストーンプログレッション | 「初の100万」等の具体的目標がエンゲージメント駆動 |

**設計原則**: 個々のクリックにインパクトを、アップグレードに変革感を。
**代表作**: Cookie Clicker

### Idle Games — 「自動化の美」

インクリメンタルメカニクスにパッシブシステムを導入。

| メカニクス | 説明 |
|-----------|------|
| パッシブ収入システム | 工場/労働者/自動ボットが継続的にリソース生成 |
| オフライン進行 | プレイヤー不在でも報酬が蓄積 → 復帰を促進 |
| 自動化レイヤー | アップグレードが段階的に手動入力を排除 |
| **プレステージメカニクス** | プログレスリセット → 永続ボーナス獲得 → メタ進行 |

**設計原則**: 「オンラインでなくても報酬がある」と感じさせる。
**代表作**: AdVenture Capitalist

### Tycoon Games — 「戦略と模擬の融合」

数値収集を経営シミュレーションに変換。

| メカニクス | 説明 |
|-----------|------|
| リソース配分 | スタッフ/資本/材料/時間制約の管理 |
| 戦略的意思決定 | 拡張vs効率、自動化vsコントロールの選択 |
| 相互接続システム | 複数生産チェーンの動的相互作用 |
| プレイヤーエージェンシー | CEO級の権限と結果責任 |

**設計原則**: 全アクションに戦略的重みがある「意味ある選択」を優先。
**代表作**: RollerCoaster Tycoon

## 統合設計フレームワーク

成功作品は3哲学をブレンドし「即時フィードバックと長期的深度」のバランスを達成。

### 開発者への設計質問

1. プレイヤーはクリック vs 戦略的思考のどちらを優先すべきか？
2. 進行にアクティブプレイが必要か、アイドル時も継続するか？
3. フォーカスは数値蓄積か、システム構築か？

### コア開発アプローチ

「小さく始める。1つのループ（クリック→報酬→アップグレード→反復）を完璧に。それを自動化。そして帝国に拡張。」
