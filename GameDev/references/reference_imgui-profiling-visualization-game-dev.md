## 背景：既存プロファイラーの課題

数値データが豊富でも、数値の羅列だけではパフォーマンスパターンを直感的に把握しにくい。

**コア洞察**: 「数値だけでは継続時間・形状・文脈が分からず、直感の構築が困難になる」

## 二段階プロファイリング戦略

```
Stage 1: 軽量可視化ツール（自作）
    ↓ 問題の兆候を検知
Stage 2: 重量ツール（Unreal Insights / PIX / RenderDoc）
    ↓ 根本原因の詳細分析
```

軽量ツールの問いは「調査すべき問題があるか？」のみ。精度より速度・明確さ・反復性を優先。

## 実装した4つのツール

| ツール | 目的 |
|--------|------|
| オンスクリーン統計マーカー | 複数のstat値をリアルタイム表示 |
| FPSモニター | 約2秒間のフレームレートをグラフ表示。フレームドロップと回復のパターンを可視化 |
| メモリデバッガー | シーン移動中のメモリ使用量を追跡。テクスチャ・マテリアル問題を深いプロファイリングなしに検出 |
| パフォーマンスモニター | フレームタイムを表示し、CPUボトルネックかGPUボトルネックかを視覚的に示す |

## Immediate Mode GUI の利点（汎用パターン）

- レイアウトオーバーヘッドが最小限
- 即座の視覚フィードバック
- ステートレス設計による高速な試作
- Retained Mode GUIより実験コストが低い

## 設計哲学

| 原則 | 内容 |
|------|------|
| 可視化優先 | グラフ・トレンドは数値より速くパターンを伝える |
| 二層戦略 | 軽量ツールは重量プロファイラの「門番」として機能 |
| 最小設定・最大明確性 | 設定なしで即使える形を優先 |
| パターン認識が主目的 | マイクロ秒精度より文脈を重視 |
| ステートレスUI | Immediate Modeで反復速度を最大化 |

**重要な教訓**: 開発ツールは網羅的である必要はない。実践者が最初に問うことに答えられれば十分。
