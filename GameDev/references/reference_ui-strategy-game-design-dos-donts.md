## UI Strategy Game Design Dos and Don'ts

著者: Josh Bycer / 公開: 2015年3月24日

### 1. マクロ/ミクロ管理の表示（Macro/Micro Management）

優れた戦略ゲームUIは、プレイヤーが「ミクロ（個別ユニット操作）」と「マクロ（全体戦略）」を同時に把握できる設計が求められる。

- **Do**: 画面下部などに統一した情報パネルを設け、頻繁な画面切り替えを最小化する
- **Don't**: 重要情報を複数のメニュー階層に分散させる
- 良い例: *Rise of Nations* — 選択ユニット情報・コスト・生産キューをひとつの下部パネルに集約

### 2. ビジュアルとホットキーのバランス（Visual and Non-Visual Commands）

上級プレイにはホットキー習得が不可欠だが、ビジュアルコマンドも同等に設計すべき。

| 用途 | 推奨 |
|------|------|
| 新規プレイヤー向け | ビジュアルアイコン + ツールチップ表示 |
| 上級プレイヤー向け | ホットキー表示 + カスタマイズ機能 |
| 緊急時の代替手段 | 常にビジュアル操作でも完結できること |

### 3. 教育的UI（Educational UI）

UIを単なる操作インターフェースとしてではなく、ゲーム知識を伝達する手段として設計する。

- ユニット統計・カウンター関係（相性）をインゲームで視覚的に提供する
- 詳細データへのアクセスが学習を加速させ、戦略的判断の質を高める
- *例*: ユニットAがユニットBに強い/弱いことを選択時にアイコンで即時フィードバック

### まとめ

優れた戦略ゲームUIは「形と機能の融合」。UIの質が戦略的体験の質を直接左右する。
情報密度の高い戦略ゲームほど、情報の優先順位付けと段階的開示が重要になる。
