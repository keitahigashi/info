## 核心的問題

「プロジェクトは最初はシンプルだが、開発者に育てられるにつれてどんどん複雑になる」。多くの開発者がプロジェクトを完成させられない。

## 主要戦略

### 1. ハードデッドラインの設定と遵守

確定した完了日を設定し、タイムラインを延長するのではなくフィーチャーをカットして達成。「完全に仕上がった製品を出荷する必要は実際にはない」。GameJolt や Itch.io でアルファ版をリリースして早期にテスト。

### 2. クリーンなタスクリストの維持

週次のプロジェクト管理が不可欠。常に問う: 「どうやってここに来たか、どこに向かうか、どうやってそこに着くか」。チーム: Trello、ソロ: Sublime Text / Notepad++ などシンプルなツール。

### 3. 反復的なゴール調整 (Iterative Goal Adjustment)

夢のゲームを一度に作ろうとせず、シンプルに始めてイテレーション。「ゴールを遠くに追いかけるのではなく、ゴールを自分に近づける」。段階的にフィーチャーを追加しながら早期にオーディエンスを構築。

### 4. 揺るぎない集中 (Unwavering Commitment)

開発者は頻繁に新しいアイデアのために現プロジェクトを放棄する。代替コンセプトは別途記録し、現作業完了後にのみ参照。「一度に1つのタスクに集中できなければ、切り替えるたびに進捗がリセットされる」。

## 結論

成功は出荷された高品質ゲームに依存する — 無数の未完成プロトタイプではない。
