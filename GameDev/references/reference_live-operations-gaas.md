## LiveOpsの核心

「ローンチ後もゲームを新鮮で収益性があり繁栄させ続ける継続的開発・コンテンツ更新・コミュニティ管理」

## 4つの基本原則

1. **継続的デリバリー** — 大規模拡張ではなく頻繁な更新
2. **長期マネタイゼーション** — プレイヤーライフタイムバリュー重視
3. **データ駆動イテレーション** — プレイヤーフィードバックで改善
4. **コミュニティエンゲージメント** — ゲームプレイと同等に重要

## イベント6類型

| タイプ | 期間 | 目的 |
|--------|------|------|
| **シーズナルイベント** | 2-4週間 | メジャーコンテンツ＋バトルパス |
| **期間限定モード** | 3-14日 | ゲームプレイバリエーション＋実験 |
| **ホリデーイベント** | 1-3週間 | 文化テーマコスメティクス |
| **フラッシュセール** | 24-72時間 | FOMO活用の収益スパイク |
| **競技シーズン** | 1-3ヶ月 | ランクプログレッション＋eスポーツ |
| **コミュニティチャレンジ** | 1-7日 | 集団エンゲージメント目標 |

72時間ウィークエンドイベント（金→日）がエンゲージメント最大化のゴールドスタンダード。

## バトルパス設計

- 50-100ティアの報酬、6-12週間シーズンでアンロック
- プレミアム$10-20で強化コスメ/限定コンテンツ
- 心理ドライバー: サンクコスト / FOMO / デイリーログイン習慣 / 無料ティアの寛大さ

## コンテンツ更新ケイデンス

| 更新タイプ | 頻度 |
|-----------|------|
| メジャーシーズナル | 6-12週間ごと |
| ミッドシーズンパッチ | シーズン3-4週目 |
| 週次サーバーメンテ | 毎週 |
| 緊急ホットフィックス | 即時 |
| パブリックテストサーバー | 随時 |

Feature flagsとリモート設定で完全クライアント更新なしの迅速テスト。

## マネタイゼーション構成比

| 収益源 | 収益割合 |
|--------|---------|
| バトルパス | 30-50% |
| コスメティクス | 40-60% |
| 期間限定オファー | FOMO駆動 |
| VIPサブスクリプション | 月額ボーナス |

動的プライシング: 個別プレイヤーの支出履歴/エンゲージメントでオファー調整。

## 重要KPI

| カテゴリ | 指標 | 目標 |
|---------|------|------|
| リテンション | D1/D7/D30 | 40%/20%/10% |
| エンゲージメント | DAU/MAU比 | 20-50% |
| マネタイゼーション | コンバージョン率 | 2-5% (F2P) |

## チーム構造（MAU別）

| 規模 | MAU | チームサイズ |
|------|-----|-----------|
| 小 | 100K-1M | 3-5人（兼務） |
| 中 | 1M-10M | 10-15人（専門家） |
| 大 | 10M+ | 20-50+人（ポッド構造） |

## 核心的メッセージ

「プレイヤーの時間とお金を尊重しつつ真の価値を届ける」持続可能なLiveOps = 短期収益抽出ではなく長期コミュニティ構築。
