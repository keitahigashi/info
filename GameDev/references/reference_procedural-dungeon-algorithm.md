## 7段階アルゴリズム

### Step 1: 部屋生成

- 円形/楕円形境界内にランダムな幅/高さの部屋を配置
- サイズ分布に**正規分布**を使用 → ダンジョン美学の制御
- `getRandomPointInCircle()` で均一空間分布
- 全位置/寸法をタイルグリッドにスナップ（丸め関数）
- 水平偏重ダンジョンには楕円を使用

### Step 2: 部屋分離（物理ベース）

- 各部屋に合致する**ソリッド物理ボディ**を追加
- 物理エンジンで平衡状態までシミュレート
- 重なりを自動解消しつつタイルグリッド整合を維持
- リアルタイムより高速なシミュレーション可能

### Step 3: メイン部屋の特定

幅/高さが平均の**1.25倍以上**の部屋をメイン部屋として選択。グラフ接続の候補に。

### Step 4: ドロネー三角形分割

メイン部屋の中心点をドロネー三角形分割アルゴリズムに入力 → 接続点の三角形を生成。後続のグラフ構築の基盤。

### Step 5: 最小全域木（MST）＋ループ追加

1. 三角形分割結果からグラフを構築
2. **最小全域木**を計算 → 全メイン部屋が到達可能に
3. ドロネーグラフから元エッジの**8-15%を追加** → オプショナルパス＋ループ生成

### Step 6: コリドー生成

グラフノード間を廊下で接続:

| 条件 | コリドー形状 |
|------|-----------|
| Y座標が近い | 水平コリドー |
| X座標が近い | 垂直コリドー |
| 両軸で異なる | **L字コリドー** |

- 並列コリドー（タイルサイズ間隔）で最小廊下幅を確保
- コリドー線と交差する非メイン部屋をコリドーコンポーネントとして統合

### Step 7: グリッド完成

残りのギャップを単一タイルセルで充填し、ダンジョン全体の歩行可能通路を完成。

## 生成パラメータの影響

| パラメータ | 影響 |
|-----------|------|
| 初期部屋分布の分散 | 低い → 均一コリドー / 高い → 多様なレイアウト |
| サイズ制約 | 部屋の最小/最大サイズがダンジョン密度に影響 |
| アスペクト比制御 | ゲームメカニクスに適したダンジョン形状を確保 |
| MST追加エッジ比率 | 高い → 多くのループ・代替ルート |
