## 核心命題：深さと複雑さの非対称関係

```
深さ ↑ → 複雑さ ↑（必要）
複雑さ ↑ → 深さ ↑（保証されない）
```

> 「複雑なものは習得が難しい。深いものは習得が難しい——しかし理由が異なる」
> — 複雑さは「量」、深さは「習得難度の質」を意味する

簡潔な表現: **「簡単に学べて、習得が難しい (easy to learn, hard to master)」** が深さの近似。

## コミュニティ調査から得た「深さ」の条件

### 必要条件

1. **有意な選択肢** — 明白でも支配的戦略でもない選択が存在する
2. **エマージェンスのある組み合わせ** — メカニクス・ダイナミクス・変数・ルールが組み合わさり多層的考慮が生まれる
3. **複数ムード先読み** — 熟練プレイヤーが数手先を計画できる
4. **スキルレベルを超えた意思決定** — すべてのスキル層で有意な判断が残る

### 「深さ」ではないもの

| 誤解 | 反論 |
|------|------|
| 選択肢の**数** | 多い選択肢は複雑さを増すが、支配戦略があれば深さはゼロに等しい |
| **実行難度**（リズムゲーム等） | 体の動きの精確さを要求するが戦略的思考を要求しない |
| **感情的インパクト**・新規性 | 驚きや感動は体験の豊かさだが深さの定義外 |
| **スキルギャップ**の大きさ | プレイヤー間の実力差は深さを測る指標ではない |

## インタープレイが生む深さ

**インタープレイ (Interplay)** = ゲーム要素間の相互影響。深さの主要生成機構。

インタープレイが不十分な場合:
- 支配戦略（dominant strategy）が出現
- 選択の意味が失われ習熟度が収束
- プレイヤーは「やることを覚える」だけになる

インタープレイを豊かにする設計ツール:
- **空間的次元**（2D/3D配置）
- **リアルタイム要素**（時間的プレッシャー）
- **ランダム変数**（毎回異なる状況）

## 明確性 (Clarity) の重要性

> 深いシステムであっても、プレイヤーが「何が起きているか」を理解できなければ有意な選択ができない。

複雑さと深さを同時に追求する場合、**情報の可読性**が設計の優先事項になる。Knights in the Nightmareの反面教師: 画面情報過多で判断が麻痺し、複雑さが深さに転化しない。

## 設計への実践示唆

1. **支配戦略の排除**: 単一最適解の存在を定期的に検証する
2. **有意な選択の設計**: すべての選択肢が「状況によっては最善」になる均衡を目指す
3. **情報設計と深さはセット**: 深いシステムほど、情報を適切に提示する仕組みが必要
4. **使われない複雑さ = 障害**: 「存在するが活用されないメカニクス」は学習負荷のみ発生させる
5. **空間・時間・ランダム性の組み合わせ**: これら3要素がエマージェントなゲームプレイを自然に生成する
