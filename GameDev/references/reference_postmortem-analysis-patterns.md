Game Developer (Ara Shirinian): https://www.gamedeveloper.com/audio/dissecting-the-postmortem-lessons-learned-from-two-years-of-game-development-self-reportage

## 研究概要

Game Developer誌の24件のポストモーテム（2008年2月〜2010年1月）を統計分析。各ポストモーテムは「成功5項目・失敗5項目」の統一フォーマット。合計**240データポイント**（成功120件、失敗120件）を収集。

## プロジェクト特性の実態

### チーム規模分布
- 最小: 5名（Age of Booty, n+）
- 最大: 265名（Far Cry 2）
- **4クラスタに集中**: 5〜10名 / 30〜40名 / 68〜90名 / 100名以上

### その他の指標
| 指標 | 値 |
|------|-----|
| マルチプラットフォーム展開率 | 約33% |
| 平均開発期間 | **2.4年** |
| 外注活用率 | 46%（11/24件） |

## 成功要因の分類

| 領域 | 順位 | 特徴 |
|------|------|------|
| **プロダクション** | **最多** | スケジュール管理と組織運営 |
| デザイン | 第2位 | ゲームデザイン上の判断 |
| 外部要因 | 第3位 | パブリッシャー関係等 |
| アート | 低い | 不釣り合いに少ない |
| テスト | 最少 | ほとんど言及されない |

**核心的発見**: プロダクションが成功の大部分を占める。

## 失敗要因の傾向

> プロダクション問題が失敗の大部分を占める。デザイン・アート・コーディングの失敗ではなく、**開発プロセスの優先順位付け・実施・管理方法**の問題。

## 研究の限界

1. **情報の不完全性**: ポストモーテム著者が選別した情報に依存
2. **信頼性の問題**: 企業のPR考慮による情報抑制の可能性

## 実践的含意

- 開発チームが共通の落とし穴を認識し、「開発地獄」の循環を断つ実証的基盤
- 経験の共有が知識蓄積の重要な手段
- ポストモーテムはチーム内で「失敗から学び、成功を祝う」プロセスとして最も価値が高い
