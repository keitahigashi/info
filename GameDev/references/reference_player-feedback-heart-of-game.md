## 概要

著者 Lee Perry（2013年5月）。ゲームを良くするための最も普遍的なアドバイスとして「フィードバック (feedback)」を挙げる。フィードバックとはボタンを押す→ゲームが反応するというループの質であり、これは単なるポリッシュではなくゲーム体験の核心部分。この感覚を優れたものにするだけで、ゲーム全体の印象が劇的に改善される。

## フィードバックとは何か

> 「ボタンを押すという単純な行為が、何度も繰り返したくなるような体験になること」

フィードバックは視覚・聴覚・触覚（コントローラー振動）・カメラ動作の4チャネルで表現され、これらが連動するほど「感触 (feel)」が向上する。

## ジャンル別実装ポイント

| ジャンル | 重要なフィードバック要素 |
|---------|----------------------|
| シューター | 命中時のヒットマーカー・敵のリアクション・弾着エフェクト |
| レーシング | FOV変化・カメラシェイク・タイヤスモーク |
| パズル | 要素消滅時のフラッシュ・音・スコアポップアップ |
| 格闘 | ヒット時のヒッチフレーム（インパクトフリーズ）・画面揺れ |
| プラットフォーマー | 着地時のカメラ揺れ・着地音・粒子エフェクト |

## 具体例：名作ゲームのフィードバック設計

### Mario 64
- ジャンプ時：上昇音 → 飛行中のトーン → 着地時の短い効果音
- 着地時：カメラの微小な下方向シェイク
- 目に見えない「地面の感触」が何千回プレイしても飽きない理由

### Gears of War
- カバーへの飛び込み：ダスト粒子 + グラント音 + カメラの前方スライド
- これらが全て同期してカバーシステムへの「重さと安心感」を演出

### Angry Birds
- 発射体の軌跡ライン：次のショットを計画させ、連射を誘発する

## 設計原則

### 「ポリッシュではなく心臓部」の意味

多くの開発者はフィードバックを「最後に磨く仕上げ要素」として扱うが、Perry はこれを誤りと指摘する：

- フィードバックがなければコアメカニクスの評価自体が不正確になる
- プレイテストで「手触りが悪い」という感想はフィードバック不足が原因の場合が多い
- **プロトタイプ段階からフィードバックを仮実装し、感触を正しく評価できる状態にする**

### 4チャネルの連動

| チャネル | 例 |
|----------|-----|
| 視覚 | パーティクル・フラッシュ・画面シェイク・ズーム |
| 聴覚 | 効果音・音程変化・SEのレイヤリング |
| 触覚 | コントローラー振動（ランブル） |
| カメラ | シェイク・前方スライド・FOV変化 |

複数チャネルを同時に反応させることで、単一チャネルの数倍の体感インパクトが得られる。

## 実践チェックリスト

- [ ] 主要アクション（攻撃・ジャンプ・アイテム取得）ごとにフィードバックが3チャネル以上で表現されているか
- [ ] フィードバックを全て無効にした状態でプレイしても「ゲームの手触り」が成立するか（成立しない＝フィードバック依存が過剰）
- [ ] 類似アクション間でフィードバックの強弱が分かれているか（強打 vs 弱攻撃）
- [ ] フィードバックのタイミングがアクションと同期しているか（0.1秒のズレも体感を損なう）
