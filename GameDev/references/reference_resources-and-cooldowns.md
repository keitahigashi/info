## リソースの基本原則

「リソースは設計によってプレイヤーを制限し、何ができるかを制約するように作られている」— リソース制約がゲーム難易度の基盤を形成。

## 普遍的リソースの分類

| リソース | 特徴 | 設計上の役割 |
|---------|------|------------|
| **ヘルス** | ほぼ全ジャンルで採用 | 最も基本的な生存制約 |
| **マネー** | 実生活の経済原則を反映 | 「より多い=より良い」の報酬感 + 意図的なジレンマ（Fable 3: 財政vs市民の幸福） |
| **時間** | 難易度調整の「簡単な方法」 | 行動速度と戦略実行の制約 |
| **ジャンル固有** | チェスの駒、弾薬、ポジション、カバー | ゲーム体験の独自性を定義 |

## MMORPGの複合リソースシステム

WoWの事例: マナ/エネルギー/レイジ/フォーカス/アルカナ/ソウル

「そのリソースの行動がキャラクターの行動を定義する」:
- **ヘルス消費型**: 非常に高リスク・高リターンの感覚
- **レイジ型**: 戦闘で蓄積 → 攻撃的プレイスタイルを促進
- **マナ型**: 有限プール → 計画的な能力使用を要求

## クールダウンの設計原則

### 本質
「エンジンの過熱比喩」— 時間ベースのリソース制限メカニズム。

### 有効性と楽しさのバランス

| 設定 | 問題 |
|------|------|
| 短いクールダウン + 強力な能力 | プレイヤーが「制御不能な力」に → チャレンジ消失 |
| 長いクールダウン + 頻繁使用可能 | 影響力低減 → 「きしむおもちゃ」効果 |

### 能力の位置づけ
「能力とそのクールダウンは料理のスパイス」— 基本ゲームプレイを強化する補助的要素であり、メインのチャレンジメカニズムではない。

## 難易度調整とリソース

### 問題のある実装
- 敵ヘルスの増加のみ → 「弾薬スポンジ」効果で退屈化

### 効果的な実装
- チェス駒の削減（直接的制約）
- マーチャント価格調整（経済的制約）
- アイテム出現率低下（可用性制約）

## Overwatchの設計哲学

Blizzardは弾薬管理を意図的に排除: 「それはゲームにとって楽しくなかった」

**教訓**: リソース管理を強要すべきではない。品質の高い設計者は「何を管理させるべきか、何をさせるべきでないかの線引き」を理解している。

## 全体的な設計バランス

- 複数リソース提供時: 一方が豊富なら他方が希少であるべき
- すべてのリソースが豊富 → ゲームが「映画」化、挑戦性消失
- 検討事項: 「管理すべきリソース数」「不完全管理への罰則」「異なるリソースの管理失敗に異なる結果があるか」
