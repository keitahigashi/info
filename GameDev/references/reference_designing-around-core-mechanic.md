## コアダイアグラムによるゲーム設計

### コアダイアグラム（同心円モデル）

ゲーム設計の構造を4層の同心円で表現するフレームワーク。

| 層 | 内容 |
|---|---|
| 最内層 | コアメカニクス（なければゲームが成立しない操作） |
| 第2層 | 副次的メカニクス（コアを補助・拡張） |
| 第3層 | プログレッションシステム |
| 最外層 | ナラティブ・世界観 |

各層は内側の層に「ゲーティング関係」を持ち、外側の層は内側の構造に依存する。

### コアメカニクスの特定方法
- 「それがなければゲームが成立しないか？」
- 「プレイ中に最も頻繁に使うか？」

この2問で判断する。例: プラットフォーマー→ジャンプ、シューター→射撃、レーシング→運転

### 設計事例比較

| ゲーム | コアメカニクス | 各層の調和度 |
|---|---|---|
| Super Mario Bros. | ジャンプ | 全層が高く調和した理想例 |
| Portal | 射撃+パズル融合 | 革新的コアの成功例 |
| Angry Birds | 投石（フリング） | 副次メカニクスとの連携が不自然な点も |
| Mass Effect | アクション↔戦略のモーダルシフト | ペーシング目的でコアダイアグラムを意図的に切替 |

### 示唆
- 記憶に残るゲーム体験は「珍しいコアメカニクスの傑出した実装」か「異なるメカニクスの驚くべき組み合わせ」から生まれやすい。
- ストラテジーゲームでは「思考・判断」がコアメカニクスになりえる（物理的操作に限らない）。
- 外側の層（ナラティブ等）はコアメカニクスの強化のために存在すべきで、独立して機能させてはならない。
