## 3つの核心目標

1. **フェアプレイの確保**: 異なるシステム間で同等の競争環境
2. **改善の機会提供**: 測定可能な進歩を見出せる仕組み
3. **継続的な最適化**: 判断力と実行力の両面で常に最適化可能

## ゲームプレイ設計

### リセット機能
- **高速リセット**: メニュー/ロゴ画面を回避しプレイ直前に復帰
- **セクション練習**: レベルセレクトやデバッグメニュー対応

### カットシーン対応（優先順位順）
1. スキップ機能（単一キープレス推奨）
2. 条件付きスキップ（既視後のみ）
3. 設定オプション化
4. オートセーブ補完

### RNG管理
- **確定的バリエーション**: すべての選択肢を等時間化
- **シード機能**: プレイヤー指定で再現性確保
- **カウント化**: 確率ではなく「100体目に必ずドロップ」

### 移動の工夫
- 多様なテクニック（二段ジャンプ、壁ジャンプ等）
- 速度と安全性のトレードオフ（スタミナ等）

## タイマー実装

### 要件
- ゲームプレイ中のフレーム数計測（ロード画面・カットシーン除外）
- 終了画面・セーブファイル選択画面での表示
- ポーズ中: 意思決定が必要なら計測継続

## 技術要件

### 固定ステップ物理エンジン
- 60FPS標準で全プレイヤー同一物理演算を保証
- ラグ時はグラフィック更新をスキップ（物理優先）

### 確定性 (Determinism)
- メモリ初期化の厳密化
- 浮動小数点演算の廃止（物理計算）
- 乱数生成器の明確なシード設定

## カテゴリ対応

| カテゴリ | 要件 |
|---------|------|
| **100%** | ゲーム内完全率表示（数値/パーセンテージ） |
| **Low%** | ソフトバリア採用、代替ルート複数化 |
| **IL** | レベル独立状態保証、専用モード |

## 実装優先順位

| 優先度 | 項目 |
|--------|------|
| **必須** | 固定フレームレート物理 / カットシーンスキップ / ゲーム内タイマー |
| **重要** | 高速リセット / 確定性保証 |
| **推奨** | IL対応 / ウィンドウモード |
