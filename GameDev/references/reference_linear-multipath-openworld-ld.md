## 3つのレベルデザインカテゴリ

### 1. リニア設計 (Linear Design)

「プレイヤーが事前定義されたパスを使ってAからBへ移動」する構造。

**エスカレーション曲線:**
1本の連続的な難易度ピークではなく、複数のビート（ピーク=激しいゲームプレイ / バレー=静かな瞬間）を配置。プレイヤーの疲弊を防ぎエンゲージメントを維持。

**ペーシング原則:**
赤エリア = **ゲームプレイ空間**、緑エリア = **静かな内省空間**。この交互配置がリズムを生む。

### 2. マルチパス設計 (Multi-Path Design)

「環境内の複数ルート」で、目的地や重要エリアをつなぐパスを提供。

**主要課題:**
- プレイヤーの混乱と方向感覚の喪失
- ルートに関係なく一貫したプログレッション維持
- プレイヤーエージェンシーを提供しつつデザイナーの意図を保持

**解決戦略:**
特定の場所と関連ゲームプレイを個別のプログレッションビートに割り当て、複数パスでも意味ある瞬間で収束させる。

**利点:** 全ルートで一貫したプレイヤープログレッション / 各パスの個別キャラクター / 構造を維持しつつエージェンシーを提供

### 3. オープンワールド / サンドボックス設計

**POI (Point of Interest) コンセプト:**
POIは磁力的引力として機能。プレイヤーは距離、相対的関心度、事前知識、過去の訪問を比較検討してPOIを選択。

**オニオン構造:**
ワールドを地域に分割し、プレイヤープログレッションをレイヤー化。外側ゾーンから内側へ、プログレッション曲線に基づいたコンテンツ体験を保証。

**POIレイヤーシステム:**

| 要素 | 説明 |
|------|------|
| 無限のアプローチ角度 | オープンワールドからの多方向接近 |
| 複数エントリポイント | プログレッションレイヤー間の複数入口 |
| 2-4の内部目標レイヤー | 段階的なアクセスポイントと難易度 |

**シンプリシティの論拠:**
コンポーネントが多すぎると「追跡が非常に困難」になる。好ましいルートを特定し、過度な複雑性をバランスするのではなく徹底的に磨くべき。

## 横断的原則

- **コミュニケーション価値**: 明確な構造図がレベルデザインを概念的に理解しないステークホルダーへの伝達を助ける
- **変数 vs 定数**: 各パターンに制御可能な変数（場所サイズ、レイヤー数、難易度）と固定定数（プログレッションビート、目標配置）が存在
