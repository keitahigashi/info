## MOBAの定義

2チーム（通常5v5）が強力なキャラクターを操作し、相手の拠点構造物を破壊する競技的チームベース体験。

## 本質的ゲームプレイ要素

### 目標構造

- 勝利条件: 相手チームのメイン建造物の破壊
- 純粋なデスマッチではなく**目標重視**の設計が差別化要因

### キャラクタープログレッション

- チャンピオン/ヒーロー/ゴッドと呼ばれる強力なプロタゴニスト
- 試合中に**経験値蓄積＋ゴールド獲得**で成長
- ゴールドでアイテム購入 → パッシブボーナス＋アクティブ能力

### ミニオンサポートシステム

- AIクリープ/ミニオンが定期的にスポーンし両拠点を繋ぐパスを移動
- 防衛タワー破壊に不可欠な支援を提供

## マップ構造

- **3レーン**: Top / Mid / Bot
- **ジャングル**: レーン間の迷路的ゾーン、サイド目標を含む

## ヒーローロール（6基本型）

| ロール | 特徴 |
|--------|------|
| Tank | 高耐久、味方保護 |
| Mage | 魔法ダメージ、範囲攻撃 |
| Marksman | 遠距離物理ダメージ |
| Healer/Support | 回復・バフ・味方支援 |
| Assassin | 高バースト、低耐久 |
| Fighter | 近接、バランス型 |

### ジャングラーとサポート

- **Jungler**: クリープ討伐＋プレイヤー待ち伏せに特化。レーンリソースを味方に残す
- **Support**: 味方の戦闘力を増幅

## バランス哲学: 完璧な不均衡

Dota 2 の「完璧な不均衡 (Perfect Imbalance)」:

- 定期的なバランス調整でメタゲームが変動
- 異なるヒーロー構成・戦略が周期的に支配→代替→シフト
- 新コンテンツリリースなしでゲームプレイの新鮮さを維持

## 設計原則

- 初心者にアクセシブルかつベテランに十分な深度
- 一見強力なメカニクス（不可視等）がスキル向上で対抗策を提示
- スキルレベル別の個別バランスパッチを回避する設計

## チーム構成

- バランスの取れたチーム構成で多様な状況に対応
- 貧弱な編成は適応力を喪失
- ドラフト戦略が試合前の重要な戦略要素
