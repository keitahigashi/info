## 深さ・複雑さ・ユニークさの三つ巴

### 核心的な定義

| 概念 | 定義 |
|------|------|
| **深さ (depth)** | 限定的なメカニクスの中で、複数の戦略的選択肢と相互作用が存在する状態 |
| **複雑さ (complexity)** | 多数の独立したシステムとルールの存在（必ずしも相互に関連していない） |
| **ユニークさ (uniqueness)** | 他のゲームと差別化する独自のメカニクスや表現 |

> 「深さは複雑さとは互換性がない。複雑さの削減は深さの喪失を意味しない」

### 理想の設計

複雑さが低い割に深さが高い状態がゲームデザインの理想。複雑さは深さへの到達を妨げるバリアになりうる。

## 比較ケーススタディ：KiN vs TWEWY

著者は DS 向け RPG 2 作品を対比して分析する。

| 要素 | Knights in the Nightmare (KiN) | The World Ends With You (TWEWY) |
|------|-------------------------------|--------------------------------|
| 導入方法 | 初日から全システム投入・138 ページのチュートリアル | 段階的な難度上昇（タッチ→デュアルコンバット→装備） |
| システムの影響 | ゲームプレイに実質的効果なし | 難易度・報酬に直結 |
| 深さ | 低い | 高い |
| プレイヤー継続性 | 数ステージで離脱 | エンドゲームまで持続 |

### KiN の失敗分析

- ユニット合成・アイテム融合など追加システムが多いが、実際のゲームプレイへの影響が感じられない
- 「スタイルと実質のズレ」：多様な選択肢があっても結果に差がない

### TWEWY の成功要因

- 各システムが難易度調整や報酬体系に直結し、プレイヤーの選択に応答する
- メカニクスを段階的に解放し、理解が深まるにつれて戦略の幅が広がる

## 複雑さを軽減しながら深さを保つ実践手法

```
1. クリーンな UI と優れた操作性
2. ツールチップ・インゲーム説明の充実
3. 重要情報への素早いアクセス
```

**好例**：『Tactics Ogre PSP』のポーズ時ツールチップ機能 ── 複雑な情報をゲームを止めずに確認できる。

## 歴史的文脈：アドベンチャーゲームの教訓

冒険ゲームの「動詞システム」の変遷：

```
1990年代初期：複雑なインタラクション（多数の動詞選択肢）
   ↓ 過剰な複雑さでユーザー離れ
TellTale ゲームス：選択肢を絞り込み・UX 改善
   ↓ ジャンルの再活性化
```

複雑さの削減はジャンルの消滅ではなく進化の手段であることを示す事例。

## 設計原則のまとめ

1. **複雑さと深さを混同しない** ── 多機能 ≠ 高深度
2. **段階的な導入** ── システムを一度に全解放せず、習熟に応じて展開
3. **各システムが結果に影響するか確認** ── 使っても使わなくても同じなら深さにならない
4. **複雑さ軽減の手段を先に検討** ── 情報設計・UI 改善で認知負荷を下げる
