## プロジェクト概要

物理シミュレーション学習目的のJavaScriptベース2D物理エンジン。三角形・四角形・円の衝突検出と応答を実装。

## 1. 慣性モーメント

### 概念
「回転のしにくさ」を表す量。2次元ではスカラー値。

### 図形別の計算式

| 図形 | 計算式 |
|------|--------|
| 三角形 | `I = (1/18)m(|r₁|² + |r₂|² + |r₃|² - r₂·r₃ - r₃·r₁ - r₁·r₂)` |
| 円 | `I = (1/2)ma²`（質量m、半径a） |

## 2. 衝突検出パイプライン

### ミンコフスキー差とサポート写像
- ミンコフスキー差: 2つの剛体の相対位置を統合表現
- サポート写像: 「任意方向で最も遠い点」を検出する関数
- 内積を活用して各図形の頂点を方向に投影

### GJKアルゴリズム (Gilbert-Johnson-Keerthi)
- サポート写像を反復的に使用
- ミンコフスキー差が**原点を含むか**を判定
- 原点包含 = 衝突状態

### EPAアルゴリズム (Expanding Polytope Algorithm)
- GJK後に**衝突深度と法線**を精密計算
- 図形膨張により接触点の正確な位置を決定
- 衝突応答に必要な情報（深度・法線・接点）を提供

## 3. 衝突点座標の特定

手法: 「物体を衝突法線方向に移動させてから接近点を求める」
- 三角形同士の最接近点を総当たりで算出（全頂点と全辺の組み合わせ）
- ローカル座標に変換して保持
- 応答計算に活用

## 4. 衝突応答の撃力公式

```
J = -vᵣ(e + 1) / [1/m₁ + 1/m₂ + n·((r₁×n)/I₁)×r₁ + n·((r₂×n)/I₂)×r₂]
```

| 記号 | 意味 |
|------|------|
| J | 撃力（impulse） |
| vᵣ | 相対速度 |
| e | 反発係数 |
| n | 衝突法線 |
| m | 質量 |
| I | 慣性モーメント |
| r | 接点からのベクトル |

## 5. ソルバー実装

- ソルバープロキシを生成し、剛体情報をコピー
- 制約セットアップで摩擦係数を算出
- **反復演算**で撃力を蓄積
- 速度・角速度を更新

## 6. 実装上の課題と解決策

### 2D角速度の3D計算との融合
スカラー角速度をZ軸回転としてベクトル化:
```javascript
vec3.cross(r, vec3(0, 0, objA.angularVelocity * t))
```

### タイムステップ管理
- 更新ループで位置・速度を逐次更新
- 誤差蓄積防止には反復制約解法を採用

## 汎用化ポイント

この実装は教育目的だが、以下の概念はエンジン非依存:

| 概念 | 汎用性 |
|------|--------|
| GJK/EPA | 凸形状の衝突検出の業界標準 |
| サポート写像 | 任意凸形状に適用可能 |
| 撃力ベースの応答 | リアルタイム物理の基本手法 |
| 反復制約ソルバー | Box2D/PhysX等でも同じ原理 |
| ミンコフスキー差 | 幾何学的衝突判定の基盤理論 |
