## なぜ手続き的生成を使うか

| 利点 | 説明 |
|------|------|
| 無限のリプレイ性 | 各プレイで異なるレイアウト |
| 開発時間削減 | 1アルゴリズムが数十の手作りマップを代替 |
| スケーラブルな難易度 | パラメータで易/中/難を調整 |
| 発見と驚き | プレイヤーもデザイナーも各回の内容を予測不能 |

**トレードオフ**: 手作りレベルの精密なペーシング・ストーリーテリング制御を犠牲に。成功作品はプロシージャル＋キュレーション（固定ボスアリーナ等）を組合せ。

## アルゴリズム1: BSP（Binary Space Partitioning）

最も一般的なダンジョン生成手法。信頼性の高い部屋ベースレイアウト。

### 手順

1. マップ全体を1つの矩形として開始
2. ランダムに分割（水平 or 垂直）
3. 各半分を再帰的に分割（最小部屋サイズに到達するまで）
4. リーフ領域にランダムパディングで部屋を配置
5. 兄弟部屋をコリドーで接続（ツリーを逆上りしながら）

**結果**: 清潔な矩形部屋＋コリドー接続。予測可能だが自然。ダンジョンクローラー/ローグライクに最適。

## アルゴリズム2: Cellular Automata（セルラーオートマタ）

有機的な洞窟空間を生成。BSPの清潔な矩形の対極。

### 手順

1. マップをランダムに充填（壁=1/床=0、約45%床確率）
2. ルールを反復適用（4-6回）: 壁隣接4+→壁、それ以外→床
3. ランダムノイズ → スムーズな有機的洞窟に進化
4. 孤立領域を除去して接続性を確保

**ルール**: 隣接8セル中、壁が5以上→壁に / 壁が4以下→床に

## アルゴリズム3: Perlin Noise 地形

滑らかで連続的なランダム値で自然な地形を生成。

- 座標ごとに-1〜1の値を生成
- 閾値で固体地面/空間を決定
- 複数オクターブを異なるスケールで重畳 → 大きな特徴＋細部

## 接続性の保証

「手続き的レベルの最も一般的な問題は到達不能エリアの生成」

### Flood Fill 検証

プレイヤー開始位置からフラッドフィルを実行。到達不能な孤立領域が存在する場合:
- 再生成
- またはブリッジ（通路追加）で接続

## ハイブリッドアプローチ

| 用途 | アルゴリズム |
|------|-----------|
| 明確な部屋＋戦利品配置 | BSP |
| 有機的洞窟探索 | Cellular Automata |
| オープンワールド/Terraria型 | Perlin Noise |
| **最も効果的** | 技法の**組合せ** |

BSPで部屋構造 → セルラーオートマタで洞窟セクション → ノイズで地形変化。

## コンテンツ配置

タイルマップ生成後:
- 有効ゾーン（スポーンから離れた床タイル/デッドエンド部屋）を特定
- 敵/アイテムをランダム配布
- スタートからの距離で重み付け → 難易度スケーリング

## 再現可能な生成

全主要RNGはシード値を受付。同一シード → 同一レベル → Minecraft式シードコードでレベル共有可能。
