## 概要

既存ゲームのメカニクス（ゲーム性、ルール構造、ギミック）を体系的に整理した設計参考資料。アナログゲームの分類からビデオゲームへの応用までをカバー。

## アナログゲームのジャンル分類

| ジャンル | 説明 | 代表例 |
|---------|------|--------|
| **アブストラクト (Abstract)** | 完全情報の抽象的なゲーム | チェス、オセロ |
| **トリックテイキング (Trick-taking)** | カード出し合いで強いカードを獲得 | ブリッジ、ハーツ |
| **ワーカープレイスメント (Worker Placement)** | 労働者配置による資源獲得 | アグリコラ |
| **拡大再生産 (Engine Building)** | 投資による段階的な成長 | カタン、テラフォーミングマーズ |
| **デッキ構築 (Deck Building)** | カードセット構築をメインメカニクスに | ドミニオン |
| **交渉・競り・ドラフト (Negotiation/Auction/Draft)** | リソース獲得の相互作用パターン | — |

## メカニクス設計要素

### ジレンマ / トレードオフ (Dilemma / Trade-off)

最適解が複数存在することで、戦術的な葛藤を生成:
- ハイリスク・ハイリターン (High-risk High-reward)
- 相互排他的な選択肢 (Mutually exclusive choices)
- リソース配分の最適化問題 (Resource allocation optimization)

### アドバンテージ生成パターン (Advantage Generation)

複数次元での利得設計:
- リソース量の優位性
- 情報量の優位性
- 確率的優位性
- 相手妨害
- 情報隠蔽

### 手番システム (Turn Systems)

- ターン制（先手有利の公平化が課題）
- 早い者勝ち (First-come First-served)
- 順序変動メカニクス (Variable turn order)

### ランダム性 (Randomness)

- ダイスロール (Dice roll)
- 山札 (Draw deck)
- 抽選箱方式 (Lottery)
- 不確定性の導入で戦略の幅を拡大

## ビデオゲームへの拡張

アナログゲームのメカニクスは基本的にビデオゲームに流用可能。コンピュータによる自動化で:
- 複雑性を管理可能に
- シングルプレイヤー向けに競争要素を緩和
- リアルタイム処理による新たなメカニクス（アクション、物理演算等）を追加
