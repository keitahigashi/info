## 出典

Game Developer / Featured Blog (Kade Dunn, 2016): https://www.gamedeveloper.com/design/game-writing-designing-non-linear-game-narratives

## 概要

インタラクティブゲームにおけるノンリニアナラティブ (non-linear narrative) の設計原則と実装ワークフローを解説。「プレイすることはユーザーとゲーム制作者の対話である」という視点から、プレイヤーエージェンシーを最大化する書き方を具体的手順で示す。

## 中心思想

- ゲームプレイは「ユーザーとクリエイターの対話 (dialogue)」
- 選択の行為がプレイヤーをパッシブな視聴者から能動的な参加者へ転換する
- 有意義な選択 (meaningful choice) を設計するには、**物語の最初から分岐を計画**する必要がある

## 設計手法 4ステップ

### 1. キャラクターから始める (Start with Character)

- 強いキャラクター造形が信頼できる物語世界の基盤
- プレイヤーが感情的に投資できるキャラクターが分岐の意味を高める
- 「このキャラクターならどう言うか」が対話選択肢の一貫性を保証する

### 2. 対話フローを事前計画する (Plan the Dialogue Flow)

- 実装前にフロー図・チャートで分岐を視覚化する
- 別ページ・別ドキュメントで各ブランチの独自性を確認
- 収束ポイントと分岐ポイントを明示してコンテンツ漏れを防ぐ

### 3. 多様な選択肢を設計する (Design Diverse Choices)

- 「同じ結論への複数ルート」ではなく、**異なるアプローチ・異なる価値観**を体現する選択肢を設計
- 各選択肢がプレイヤーキャラクターの異なる側面を表す
- 例: 直接的/外交的/欺く/黙る などの多角的アプローチ

### 4. 多段階ワークフロー (Multi-Stage Workflow)

| ステップ | ツール・方法 |
|---------|-----------|
| ドラフト | 紙・テキストエディタ |
| 分岐可視化 | Twine、インクルスキー等のインタラクティブ執筆ツール |
| 実装 | ゲームエンジン（RPG Maker等） |
| テスト | 全対話パスの動作確認 |

## ポイント

- ノンリニアナラティブの最大の失敗は「後付けで分岐を追加すること」—最初から計画する
- プレイヤーの選択が**結果に影響する**ことをシステムレベルで保証する必要がある
- 複雑な分岐よりも「少数の意味のある分岐」の方が体験品質が高い
