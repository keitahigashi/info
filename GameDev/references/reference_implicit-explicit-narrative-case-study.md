## 暗示的ナラティブと明示的ナラティブのケーススタディ

著者: Douglas Lynn (2011年5月5日)

### 2種類のナラティブ定義

| 種類 | 定義 | 伝達手段 |
|------|------|----------|
| 暗示的ナラティブ (Implicit Narrative) | プレイヤーの選択・行動・結果を通じて形成される物語 | インタラクション・環境・プレイヤー自身の解釈 |
| 明示的ナラティブ (Explicit Narrative) | 開発者が直接定義・提示する物語 | テキスト・カットシーン・ダイアログ |

### メトロイドシリーズによる実証

#### 暗示的ナラティブの成功例（25年間）

- プレイヤーは主人公サムスの人格をほぼ自由に解釈できるよう設計
- **メトロイド プライム三部作**: スキャンバイザーで外部情報・世界の歴史を提供しつつ、サムスは沈黙を保つ
  - 世界情報は豊富に提供（明示的）
  - サムス自身の感情・思考は提供しない（暗示的）
  - → プレイヤーが「自分の感情」をサムスに投影できる空間が生まれる

#### 明示的ナラティブへの移行と摩擦（2010年）

- **メトロイド アザーエム**: 初めてサムスに明確なキャラクター（トラウマ・感情・台詞）を付与
- **結果**: ファンの長年の解釈と衝突し、強い反発を招く
- **原因**: 25年間プレイヤーが構築してきた「自分だけのサムス」と、開発者が設定したサムスの不一致

### プレイヤー投影型キャラクター設計の原則

著者はプレイヤーとゲームの関係を「対話 (dialogue)」として説明:

```
プレイヤーの行動 → ゲームの制約内での反応 → 「キャラクターならこうする」という解釈
     ↓
プレイヤーがキャラクターの思考様式を内面化する心理的メカニズム
```

**ゲームの独自性**: プレイヤーが「自分自身と最も共感しやすい」キャラクターを体験できる唯一のメディア。この特性は暗示的ナラティブの設計によって最大化される。

### 設計上の教訓

1. **キャラクターの空白設計**: 主人公の個性を完全に定義しない「解釈の余白」を意図的に設ける
2. **世界情報と人格情報の分離**: 世界の歴史・仕組みは豊富に提示（明示的）しつつ、主人公の内面は最小限に留める（暗示的）
3. **長期シリーズでのナラティブ変更コスト**: ファンが長年構築してきた「暗示的ナラティブ」を後から明示的に塗り替えると、強い拒否反応を生む
4. **投影型vs個性型のトレードオフ**: キャラクターに強い個性を付与するほど感情移入の対象は「キャラクター自身」になり、プレイヤー投影は薄れる。ゲームのジャンルや目的に応じてどちらを優先するか設計段階で決定する

### 他ジャンルへの応用

- **RPGの主人公設計**: 無口な主人公（DQ、ポケモン）vs 個性的な主人公（FF、ペルソナ）の設計判断はこの暗示的/明示的ナラティブのトレードオフと直結
- **オープンワールド**: プレイヤーが「自分の物語」を作れる世界は、暗示的ナラティブの大規模実装
- **メタフィクション的活用**: キャラクターが「誰か」という曖昧さを意図的にテーマにした作品（Undertale等）も暗示的ナラティブの応用
