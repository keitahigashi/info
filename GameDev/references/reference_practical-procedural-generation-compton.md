## 核心概念：10万杯のオートミール問題

> 「数学的に異なる100,000通りのオートミールより、
>  プレイヤーが識別できる意味ある1,000通りの方が価値がある」
> — Kate Compton

プロシージャル生成 (procedural generation) の目標は**バリエーション数の最大化ではなく、意味ある多様性**の生成。無数の変形が生じても、プレイヤーが体験上の差異を感知できなければ意味がない。

## 設計空間の理解

生成システムを設計する前に、生成したいものの**設計空間 (design space)** を定義する：

| 次元 | 例（ダンジョン生成） |
|-----|------------------|
| 形状 | 部屋サイズ・廊下幅・形状タイプ |
| 接続性 | 分岐率・ループ率・デッドエンド数 |
| コンテンツ | 敵配置・宝箱密度・ギミック種別 |
| 難度 | 敵強度分布・回復アイテム頻度 |

すべての次元を一度に生成しようとせず、**最も重要な次元を先に特定**する。

## 5ステッププロセス

1. **設計空間の理解** → 何を生成するか、どんな軸があるかを列挙
2. **制約条件の列挙** → 「これは絶対あってはならない」を先に定義
3. **プロセスの理解** → 人間が手で作るときの手順を観察・言語化
4. **生成方法の選択** → 下記の手法から適切なものを選択
5. **反復と柔軟性** → 正しく実装するより「機能することが大切」

## 生成手法の選択肢（IKEA型設計戦略）

### タイル型生成 (Tile-based)
- 領域を分割し、各タイルに異なるロジックを適用
- 例：ダンジョンエリアごとに異なるテーマ・難度
- 利点：局所的な制御が容易

### 文法ベース生成 (Grammar-based)
- 言語の文法規則のように、記号を置換して構造を生成
- 例：`LEVEL → INTRO MIDDLE BOSS` → 各記号をさらに展開
- L-System（植物生成）、テキスト生成に有効

### ランダム分布 (Random Distribution)
- キュレーション可能なランダム配置
- **ブルートフォース法**: 大量生成して良いものだけ採用（Spore での実績）
- 分布の形状（正規分布・指数分布等）で体験の傾向を制御

## 実践的アドバイス

### 「正しく実装しなくていい」

Comptonは「完璧なアルゴリズムを探さず、まず動かせ」と主張。Spore開発での経験から:

```
手法: ランダム生成 → 評価関数でフィルタ → 合格したものを使用
効果: 理論的最適解より、実際に"良く見える"結果を保証
```

### 制約を先に決める

「生成してから検査」より「制約を先に組み込む」方が効率的。特に必須条件（スタートとゴールが接続されている等）は生成アルゴリズム内部に組み込む。

### 意味的一貫性の確保

- ランダムな組み合わせが「砂漠のバイオームに雪の敵」のような矛盾を生まないよう、**タグシステム**で意味的互換性を管理
- 生成後バリデーションより、生成時の制約設計が根本的解決

## 適用判断の基準

| 状況 | 判断 |
|-----|------|
| 同じコンテンツを繰り返すゲーム | 適用推奨 |
| 1回限りの体験を重視するゲーム | 手作りの方が適切な場合も |
| 設計空間の定義が困難 | 先に手動で作成してパターン抽出 |
| 意味ある変形が難しい | 表面的バリエーションに留まるリスク |
