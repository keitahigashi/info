## SBMMの3つの主要目標

| 目標 | 説明 |
|------|------|
| プレイヤーリテンション | バランスの取れた競争がユーザー満足度と継続的エンゲージメントを向上 |
| 競争的公正性 | パフォーマンスが試合結果を決定し、偶然ではない |
| エンゲージメント | 勝利可能な試合がプレイ時間延長を促進 |

## スキルレーティングシステム

### 2種類のレーティング

- **動的レーティング (Dynamic Rating)**: 試合結果に基づいて毎試合調整
- **ベースレーティング (Base Rating)**: 十分なデータ蓄積まで新規プレイヤーに適用

### 主要アルゴリズム

- **Elo**: Árpád Élő教授（1960年）が考案。ゼロサムゲームでの相対スキル計算
- **Glicko-2**: Rating + Rating Deviation + Volatility の3指標。ボラティリティファーミングの問題あり
- **TrueSkill**: Microsoft Xbox 360実装。チームベースゲーム向けElo拡張
- **Elo-MMR**: Massive（線形スケール）・Monotonic（インセンティブ互換）・Robust（変動制限）

## マッチメイキングアルゴリズム3種

| アルゴリズム | 特性 | 適用場面 |
|------------|------|---------|
| Greedy | 速度優先、キュー順にマッチ | 低レイテンシ要求 |
| Hungarian | 複数変数のコスト最適化ペアリング | 精密マッチング |
| Stable Marriage | 相互選好マッチング | チーム編成 |

## マッチメイキングプール

プレイヤーをスキルベースのバケットに分類し、キュー時間短縮とバランスを両立。地理的位置・レイテンシ最適化も考慮。

## 主要課題

### 公平性 vs 待ち時間

- 厳格すぎるシステム → キュー遅延増加
- 緩すぎるシステム → アンバランスなマッチ
- 分析（リテンション率/スキル差/ボット検出）で最適化

### スマーフィング (Smurfing)

経験豊富なプレイヤーが新アカウントで低スキル帯に参入する問題。検出アルゴリズムによるレーティング急速調整で対策。

### ELO Hell

チームベースゲームで特に発生。個人パフォーマンスよりチーム結果が重要なため、特定ティアに固定される。

## 高度な機能

- **EOMM (Engagement Optimized Matchmaking)**: プレイ時間・連勝を考慮したエンゲージメント最適化
- **クロスプラットフォーム正規化**: 入力デバイスのアドバンテージ差を調整
- **動的パラメータ調整**: プレイヤーフィードバックに基づく継続的チューニング

## 成功指標

- マッチ品質（勝率/満足度/リテンション）
- キュー効率（平均待ち時間）
- スキルプログレッション公平性
