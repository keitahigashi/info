## ブレンドツリーの基本概念

ランタイムパラメータに基づいて複数のアニメーションクリップをスムーズに補間するノードベースシステム。急激な遷移ではなく、数値変化に応じたシームレスな補間を実現。

## 1D ブレンドツリー: 速度ベースロコモーション

単一軸（速度）でロコモーション制御:

| クリップ | 速度閾値 |
|---------|---------|
| Idle | 0 cm/s |
| Walk | ~150 cm/s |
| Jog | ~300 cm/s |
| Run | ~500 cm/s |
| Sprint | ~600+ cm/s |

### ストライドミスマッチ問題

歩行（短い歩幅）と走行（長い歩幅）の中間ブレンドで「スライディング/スケーティング」アーティファクト発生。

**解決策**:
1. アニメーション速度スケーリング（再生速度を接地に合わせる）
2. 閾値チューニング（自然な歩幅が一致するポイントでブレンド）
3. IKフットプレースメント（足を地面にピン留め）

## 2D ブレンドツリー: 方向移動

- X軸: 横方向速度 / Y軸: 前後方向速度
- **重心座標補間 (barycentric interpolation)**: 最も近い3サンプルポイント間でブレンド

### サンプル数推奨

| 品質 | サンプル数 |
|------|----------|
| 最小 | 5（中央+4方位） |
| 標準 | 9（中央+4方位+4斜め） |
| AAA高忠実度 | 16-24 |

## 加算ブレンドレイヤー (Additive Blending)

ベースアニメーションを**置換せず差分を重畳**。参照ポーズ（T/Aポーズ）との差分としてデルタを表現。

### 主な用途
- **エイムオフセット**: 水平/垂直照準角度のロコモーション重畳
- **ヒットリアクション**: 移動状態への短時間ボディフリンチ
- **呼吸**: アイドル/歩行時の胸部膨張
- **表情アニメーション**: ニュートラル上への感情重畳
- **武器スウェイ**: プロシージャル/アニメーション駆動の武器動き

## ブレンドウェイトシステム

| タイプ | 動作 | 用途 |
|--------|------|------|
| Normalized | 重み合計=1.0に正規化 | ロコモーション・方向移動 |
| Direct (Non-Normalized) | 正規化なし、1.0超過可 | 誇張スタイリスティック効果 |

## ステートマシンとの統合

**相補的関係**で階層的に機能:

- **ステートマシン**: 離散的状態を管理（Locomotion/Combat/Climbing/Swimming/Dead）
- **ブレンドツリー**: 状態**内**の連続的変化を処理（Walk→Runのスピードブレンド等）

### レイヤードアニメーション

下半身: ステートマシンによるロコモーション
上半身: エイム/ジェスチャーレイヤー（ボーンマスクでブレンド）

## よくある落とし穴と対策

| 問題 | 原因 | 対策 |
|------|------|------|
| テレポート/ドリフト | ルートモーション設定の不一致 | 全クリップで統一 |
| 斜め方向のアーティファクト | 2Dサンプル不足 | 45°中間サンプル追加 |
| フット接触のデシンク | クリップ長の不一致 | 同一フレーム数で制作/同期グループ使用 |
| 不自然な遷移 | 全遷移にブレンドツリー使用 | 離散的変化（武器引抜/死亡）は短いクロスフェード |

## パフォーマンス最適化

- **クリップサンプリング**: 寄与クリップは毎フレームサンプリング → 閾値をタイトに設定
- **LODシステム**: 距離に応じたブレンドレイヤー削減
  - 20m以内: フル2D + 加算（ヒーローキャラ）
  - 中距離: 簡易1Dロコモーション
  - 遠距離: 単一クリップ/モーションマッチドポーズ
- **スレッドセーフティ**: マルチスレッド評価の活用
- **圧縮**: エンジンネイティブ圧縮をロコモーションクリップに適用
