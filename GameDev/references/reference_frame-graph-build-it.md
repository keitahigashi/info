## フレームグラフ実装ガイド

### 概要

2026年2月10日公開。フレームグラフ4部作の第2部。理論を実際に動くC++17コードへ落とし込む実装ガイド。90行のスキャフォールドから500行の完全版まで3段階のイテレーションで構築する。

### 3段階の実装イテレーション

#### Version 1：スキャフォールド（約90行）
- パスとリソースの宣言・逐次実行の基本骨格
- 依存関係追跡なし、バリアなし
- 目的：最小動作フレームを確立する

#### Version 2：依存関係とバリア（約260行）
- **リソースバージョニング**：書き込みごとにバージョン番号をインクリメントし、読み込みは現在バージョンに紐付く
- **依存エッジの自動生成**：RAW（Read After Write）・WAR・WAWの依存ハザードを自動検出
- **Kahnのアルゴリズム**によるトポロジカルソート
- **デッドパスカリング**：後段から逆向きにグラフを辿り不要パスを除去
- **自動バリア挿入**：状態遷移に基づきバリアをコンパイル時に事前計算

#### Version 3：ライフタイムとエイリアシング（約500行）
- **ライフタイム解析**：リソースごとの初回使用パス・最終使用パスを記録
- **グリーディフリーリストアロケータ**：非重複ライフタイムのリソースが物理メモリを再利用
- ディファードレンダリングパイプラインでVRAMを約50%削減

```
// 宣言フェーズ：依存関係をラムダで登録
// コンパイルフェーズ：全バリアと実行順を事前計算
// 実行フェーズ：GPUコマンドの純粋再生
```

### 技術的ポイント

- **リソースバージョニング**で明示的な依存宣言不要でエッジを自動生成
- コンパイル時に全バリアが確定することで、実行時の意思決定コストがゼロ
- C++17標準機能のみで動作し、特定エンジンへの依存がない
- 各バージョンは前バージョンの上に積み上げる構造で段階的な理解が可能

### 診断出力の内容

- バリア遷移の一覧
- メモリ削減率（%）
