## AI計画システムの4手法比較

### 1. Classical Planning（古典的計画法）

- 初期状態からゴール状態への**オペレータ列**で意思決定をモデル化
- 構成要素: 初期状態 / ゴール状態 / オペレータ（状態変換アクション）
- **適用**: パズル・線形チャレンジ
- **弱点**: 計算コストが高い、動的環境に弱い

### 2. GOAP（Goal-Oriented Action Planning）

- F.E.A.R.（2005）で実用化。NPCが高レベル目標を満たすアクション列を**動的に構築**
- 前提条件（preconditions）と効果（effects）を評価し、状況に応じたアクション列を生成
- **強み**: 創発的で多様な行動、ハンドスクリプト不要
- **弱点**: アクション・前提条件の設計が緻密さを要求、デバッグ困難

### 3. Utility-Based Planning（ユーティリティベース計画）

- 「何が可能か」ではなく「何が最も価値があるか」にフォーカス
- 各アクションを現在の条件に基づいて**スコアリング**し、最高スコアのアクションを選択
- **強み**: 常に再優先度付け、リソース管理・戦略ゲームに最適、カーブ調整で容易にチューニング
- **弱点**: スコア急変による「ビクビク」行動、数学的バランスが必要

| 空腹かつ所持金少の例 | スコア |
|---------------------|--------|
| 冷蔵庫から自炊 | 0.8 |
| デリバリー注文 | 0.6 |
| 外食 | 0.4 |

### 4. Behavior Tree（ビヘイビアツリー）

- シーケンス・セレクター・デコレーターの**階層的決定構造**
- **強み**: 視覚的に明確、デバッグ・修正容易、知覚・移動システムとの統合がスムーズ
- **弱点**: 巨大化すると「スパゲッティツリー」に、GOAPやUtilityより創発性が低い

## 比較マトリクス

| システム | 最適用途 | 主な強み | 主な課題 |
|---------|---------|---------|---------|
| Classical | 構造化問題 | 論理的推論 | 計算コスト |
| GOAP | 戦闘・動的NPC | 創発的行動 | セットアップ複雑性 |
| Utility | リソース管理 | 継続的適応 | バランスチューニング |
| Behavior Tree | 戦術AI | 構造的明快さ | スケーラビリティ |

## ハイブリッド統合戦略

多くのスタジオは複合アプローチを採用:

1. **Utilityシステム** → 全体の優先度を決定
2. **ビヘイビアツリー** → 選択された優先度を実行
3. **知覚システム** → 両レイヤーにコンテキストデータを供給

## 選定フレームワーク

- **創発的・複数ステップ解** → GOAP
- **常時変動する優先度** → Utility Planning
- **構造化・予測可能な行動** → Behavior Trees
- **単純なパズル的ロジック** → Classical Planning
