## インタラクティブストーリーの仕組み

著者: Alan Jack (2011)

### 中心的な主張

**ゲームにおけるナラティブを理解するには、「ナラティブが存在する方法」と「ナラティブが存在しない方法」の両方を理解することが鍵となる。**

プレイヤーが最も記憶に残すのはカットシーンではなく、インタラクティブなゲームプレイの瞬間である（研究知見）。

### 2つのナラティブの在り方

#### ナラティブが「存在する」方法
ゲーム内ナラティブの基本構造:
- **主人公** = プレイヤー（目標を持つ）
- **障害** = ルール・敵対者（目標達成を妨げる）
- **物語** = プレイヤーが目標に向かって行動する過程で自然に生成される

#### 暗示的ナラティブ (Implied Narrative)
映画の「暗示的ナラティブ（見せない要素が想像力を刺激する）」と同様に、ゲームのストーリーはゲーム自体に「存在する」のではなく、**プレイ中にプレイヤーが「創出する」**。

ナラティブはデータベースに書き込まれた情報ではなく、プレイヤーの行動と選択から生まれるもの。

### ポイントベースナラティブ (Point-Based Storytelling)

開発者が「1つの統一的な物語」を作成するのではなく、**複数の小さな物語（ポイント）を作成し、プレイヤーが自分の行動を通じて全体的なナラティブを構築する**手法。

```
従来のアプローチ:
開発者 → 一本道の統一ストーリー → プレイヤーが体験

ポイントベースアプローチ:
開発者 → 多数の小さな物語ポイント → プレイヤーが自由に結びつけ → 個別のナラティブ体験
```

### 具体例: Fallout: New Vegas

- プレイヤーを曖昧なプロンプトで開放世界に解放
- 「疎に繋がったイベント間を移動させる」設計
- 統一されたストーリーの幻想をプレイヤー自身が構築
- 各プレイヤーが「自分だけの物語」として記憶する

### 設計上の示唆

1. **ゲームプレイにナラティブを委ねる**: カットシーンへの依存を減らし、インタラクティブな瞬間にナラティブの重心を置く
2. **空白を意図的に設計する**: すべてを説明せず、プレイヤーの解釈余地を残す「暗示的ナラティブ」空間を設ける
3. **ポイントの密度と接続性**: 個々のナラティブポイントの質と、ポイント間の有機的なつながりの両方を設計する
4. **プレイヤーを主人公にする**: 主人公の行動をプレイヤーが選択することで、感情的関与が生まれる
