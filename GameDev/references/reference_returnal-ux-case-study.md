## 1. ゲームプレイファーストUI原則

### コア哲学
- ゲームプレイ重視を設計の最優先事項に
- メニューは「最小限でクリーン」に保つ
- **タイトル画面・メインメニューを廃止**し、直接アクションへ

### HUD情報設計
- 重要情報は**レティクル周辺**の即座の視認範囲に配置
- ゲーム世界内に**ディエジェティック表示**を統合
- 非必須情報は画面周辺に配置

### 危機管理コミュニケーション
低体力状態では複数感覚を動員:
- ヘルメットガラス破損アニメーション
- デジタルグリッチ効果
- 警告音声・VFX
- ハプティックフィードバック

## 2. 3Dマップシステム

- 従来的な2Dコンパスでは不十分（高い垂直性レベル設計）
- **正確性・視覚的一貫性・パフォーマンス**の3要素を両立
- プレイヤーナビゲーション + 戦闘フロー計画をサポート

## 3. オーディオ・ハプティクス統合

| 機能 | 実装 |
|------|------|
| 武器チャージ | 上昇する充電音 → 満充電で独特なUI音 + ハプティックパルス |
| Malignantアイテム | 近接時に「心臓鼓動」的なハプティックパルス |
| パラサイト装着 | 独特なハプティックテクスチャ + 音声キュー |
| スキャナー | ホログラム投影 + ハプティック + 効果音 |

## 4. レトロフューチャリスティック美学

### デザイン背景
- キャラクター「Selene」の会社・利用可能技術を反映
- **厳密に工業的で機能的**な表現

### ビジュアル戦略
- 1970年代美学（科学恐怖映画の影響）
- CRTモニター・夜間視野機器からの「幽霊的な青緑色」
- 損傷・グリッチエフェクトで劣化感を演出

### UI制限原則
- 最小限の方向性モーション
- フリッカー点滅のみ
- **層状構造なし（ポップアップ排除）**
- 厳密な色パレット

## 5. アクセシビリティ・カスタマイズ

### コントローラー設定
- オプションメニューの最初にコントローラーマップ表示
- 直感的なリマッピング + リアルタイムプレビュー

### 色覚多様性対応
- 色盲モード3種（Deuteranopia/Protanopia/Tritanopia）
- **強度スライダー**による微調整
- アイテムビーム色の代替パレット

### サブタイトル
- サイズ・色・言語・話者名・背景不透明度調整可能
- メニュー内プレビュー機能

### 包含的設計方針
- 「専用アクセシビリティカテゴリ」を設けず、**全プレイヤー向け設定**として統合
