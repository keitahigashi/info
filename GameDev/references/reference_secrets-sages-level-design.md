## 核心設計原則（業界知見集）

### フォーカスと継続性（Tim Willits, id Software）
各レベルは明確な単一の目標を持つべきで、技術的変化や制作プレッシャーによってその目標を見失わないことが重要。

### リスクと報酬のサイクル
新エリアへ進むたびに課題（リスク）を配置し、重要に見えるエリアには必ず報酬を用意する。「重要そうに見えて何もない」空間はプレイヤーの期待を裏切る。

### 環境的雰囲気とビジュアルテーザー (visual teaser)
- レベルはプレイヤーの期待に合致した見た目を持つべき
- 窓越しに見える到達不可能なエリアなど「より広い世界が存在すること」を示す視覚的誘惑が効果的

### ペーシング（Cliff Bleszinski, Epic MegaGames）
> 「恐怖よりペーシングの方が重要。長い静寂の後に突然の脅威が来ると、プレイヤーは装備が整う」

### 目標・ゲームプレイ・雰囲気の三角形（Marc Laidlaw, Valve Software）
Half-Life では各レベルにこの3要素を一貫して統合した。

## Paul Jaquaysの26のレベルデザインTips（主要項目）

| カテゴリ | Tips |
|---------|------|
| プリプロダクション | スケッチを作成し、大きな幾何学的ブロックでブロックアウトしてからディテールを追加 |
| 再利用性 | モジュール設計で再利用可能なアーキテクチャピースを作成 |
| 反復 | マップを頻繁にコンパイルして問題を早期に発見 |
| フォーカス | シングルプレイヤーマップには全てのゲーム機能を含めない |
| 方向感覚 | 大規模マップに異なるロケーションを配置してプレイヤーのナビゲーションを支援 |
| AI | 敵の巡回動作や自然なAI行動を表現 |
| 視認性 | プレイヤーを完全な暗闇で行動させない |

## デスマッチマップの5様式

| 様式 | 特徴 |
|------|------|
| アリーナ型 | 中央エリアで戦闘集中 |
| サーキュラー型 | 折り返し不要の循環設計 |
| リニア型 | 少数の代替経路を持つ道順型 |
| ロケーション型 | ユニークで識別可能なエリア |
| テーマ中心型 | 特定要素（水・溶岩など）を強調 |

## 相互作用性の設計（The Levelord）

現代の FPS は単なる射撃ゲームではなく「相互作用性の高い環境」。
- 破壊可能オブジェクト
- パズル解法
- 隠しエリアの発見

これらがプレイヤーに能動的な発見体験を提供する。

## 認証性 (authenticity) の重要性

プレイヤーの没入感は現実感に大きく依存する。一貫したテーマと現実的要素の組み合わせが必要。内部的に整合性のある世界観設計が長期的な没入を支える。

## 汎用化ポイント

1999年の記事だが、「明確な目標・リスクと報酬・ペーシング・モジュール設計」の原則は現代のどのジャンルにも適用可能。特に「ビジュアルテーザー」と「サーキュラー動線」は現代のオープンワールド・ダンジョン設計でも核心的概念として生きている。
