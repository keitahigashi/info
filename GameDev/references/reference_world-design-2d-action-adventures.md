## ジャンル定義

アクションアドベンチャーは「物理的なナビゲーションと戦闘（アクション要素）」と「詳細なナラティブ・キャラクター・インベントリ・ダイアログ（アドベンチャー要素）」の融合ジャンル。サバイバルホラー・ソウルズライク・メトロイドヴァニアを包含する広義の分類。

## ジャンル進化の歴史

| 時期 | 出来事 |
|------|-------|
| 1970年代 | Pong・Space Invaders: 物理的衝突ベースゲームプレイの確立 |
| 1976-1980 | Colossal Cave Adventure・Zork: テキストアドベンチャーによるナラティブ主導インタラクションの確立 |
| 1979-1980 | Superman (1979)・Adventure (1980): アクションとインベントリの融合、ジャンル誕生 |
| 1986 | The Legend of Zelda: 探索可能なオーバーワールド・ダンジョン・アイテムベース進行・ボス戦の確立 |
| 1986 | Metroid: プラットフォームと探索・能力ベース進行の融合 |
| 1994 | Super Metroid: スキルゲート設計と環境ストーリーテリングの完成形 |
| 1997 | Castlevania: Symphony of the Night: RPG要素の統合 |
| 2000年代〜 | Cave Story・Hollow Knight等インディ作品がジャンルを継承・進化 |

### ジャンル結晶化の要因（Zelda 1986）

1. 探索可能なオーバーワールド
2. 隠されたダンジョン（アンダーワールド）
3. アイテムベースの進行（特定アイテムでボスを攻略）
4. 永続的な世界状態（解錠された扉・倒した敵が維持される）
5. プレイヤーの能力拡大による後退的探索

## ワールドデザインの8核心要素

| # | 要素 | 説明 |
|---|------|------|
| 1 | **視点選択（トップダウン vs サイドスクロール）** | トップダウンは広大な探索に優れるが垂直深度の表現が難しい。サイドスクロールは垂直メカニクスを活かせるが視覚的単調さに注意 |
| 2 | **能力ベース進行** | 新能力の取得がマップの新領域へのアクセスを解除する。アイテム追跡と明示的な設計シグナルが必須 |
| 3 | **非線形設計** | 必須経路とオプション経路を組み合わせた分岐構造。線形進行トラップを防ぐ |
| 4 | **エンカウンター設計** | マクロ（世界構造）からミクロ（個室・スクリーン）へのスケール切替が必要 |
| 5 | **永続的環境** | プレイヤーの行動（解錠・クリア）が世界に記録される。繰り返しリソース消費を防ぐ |
| 6 | **探索の誘惑** | アイテム配置と経路制限で発見意欲を喚起。「見えるが届かない目標」が動機を生む |
| 7 | **制御されたバックトラック** | 不可避だが単調にならないよう、ショートカット・ファストトラベル・多様な経路で管理 |
| 8 | **環境ナラティブ** | ダイアログではなく環境配置でストーリーを伝える |

## 構造パターン

### オーバーワールド / アンダーワールド

英雄文学（ギルガメシュ・オデュッセイア等）に根ざした古典的構造。

- **オーバーワールド**: 広大な探索空間、NPC・乗り物・キャラクター間相互作用
- **アンダーワールド（ダンジョン）**: 焦点化された密度の高いチャレンジ

### ハブアンドスポーク (Hub-and-Spoke)

Wonder Boy III が開拓した構造。中央ハブから複数のスポーク領域に分岐し、能力取得とともに新領域が解放される。

### スキルゲート設計（Super Metroid の完成形）

アイテム取得直後に「なぜそのアイテムが必要か」を即座に実証する空間設計。高ジャンプブーツ取得→その場で高い場所に到達→以前入れなかった部屋へ誘導。脱出経路を事前に組み込み、デザイナーの意図を体験で学ばせる。

## 制約ベース設計の利点

GB Studioを実習エンジンに採用した理由:
- 16×16ピクセルスプライト・4色パレットという制約が「技術的複雑さではなくメカニクスの反復改善」を強制
- 小規模プロジェクトを迅速に完成させられる → デザイナーの自信形成
- 制約の中でのクリエイティビティがインディ開発の現実に即している

> 「制約は創造性の敵ではなく、必要なメカニクスに集中させる力だ」

## 設計実践チェックリスト

- [ ] アビリティゲートの地図とアンロック順序を事前に計画したか
- [ ] スキルゲート設置直後に「なぜそのアビリティが必要か」を体験で示せているか
- [ ] バックトラックエリアにショートカットまたは新ルートを設けたか
- [ ] 非線形探索で詰まるプレイヤーへの環境的ヒントを用意したか
- [ ] 永続的状態（解錠・収集）を確実に記録する仕組みがあるか
- [ ] 複数のデザイナーが協業する場合、共通のワールド構造ドキュメントがあるか
