## 概要

JellyCar Worlds開発者Tim FitzRandolph（@walaber）によるソフトボディ物理エンジンの設計・実装詳解（2022年12月）。ポイントマス・スプリング・形状マッチング (shape matching) を段階的に積み重ねることで、変形・圧縮・回復が可能な物体を実現する手法をエンジン非依存で解説する。

## コアアーキテクチャ

```
ポイントマス群 ─→ スプリング力計算 ─→ 形状マッチング ─→ 衝突検出 ─→ 位置更新
     ↑______________ガス圧力フィードバック_________________________|
```

## 主要コンポーネント

| 要素 | 役割 |
|------|------|
| ポイントマス (Point Mass) | 物体を構成する質点。重力・力を蓄積し速度・位置を更新 |
| エッジスプリング | 物体外周の形状を定義するバネ |
| 内部スプリング | 構造的サポートと形状維持 |
| 形状マッチング | 不可視の剛体フレームで変形後の元形状への復元力を付与 |
| ガス圧力 | タイヤ・風船などの膨張シミュレーション（体積偏差に比例）|

## 衝突検出アルゴリズム

**レイキャスト法（点-多角形判定）**
- テスト点から外部の固定点へ線を引き、対象多角形との交差回数を数える
- 奇数回交差 = 点は多角形内部
- A→B、B→Aの双方向テストで相互貫通を防止

## 形状マッチングの重要性

「内側反転問題」（圧縮バネが薄い物体を不安定化する現象）の解決策：
1. ポイント群の平均位置・角度を算出して不可視の剛体フレームを配置
2. フレームが変形形状に追従し、各点に元の位置への復元バネ力を付与
3. 細長い形状には複数フレームを分割配置してローカル変形に対応

## ジョイントシステム

- **溶接ジョイント (Weld Joint)**：複数オブジェクト間の点マスを直接固定接続
- **ピンジョイント (Pin Joint)**：加重平均アンカーによる柔軟な接続点（車体と車輪の接続等）

## 実装上の注意点

- スプリング力に上限クランプを設定し、過剰変形時の非現実的な力を防ぐ
- 衝突解決はリジッドリポジショニングではなく「ソフトな変形」を活用
- コードより概念理解を優先した解説で、独自物理エンジン実装の出発点として参照可能
