## UIアクセシビリティ実践チェックリスト2026

### 5大チェック領域

| 領域 | 基準値 / 要件 |
|---|---|
| **色・コントラスト** | 通常テキスト 4.5:1 以上のコントラスト比。男性の8%が色覚異常を持つため、色のみへの依存を禁止 |
| **タイポグラフィ** | 本文フォント最低16px・行間1.5倍。世界人口の15%が難読症を抱える点を考慮 |
| **インタラクティブ要素** | タッチターゲット最低44×44px。フォーカス状態の可視化必須。エラーメッセージは具体的に |
| **モーション・アニメーション** | `prefers-reduced-motion` を尊重。コンテンツが毎秒3回以上点滅することを禁止（発作リスク） |
| **ナビゲーション・構造** | 見出し階層を飛ばさない。キーボードのみで全操作を可能にする |

### 30分テストプロトコル

1. コントラストチェック（5分）
2. キーボードナビゲーション確認（10分）
3. スクリーンリーダーテスト（10分）
4. 200%ズームテスト（3分）
5. 視覚的ヒエラルキー評価（1分）

### 設計者の典型的ミス

- フォーカスをトラップするモーダルダイアログ
- カスタムコンポーネントへのARIAラベル漏れ
- アイコンのみのボタン（テキスト代替なし）
- ダークモードでのコントラスト要件見落とし

### インクルーシブデザインの観点

障害を持つユーザーだけでなく一時的障害（片手使用・騒音環境）も包含。

### ゲームUIへの適用

- HUD要素・インベントリ画面・設定メニューで4.5:1コントラスト確保
- コントローラー操作時のフォーカスリング可視化
- モーション酔い対策としての揺れ演出オプション化
- 色覚異常モード・コントロール再マッピング・字幕オプションを設計段階から組み込む
