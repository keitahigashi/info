## 基本概念

### 感情の定義
性別、年齢、文化的背景に関わらず、全ての人で同じ方法で引き起こされ表現される生理的反応。5つのコア感情: 怒り、悲しみ、喜び、恐怖、嫌悪感。

### 感情 vs 感情状態
| 概念 | 特徴 | 持続時間 |
|------|------|---------|
| **感情 (Emotion)** | 短時間の生理的反応 | 秒〜分 |
| **感情状態 (Feeling)** | 意識的経験に基づく（知識・過去の経験含む） | 分〜時間 |

## 感情設計の6ステップ実装手順

1. プレイヤーに伝えたい感情を決定
2. ゲームジャンルとプレイヤーニーズに合致させる
3. 感情を喚起する場面を選定
4. その瞬間の感情を特定
5. 補完可能な感情を把握
6. メカニクスを追加・カスタマイズ

## 理論1: マズローの欲求階層説

プレイヤーが各階層レベルで期待する感情をリスト化し、望ましくない感情も並列で記録。

**Minecraftの例**: 全階層をカバーすることで広範な層に対応（安全欲求→自己実現まで）。

## 理論2: プラチック心理進化理論

感情トリガーのメカニクス選定に活用。5段階プロセス:
刺激 → 認識 → 感情 → 反応 → 期待

### 各感情のトリガー設計

| 感情 | トリガー条件 | ゲーム適用例 |
|------|-----------|------------|
| **恐怖** | 危険で対抗不可な脅威 | 強敵、環境変化 |
| **怒り** | 倒せる敵対対象 | 破壊可能な障害物 |
| **喜び** | リソース獲得 | アイテム、友好関係 |
| **悲しみ** | リソース喪失 | コスト支払い |
| **信頼** | 長期的友好関係 | ツールの的確性 |
| **嫌悪感** | 受動的接触 | LD警告要素 |
| **期待/驚き** | 探索活動 | 安全な環境での発見 |

## 理論3: ケラマンとプラチックの感情強度モデル

- 感情を同心円図で可視化（中心=高強度、周辺=低強度）
- 対極感情の組み合わせ（悲嘆↔歓喜、恐怖↔激怒）
- **複数感情の混合**で具体的な感情状態を予測

**F2Pの例**: ポップアップオファー = 怒り（中断） + 喜び（有利オファー） → 「期待感」

## 重要な設計原則

1. **感情の過負荷回避**: プレイヤーの感情的負担を監視
2. **期待と現実のバランス**: 常に期待を破裂させると挫折
3. **ニュートラル局面の必要性**: 継続的な高感情状態は避ける
