## ハイブリッドカジュアルゲームの設計原則

### ジャンルの定義

**ハイブリッドカジュアル (hybrid-casual)**: ハイパーカジュアルゲームのアクセシビリティに、より深いエンゲージメントのメカニクスを組み合わせたジャンル。

- ハイパーカジュアル: 広告収益主体、短ライフサイクル
- ハイブリッドカジュアル: IAP (アプリ内課金) + 広告の併用、長期 LTV (顧客生涯価値) を狙う

### 原則1: 強固で楽しいコアループを構築する

Tower Destiny のコアループ設計例:
1. エネルギー消費 → ゾンビを倒しながら前進
2. 各種パラメータをアップグレード
3. 強化された状態で戻り、より遠くまで進む
4. 他のメタゲーム層（チャレンジ・クエスト）を探索

このループの設計ポイント:
- UI が進捗を視覚的に表現（タワーの外観変化・移動距離の数値）
- 短期目標（レベルクリア）と長期目標（ボス撃破）の組み合わせ
- 複数のアップグレードと装備でリプレイアビリティ (replayability) を確保

**ゲームフィール (juicy game feel)**: 美しいグラフィック、すべてのインタラクションへのアニメーション、主要アクションへの視覚的報酬が中毒性を生む。

### 原則2: 多様で一貫したメカニクスを段階的に導入する

Wasteland Life のメカニクス導入設計:

| ステージ | メカニクス | プレイヤーの学習 |
|---------|-----------|----------------|
| 初期 | 砂を掃除機で除去 | ゲームの基本動作理解 |
| 中期 | ツルハシで物を壊す | 2つ目のツールと進化軸の解放 |
| 後期 | 両メカニクスの組み合わせ | 探索・収集の複合的アクション |

設計の鉄則:
- 新しいメカニクスは既存ゲームデザインとテーマに沿っていること
- 一度に多くのメカニクスを開放すると KPI が悪化する
- 各メカニクスが新しい進行軸（progression axis）を提供する

### 原則3: シームレスなマネタイズ設計

Tower War のマネタイズ導入タイムライン:
- レベル 1-4: メカニクス学習に集中、広告なし
- レベル 4: フィーバーモードを無料体験させる
- レベル 6: IAP セクションを開放（実際に使う準備が整った頃）
- 以降: 広告・IAP を自然なゲームフローに組み込む

目標: 「フロー状態 (flow state) を維持しながら収益化する」

### 原則4: バランス調整とゲーム深度

シンプルさと複雑さのバランスこそがリテンション向上の鍵:
- プレイヤーを圧倒せず、しかし浅すぎず
- 深度 (depth) = 同じシステムで発見し続けられる層の厚み
- 追加メカニクスは選択肢を増やすが、コアファンタジーを損なわないこと

### 汎用化ポイント

ハイブリッドカジュアルの設計原則はジャンルを超えて適用可能:
- 「コアループ → 段階的複雑化 → 報酬とゲームフィール」の構造はあらゆるゲームに通じる
- マネタイズ位置の設計はエンゲージメントカーブ設計と同じ原理
- プレイヤーの「準備が整った」タイミングでシステムを開放する点は、チュートリアル設計にも応用できる
