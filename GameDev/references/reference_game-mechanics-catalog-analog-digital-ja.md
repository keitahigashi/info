### 2. ゲーム設計の基本要素

**勝利条件の種類**

- 勝利点の最多獲得
- 早抜け（先着ゴール）
- エリアマジョリティ（最多配置）
- セットコレクション（役の完成）
- 隠情報の推測成功
- 全員脱落型（排除式）

**リプレイ性の確保手法**

- カード・能力・役割の組み合わせをゲームごとに変更
- ステージ構築式（カルカソンヌ型）
- 会話・交渉主体設計

---

### 3. メカニクスパターン詳細

**相互作用パターン**

- 手札交換、隠匿情報の推理とブラフ
- 共有リソースの奪い合い、投票、交渉・競り

**ジレンマ/トレードオフ設計原則**

> **「最適解が存在してはならない」**

- ハイリスク・ハイリターン
- リソース配分の最適化困難性
- 三すくみパターン

**コスト/制約体系**

- 高コスト/高効果（対称投資）
- フードサプライ（継続運用コスト）
- 使用回数制限、条件付きアクティベーション
- クールダウン、トレードオフ（AするとBできない）

---

### 4. ビデオゲームメカニクスへの展開

**アナログとの主な差分**

- コンピュータが複雑な計算を代行 → 実現可能なメカニクスの幅が大幅に拡大
- 非公開情報の自動管理（ゲームマスター不要）
- シングルプレイヤーでは「サービス設計」が可能（対人ジレンマを意図的に破壊して爽快感を提供）

**主要ジャンル系統**

| カテゴリ | ジャンル |
|---|---|
| アクション系 | アクションアドベンチャー・プラットフォーマー・ステルス・格闘 |
| シューター系 | FPS/TPS・シューティング |
| RPG系 | JRPG・MMO・ハック&スラッシュ |
| ストラテジー系 | TBS・SRPG・RTS・タワーディフェンス |
| シミュレーション系 | リソース管理・ライフシム・恋愛・ペット |
| その他 | パズル・リズム・レース・パーティ |

**ランダム性の実装比較**

| 方式 | 特性 | 向き |
|---|---|---|
| ダイスロール | 正規分布収束、試行ごとに独立 | 即興性・スリル |
| 山札方式（抽選箱型） | 期待値を保証、ドライラン防止 | 長期ゲーム |

**カードメカニクス活用**

- ドラフト選択：プレイヤーが能動的に選択
- デッキ構築：進化する意思決定システム
- 非公開情報活用：読み合い・心理戦
