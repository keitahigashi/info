## 核心的テーゼ

スキルツリーの普及は正当性を保証しない。各ゲームが意図的な設計でこのメカニクスを「獲得」すべき。多くは未検討の実装により、潜在的な設計可能性を浪費している。

## 7つの設計原則

### Key #1: 意味ある選択のコミットメント
プレイスタイル発展のための影響力ある長期的選択を強制。些細なステータス調整ではなく、ゲームプレイ方向を実質的に変える決定。

### Key #2: プレイヤーエンパワーメント
自己表現を促進し、キャラクターへの投影を支援。定型アーキタイプではなく自己同一化を促進。

### Key #3: ゲームプレイ強化
特定プレイスタイルを強化し、ゲームプレイを開放し、表現の自由度を拡大。

### Key #4: バランスの取れたスコープ
選択麻痺を避けつつ明確な特化パスを開く。15〜40ノードが中規模ゲームに適切。

### Key #5: リスペック戦略
高コストの完全リスペック → 取り返しのつかないミスからの回復を許容しつつ、投資の結果を維持。

### Key #6: 完了哲学
非RPGでは: キャンペーン中は特化を強制 → クリア後にフルツリー完了を解放。ハード選択とコンプリーショニスト満足の両立。

### Key #7: イノベーション必須
未開拓領域への冒険を推奨。既存実装の複製ではなく実験を。

## スキル品質評価フレームワーク

| 品質 | 特徴 | 例 |
|------|------|-----|
| **Bad** | ゲームプレイパターン/行動変化なし | 「+5% XP獲得」 |
| **OK** | 機械的制限を解除、ツール使用範囲を微拡大 | 「エリート敵を暗殺可能」 |
| **Good** | ゲームプレイを再発明、新メカニクス導入、ファンタジー表現 | 「倒した敵を召喚して味方に」 |

### 動詞テスト (The Verb Test)

- 良いスキルは**動詞を含む**
- 素晴らしいスキルは**ユニークな動詞を含む**
- アクション指向の記述 = 機械的実体の指標
- ユニークな動詞 = ステータス反復ではなく真に新しいメカニクス

## アンチパターン

### 極端1: コア能力のゲーティング
基本メカニクス（移動オプション/基本攻撃）をスキルツリーに制限 → フラストレーションを生む快適機能の否定。

### 極端2: 使用不能な特殊性
Far Cry 3の「下から二重暗殺」: 使用条件が極めて特殊で、ゲーム内にその条件が発生するシナリオが存在しない。開発コストのみでプレイヤーエンゲージメントゼロ。

## 実践的分析

| ゲーム | 意味あるスキル率 |
|--------|----------------|
| Control | 17% — 大半がゲームプレイ影響なしのステータス調整 |
| AC: Origins | 76% — 能力がプレイヤーアプローチを根本的に変更 |

## 未開拓設計領域

- 不可視/ランダム化スキル発見（リスクテイク促進）
- ネガティブ結果スキル（正のボーナスとのバランス）
- 「毒を選べ」ツリー（プレイヤー脆弱性選択の強制）
- 相互連結/ルーンベースのスキル関係
- 曖昧な記述（プレイヤー実験と解読の要求）
