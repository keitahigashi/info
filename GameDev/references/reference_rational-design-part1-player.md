## 設計哲学の基本方針

> 「ゲーム設計とは"良いアイデアを生み出すこと"ではなく、"ユーザーのニーズに応える解決策を創造すること"」

デザイナー自身の好みではなく、プレイヤーの深い理解を出発点とする。

## プレイヤーモデリングの4フレームワーク

### 1. バートルのプレイヤータイプ (Bartle, 1996)
| タイプ | 行動傾向 |
|--------|---------|
| Killers | 他プレイヤーとの競争・支配 |
| Achievers | 目標達成・コレクション |
| Explorers | 世界探索・発見 |
| Socialisers | 他プレイヤーとの交流 |

各機能設計時に「どのタイプに向けているか」を意図的に検討する。

### 2. ラザーロの4つの楽しみの鍵 (Lazarro, 2004)
| カテゴリ | 内容 |
|---------|------|
| Hard Fun | 困難な目標への挑戦・達成感 |
| Easy Fun | 好奇心・探索・空想 |
| Altered States | 感情・感覚の変容体験 |
| The People | 協力・競争・社会的つながり |

プレイセッション中にプレイヤーは複数モードを行き来する。最適設計は3種以上の楽しみを備えること。

### 3. マズローのニーズ階層 (Maslow, 1954)
- 基本的欲求から高次の自己実現へ進行する順序性
- 機能の実装優先順位付けに応用（例：ギルドシステムより先にヘルスシステムを教える）

### 4. その他の参考モデル
- ビッグファイブ性格特性
- ライス16の基本欲求
- フィッシャーの性格タイプシステム

## フロー理論の実装

### マイクロフロー（瞬間的没入）の3条件
1. 明確な目標と進捗が見える構造
2. 即座かつ明確なフィードバック
3. スキルと課題のバランス

**構築の3要素:**
- 入力パターンをリズミカルな「スコア」として設計
- 成功の連続を促進し安心感を提供
- 段階的な正強化フィードバック（「Nice!」→「+10XP」など）

### マクロフロー（長期的進行）
プレイヤーのスキル向上に合わせて難度を調整し、退屈と不安の間の「フローチャネル (flow channel)」を維持する。

## O.C.R. ループフレームワーク

各ループを以下で定義する:

| 要素 | 内容 |
|------|------|
| Objective | プレイヤーが達成を目指すもの |
| Challenge | 必要な機械的スキル |
| Reward | 成就時の報酬 |

時間軸で分類: 短期（秒単位）→ 中期（分単位）→ 長期（時間単位）

ループを視覚化して入れ子構造を示すグラフを作成し、動機づけ戦略の全体像を評価する。

## 重要な注意点

- どのモデルも完全ではない。複数モデルの組み合わせが有効
- 「最善のモデル」探しよりも各モデルの強みを活用する姿勢が重要
- DDA（動的難度調整）は制作短縮時に削られやすいが、意図的に設計すべき重要要素
