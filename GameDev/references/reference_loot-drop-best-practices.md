## ルートテーブルの基本構造 (Loot Table Structure)

| 要素 | 説明 |
|------|------|
| **アイテム** | プレイヤーに付与するもの |
| **ウェイト (weight)** | 相対確率（1〜10,000の整数）|
| **ロール機構** | 合計ウェイトから範囲を割り当て、乱数で選別 |

例: 剣（ウェイト10）/ 盾（40）/ なし（50）→ 合計100で範囲分割

## 5つのバリエーション (Loot Table Variations)

1. **置換なしロール (Roll Without Replacement)** — Tetrisのように、選ばれたアイテムの出現確率を減少させ最終的に全種獲得を保証する
2. **確定ドロップ (Guaranteed Drop)** — N回のロール後に必ずアイテムを獲得させる天井 (pity) 機構
3. **階層型テーブル (Layered Table)** — 「日次コイン上限100枚」など複数制約を重ねるリソース管理
4. **条件付きドロップ (Conditional Drop)** — プレイヤーレベル・与ダメージ量などを検証してから抽選する
5. **修飾子 (Modifier)** — スキルに応じてドロップ量や確率に倍率をかける

## 設計原則

- **複雑さの最小化**: プレイヤー体験に必要な最低限の精度のみ実装する
- 手続き型生成・AI行動選択などにも応用可能な「確率的システムの第一段階モデル」
- 小さいテーブルを組み合わせてモジュール設計するほうがデバッグが容易

## 汎用化ポイント

ルートテーブルの構造はアイテムドロップに限らず、敵出現パターン・クエスト報酬・AIの行動選択など、あらゆる「重み付き確率選択」に転用できる。
