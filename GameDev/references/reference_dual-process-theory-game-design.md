## デュアルプロセス理論とは

ノーベル賞受賞者ダニエル・カーネマン（Daniel Kahneman）が提唱した人間の意思決定の2つのモード。

| | System 1（直感的思考） | System 2（論理的思考） |
|--|--|--|
| 処理方式 | 無意識・自動的 | 意図的・制御可能 |
| 速度 | 高速 | 低速 |
| 基盤 | 直感・gut feeling | 理由・論理 |
| 精度 | 誤りやすい | より正確 |
| 努力 | 不要 | 注意力が必要 |
| 適した状況 | 選択肢が多い・時間が短い | 選択肢を絞り込んだ後 |

## ゲームデザインへの応用

### ジャンル別の活用戦略

**ストラテジー・シティビルダー**

- レイヤード化されたメニューで複雑な選択を段階的に提示する
- System 1 でカテゴリを選択 → System 2 で最終決定する2段階設計
- Tropico 4 の建物カテゴリ分類が好例：認知負荷を軽減しながら戦略的思考を促す

**パズルゲーム**

- System 2 思考を重視した設計
- 時間制限は慎重に設定する（時間圧があると System 1 に切り替わる）
- 直感的な手がかりも用意し System 1 と System 2 を行き来できるようにする

**ドレスアップ・カジュアル**

- 豊富な選択肢で創造性を支援しながらメニュー階層化で過剰感を緩和する
- ランダムボタンなどの「System 1 トリック」で選択疲れを防ぐ

### デザイナーへの推奨事項

1. **時間圧のある場面**では System 1 を活かす設計（直感的操作）
2. **複雑な戦略ゲーム**ではレイヤード決定システムを導入する
3. **アドバイザー機能**や提案表示で直感的判断をサポートする
4. **System 1 で下す決定**は可逆性を確保する（取り消し・やり直し）
5. ゲームジャンルに応じて System 1 / System 2 のバランスを意識して設計する

## 認知負荷（cognitive load）との関係

選択肢の過多は System 2 の処理能力を超えて決定疲れ（decision fatigue）を引き起こす。段階的な情報提示・カテゴリ化・デフォルト設定はすべて「System 2 の負荷を System 1 に移譲する」設計パターンとして理解できる。
