## パラダイムはツールであり、目的ではない

### 「パラダイムの罠 (Paradigm Trap)」

デザイナーが特定の設計手法（オープンワールド・サイドスクローラーなど）を最終目標として設定すると：

- 予算・人員・時間の実現可能性の検討が後退する
- プロジェクト全体が制御不能になるリスクが生まれる
- 「このパラダイムで作りたい」という出発点は罠になる

### 「ツールキット解法 (Toolkit Solution)」

利用可能なリソース（時間・予算・人員）を起点に、最適な設計パラダイムを選択・組み合わせるアプローチ：

```
リソース制約を明確化
  ↓
参照ゲームから要素を選別
  ↓
各要素の開発コストを評価
  ↓
スコープ内で完成可能な組み合わせを決定
```

## アクション RPG 設計の具体例

著者は新規アクション RPG の設計プロセスを事例として詳述する。

### 参照ゲームと選択結果

| ゲーム | 検討した要素 | 採用/不採用 | 理由 |
|--------|------------|-----------|------|
| Dark Souls 3 | 戦闘システム | **採用** | スコープ内で実現可能 |
| The Witcher 3 | 探索要素（ノヴィグラド規模のオープンワールド） | **不採用** | 開発 3〜5 年・300 人・数千万ドルが必要 |
| Breath of the Wild | パズル解法・マップの段階的開示 | **採用** | 空間設計のアプローチが応用可能 |
| Path of Exile | プレイヤーの自由度・能力ツリー | **採用** | スコープ内で実現可能 |

### 最終選択：メトロイドヴァニア設計

ダークソウル的な相互接続型ハンドクラフトレベル設計を選択した理由：

1. 開発コストがオープンワールドより大幅に低い
2. すべてのレベル要素がゲームプレイに貢献する
3. スコープ管理が容易

## オープンワールドのコスト分析

**The Witcher 3 のノヴィグラド都市を作る場合の概算コスト：**
- 開発期間：3〜5 年
- チーム規模：300 人程度
- 予算：数千万ドル規模

→ インディー開発には非現実的。「オープンワールドを作りたい」という出発点は多くの場合スコープクリープの原因になる。

## 5つの設計原則

1. **デザインパラダイムは手段であり、目的ではない**
2. **パラダイムを活用しつつ、その支配を受けない**
3. **実際のリソース制約を設計に反映させる**
4. **スコープ内で完成可能なゲームを目指す**
5. **既存の優れた作品から学ぶことは有効（模倣ではなく応用）**

## 創造的プロセスの推奨フロー

```
1. 既知の要素の組み合わせから出発
2. 各要素の開発コストを見積もる
3. リソースを超える要素を削除
4. 残った組み合わせでプロトタイプを作成
5. 独自性は削除プロセスの中で自然に生まれる
```

> 「完成可能なゲームを作ることが、まず最初の成功条件」という実用的な視点がこの記事の核心。
