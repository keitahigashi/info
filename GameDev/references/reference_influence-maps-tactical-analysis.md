## 定義と目的

AIシステムがゲーム世界を理解・操作するための空間表現。世界データを読み取り可能なフォーマットで表示し、自律エージェントに**行動可能なインテリジェンス**を提供。「戦術分析 (tactical analysis)」とも呼ばれる。

## 主な応用領域

| 領域 | 説明 |
|------|------|
| 地形分析 | 通行困難エリア・視界ゾーンの特定 |
| 敵領土マッピング | RTS弱点分析・戦略的進攻ルート |
| 分隊戦術 | FPS含むチーム移動・ポジショニング協調 |

## 3つの拡散手法

### 1. Limited Radius of Effect（限定半径効果）

影響が基点ノードから外側に拡散し減少:

```
influence = max_value - (max_value * (distance / max_distance))
```

### 2. Gaussian Convolution Filtering（ガウシアン畳み込み）

グラフィクスベースのアプローチ。セルが周囲の影響によってカーネルマッピングで重み付け。

### 3. Map Flooding（マップフラッディング）

最も計算効率が高い。影響が最大値で均一に拡散。ただし戦術的リスクあり（精密さに欠ける）。

### 実装例（GameMaker・ノードベース）

3層のマップをレイヤリング:

| レイヤー | 内容 | 更新頻度 |
|---------|------|---------|
| 地形影響 | 狭い通路を不利としてマーク | 静的（レベル初期化時1回） |
| 危険ゾーン | プレイヤー近接からの伝播（1ノード外側） | 動的 |
| 視界マップ | プレイヤー可視ノードに均一影響 | 低頻度（最適化のため） |

## パフォーマンス最適化

| 手法 | 説明 |
|------|------|
| 静的プロパティの事前計算 | レベル初期化時に1回だけ計算 |
| 中断可能な更新 | ゆっくり変化する影響は処理を中断可能に |
| フレーム分散 | 動的マップをフレームごとに1つずつ更新 |
| 差分更新 | 変化したノードのみ再計算 |

## 更新頻度ガイドライン

| マップタイプ | 推奨Hz |
|------------|--------|
| 戦略マップ（全体状況） | 0.5〜1 Hz |
| 戦術マップ（個別AI） | 2〜5 Hz |
