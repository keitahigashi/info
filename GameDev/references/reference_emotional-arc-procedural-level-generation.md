## 出典

arXiv (2025年8月): https://arxiv.org/abs/2508.02132

## 感情アーク（Emotional Arc）の普遍性

Kurt Vonnegutの物語構造研究に由来する感情アーク理論を出発点とする。「Rise（高揚）」と「Fall（緊張）」という2つの基本感情パターンがあらゆる文化・メディアのストーリーに共通して存在するという構造主義ナラティブ論の知見を、ゲームレベル生成に統合する。

この理論的前提は「All Stories Are One Story（すべての物語は一つの物語）」という命題として定式化されており、感情アークを通じてナラティブとゲームプレイを共通の設計言語で扱えることを示唆する。

## フレームワークの設計原理

### 感情アークとゲームプレイの統合
- 各ストーリーノードに対してキャラクター・アイテム・ゲームプレイ属性（体力・攻撃力等）を感情的軌跡に応じて自動生成
- 分岐するストーリーグラフ構造により、プレイヤーの選択に応じて難易度・体験密度が動的に調整される
- 感情アークの「山谷」に合わせた難易度カーブの自動設計が可能

### 設計図としての感情アーク
```
感情的高揚期（Rise）→ エンカウント強度を上げる・新要素を投入
感情的緊張期（Fall） → 探索・回復フェーズ・情報整理の機会を与える
```

## 技術的実装

LLM（大規模言語モデル）とアダプティブエンティティ生成を組み合わせたアクションRPGプロトタイプで実装・検証。ナラティブ計画 (Narrative Planning) とゲームイベント生成を統合したパイプラインを提案する。

ユーザー評価でエンゲージメント・ナラティブ整合性・感情的インパクトが著しく向上したことを確認している。

## 設計者への示唆

### ナラティブと難易度の統合設計
従来はゲームプレイの難易度曲線とナラティブの緊張曲線を独立して設計することが多かったが、感情アークという共通軸を用いることで両者を統合設計できる。これにより体験の一貫性（ルドナラティブ整合性）が向上する。

### 手動設計への応用
プロシージャル生成を使わない手動レベル設計でも、感情アークは以下のように機能する：
- 物語上の緊張の山場に合わせてエンカウントの難易度を設計する
- 物語上の解放・休息フェーズに探索・収集・対話パートを配置する
- キャラクターのアーク（成長・喪失・変化）をゲームプレイの要求変化と連動させる

### プロシージャル生成設計への応用
感情アークをパラメータとして利用することで、文脈に応じた多様なコンテンツを一貫した体験として生成できる。単なるランダム生成ではなく、感情的文脈に基づく「意味のある」バリエーションを実現する。
