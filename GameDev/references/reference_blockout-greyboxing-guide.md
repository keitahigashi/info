## 定義と目的

ブロックアウト（blockmesh / graybox）は、詳細なアートアセットなしにシンプルな幾何形状で構築される3Dレベルのラフドラフト。「安い（修正容易な）」段階で設計を証明し、「高い（最終アート）」段階に移行する。

## 5段階ワークフロー

1. **レイアウトをスケッチ** — シンプルな描画で計画の基盤を確立
2. **グラウンドプレーン・スケールフィギュア・壁を追加** — 空間コンテキストとプレイヤー比率を確立
3. **プレイテスト** — フル重力・コリジョンで空間を歩く（エディタで飛ばない）
4. **分岐してイテレーション** — フィードバックに基づきセクションを再構築
5. **ステップ4を繰り返す**

## 4つの重点領域

| 領域 | 内容 |
|------|------|
| **マッシング (Massing)** | 形状によるボリュームと重みの伝達 |
| **メトリクス (Metrics)** | スケール・寸法・プロポーションの確立 |
| **ウェイファインディング (Wayfinding)** | プレイヤーナビゲーションとマップ構造の学習 |
| **プレイテスト (Playtesting)** | エンカウンター機能・バランス・目的達成の検証 |

## イテレーションの哲学

「99%の場合、ブロックアウトはプレイテストを生き残らない」。部屋の削除・空間の拡大・セクション全体の再構築を受け入れる。Neon White の例: 出荷100+レベルに対し約2倍がボツ、1レベルに3年間で50+の変更。

## 構築手法5種

| 手法 | 特徴 | 適したケース |
|------|------|------------|
| **プリミティブ** | 基本キューブ・ボックスの配置 | シンプルだが制限あり |
| **ブラシ/モデリング** | エディタ内ローポリ形状 | 最大のコントロール |
| **モジュラーキット** | プリメイドピースのスナップ | 初心者推奨 |
| **スカルプティング** | 有機的形状のペイント | 地形・ランドスケープ |
| **スプライン** | パスに沿ったジオメトリ生成 | 道路・河川 |

## 実践的なコツ

- 壁はスケールフィギュアの150-200%の高さ
- 通路・ドアは幅の最低2倍
- マップが小さい → 外壁を拡大してから中央を埋める
- マップが大きい → 不要な部屋を削除・圧縮
- 不安で始められない → まず1部屋だけ作る
