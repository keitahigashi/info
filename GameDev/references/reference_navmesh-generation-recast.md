## Recast Navigation とは

ナビゲーションメッシュ（NavMesh）の構築・ロード・ストリーミング・パスファインディング・AIエージェント制御を自動化するツールセット。2009年にMikko Mononenにより開発。

### 採用エンジン
- Unity
- Unreal Engine
- Godot
- O3DE
- 多数のAAAおよびインディーゲーム

## ナビメッシュの基本概念

ゲームワールドの「走行可能エリアの簡略化されたビュー」。AIパスファインディングに最適化された表現。

- 三角形がグラフのノードとなる
- 隣接ポリゴンのエッジが接続を定義
- **String-Pulling Algorithm**: ポリゴンパスを線分の列に精緻化

## エージェント属性

ナビメッシュは特定のエージェントタイプに合わせてカスタマイズされる:

| 属性 | 説明 | 例 |
|------|------|-----|
| サイズ（半径・高さ） | シリンダー/カプセルで定義 | 人型: r=0.3, h=1.8 |
| 最大勾配 (Max Slope) | 歩行可能な最急角度 | 45度 |
| 段差高さ (Step Height) | 乗り越え可能な最大障害物高 | 0.4m |

異なるエージェントタイプには別個のナビメッシュが必要（大型モンスター vs 小型キャラクター）。

## ナビメッシュ生成パイプライン（4段階）

### Phase 1: ボクセル化 (Voxelization)
- 入力ジオメトリ（三角形メッシュ）をボクセルグリッドに変換
- 歩行可能な三角形をマーク
- **利点**: 大量の重複ジオメトリと小さな三角形に頑健に対応

### Phase 2: フィルタリング (Filtering)
- 低いハング障害物（天井が低い場所）を除去
- 台帳スパン（ledge spans）を処理
- エージェントの高さに基づき通過不能なスパンを除外

### Phase 3: コンパクト化と領域分割 (Compaction & Region Building)
- ボクセルデータを効率的に統合
- 距離フィールドの計算
- 連結した歩行可能領域を構築

### Phase 4: ポリゴン化 (Polygonization)
- ボクセルからナビメッシュポリゴンを三角測量
- メッシュディテール情報を追加
- 最終的な凸ポリゴンメッシュを出力

```
入力ジオメトリ → [ボクセル化] → [フィルタリング] → [領域分割] → [ポリゴン化] → NavMesh
```

## メッシュタイプ

### シングルメッシュ (Single NavMesh)
- 静的で単純な環境に適合
- 生成が高速
- ワールド全体を1つのメッシュで表現

### タイルメッシュ (Tiled NavMesh)
- 大規模・動的環境に対応
- **部分的再ベイク**: 変更箇所のタイルのみ再生成
- **階層的パスプランニング**: 大域→局所の2段階検索
- **データストリーミング**: オープンワールドでの段階的読み込み
- 複雑だが拡張性が高い

## Detour（ランタイムコンポーネント）

Recastが生成したナビメッシュ上で動作するランタイムライブラリ:

- **A*パスファインディング**: ポリゴンベースのグラフ検索
- **空間クエリ**: 最近接ポリゴン検索、レイキャスト
- **ポリゴンフィルタリング**: 通行可能/不可のフラグ制御

## DetourCrowd（群衆シミュレーション）

- エージェント移動と衝突回避
- 群衆シミュレーション
- **RVO (Reciprocal Velocity Obstacles)**: 相互回避

## 生成プロセスの利点

- ボクセル化により入力ジオメトリの品質に対して頑健
- デバッグ・可視化が容易（各段階を視覚的に確認可能）
- カスタマイズ性と明確なパフォーマンス/品質トレードオフ
- 業界標準として広く検証済み

## 汎用化ポイント

Recast/Detourはエンジン固有の実装だが、以下の概念はエンジン非依存:
- ボクセル化ベースのナビメッシュ生成パイプライン
- エージェント属性によるメッシュカスタマイズ
- タイルベースの動的更新戦略
- String-Pullingによるパス平滑化
- 階層的パスプランニング
