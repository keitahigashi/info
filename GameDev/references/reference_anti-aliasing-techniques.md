## エイリアシングとは

斜め線のジャギー（階段状アーティファクト）。VRではHMDの微動と組合せでフリッカリングが発生し不快感を増大。

## 2グループ分類

| グループ | 手法 | パフォーマンス影響 |
|---------|------|---------------|
| ポストプロセス（低コスト） | FXAA, SMAA | 軽微 |
| サンプリングベース（高コスト） | MSAA, SSAA, TXAA | 中〜重い |

## 5手法詳細

### FXAA (Fast Approximate AA)

- **仕組み**: 輝度の急激な変化を持つピクセル群をブラー
- **パフォーマンス**: 最小影響、最も効率的
- **長所**: 最速実行
- **短所**: テクスチャディテールもブラー、コントラスト低下、動的シーンに弱い
- **推奨**: モバイル / 前後カメラ移動なしのPC

### SMAA (Subpixel Morphological AA)

- **仕組み**: 線/曲線/オブジェクト境界のパターンを認識し、方向に沿ってブラー
- **パフォーマンス**: FXAAより高負荷だが「極めて高速」
- **長所**: FXAAよりジャギー削減大＋ブラー少、ジオメトリ認識
- **短所**: 静的画像に最適
- **推奨**: ディファードレンダリング（MSAA不可時）

### MSAA (MultiSample AA)

- **仕組み**: ポリゴンエッジのみでサンプル数を増加（2-16サンプル）、複数ピクセルの平均値
- **パフォーマンス**: 品質比で比較的効率的
- **長所**: 画像ブラーなし、SSAAより低負荷
- **短所**: ポリゴン境界のみ有効、スペキュラ/透明エイリアシング非対応、**フォワードレンダリング限定**
- **推奨**: フォワードレンダリングパイプライン

### SSAA (SuperSampling AA)

- **仕組み**: 高解像度でレンダリング→ダウンサンプル
- **パフォーマンス**: **最も高コスト**（2X解像度で4倍負荷）
- **長所**: 全画像の卓越した品質、テクスチャ/境界両方のアーティファクト除去
- **短所**: 深刻なパフォーマンスペナルティ
- **推奨**: ハイエンドシステム / オフラインレンダリング

### TXAA (Temporal Approximate AA)

- **仕組み**: SSAA＋MSAAのハイブリッド。毎フレーム0.5-1ピクセルのカメラジッター → 時間経過で同一ピクセルの複数画像を取得
- **パフォーマンス**: SMAAより高い（モーションベクターレンダリング必要）
- **長所**: 静的＋動的エイリアシング両方に対応、FXAAとの組合せ可
- **短所**: 高速動作シーンでテンポラルブラー → シャープニングフィルタ必要
- **推奨**: 現代ゲーム（UE4/5, Unity HDRP, id Tech 7のデフォルト）

## パフォーマンス影響

| 手法 | FPS回復（OFF時） |
|------|----------------|
| TAA→Off | 8-15% |
| MSAA 4x→Off | 20-40% |
| FXAA/SMAA | ほぼ無視可能 |

## 選定ガイド

| 条件 | 推奨 |
|------|------|
| ハードウェア制限 | FXAA |
| ディファードレンダリング | SMAA |
| 高性能GPU | TXAA |
| 最高品質（性能無視） | SSAA 2X-4X |
