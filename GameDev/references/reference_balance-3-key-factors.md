## バランスの三角形フレームワーク

ゲームバランスに共通する3変数を三角形として定義する。

### 1. パワー (Power)

- 能力・スキルが与えるダメージ量など直接的な威力
- 最も定量化しやすい要素（「このスペルは10ダメージ、別のスペルは20ダメージ」）
- パワー単体だけではバランスは成立しない

### 2. ユーティリティ (Utility)

- ダメージ以外の追加効果・特殊効果
  - 回復・バフ/デバフ・スタン効果・トラップ・追加行動数など
- **原則**: 高パワーの能力は低ユーティリティを持つべきで、逆もしかり

### 3. コスト (Cost)

- 能力使用の代償（リソース消費・クールダウン・行動時間）
- **原則**: コストは能力の影響度に比例する

```
    パワー
   /      \
  /        \
コスト ― ユーティリティ
```

## 偽りの選択肢 (False Choice)

バランス崩壊の典型的パターン:
- **強すぎる**: 「使わないわけにはいかない」選択肢
- **弱すぎる**: 「使う意味がない」選択肢

すべての選択肢に価値を持たせることが目標。

## 具体例

### XCOM シリーズ
「確率を無視する攻撃」のような高パワー能力 → 「ミッション中の使用回数制限」というコストで調整

### CCG（Gwentなど）
- 高パワーカード → 限定的な効果
- 強力な効果 → 低い数値ステータス

## 設計の実践ポイント

- 高レベル能力は低レベル能力より長いクールダウンを設定し、低レベル能力の有用性を維持する
- コストの過剰設定は「温存」行動を誘発するため避ける
- マルチプレイゲームではとくに三角形の均衡が重要
