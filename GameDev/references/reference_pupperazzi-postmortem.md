## 出典

Game Developer (Sundae Month, 2024): https://www.gamedeveloper.com/production/postmortem-sundae-month-s-pupperazzi

## プロジェクト概要

- **ゲーム**：Pupperazzi（犬の写真撮影ゲーム）
- **スタジオ**：Sundae Month（大学時代に創設された学生主導スタジオ）
- **リリース日**：2022 年 1 月（Steam・Epic・Xbox・Game Pass 対応）
- **評価**：Steam で 96% の肯定的評価

## 開発の経緯

### 出発点

当初は競技的な犬発見ゲームとして着想。2 ヶ月で試作版が完成したが、品質に関する内部紛争により開発が一時中断。

### 「混乱のサイクル (Chaos Cycle)」

```
複数の内部プロジェクトと契約業務を同時進行
  ↓
リソースの分散
  ↓
開発の停滞
  ↓
モラールの低下・チーム文化の悪化
  ↓
創造的投資の回避・燃え尽き (burnout)
  ↓
「秘密のプリプロダクション」の繰り返し
（常に初期段階にいることに気づかない）
```

「"秘密の前制作"と呼ぶもの」の中で、スタジオは実際には常に初期段階にあることに気づいていなかった。

### 転機：焦点化

2019 年中盤、スタジオは Pupperazzi への全リソース集中を決定：
- 内部プロジェクトの中止
- 人員整理
- オーナーの 1 人が個人事業「Weathered Sweater」として分離独立

2020 年初頭：完全なゲームループが初めて完成。その後、Kitfox が出版社として参加。

## 成功要因

| 要因 | 詳細 |
|------|------|
| 高い肯定的評価 | Steam で 96%（チーム関係の維持とゲームの魅力が寄与） |
| プレイヤーの自発的拡散 | ソーシャルメディアで犬写真を共有する高エンゲージメント |
| Game Pass 対応 | 財政的成功の実質的な支柱（次項参照） |

## 失敗・課題

| 課題 | 詳細 |
|------|------|
| **Game Pass 依存** | GamePass なしでは「財政的に成功していなかった」と明言。単体販売だけでは収益化困難だった |
| **コンソール移植の不備** | コンソール移植経験が不十分でパフォーマンス・技術的問題が発生 |
| **チーム経験不足** | 3 名のオーナーと最大 12 名の協力者という学生主導体制の限界 |

## 重要な教訓

### 1. 焦点化の重要性

複数プロジェクトの並行は「混乱のサイクル」を生む。1 つのプロジェクトへの全リソース集中が転機になった。

### 2. プロダクション体制を最初から構築する

「秘密のプリプロダクション」（正式なプロジェクト化を先送りにした曖昧な状態）は進捗の錯覚を生む。明確なプロダクション体制と意思決定プロセスが必要。

### 3. プラットフォーム依存リスクを把握する

Game Pass のような収益モデルへの依存は、交渉力の低下や将来の不確実性につながる。販売戦略の多角化が望ましい。

### 4. チームの小ささは強みにも弱みにもなる

学生主導の小規模チームは機動性が高いが、専門知識（コンソール移植・マーケティング等）の欠如が後に問題となる。出版社サポートの早期確保が重要。

## 「混乱のサイクル」からの脱出フレームワーク

```
1. 現在のプロジェクト数を数え、実質的に進んでいるものを 1 つに絞る
2. 並行プロジェクトを中止または分離する
3. 明確なプロダクション体制（ロードマップ・マイルストーン）を設ける
4. チーム文化への意識的な投資（定期レトロスペクティブ等）
5. 収益戦略を早期に決定する（Game Pass 依存 vs 直接販売 vs 出版社契約）
```
