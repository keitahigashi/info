## コア原則: メトリクス (Metrics)

メトリクスは戦闘レベルデザインの背骨。「レベルの空間、カバーの高さ、廊下の幅などすべてを決定する」定量的寸法。

### 基礎計測領域
- プレイヤー移動速度
- ジャンプ高さ（シングル/ダブル）
- 廊下幅
- ドア・窓の寸法
- 複数フロア間の垂直間隔

### テスト空間の構築
メトリクス確定前に「チームメイトとテストするための空間を作成」すべき。ジム的環境で推測ではなく協働実験を通じて適切な感覚を決定する。

### ビジュアル区別の原則
主要建築要素は二次的要素の**約2倍のサイズ**にすべき。1メートル差ではビジュアルインパクトが不足、2倍にすることでプレイヤーの明確な認識を確保。

## 戦闘空間メトリクスの5要素

| 要素 | 説明 |
|------|------|
| **カバーの高さと幅** | 使用可能なカバーの明確な定義 |
| **カバー間隔 (バッファゾーン)** | カバーポイント間の距離制御。**最も重要なメトリクスの一つ** |
| **建築物上のカバールール** | ドア・窓・構造物とカバーの統合 |
| **武器射程** | 多様なプレイスタイルを促進する空間設計 |
| **敵アーキタイプ** | 異なるAI行動を考慮した配置 |

## カバー設計の詳細

### 間隔設計
- 適切な間隔がプレイヤーに実行可能なカバールートを理解させる
- AIが効果的にナビゲート・脅威を与えられる距離を確保

### 建築要素との統合
- ドア・窓近くにカバーを配置し、プレイヤーが無防備に空間に入ることを防止
- 窓 = スナイパーポジション
- ドア = 防御的カバーが必要

## 武器射程と空間設計

武器の有効距離が異なるため、レベル空間は「特定のプレイスタイルを促進」すべき:
- **スナイパーネスト**: 長距離交戦用
- **フランクルート**: 近接ショットガンプレイ用
- 空間設計がプレイスタイルの多様性を支える

## ドキュメンテーション基準

- メトリクスドキュメントは「非常に読みやすく」、明確な視覚スケール付き
- 参照テーブルを含め、「何ページもスクロールせずに」情報を見つけられるように
- 設計者はアーキテクチャがメトリクスに準拠しているか常に検証

## 統合設計の原則

「建築の信憑性とテーマを、ゲームプレイの優れた感覚と組み合わせる」。戦闘空間とナラティブ環境を別物として扱わず統合する。

## 適用範囲

本フレームワークは**射撃ベースの遠距離戦闘**に特化。近接戦闘、ターン制戦闘システム、異なるカメラパースペクティブ（一人称/三人称/アイソメトリック）には別のアプローチが必要。
