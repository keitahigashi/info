## SSMフレームワークとは

Amnesia・PENUMBRAシリーズ開発元Frictional GamesのThomas Gripが提唱するゲームデザイン分析フレームワーク。ゲームが機能する3つの空間を定義する。

## 3空間の定義

### System（システム空間）
コードと物理演算が存在する領域。MDAフレームワークのメカニクス＋ダイナミクスに相当。

- **メカニクス**: ゲームのルールと基礎システム
- **ダイナミクス**: ルールが動作したときに生じるインタラクション

### Story（ストーリー空間）
システムに文脈と意味を与える領域。プレイヤーが「見聞きする」すべてを包含。

- **ミズアンセーヌ (Mise-en-scène)**: 視覚・聴覚による表現（グラフィック・音楽・SE・アニメーション）
- **ドラマ**: 台本的なナラティブ要素（ストーリー・キャラクター・対話）

### Mental Model（メンタルモデル空間）
プレイヤーの個人的経験と理解。MDAのAesthetics（美的体験）より広い概念。

- **アフォーダンス (Affordance)**: プレイヤーが知覚する「できること」の集合
- **スキーマ (Schema)**: 行動パターンの認識・記憶

## ゲームループ

```
プレイヤー入力
  → System（物理処理）
  → Story（感覚出力）
  → Mental Model（認識更新）
  → 次の入力決定
```

プレイヤーは常に「想像上のゲーム」（自身のメンタルモデル）を基準にプレイする。

## 設計への応用

### メンタルモデルのコントロール
プレイヤーのメンタルモデルに意図的な誤解を植え付け、後で覆すことで驚きを生む。

**例: ウサギのコンパニオン設計**
- Story空間: 愛くるしい仲間キャラクター
- System空間: プレイヤーを守る「盾」
- 「ウサギが盾として機能していた」と発覚した瞬間、メンタルモデルが更新され強い感情反応が生まれる

### MDAとの違い
MDAはデザイナー視点（M→D→A）の一方向モデル。SSMはプレイヤー体験を循環として捉え、システム・ストーリー・メンタルモデルが相互に影響する双方向モデル。

## 活用場面

| 課題 | SSM視点での診断 |
|------|---------------|
| プレイヤーが操作に困惑する | Mental Model ↔ System のアフォーダンス不整合 |
| 世界観に没入できない | Story空間のミズアンセーヌ不足 |
| ゲームプレイが単調 | Systemのダイナミクス多様性不足 |
| 感情的な山場がない | 3空間の連携が設計されていない |
