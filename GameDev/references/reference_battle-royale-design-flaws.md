## バトルロイヤルが成功した理由

### ペーシング構造
「試合は素早く終わることも、長くなることも。ピークと谷がある」

- 序盤: リソース収集フェーズ
- 終盤: 強度が増加

### 制御されたランダム化
縮小マップがリングの配置によりプレイを変動させ、根本的メカニクス変更なしに停滞を防止。

## 設計欠陥（3つ）

### 1. 反復戦略の形成
固定マップレイアウトがプレイヤーに一貫して機能する「勝利の公式」を開発させる → 試合間の真の変動が減少。

### 2. 停滞するルートシステム
トップティア装備の集中が支配的戦略を生む。「最初に入手した者が高確率で支配する」→ 実行可能なプレイスタイルと戦術オプションが制限。

### 3. キャラクター差別化の不足
Apex Legendsが分隊ベースチャンピオンを導入したが、意味のあるメカニクス的多様性には十分に拡張されていない。

## 改善提案（4つ）

| 提案 | 詳細 |
|------|------|
| **ランダム化マップ** | 手続き的地形生成で2層のランダム化（マップ構造＋リング） |
| **拡張ルートプール** | 装備を手続き生成するか、「単一ゲームに全てが出現できない」ほどのバリエーション |
| **強化キャラクターシステム** | MOBA/TF2アプローチに倣った深いカスタマイズ |
| **分隊シナジー** | 複雑なチームインタラクションを可能にする深い協力メカニクス |

## 業界への警告

テンプレート追従の継続は「MMO型クラッシュ」のリスク → ジャンル内に未開拓の設計可能性が残存。
