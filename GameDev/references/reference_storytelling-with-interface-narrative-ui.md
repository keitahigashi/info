## インターフェースで語る物語 — ゲームUIのナラティブデザイン

### 背景
Dead Space（2008）がホログラフィックディスプレイを導入して以来、「ダイエジェティックUI (diegetic UI)」が普及。しかし、コンテキスチュアルUIが唯一解ではなく、真の目的は「物語そのもの」にある。

### 物語的UI要素の4構成要素

| 要素 | 説明 | 実例 |
|------|------|------|
| **キャラクター** | プレイヤーの注意を引き、感情を伝える | Tomb Raider の身体UIシステム |
| **オブジェクト** | 環境描写を通じた物語表現 | GTA5 の携帯電話UI |
| **雰囲気 (Atmosphere)** | 視覚効果による没入感・世界観演出 | Identity V の曇りガラス効果 |
| **イベント** | インタラクティブな要素の組み合わせによる物語進行 | 状況依存的なUIの変化 |

### ナラティブUI設計の4原則

1. **世界観との整合性** — UIがゲームの設定・世界観に溶け込んでいること
2. **感情設計の一致** — UI演出がゲームの感情的文脈に合っていること
3. **不可能な三角形のバランス** — 「シンプルさ」「操作性」「情報量」の三要素のトレードオフを認識して調整
4. **段階的設計プロセス** — システム構造の把握 → シナリオ設計 → UIの物語的実装の順で進む

### 3ステップデザインプロセス

1. **システム構造の整理** — ゲームの中核インターフェース要素を特定
2. **ゲーム内シナリオの設計** — UIが機能するシチュエーション・文脈を定義
3. **物語的要素の実装** — 4構成要素×各デザイン原則に基づくUI実装

### 結論
コンテキスチュアルデザインだけが物語的UIの解ではない。機能性と物語性を統合することが優れたUIデザインの本質。
