## 設計原則の違い

### WebUI（ビジネスアプリケーション向け）

- **優先事項**: 再現性、標準化、アクセシビリティ
- 業界標準フレームワーク（Material Design、Bootstrap等）を採用し、学習コストを最小化
- 大規模な変更は「ユーザー離脱のリスク」として認識
- 互換性と安定性を重視

### GameUI（エンターテイメント向け）

- **優先事項**: 没入感 (immersion)、ブランド表現、世界観構築 (worldbuilding)
- 各タイトル固有のデザインが特徴
- UIアニメーション、サウンド、ハプティクス (haptics) を通じた多層的フィードバックが「体験価値」を高める
- UIの更新自体が「新しい体験」として肯定的に受け入れられる文化

## 具体的な実装の差異

### アニメーション・動き

- **WebUI**: 状態変化を示す場合のみ（最小限）
- **GameUI**: メニューやローディング画面でも「ユーザーを世界に繋ぎ止める」演出を常に含む

### インタラクション設計

- **WebUI**: 「OKボタン」のような明確な選択肢で誤操作を防止
- **GameUI**: 「画面の任意の場所をタップして遷移」など、テンポや快適性を優先した柔軟な実装

## 両分野の知見の相互活用

### Web へのゲーム的アプローチ

- SaaS等で「ブランド価値を表現するUIとしての体験」が重要視
- 単なる情報提供を超えた「記憶に残るサービス設計」が求められている

### ゲームへのWeb的哲学

- クロスプラットフォーム対応の拡大に伴い、多様なデバイス・ユーザー層への対応が必須
- 普遍的ユーザビリティ (universal usability) がGameUIにも求められている

## 汎用化ポイント

本記事はWeb/ゲーム双方の実務経験に基づく比較であり、UIデザインの**目的に応じた設計原則の使い分け**をエンジン非依存で考える材料となる。ゲームUI設計時に「没入感とユーザビリティのバランス」を検討する際に参照できる。
