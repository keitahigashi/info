## 概要

「ゲームアウトカムズプロジェクト (Game Outcomes Project)」の第3部。数百のゲーム開発プロジェクトを横断調査し、批評的評価・財務的成功・スケジュール遵守・社内目標達成という複数の成果指標とさまざまな開発要因の相関係数を算出した実証研究。

## 成功と相関する要因（相関係数の高い順）

### 強い相関（0.50以上）
| 要因 | 相関係数 | 説明 |
|---|---|---|
| デザインリスク管理 (design risk management) | 0.57+ | デザイン変更への対応方法・頻度の管理 |
| チームフォーカス (team focus) | 0.50 | 共通ビジョンの共有と意思疎通の一貫性 |

### 中程度の相関（0.36〜0.44）
| 要因 | 相関係数 |
|---|---|
| クランチの回避 (crunch avoidance) | 0.44 |
| チーム安定性 (team stability) | 0.39 |
| コミュニケーション | 0.38 |
| 失敗への組織的姿勢 (organizational attitude to failure) | 0.37 |
| 相互尊重 (mutual respect) | 0.37 |
| プロジェクト計画 (project planning) | 0.36 |
| 技術リスク管理 (technical risk management) | 0.36 |

## 相関が見られなかった（影響なし）の要因

- アウトソーシングの活用
- クロスファンクショナルチームの採用
- **開発手法の選択（ウォーターフォール vs アジャイル）**

## 重要な洞察

### 「ビジョンへの共感」の条件付き相関
- 「チームメンバーがゲームのビジョンを深く気にかけていた」という項目:
  - 批評的評価・内部目標達成 → **強い正の相関**
  - スケジュール遵守 → **ほぼ相関なし**
- つまり「情熱と共感」はゲームの質を高めるが、時間通りに終わる保証はしない

### デザイン文書の有用性は予想より低い
- 設計文書の整備そのものは成功指標との相関が弱かった
- 文書の内容・活用方法が重要であり、文書の「存在」だけでは不十分

### クランチは成果を改善しない
- 長時間労働はスケジュール改善にも品質向上にも寄与しないことがデータで示された

## 実践的含意

1. **デザインリスクの優先対処**: 仕様変更・機能追加の管理プロセスに最優先で投資する
2. **チームビジョンの共有**: 全員がゲームの目標・方向性を同じように理解する仕組みを作る
3. **アジャイル神話の解体**: 手法の選択より、リスク管理・コミュニケーション・安定性が重要
