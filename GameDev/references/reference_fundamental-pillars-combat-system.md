## コンバットシステムの基本設計理論

### 核心原則

優れたコンバットシステムの目標:
> 「プレイヤーが正しい時に正しい能力を使うという**賢い選択**を迫られること」

達成したいこと:
- 各行動が独自の意味と機能を持つ
- リスクと報酬が適切にバランスされている
- プレイヤーが「自分の判断が正しかった」という知的充足感を得られる

### 能力設計の2つの軸

#### 軸1: 独自機能（Unique Function）
各能力は他の能力では代替できない固有の役割を持つ。

| 能力 | 独自機能 | 使用条件 |
|------|---------|---------|
| 近接攻撃 | 最大ダメージ・確実な命中 | 敵が近距離にいる場合 |
| グレネード | カバー越し・範囲攻撃 | 敵が遮蔽物の後ろにいる場合 |
| アイアンサイト射撃 | 遠距離精密攻撃 | 敵が遠距離・静止している場合 |
| カバー射撃 | 安全な継続ダメージ | 接近が困難な場合 |

#### 軸2: リスクと報酬（Risk/Reward）
高い効果には相応のコスト・リスクが伴う設計。

- 近接攻撃は最大ダメージだが、距離制限・被弾リスクがある
- グレネードは強力だが消耗品で本数制限がある
- アイアンサイトは精密だが視野狭窄・移動制限がある

### プレイヤーが解くべき3つのチャレンジ

1. **距離評価**: 現在の敵との間隔に最適な能力はどれか
2. **時間評価**: 能力の実行にかかる時間と、その間の被弾リスクのトレードオフ
3. **戦術的予測**: 複数能力の組み合わせシーケンスの最適解を見つける

### 敵設計：段階的学習ツールとしての役割

#### 基本原則

敵は単なる障害物でなく、**プレイヤーに特定の能力の使い方を教える教師**として機能する。

#### 敵アーキタイプと学習内容

| 敵タイプ | 要求される対応 | プレイヤーが学ぶこと |
|---------|-------------|-----------------|
| シールド持ち敵 | 精密攻撃・弱点狙い | 基本攻撃の使い分け |
| 重装甲敵 | 強力な攻撃・特殊能力の使用 | 強力能力の適切な使用タイミング |
| 遠距離スナイパー | カバー活用・接近戦略 | 距離管理・空間移動 |
| 爆弾型敵 | 素早い排除・距離確保 | 脅威優先順位の判断 |

#### サブクラスによる難易度スケーリング

基本敵タイプに新しい能力を追加することで、基本ルールを維持しながら複雑性を増加:

```
基本スナイパー敵
  → スナイパー + シールド（精密攻撃とカバー突破の両立）
  → スナイパー + 仲間召喚（優先排除の判断強化）
```

### 戦闘設計のフロー

```
能力セットの定義
  ↓
各能力の独自機能を明確化（重複を排除）
  ↓
リスク/報酬バランスを設定
  ↓
各能力を「教える」敵タイプを設計
  ↓
組み合わせシナリオで戦術的深みを検証
```

### まとめ

優れたコンバットシステムは:
- 多様な戦闘体験を提供する（単調な繰り返しを避ける）
- プレイヤーに知的充足感をもたらす（「自分で判断した感覚」）
- 能力の独自性と明確なリスク/報酬構造で実現する
