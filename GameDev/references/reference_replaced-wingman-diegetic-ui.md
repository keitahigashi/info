## ディスプレイブロート (Display Bloat) 問題

近代ゲームでは HUD 要素が増加し続け、**ディスプレイブロート (display bloat)** — 画面に情報が過剰に積み重なって没入感・視認性が低下する現象 — が深刻化している。開発が進むほどデータ表示の必要性が増し、後付けで HUD 要素を追加し続けた結果として発生しやすい。

## Replaced の解決策：Wingman

Replaced（サイバーパンク系アクション RPG）の開発チームは、必要な情報を「**Wingman**」というゲーム内デバイスに統合することで問題を解決した。

| 要素 | 内容 |
|------|------|
| **形態** | プレイヤーキャラクターが装備するゲーム内ガジェット |
| **機能** | HP・リソース・マーカーなどの情報を一元集約 |
| **統合** | ゲーム世界の論理（lore）に根ざした設計で自然に見える |
| **美的一貫性** | ゲームのビジュアルスタイルと完全に一致 |

## ダイエジェティック統合の利点

- **没入感の維持**：情報が HUD オーバーレイではなくゲーム世界の一部として機能
- **認知負荷の軽減**：プレイヤーはキャラクターの視点を維持したまま情報を取得
- **ナラティブ補強**：デバイスの存在がゲーム世界のロアを強化
- **スケーラビリティ**：情報追加時も Wingman に集約するだけでブロートを防止

## 既存ダイエジェティック UI との比較

| タイトル | 手法 | 特徴 |
|---------|------|------|
| Dead Space | RIG スーツ | HP/弾薬を背中に表示 |
| Metro 2033 | 腕時計 | 放射線・時間の表示 |
| Far Cry 2 | 物理マップ | 実際に地図を手に持つ |
| **Replaced** | Wingman | 情報を単一デバイスに集約してブロート回避 |

## 設計上の教訓

1. **早期に情報アーキテクチャを設計する**：後付けで HUD を追加し続けるとブロートは避けられない
2. **世界観との整合性**：ガジェットや道具としてUIを設計するとナラティブ一貫性が生まれる
3. **美的目標とUIを統一する**：「スタイリッシュなガジェット」というコンセプトが UI と体験デザインを橋渡しした
