## 開発フェーズ概要

ゲーム開発は「ざっくり7段階」に分けられる。全体期間が10年以上に及ぶプロジェクトもあり、各段階で達成度を明確に設定することが重要。

## 7つの開発段階

### 1. 企画立案（全工程の約5%）
- ゲームのコンセプトとビジュアル資料の作成
- 開発開始の基礎となる段階
- ターゲットとゲームの方向性を決定

### 2. プロト版（全工程の約20%）
- 「ビジュアルは仮でもいいので企画したゲームの面白さや売りが最低限分かる物を」作成
- ゲームの本質的な魅力を検証する重要なマイルストーン
- コアゲームプレイの楽しさを実証する段階

### 3. アルファ版（全工程の約30%）
- 全機能実装バージョン（大前提）
- テスト工程を開始するためのマイルストーン
- 主要な機能がすべて動作する状態

### 4. ベータ版（全工程の約60%）
- 全工程が完了し、外部へ成果物を出す準備が整った状態
- 「評価完了バージョン」
- プログラム製作からテストまで全工程が完了

### 5-7. マスター・ゴールド・リリース
- 最終調整と品質保証
- リリース候補版の確定
- 出荷

## 進捗管理の目安

| フェーズ | 進捗率 | 目的 |
|---------|--------|------|
| 企画 | ~5% | コンセプト確定 |
| プロト | ~20% | コアの面白さ検証 |
| アルファ | ~30% | 全機能実装・テスト開始 |
| ベータ | ~60% | 評価完了・外部出し準備 |
| マスター | ~90%+ | 最終品質保証 |

## 設計上のポイント

- 開発期間は長期化しやすいため、**タイミングと完成度を明確に設定**
- プロジェクト規模に応じた柔軟な対応が必要
- 長期開発では**進捗の可視化**が成功の鍵
- マイルストーンはスケジュールとは異なり「達成すべき品質基準」を表す
