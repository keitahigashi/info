## 2D画像処理から3D生成への拡張

オリジナルWFC（画像処理）を3Dモデル生成に適応。3Dマトリクス内の各セルにタイルを配置し、シームレスに接続。

## コアプロセス

### Collapse（収縮）

1. 全マトリクスセルが任意のタイルセットピースを保持可能な状態で開始
2. 有効タイルオプションが最少のセル（**最低エントロピー**）を特定
3. **重み付き確率**でランダム選択 → collapse
4. 隣接セルを制約

### Wave Propagation（波動伝播）

セル collapse 後、隣接セルが可能性空間を更新 → マトリクス全体に波及 → 安定化まで継続

## タイルセット設計要件

### 頂点マッチング

- タイルの境界ボリュームエッジで頂点座標を比較
- ベースタイルのプログラム的回転を生成
- 有効な接続を文書化する**隣接リスト**を構築

### 境界条件

初期生成の問題: 内部要素がエッジに露出

**解決策**: 外部面のみに接続する空タイルを追加 → 全境界セルをこの空タイルで collapse → 構造をマトリクス体積内に封じ込め

### マルチセルタイル

1〜2個の隣接のみに接続するタイルを作成すると、特別な処理なしに「複数セルにまたがる」効果が自然に出現 — 隣接ルールからの制約が自動的にこのパターンを生成

## 生成結果

- 同一タイルセットから多様な建築デザインを生成
- 複数の屋根構成、異なる壁配置、構造的バリエーション

## 課題と限界

| 課題 | 説明 |
|------|------|
| アクセシビリティ | 生成構造が到達不能・不完全な場合がある |
| タイルセット依存 | 品質はアーティスト制作タイルセットに大きく依存 |
| 通行可能性保証なし | ドア・到達可能性要件は計算コストの高い後処理とバックトラッキングが必要 |
| パフォーマンス | ワーカースレッドとコルーチンで最適化してもディレイ発生 |

## 実践的推奨

- **使用タイミング**: ロード画面・ワールドストリーミングチャンク生成
- **シリアライゼーション**: 生成後オブジェクトIDとして保存し、再生成コスト回避
- **設計哲学**: アルゴリズムはアーティスティックインプットを増幅するが、貧弱なデザインを克服はできない

## 「ローカル類似性」の乖離

この3D実装はオリジナルWFCの「入力パターンとのローカル類似性」概念を放棄。理論的完全性と実用的実装容易性のトレードオフ。将来的には3Dトレーニングモデルの統合で「ローカル類似性」の再獲得が可能。
