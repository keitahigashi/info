## 出典

Game Developer (Mike Lopez, 2008): https://www.gamedeveloper.com/design/gameplay-fundamentals-revisited-part-2-building-a-pacing-structure

## 核心問題

「その場しのぎ」的レベル設計では開発後期に大規模な作り直しが発生する（BioShockは最終9ヶ月でミッション全面改作）。事前に紙面上でペーシングを設計することで修正コストを大幅削減できる。

## 9段階の実装フレームワーク

### 1. ブレインストーミング
- レベルロケーション：計画数の2〜3倍を提案
- 高アクションイベント：レベルあたり5〜6個を目標
- 汎用イベント：主要メカニクスを活用した再利用可能な要素

### 2. 優先順位付け
スケジュール・リソース制約に基づくタスク推定。保守的な削減（10〜20%の余裕を確保）。

### 3. ストーリーフレームワーク
高レベルのプロット要点とミッション目標を概要化する。

### 4. レベルとキーイベントのシーケンシング
緊張度スケール（1〜10）による相対的な強度評価と階層化。

### 5. プロット固有イベントと緊張度の谷間 (valley)
各主要アクションイベントの前後に低強度の谷間を配置してコントラストを生成する。

### 6. 時間メトリクス (time metrics)
- レベル長は同一またはほぼ増加傾向
- ピーク間隔は均等または漸減
- 30秒単位のグリッドで可視化

### 7. トップダウン傾向評価
期待値から±20%の異常点を特定して調整する。

### 8. 初期レベル構築
設計仕様に基づくプロトタイプ実装。レベルビルダーはピーク間の時間メトリクスを距離に変換し谷間配置を決定する。

### 9. レビューと反復
4〜6週間ごとの定期的な強度評価と設計仕様の調整。

## 核心設計原則

### 緊張度とペーシングのコントラスト理論
```
高強度イベント → 低強度の休息場面 → 高強度イベント → ...
```
プレイヤーの心理的安全性と解放感を交互に作り出すことで没入感を維持する。

### 環境による視覚的ペーシング
テーマパーク・都市計画の原則を応用:
- 視線誘導 (sight lines)
- 窮屈感と開放感の対比
- 環境ギミックによる緊張の操作

## 実装上の利点

1. 紙面上での反復は実装後の修正より遥かに安価
2. 明確なターゲット基準により創造的フォーカスが集中する
3. スケジュール予測可能性の向上と製作効率の最適化
