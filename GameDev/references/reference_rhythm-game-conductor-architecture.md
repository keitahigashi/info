## 核心的課題

「人間はリズミックタイミングの逸脱に極めて敏感」→ リズムゲームのあらゆるアクション/移動/入力は音楽に**直接同期**が必須。

## Conductorクラス: コアアーキテクチャ

### 主要変数

| 変数 | 説明 |
|------|------|
| `songBpm` | トラックのBPM |
| `secPerBeat` | ビートあたり秒数（60/BPM） |
| `songPosition` | 現在再生位置（秒） |
| `songPositionInBeats` | ビート表現の位置 |
| `dspSongTime` | オーディオ開始時のタイムスタンプ |

### 初期化と更新

起動時にオーディオ開始タイミングを記録。毎フレーム:

```
経過時間 = 現在DSP時間 − 初期DSPタイムスタンプ
ビート位置 = 経過時間 / secPerBeat
```

→ ゲーム要素をフレームタイミングではなく**オーディオ再生にロック**。

## firstBeatOffset: 実世界音源への対処

実際のオーディオファイルは最初のビート前に無音を含むことが多い。

**解決策**: `firstBeatOffset` パラメータで全タイミング計算をシフト → ゲーム進行に伴う同期ドリフトを防止。音源ファイルごとに手動決定が必要。

## 同期を妨げる要因

| 要因 | 説明 |
|------|------|
| オーディオバッファ | 再生開始の遅延 |
| DSPタイミング | オーディオ処理のタイミング粒度 |
| 入力レイテンシ | コントローラー/キーボードの物理的遅延 |
| ディスプレイラグ | フレーム描画の遅延 |
| タイミングドリフト | 長時間プレイでの累積誤差 |

## ループメカニクス

リピートするトラック用にループ境界でビートカウントをリセット → 繰り返しオーディオセグメント内でのレベルデザインが可能。

## 設計哲学: ビート抽象化

生のミリ秒ではなく**ビート数**で設計仕様を記述。開発者がミリ秒計算ではなく**音楽的用語で思考**できる抽象化。

## キャリブレーション

見ているもの/聞いているもの/入力するもの間の**わずかな遅延でも体験全体が崩壊**。プレイヤーごとのキャリブレーション設定が不可欠。
