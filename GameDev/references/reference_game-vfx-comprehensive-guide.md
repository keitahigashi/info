## VFXの3本柱

### 1. パーティクルシステム
- 小さなパーティクルの集合で効果を生成
- サイズ、色、速度、寿命を調整可能
- 煙、火、魔法エフェクトに使用

### 2. シェーダー
- GPU駆動プログラム
- リアルタイム映像処理
- 動的ライティング、反射、屈折

### 3. 物理シミュレーション
- 現実的な動きと相互作用
- 水、風などの環境効果

## VFX設計の5つの重要要素

| 要素 | 説明 | 例 |
|------|------|-----|
| **色** | 統一されたカラーシステム | 緑=治癒、赤=危険 |
| **コントラスト** | 強度表現 | 明るい色=強力 |
| **形状** | 形態言語 | 円形=友好的、尖った形=危険 |
| **タイミング** | アニメーション原理による説得力 | 予備動作、誇張 |
| **関連性** | 現実世界との結びつけ | 炎=熱、氷=寒 |

## ゲームVFX vs 映画VFX

| 観点 | ゲーム | 映画 |
|------|--------|------|
| 視点 | あらゆる角度から表示 | 固定視点 |
| リソース | 制限あり（FPS維持） | 制限なし |
| 目的 | **プレイヤーへの情報伝達** | 視覚的補強 |
| 制作ツール | ゲームエンジン統合 | 単独ソフト |

## ジャンル別VFX要件

| ジャンル | 重点 |
|---------|------|
| アクション | 爆発、銃撃、激烈なアニメーション |
| RPG | ライティング、天候、環境効果 |
| パズル | UI動画、サブタルなエフェクト |

## 制作プロセス

1. **コンセプト企画**: デザイナー・アーティスト・ライターの協働
2. **創作と動画化**: デジタルツールでの資産制作
3. **統合とプログラミング**: ゲームエンジンへの組み込み
4. **テストと改善**: フィードバックに基づく調整

## 技術ツール

**UE5 Niagara**: 流体シミュレーション、衝突判定、カスタムモジュール機能。VFX開発の可能性を大幅に拡張。
