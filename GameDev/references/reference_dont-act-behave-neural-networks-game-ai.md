## ゲームAIにおけるニューラルネットワークの実践的活用

### 従来アプローチの問題点（AANN）

**AANN（Atomic Action Neural Network）**: ニューラルネットワークが直接的な原子的操作（移動・攻撃・ジャンプなど）を制御する方式。

**主な課題**:
- リアルタイム実行時の計算負荷が高く、多くのGPU処理を要する
- 「ジッター (jitter)」現象：フレームごとに微妙に異なる判断を下し、動きが不安定になる
- 複数エージェントの協調が困難
- AIの動作予測・デバッグが難しい

### 提案アーキテクチャ：BCN（行動制御ネットワーク）

**核心コンセプト**: ニューラルネットワークに「原子的操作の実行」ではなく「高レベルな行動計画の選択」を担わせ、実際の実行は専門化された既存システムに委ねる。

**層状アーキテクチャ**:
```
[BCN層] ニューラルネットワーク: 行動シーケンスの生成・選択
         ↓
[実行層] 既存の特化システム: 行動を実際に実行
         ↓
[制御層] 物理演算・アニメーション・衝突判定
```

### 実装事例

#### ケース1: 二段ジャンプによる移動AI

- BCNが「通過すべき中間点のシーケンス」を生成
- 別の移動スクリプトが中間点間の実際の移動を担う

**効果**:
- CPU使用率の大幅削減
- ジッター排除（中間点は固定されるため）
- 手続き生成 (procedural generation) レベルでの事前経路計画が可能

#### ケース2: サバイバルゲームAIの日常ルーティン

6000ステップのトレーニングで自律的な行動パターンを習得:

| 時間帯 | AIの自律的行動 |
|--------|-------------|
| 昼間 | 資源（食料・素材）の収集 |
| 夕方 | 火を起こす・矢を作る |
| 夜間 | 睡眠・水分補給 |

#### ケース3: Rocket League風AI設計

```
高レベル判断層: 「エアリアル攻撃を実行すべきか」
    ↓
特化AANN: ボール追従・空中姿勢制御
    ↓
低レベルシステム: 車の物理制御
```

### 設計原則

**ニューラルネットワークの適切な役割**:
- 判断・計画 → BCNが担当
- 実行・制御 → 既存の確実なシステムが担当

**重要な注意点**:
- ニューラルネットワークは「すべてのAIシステムを置き換える」ものでない
- 制御不可能性（unexpected behavior）を設計段階から考慮する
- 単一の大きなネットワークより、**複数の特化した小さなネットワーク**が効率的

### 未来の拡張可能性

| 方向性 | 説明 |
|--------|------|
| 協調学習 | 複数エージェント間の戦略的協力をBCNで学習 |
| 先読み計画 | 予測ネットワークで複数ステップ先のシミュレーション |
| 逆向き学習 | プレイヤーの行動パターンを解析し学習データを自動生成 |
