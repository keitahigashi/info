## メカニクス導入の失敗パターンと改善策

著者Max Pearsが、ゲーム開発者が陥りがちな「メカニクス導入の失敗パターン」を5つ列挙し、それぞれの改善策を解説した実践的記事。

### 5つの失敗パターン

1. **複数メカニクスの同時導入** — プレイヤーが処理しきれず混乱する
2. **習得中のゲームオーバー設定** — 失敗ペナルティが学習意欲を削ぐ
3. **危機的状況での学習強要** — 戦闘中など緊張状態では新情報を吸収できない
4. **フィードバック・指示の欠如** — 何が正しい行動かプレイヤーが判断できない
5. **テキスト中心の退屈なチュートリアル** — 読み飛ばされ学習効果がゼロになる

### 改善策の核心

- **1つずつ、安全な環境で段階的に導入する**
- 「ショー・ドント・テル (show don't tell)」——ゲーム内環境設計で自然に学習させる
- マリオシリーズ（Goomba登場シーン）やHalf-Life 2（重力銃の段階的教示）を良例として分析

### 設計指針

- メカニクスを導入する順序はコアループの習熟度に合わせて設計する
- プレイヤーが「試す → 結果を見る → 理解する」サイクルを完結できる安全ゾーンを用意する
- フィードバック（視覚・音響・振動）はメカニクス習得の確認ループとして機能させる
