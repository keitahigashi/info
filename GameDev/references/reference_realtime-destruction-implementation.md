## 破壊の3タイプ

| タイプ | 説明 |
|--------|------|
| **Fracture（破砕）** | オブジェクトが小片に分割。スケールが粒度を決定 |
| **Cut（切断）** | 鋭利な物体が平面カッターで固有の分離片を生成 |
| **Tear（引裂）** | 布等の可鍛性素材の緩い制約破断。硬い破砕とは区別 |

## 3つの実装アプローチ

### 1. Pre-baked（事前ベイク）

アーティストが事前にピースを組立て、ランタイムで除去。**最もアートフレンドリー**で汎用的。

### 2. Procedural（手続き的）

ランタイムアルゴリズムでピース生成。計算負荷高いが表現力最大。メッシュ複雑度の厳密な制御が必要。

### 3. Voxel（ボクセル）

ボクセルプリミティブから破壊可能オブジェクト構築。普遍的破壊可能性だがカスタム物理コード＋アートスタイル制限。

## 事前ベイクの技術システム

### 複合物理形状（Compound Physics Shapes）

各破壊可能コンポーネントに個別物理ジオメトリ。破壊時のサブ構造保持が核心課題。

### グラフ構造によるコンポーネント接続

コンポーネント間の接続をグラフ理論で追跡。

### 島検出（Island Finding）

コンポーネント分離時、**切断されたサブグラフ（島）を特定** → 独立物理ボディとして生成。

## パフォーマンス最適化（3手法）

| 手法 | 説明 |
|------|------|
| **隣接行列** | エッジ走査からN操作に近傍検索を加速 |
| **タイムスライシング** | グラフ検索をフレーム分散してスタッター削減 |
| **マルチスレッディング** | 検索を別スレッドにオフロード |
