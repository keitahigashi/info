## 出典

ポップインサイト: https://popinsight.jp/blog/?p=80524

## 1. 直感的フィードバック設計

**Call of Duty事例**: 体力ゲージを数字で表示せず、ダメージ時に「操作キャラが痛みにあえぎ、スクリーン縁が血の色に染まる」ビジュアル表現。プレイヤーは周囲に集中し続けられる。

**原則**: 情報を直感的・感情的に伝える設計がユーザー体験を高める。

## 2. 世界観統合メニュー

**Skyrim事例**: スキルツリーを「星座として表示」。機能は他ゲームと同等だが、ファンタジー世界観に合致した表現で没頭度が深まる。

**原則**: UIが製品のアイデンティティを正確に表現しているかを検証する。

## 3. 段階的情報提示

**ゼルダの伝説 BotW事例**: カエルに焦点を当てると種別情報が表示。「見つけやすすぎず、見つけられない程難しくもない」バランス。

**原則**: 複数のPOI（注目点）で段階的にユーザーを誘導する。

## 4. カスタマイズによる複雑性の解決

**World of Warcraft事例**: 複雑なゲームを「ユーザーに自分仕様にカスタマイズさせる」アプローチで対応。万能な解決より個別対応を優先。

**原則**: ユーザーに主要機能をカスタマイズさせる自由度を提供する。

## Webデザインへの応用

- UIが製品を正確に表現しているか検証
- 複数POIで段階的にユーザーを誘導
- 主要機能のカスタマイズの自由度を提供
- 優れたUIは「使いやすさ」だけでなく「楽しさ」も提供できる
