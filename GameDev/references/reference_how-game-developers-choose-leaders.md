## 概要

著者 Seth Spaulding（Firaxis Games アートディレクター）、2009年3月。書籍『Team Leadership In The Game Industry』からの抜粋。リーダー選出の理想プロセスと現実の乖離を5つのケーススタディ（Rick・Victor・Xavier・Yvette・Zeke/Alan）で解剖。スキルのみに基づく選抜・縁故主義・サポート不足が連鎖的に組織を崩壊させる構造を明らかにする。

## 理想と現実のギャップ

**理想的リーダー育成プロセス：**
有能な社員を早期から段階的にメンタリング → 準備が整った時点で昇進

**現実の障害：**
- 人員不足による「誰かがすぐ管理職になる必要がある」圧力
- 政治的配慮・縁故関係による非合理的な選抜
- リーダーシップスキルと専門スキルの混同

## 5つの失敗ケーススタディ

### Case 1: Rick — 外部実績者の文化的ミスマッチ

**状況：** 他業界での高実績者をゲームスタジオのリードとして外部招聘

**失敗メカニズム：**
- ゲーム業界知識の欠如 → チームの信頼喪失
- 経営陣が「投資した採用を認めたくない」という心理で問題を放置
- 最終的にスタッフの大量離職

**教訓：** 業界外の実績は評価基準の一部に過ぎない。文化・業務知識の適合性を採用前に厳密に検証する。

### Case 2: Victor — 兼任とスキル喪失への恐怖

**状況：** 優秀な専門家を管理職と制作担当の兼任に配置

**失敗メカニズム：**
- 二重役割の非効率化
- 「制作を続けていないと自分のスキルが失われる」という心理的障壁
- 必要な役割変更を本人が拒否し続けた結果、管理も制作も機能不全

**教訓：** リーダーへの昇進は「専門家としての自分を失う」という恐怖を伴う。このトランジションを心理的にサポートする体制が必要。

### Case 3: Xavier — 問題の先送りと転属繰り返し

**状況：** パフォーマンス問題を持つ社員を「配置転換」で解決しようとした

**失敗メカニズム：**
- 直接的フィードバックを避け続けた結果、問題が複数部門に拡散
- 各部門が「前の部門でうまくいかなかった人」として引き受けを嫌がる文化が形成
- 最終的に雇用契約終了となったが、早期介入で防げた

**教訓：** 問題の先送りは被害を拡大する。誠実で直接的なパフォーマンスフィードバックを早期に行う。

### Case 4: Yvette — サポートなき昇進と燃え尽き

**状況：** 高い専門スキルを持つ社員をサポートなしで昇進させた

**失敗メカニズム：**
- 管理職業務の学習コストをマネジメントがカバーせず
- 「自分でやった方が早い」として個人作業量が増大
- 燃え尽き症候群 (burnout) による退職

**教訓：** 昇進後の継続的メンタリング・コーチングが必須。「任命 = 完了」ではない。

### Case 5: Zeke & Alan — 二人制リーダーシップの曖昧さ

**状況：** 適切な単独候補者がいないため、二人で共同管理職を設けた

**失敗メカニズム：**
- 意思決定権限の曖昧さ → 両者が互いの判断を覆す
- チームが「どちらに従うべきか分からない」状況に陥る
- 権限構造の欠如が組織全体の生産性を低下させた

**教訓：** 共同リーダーシップが機能するには極めて明確な役割分担と権限定義が必要。適切な候補者がいない場合、外部採用や育成期間の設定が優先される。

## リーダー選出の原則（Joe Minton インタビューより）

元スタジオ代表が強調した主要原則：

1. **友情と友好性の区別** — 「友達として好き」と「この人についていける」は別物
2. **透明な経営情報の共有** — リーダー候補に組織の課題・制約を早期に伝える
3. **一貫した意思決定** — 状況によってルールを変えない。「例外」が文化を壊す
4. **効果的なメンタリング** — 昇進後 6 ヶ月間は週次で 1on1 を実施するなど構造化する

## 採用・育成チェックリスト

- [ ] 候補者の専門スキルに加え、「業界・文化適合性」を評価したか
- [ ] リーダー昇進の「責任範囲」と「期待値」を文書化して共有したか
- [ ] 昇進後のメンタリング・サポート体制が設計されているか
- [ ] パフォーマンス問題に対して直接フィードバックを行う仕組みがあるか
- [ ] 共同リーダーシップを採用する場合、役割と権限が明文化されているか
