## コア最適化領域

### 1. ボトルネック特定

「ロードシーケンス中にゲームをプロファイルし、大きなアセット/同期操作/不要なリソース取得を特定」

ツール: Unity Profiler / Unreal Session Frontend

### 2. 非同期ロード

ノンブロッキングアセットロードでゲームプレイフリーズを防止。

- Unity: **Addressables** / Asset Bundles
- Unreal: **Asset Manager**

オープンワールドではプレイヤーが新ゾーンに接近するとジオメトリ/テクスチャがオンデマンドでロード。

### 3. アセットバンドル化

関連リソースをバンドル（Unity Asset Bundles / Unreal PAK）にグループ化 → 個別ファイル操作を削減しディスクパフォーマンス向上。

Addressables: 物理的位置制約なしのリモートロード＋依存関係管理。

### 4. 圧縮テクニック

| アセット | 推奨フォーマット |
|---------|---------------|
| テクスチャ（デスクトップ） | DXT |
| テクスチャ（モバイル） | ETC / ASTC |
| オーディオ | Ogg Vorbis / AAC |

ファイルサイズ / メモリフットプリント / デコード時間のトレードオフを考量。

### 5. オブジェクトプーリング

事前確保インスタンスの再利用 → 生成/破棄オーバーヘッド削減。弾丸/敵等の頻繁スポーン要素に特に有効。

### 6. メモリライフサイクル制御

| 原則 | 説明 |
|------|------|
| 優先度付きロード | 重要アセットを最初に |
| 遅延ロード | 二次要素を後回し |
| 積極的アンロード | 未使用リソースを能動的に解放 |
| 手動管理 | レベル遷移間のアセット永続化を精密制御 |

### 7. キャッシングメカニズム

頻繁アクセスアセットのローカル保存 → 再ダウンロード/帯域幅削減。永続データパスでゲーム状態とUGCをセッション間保持。

## ワークフロー統合

最適化を**継続的開発プラクティス**として扱う:
- 未最適化アセットの自動チェック
- タスクトラッキングで進捗監視
- ビルドパイプラインへの検証統合
