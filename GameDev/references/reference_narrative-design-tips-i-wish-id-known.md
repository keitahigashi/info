## ゲームナラティブデザインの4つの実践的ヒント

### 1. ストーリーはゲームプレイから始める（Verbing）

**核心原則**: 物語がゲームプレイに付け加えられるのでなく、ゲームプレイが物語を定義する。

**Verbingプロセス**:
1. プレイヤーが取れるすべての行動（動詞）をリストアップする
2. 各行動がキャラクターについて何を語るかを問う
3. 行動そのものがキャラクターの性格・価値観を示す語り手になる

**具体例**:
- 『ラスト・オブ・アス』: 近接戦闘・隠密・資源節約というゲームプレイがジョエルの生存者としての性格を語る
- 『ダークソウル』: 死とリスポーンのメカニクス自体が世界観の核心的語り手

### 2. ストーリーはテキストだけではない

**語りの多チャンネル性**:

| チャンネル | 語りの具体例 |
|----------|------------|
| 環境美術 | 廃墟の状態・配置が過去の出来事を示す |
| 敵デザイン | 敵の外見・行動がロア（lore）を補完する |
| 音響設計 | BGMや環境音が場所の感情的意味を伝える |
| UI/HUD | インターフェース設計がゲーム世界への没入度に影響する |

**重要な原則**: 視覚的・音響的要素で伝えられることは、テキストで繰り返さない。複数チャンネルの相乗効果でストーリーが完成する。

### 3. ライティングはプレイヤー行動へのフィードバック

**フィードバックループとしてのライティング**:

```
プレイヤー入力 → ゲームプレイ結果 → ナラティブフィードバック
```

- カットシーンや会話は「プレイヤーが達成したことへの応答」として機能させる
- ゲームプレイを中断するのでなく、ゲームプレイに報いる位置づけ
- レベルクリア後・ボス撃破後など、自然なリズムに配置する

**避けるべきパターン**:
- ゲームプレイ中の長い会話割り込み（操作感を損なう）
- ゲームプレイと無関係な物語情報の押し付け

### 4. シーケンスではなく環境に焦点を当てる

**環境としてのゲーム設計**:

- ゲームは映画と異なり、プレイヤーが「次に何が起こるか」を決定する
- 線形のプロット順序に頼りすぎると、インタラクティブ性が失われる
- 環境設計でプレイヤーを自然に誘導しながら、探索の自由を担保する

**実践的アプローチ**:

| 設計レイヤー | 実装例 |
|------------|--------|
| 空間的語り | NPCの配置・ドアの状態・アイテムの位置で出来事を示す |
| 発見型情報 | 日記・メモ・環境美術をプレイヤーが能動的に発見する |
| 分岐への対応 | どの順序でプレイヤーが来ても意味が通じる設計 |

**参照事例**: 『ゼルダ』シリーズの環境誘導（言葉なしで次の目的地を示す）・『Civilization 5』の創発的なプレイヤーストーリー生成

### まとめ

- ゲームプレイと物語は一体化すべき。後から物語を追加するのではない
- 語り手はテキストだけでない。空間・音・視覚・メカニクスすべてが語る
- ライティングはリアクティブ（反応的）に機能させ、プレイヤーの主体感を損なわない
- 環境設計がナラティブデザインの核心ツールになる
