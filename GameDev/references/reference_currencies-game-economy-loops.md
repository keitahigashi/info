## 概要

ゲーム経済（ゲームエコノミー, game economy）とはゴールドだけでなく、戦場に残った弾薬・NPCの好感度まで含む広義の「通貨」システムである。本記事はコアループ図に通貨の増減を注記する簡易アノテーション手法を提案し、Machinations ツールによるシミュレーションまでのワークフローを示す。

## 通貨アノテーションシステム

コアループ図の各ステップに以下の記法で通貨の増減を記入する。

| 記法 | 意味 |
|------|------|
| 緑字 `++` / `+` | ソース（source）: 通貨が経済に追加される |
| 赤字 `--` / `–` | シンク（sink）: 通貨が経済から除去される |
| 大文字 | 主要な変化 |
| 小文字 | 軽微な変化 |

**ソース (source)**: ゴールド獲得・XP獲得など経済に通貨を注入する仕組み
**シンク (sink)**: 装備購入・ガチャ消費など通貨を恒久的に除去し、インフレ（inflation）を抑制する仕組み

## ケーススタディ: Star Wars: Galaxy of Heroes

バトル完了ループのアノテーション例:
- エネルギー消費（シンク）→ バトル実行
- 経験値・キャラクター強化素材（ソース）が獲得される
- クリスタル（ハード通貨）消費でインスタント完了（シンク）
- ソフト通貨インフレを防止しつつグラインド疲労を軽減

## Machinations による高度モデリング

Joris Dormans 考案の対話型経済シミュレーターで、以下を実現できる:
- **ソース・シンクの接続を視覚的に定義**してループ構造を確認
- 現実のプレイデータの平均値・極値を注入して**予測分析**
- 正のフィードバックループ（positive feedback loop）の過強化をシミュレーション前に検出

### RPG 経済モデルの分析例

XP → レベル → HP/PP/マナ割り振りの Machinations ダイアグラムで「単一スキル集中 (hyper-specialization)」が最適解になるという設計問題を事前発見できた。

## ソフト通貨とハード通貨

| 種別 | 特徴 |
|------|------|
| ソフト通貨（soft currency） | プレイ報酬として大量供給、インフレしやすい |
| ハード通貨（hard currency） | プレミアム購入など希少、購買行動に直結 |

ソフト通貨のシンクを設計する核心課題: **プレイヤーが継続的に欲しいと感じるものを消費先として提供すること**。

## 分析統合のワークフロー

1. コアループ図を作成しソース・シンクをアノテーション
2. Machinations でループを対話的に再現
3. ソフトローンチのデータ（プレイヤー平均・上位10%）をスプレッドシートから注入
4. 経済崩壊（破産・ハイパーインフレ）のリスク箇所を早期特定
5. バランスシートとして管理・更新
