## 研究概要

66のゲームサンプルに基づき、ミニマップの最適な配置位置を分析。UI/UXデザイナー向けの実践的ガイドライン。

## ジャンル別の慣例分析

| ジャンル | 主要配置 | 備考 |
|---------|---------|------|
| MOBA | 画面下部に統一 | 最も一貫 |
| ストラテジー | 左下が主流 | RTSは右上も |
| MMORPG | 右上に集中 | |
| レーシング | 左側が優位 | |
| FPS | 左側が主要位置 | |

**重要な発見**: 同一シリーズでも配置に一貫性がなく、**明確な業界標準が存在しない**。

## 配置の心理学的根拠

- 西洋文化圏では**左側の方が視線の注視時間が長い**
- 高速ゲームでの反応時間短縮に寄与
- ミニマップとコアゲームプレイ要素の距離を最小化すべき

## 推奨設計原則

### 必須表示要素（優先度順）
1. 障害物と通路の視覚化
2. プレイヤー位置
3. 敵・味方位置
4. 関心地点（チェックポイント等）

### 追加可能要素
- 戦争の霧 (Fog of War)
- ユニット状態表示
- 地形情報

### 認知的負荷軽減策
- コア要素との距離を最小化
- **色盲対応**の配色検討
- 要素間の高コントラスト設定
- シンプルなメタファー使用

## インタラクション機能

- ウェイポイント設定
- ピング/アラート機能
- ズーム機能
- マップ回転/固定切り替え

## 核心的示唆

ユーザーの習慣形成と視線動線の効率性が、単なる美的配置より**機能的に優先されるべき**。
