## 概要

ゲームバランス設計における重要概念「スプレッド (spread)」を定義し、その規模が開発難度・プレイヤー体験に与える影響を理論化する。Red Hook Studios 共同創設者 Tyler Sigman による実践知見。

## 核心概念: スプレッド (spread)

| 用語 | 定義 |
|------|------|
| メカニクス (mechanic) | ゲーム内の任意の機能・能力（射撃精度、武器ダメージ等） |
| 基本値 (base value) | メカニクスの最低レベルの値 |
| 最終値 (final value) | 最高能力段階での値 |
| スプレッド (spread) | 最終値 − 基本値 = 能力成長幅 |

## スプレッド規模の比較

### 低スプレッド設計

**利点**:
- バランス調整が容易（難易度曲線とプレイヤー能力曲線の交差リスクが低い）
- ゲームが「崩壊 (breaking)」するリスクを抑制
- 競争環境では10%の差でも試合結果を左右する戦略的緊張感を維持

**課題**:
- プレイヤーのアップグレード報酬感 (upgrade satisfaction) が薄れる可能性
- 視覚・演出・購入体験などで報酬感を補完する設計が必要

### 高スプレッド設計

**利点**:
- プレイヤーに強い成長感・パワーファンタジーを提供
- D&D系RPGの典型（1レベル: 10HP → 12レベル: 120HP）

**課題**:
- 難易度曲線の設計が複雑化
- テストリソースの大量消費
- バランス崩壊リスクが高まる

## 実例

### Age of Empires: The Age of Kings DS

意図的な**低スプレッド設計**を採用:
- 個別アップグレードは5〜10%の段階的改善
- 理由: テストリソースが限定的だったため、バランス崩壊リスクを最小化
- 小幅改善でも競争ゲームではロールが変わる可能性を活用

### Mass Effect 2

- +10%以下の増分アップグレード
- 敵の動的スケーリング (dynamic scaling) でアップグレード影響を吸収
- 具体的な数値をUIに表示せず演出を優先し、低スプレッドの報酬感不足を補完

### Classic AD&D

高スプレッドの典型:
- HPが核心メカニクス（1〜12レベルで10倍成長）
- モンスター・遭遇設計をすべてこのスプレッドに合わせて構築

## 設計判断フレームワーク

```
スプレッドの選択 = f(テスト能力, プレイヤー期待, ゲームジャンル)
```

1. **テストリソースが限定的** → 低スプレッドを選択してバランス崩壊リスクを抑制
2. **パワーファンタジーが核心** → 高スプレッドを選択し演出で補強
3. **競争ゲーム** → 低〜中スプレッドで小差が戦略的に意味を持つ設計
4. **RPG/成長型ゲーム** → 中〜高スプレッドで達成感を維持

## バランス設計における原則

- スプレッドの大小は「どちらが優れているか」ではなく**ゲームの核心メカニクスと開発制約から決定する**
- 低スプレッドでも**演出・UI・購入体験**で報酬感を補完できる
- 競争環境では小さな数値差がプレイヤーにとって大きな意味を持つ
- テスト工数を見積もった上でスプレッド規模を決める（大スプレッド = 大テスト）
