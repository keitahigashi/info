## コア原則

「マーケティングは最終段階ではない。ローンチ時にスイッチを入れるものではない」。開発サイクル全体を通じた系統的・並行的な実行が必要。

## 5フェーズ

### 1. ゲームポジショニングの早期定義

| 要素 | 説明 |
|------|------|
| **ターゲットオーディエンス** | 漠然としたデモグラフィクスではなく特定コミュニティ（「ターン制タクティクスファン」等） |
| **コアフック** | 「誰かがこのゲームを試す理由」を一文で明確化 |
| **スタジオボイス** | 一貫した公開プレゼンテーションスタイル（技術的/ユーモラス/様式化） |
| **ビジネス目標** | ユニット販売/パブリッシャー関心/コミュニティ成長のどれを優先するか |

### 2. 開発中のコミュニティ構築

**インフラ構築:**
- Webサイト・ソーシャルハンドルの確保
- メーリングリスト（自己所有のオーディエンスリーチ）
- Discord サーバーでの早期ファンエンゲージメント

**コンテンツルーティン:**
週2-3回の投稿（開発GIF、devlog、コンセプトアート）。「コンテンツはポリッシュではなくプロセスに焦点を当てるべき」。

**Steamページ:**
早期公開でウィッシュリストを蓄積。「早く公開するほど、ウィッシュリスト生成期間が長くなる」。

### 3. 中間開発: 可視性と信頼性

- **パブリックデモ**: 5分以内にコアメカニクスを体験できるバーティカルスライス
- **クリエイターアウトリーチ**: 具体的なコンテンツに言及、ジャンル適合を説明、150語以内のピッチ
- **プレス＆イベント**: ニュースに値する瞬間（トレーラー/デモリリース）にタイミングを合わせる。Steam Next Fest・地域ショーケースに応募

### 4. ローンチ: モメンタム最大化

「ローンチ後72時間がストアフロント、特にSteamでの可視性に重要」。

**主要アクティビティ:**
- ローンチ1週間前に最強トレーラーリリース
- Discord/メール/Twitter/TikTok/YouTubeで協調アナウンス
- 信頼できるストリーマーに早期キー提供
- バグレポート・コミュニティエンゲージメントに迅速対応
- 他インディーとのバンドルディール検討

### 5. ポストローンチ: サポートと拡張

- パッチノートをニュースビートとして扱う
- 無料コンテンツ更新（新モード、コスメティクス）を計画
- 季節的Steamイベントやキュレーションショーケースに応募
- ファンアートや開発者インサイトを共有

**ロングテール戦略:** 多くのゲームがローンチ数ヶ月後に持続的アップデートとコミュニティ活動で最大販売を経験。
