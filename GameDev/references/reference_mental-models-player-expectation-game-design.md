## メンタルモデル (Mental Model) の本質

- メンタルモデル：脳が感覚データから構築する外界の**心的シミュレーション**
- 「現実は脳が作り出したフィクション」：プレイヤーが知覚するゲーム世界は物理的実装ではなく脳の解釈
- **含意**：グラフィクスの視覚的品質より、心的シミュレーションの一貫性と予測可能性が体験の質を決定する

## デザイナーのモデル vs プレイヤーのモデルのズレ

### 事例：『Penumbra』の武器システム

| 主体 | メンタルモデル |
|------|--------------|
| デザイナーの意図 | 武器 = 現実的な防衛手段（使用は緊急時のみ） |
| プレイヤー解釈A | 武器 = ゲーム的な戦闘メカニクス（積極的に戦う） |
| プレイヤー解釈B | 武器 = 謎解きのツール（環境パズルに使用） |

- 同一システムでも、異なるメンタルモデルによりプレイヤー行動が大きく分岐する
- 設計者が想定しない使い方が生まれる根本原因

## ゲーム設計への応用

### 1. 一貫したルール体系の構築

- オブジェクトとの相互作用が**予測可能**であること
- 「このオブジェクトはこう動く」というモデルをプレイヤーが安定して形成できる
- ルールの例外が多いほどメンタルモデルが崩壊し、認知的負荷 (cognitive load) が増大する

### 2. 環境の内部的属性の一貫性

- ピクセルレベルの画像品質より、オブジェクトの「ふるまいの一貫性」が優先される
- プレイヤーが世界を「生きている」と感じる条件は、属性の一貫した適用にある
- 例：火は常に燃えるもの、水は常に濡れるもの → 例外なく適用

### 3. 認知的期待管理（初期情報提供）

**事例：『Amnesia: The Dark Descent』**
- ゲーム開始時に「このゲームのプレイ方法についての通知」を提示
- 効果：プレイヤーの初期メンタルモデル形成を支援し、認知的不協和 (cognitive dissonance) を軽減
- ホラー体験の没入度が著しく向上

```
プレイヤーがゲームを開始
    ↓
初期情報でメンタルモデルの「骨格」を提供
    ↓
実際の相互作用でモデルを精緻化・検証
    ↓
予測どおりに動く = 没入感向上
予測を外れる = 驚き / 混乱（意図的なら有効）
```

## フィードバックシステムとの関係

- フィードバックの主な役割：プレイヤーのメンタルモデル形成の**ガイド**
- 明確な因果関係（行動→結果）を視覚・音響でサポート
- フィードバックの遅延・曖昧さはモデル形成を阻害する

## 設計実践の指針

| 場面 | 適用 |
|------|------|
| チュートリアル設計 | プレイヤーの既存モデルとゲームルールのズレを把握し、早期にギャップを埋める |
| UI設計 | アイコン・表示がプレイヤーの直感的理解と一致しているか検証する |
| 操作設計 | コントローラーの動作とゲーム内結果が同一メンタルモデルを共有しているか確認する |
| 新メカニクス導入 | 既存モデルを活用した「類推可能な設計」から始める |

## 設計上の現実的な制約

- デザイナーとプレイヤーのモデルの**完全な合致は不可能**
- 目標：大きなズレを最小化し、意図的なズレ（驚き・発見）のみを残す
- 多様なプレイヤーの解釈を予見したうえで、一貫性の範囲内で設計を行う
