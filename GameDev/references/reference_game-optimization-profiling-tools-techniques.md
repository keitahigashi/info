## 核心哲学

「最適化に強いスタジオは、より一生懸命最適化するのではなく、リグレッションをより早期に可視化する。QAタイムではなくサブミットタイム（提出時）に」

## 三層最適化フレームワーク

```
Layer 1: プロファイリング (Profiling)
    ↓ ボトルネック特定
Layer 2: バリデーション (Validation)
    ↓ 品質ゲート・リグレッション防止
Layer 3: 自動化 (Automation)
    ↓ ビルド横断でのトレンド追跡
```

## Layer 1: プロファイリングツール一覧

### エンジン統合プロファイラー
- **Unreal Insights**: エンジンサブシステム横断の詳細タイミングキャプチャ
- **Unity Profiler**: フレームレベルのCPU/GPU分析、オーディオ・物理・ライティングモジュール内蔵

### ハードウェアレベルプロファイラー
- **RenderDoc**: 無料・クロスプラットフォームGPUデバッガ。ドローコール・シェーダーコスト特定
- **Microsoft PIX**: Xbox / DirectX 12向けパフォーマンス分析
- **NVIDIA Nsight Graphics**: 複数グラフィックスAPIのレンダリングパイプラインプロファイリング
- **Intel VTune Profiler**: Intelプラットフォーム向けシステム・フレームレベル分析

## Layer 2: アセットバリデーション（品質ゲート）

提出時に自動チェックを実行してパフォーマンス問題の混入を防止：

| チェック項目 | 内容 |
|------------|------|
| ポリゴン・トライアングル数 | 上限値バリデーション |
| LOD設定 | 正しい設定の検証 |
| テクスチャ圧縮 | フォーマット仕様への適合 |
| メモリフラグ | 適切な管理 |
| 依存関係整合性 | 参照の検証 |

## Layer 3: 自動化によるトレンド追跡

ナイトリービルド横断での継続的モニタリング：

> 「単一のプロファイルキャプチャは1つのビルドを語る。2週間のトレンドはチームが何をしているかを語る」

Virtuos Labs が開発した「Goliath」システム事例：フレームタイムデータをシステム別に表示し、機能統合直後のパフォーマンス影響を即座に検知。

## 実装タイムライン推奨

| フェーズ | 優先実装内容 |
|---------|------------|
| プロダクション初期 | ゲームプレイ優先、インフラ投資は後回し |
| コンテンツスケールアップ直前 | バリデーション・トラッキングシステムを確立（最適タイミング） |
| アセットパイプライン設計時 | アセットバリデーターを早期実装 |

## 最終目標

フレームレート目標の達成を超えた最終目標は「パフォーマンスへの影響を理解した上で、ゲーム開発においてスマートな判断を下せること」。
