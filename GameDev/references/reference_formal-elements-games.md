Game Design Concepts (Ian Schreiber): https://gamedesignconcepts.wordpress.com/2009/07/06/level-3-formal-elements-of-games/

## 概要

ゲームは相互接続されたシステム。1つの形式要素を変更するだけでゲームプレイが大きく変わる。Tracy Fullerton の定義: 「プレイヤーを構造化された紛争に引き込み、その不確実性を不平等な結果で解決する、閉じた形式的システム」。

## プレイヤー構造 (7タイプ)

| 構造 | 説明 | 例 |
|------|------|-----|
| **ソリテア** | 1人 vs システム | マインスイーパー |
| **ヘッドトゥヘッド** | 2人対戦 | チェス、囲碁 |
| **協力 (PvE)** | 複数人 vs システム | WoW |
| **1対多** | 1人 vs チーム | Scotland Yard |
| **フリーフォーオール** | 複数独立競争者 | モノポリー |
| **チーム対戦** | グループ同士の競争 | スポーツ、ブリッジ |
| **プレデター・プレイ** | 円形構造で隣を攻撃 | Assassination |

## 目標タイプ (8分類)

| タイプ | 例 |
|--------|-----|
| 捕獲/破壊 | チェス |
| 領土支配 | RISK, Diplomacy |
| コレクション | 花札, Bohnanza |

| 空間整列 | テトリス |

| 否定（禁止行動の回避） | ツイスター, ジェンガ |

## ルール分類 (3カテゴリ)

| カテゴリ | 説明 |
|---------|------|
| **操作的ルール (Operational)** | プレイヤーが従う可視的手順 |
| **構成的ルール (Constituative)** | 表面メカニクスの下にある抽象的論理 |
| **暗黙的ルール (Implied)** | 書かれていない社会的合意（時間制限、スポーツマンシップ） |

## リソースとリソース管理

プレイヤーが制御する要素: 通貨/HP、領土所有、保有情報、利用可能時間、収集オブジェクト

## 情報構造

| タイプ | 説明 | 例 |
|--------|------|-----|
| **完全情報** | 全プレイヤーが全て見える | チェス |
| **私的情報** | 隠された手札 | ポーカー |
| **特権情報** | 1プレイヤーの秘密知識 | — |
| **隠蔽情報** | 全プレイヤーに未知の事実 | Clue |
| **戦場の霧** | 隠されたマップセクション | RTS |

## ターン構造

- **ターンベース**: 1プレイヤーが行動し交代
- **同時**: 複数プレイヤーが同時行動
- **リアルタイム**: アクションが連続発生
- **可変ターン順**: 条件に基づく動的順序

## 3ステップ分析フレームワーク

| ステップ | 目的 |
|---------|------|
| **1. 記述** | 形式要素を解釈なしに客観的に記述 |
| **2. 分析** | 相互作用と結果のプレイ体験の有効性を分析 |
| **3. 検討** | デザイナーの選択と代替可能性を検討 |

## 設計原則

- **ゲームはシステム**: 要素間の相互依存を理解する。1つの変更がカスケード効果を持つ
- **プレイが理解を明かす**: 理論的分析より直接体験がメカニクスの相互作用を実証
- **意図的要素選択**: 各コンポーネントが意図的設計目標に寄与。未定義の形式要素は凝集性の欠如を生む
- **フィードバック**: 明確な知覚可能結果がプレイヤーのアクションと帰結を結びつける
- **プレイヤーの意図**: 計画の立案と実行を許可することがエンゲージメントを強化
