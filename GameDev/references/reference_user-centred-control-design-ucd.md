## UCD×コントロール設計の方法論

著者: Samantha Stahlke、2017年10月9日、Game Developer Featured Blog

コントロール設計は実装前から「ユーザー中心設計（User-Centered Design / UCD）」の4フェーズで構造化すべき。

---

### UCD 4フェーズの適用

| フェーズ | 内容 |
|---|---|
| ①理解 | ターゲットユーザーの特性分析（年齢層・スキルレベル・好みのジャンル等） |
| ②要件定義 | 5カテゴリの要件を明確化（下表） |
| ③プロトタイピング | 複数のコントロールバリアントを設計・比較 |
| ④評価 | アルファテスト＋ゲームプレイ録画＋アンケート＋インタビュー |

---

### 5カテゴリの要件定義フレームワーク

| 要件種別 | 内容 |
|---|---|
| 機能要件 | 移動・ジャンプ・オブジェクト操作・カメラ制御の入力マッピング |
| 非機能要件 | キーボード/ゲームパッド対応、レスポンシブ性と流動性の重視 |
| データ要件 | カメラ・状態・物理・インタラクティブオブジェクト位置との連携 |
| コンテキスト要件 | 熟練プレイヤーがデバイスを見ずに操作できること |
| ユーザビリティ要件 | 存在感を消せる・学習しやすい・ゲームプレイを強化する操作感 |

---

### 3種類のコントロールバリアント設計事例

**① 人型キャラクター**

- 速度直結マッピング（アニメーション加速度に同期）
- 「即座に方向転換」の実現（厳密な物理法則より操作感優先）
- Fitts則的にタイトな設計

**② 物理オブジェクト（ボール・球体）**

- 速度ではなく力（フォース）を適用
- ジャンプ・衝突を通じた角運動量の保持
- テストプレイで「物理的なリアル挙動」を好む傾向が確認された

**③ 飛行体（紙飛行機）**

- 乗り物系とフライトシミュレーター系のハイブリッドアプローチ
- スロットル＋ヨーを通常移動軸にマッピング
- ピッチ＋ロールをカメラ制御軸に解放

### 実践的知見

「UCDアプローチは設計品質とチーム効率を同時に向上させた」。実装前の要件定義フェーズへの投資が後工程のコストを大幅に削減する。コントロール設計を「作りながら考える」のではなく、HCI手法を借用して設計プロセス自体を構造化することで、バリアント間の比較・評価が可能になる。
