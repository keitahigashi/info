## レベルデザインの基礎

### レベルとは何か

- **定義**：プレイヤーのインタラクションのための舞台（ステージ、マップ、ミッション等とも呼ぶ）
- レベルはプレイヤーに課題（戦闘・パズル・探索など）を提示する空間
- デザイナーは「プレイヤーが活動する領域」を意図的に設計する

### プレイスペース設計の原則

**アフォーダンス理論 (Affordance Theory) の活用**

物体の視覚的特性からプレイヤーが「何ができるか」を直感的に理解できるよう設計する：

- ドアは「通れる」、壁は「通れない」ことを見た目で伝える
- インタラクト可能なオブジェクトは非インタラクト可能と視覚的に区別する
- 一貫したビジュアル言語 (visual language) を全レベルで維持する

**視覚的誘導テクニック**

| テクニック | 説明 |
|-----------|------|
| カラーコーディング | 重要オブジェクト・危険地帯・目標を色で区別 |
| 照明 (lighting) | 明るいエリアは進行ルート、暗いエリアは危険を示唆 |
| リーディングライン (leading lines) | 建物の形・道・フェンス等で視線を目標方向へ誘導 |
| ウェイポイント/サイン | 迷子防止のための標識・マーカー |

### Show, Don't Tell の原則

- 「何をすべきか」は視覚的に示す
- 「どうやるか」はプレイヤーに発見させる
- 優れた誘導はプレイヤーが「自分で気づいた」と感じさせる

### ナビゲーション設計

- 適切なナビゲーションはプレイヤーからコントロールを奪わない
- プレイヤーが「どこに行けばよいか」を常に把握できる設計が基本
- 複雑さを増すときは新しい視覚言語を段階的に導入する

### レベルプログレッション

```
簡単な課題 → スキル習得 → より難しい課題 → 習熟確認 → 次のスキル導入
```

- 段階的な難易度上昇でプレイヤーの準備不足を防ぐ
- 新メカニクスは安全な環境で紹介し、危険な状況で試させる（サンドボックス→実戦）
- ペーシング制御：高強度と低強度を交互に配置し緊張と緩和を生む

### 空間の読みやすさ (Spatial Readability)

- レイアウトはトップダウン視点での論理性より、プレイヤー視点での明瞭さを優先
- 「500回プレイしたデザイナー」と「初見プレイヤー」の視点ギャップを常に意識する
- プレイテストで「迷子ルート」を特定し、視覚的手がかりを追加する
