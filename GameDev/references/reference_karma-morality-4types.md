## カルマシステムの4つの主要タイプ

### 1. バイナリ善悪システム — inFamousシリーズ

| 項目 | 内容 |
|------|------|
| 構造 | 「ヒーロー」vs「インファマス」のスライダー |
| 利点 | 開発側が2つの明確な経路にコンテンツを集中可能 |
| 欠点 | 実質的な選択肢の幻想。動的なストーリー展開がない |

プレイヤーが初期選択で道徳的方向に固定される傾向がある。

### 2. 独立スケール型 — Mass Effectシリーズ

| 項目 | 内容 |
|------|------|
| 構造 | 「パラゴン (Paragon)」と「レネゲード (Renegade)」を別々のスケールで測定 |
| 利点 | 道徳的に複雑な決定が可能。真のロールプレイを実現 |
| 欠点 | 膨大なコンテンツ制作が必要。矛盾のない物語の維持が困難 |

善と悪が相殺されず、両方を蓄積できる設計。

### 3. 承認システム — Dragon Age: Origins

| 項目 | 内容 |
|------|------|
| 構造 | パーティメンバーごとに -100〜+100 の独立した評価スコア |
| 利点 | 個人的な対人関係に基づいた決定。より人間的で現実的 |
| 欠点 | パーティメンバー制度を持つゲームに限定される |

「善悪」ではなく「この人はあなたをどう思うか」を軸にした設計。

### 4. 統合的選択 — 複数ゲーム

| 項目 | 内容 |
|------|------|
| 構造 | ゲーム終盤の単一の重大な決定でエンディングが変わる |
| 利点 | 実装が軽量。少ないコンテンツで大きなストーリー変化 |
| 欠点 | プレイヤーの実際の影響は物語の最後のみ。ロールプレイ機会が限定的 |

## 設計上の主要な考慮点

| 要素 | 説明 |
|------|------|
| **プレイヤーエージェンシー** | 選択がストーリーに影響する度合い |
| **コンテンツ負荷** | 実装に必要な開発リソース |
| **物語の一貫性** | 複数の選択肢を統合した際の説得力 |

## タイプ別の適用指針

| プロジェクト規模 | 推奨タイプ | 理由 |
|----------------|-----------|------|
| 大規模AAA | 独立スケール型 or 承認システム | リソースを投入して深い分岐を実現可能 |
| 中規模 | バイナリ型 | 2ルートに集中してコンテンツ密度を確保 |
| 小規模インディー | 統合的選択 | 最小コストで意味あるエンディング分岐 |

## 結論

完璧なシステムはなく、ゲームの予算規模・目的・ジャンルに応じて最適なアプローチを選択すべき。複数のアプローチを組み合わせるハイブリッド設計も有効。
