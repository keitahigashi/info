## アップグレードシステムの3機能

| 機能 | 説明 |
|------|------|
| **心理的報酬** | 「安定したドーパミンの点滴」— スケジュールされた進行報酬 |
| **プレイヤー表現** | プレイスタイルとキャラクター能力のカスタマイズ |
| **意思決定** | 異なる選択肢間の有意義な選択の創出 |

## 面白い選択の4属性

### 1. 異なる & バランスが取れている
- 選択肢は質的に異なりつつ等しく実用的であるべき
- **事例**: 宇宙シューター — ミサイル（即時だが射程制限）vs レーザー（遅いが全ダメージ）
- 異なるメカニクスが異なる戦略を要求する設計

### 2. 選択肢を限定する
- 過度な選択肢は「選択の麻痺」(Choice Paralysis) を生む
- 「36種類の石鹸から1つ選ぶ」状況 → 豊富さが能力を麻痺させる
- 3-5の明確に差別化された選択肢が最適

### 3. 明確性
- 各選択肢の機能が透明であること
- **悪い例**: 「+15 Finesse」（曖昧な統計値）
- **良い例**: 「クリティカルヒット確率+15%」（明示的な効果）

### 4. 不可逆性（コミットメント）
- 選択が元に戻せない場合、決断の重みが増す
- ただし不可逆性が高すぎると実験を阻害

## 装備システムの特性

| 特性 | 説明 |
|------|------|
| **売却可能性** | 部分的な払い戻しでコミットメントの不安を軽減 |
| **共有可能性** | パーティメンバー間でアイテム移転 → 全体投資の感覚 |
| **具体性** | 物理的アイテムは抽象的ステータスより報酬感が高い |
| **非累積性** | 単一スロット制限で装備タイミングの意思決定を創出 |

## プログレッション曲線のバランス

### カスタマイズによる深み
- プレイヤーとモンスターが同一ペースでレベルアップ → ゲームプレイが静的
- プレイヤーに特化の自由を許可 → 「リプレイ価値と多様なプレイスタイル」を創出
- 強みと弱みの特化 → 適応的な戦術が必要

### パワースケーリング
- プレイヤーと敵の相対的なパワー推移を監視
- 不均等な進行速度 → ゲームが「簡単すぎるか難しすぎる」リスク
- インフレ制御が長期的バランスの鍵

## 設計上のテイクアウェイ

- アップグレードは「数値的強化」ではなく「プレイスタイルの変化」を提供すべき
- 選択肢の「量」より「質的差異」が重要
- 明確性とコミットメントのバランスが鍵
- 装備の具体性（物理アイテム）はスキルツリーの抽象性を補完する
