## ディエジェシスとは

**ディエジェシス (diegesis)** は古代ギリシャの劇場に由来する概念。ゲーム設計においては「ゲーム世界の内側に存在するか、外側に存在するか」を分類するためのフレームワーク。

| 分類 | 定義 | 例 |
|------|------|-----|
| **ディエジェティック (diegetic)** | ゲーム世界内に存在し、キャラクターが認識可能 | Dead Space のスーツ投影 HUD |
| **ノンディエジェティック (non-diegetic)** | プレイヤー専用の情報で、ゲーム世界には存在しない | 抽象的な体力バー、ポーズメニュー |
| **メタ的 (meta)** | ゲーム世界との曖昧な境界 | バイオハザードのタイプライター保存 |

## 具体的ゲーム事例

### GoldenEye 007
- ボンドの腕時計がインベントリ・メニューとして機能
- **効果**: 「キャラクターが操作デバイスを認識している」という一体感
- プレイヤーの UI 操作がゲーム世界のアクションとして描写される

### Half-Life
- HEV スーツ装備時のみ HUD が表示される
- **効果**: キャラクターとプレイヤーが同じ視覚情報を共有
- スーツを脱いだ状態では HUD が消える論理的一貫性

### Overwatch
- キャラクター武器の弾数表示・能力クールダウンを武器自体に組み込み
- **効果**: パーソナリティ表現とゲームフィードバックの両立
- ゲーム世界に「存在する」情報としてプレイヤーに伝達

### Portal
- エレベータ移動時、ゲーム内音楽が物理的距離に応じて減衰
- **効果**: 音響要素が現実の物理法則に従うことで世界の真実性が向上
- プレイヤーが「世界に存在している」感覚を強化

### Dead Space（最高度のディエジェティック設計）
- 全 HUD 要素（体力・弾薬・マップ・インベントリ・メニュー）がスーツの投影として実装
- **効果**: ゲームプレイ中に UI のために没入感が途切れない
- ゲーム世界の外から「情報を読む」感覚が皆無

## 設計パターン

| パターン | 目的 | 実装例 |
|---------|------|--------|
| **ディエジェティック切り替え** | 感情的重要シーンでの没入感最大化 | HP 低下時の視覚・音響フィルタ変化 |
| **ローディング隠蔽** | トランジションの断絶防止 | RE2 の門開閉アニメーション |
| **キャラクター状態の視覚反映** | 非言語フィードバック | 低 HP 時の歩行アニメーション変化 |
| **制限的な保存機能** | 緊張感の生成 | タイプライター・キオスクによる保存 |

## 注意点: ディエジェシスは品質を保証しない

ディエジェティック = 高品質ではない。

- **戦略ゲーム** (Europa Universalis IV 等): ノンディエジェティック UI が大量の情報を効率的に伝達し、ゲームとして成功
- **VR 体験**: ディエジェティック要素が効果的（手元に UI を配置等）
- **カジュアルゲーム**: シンプルなノンディエジェティック UI が学習コストを下げる

> 「ディエジェシスの適用基準はゲーム体験の向上である」

## 選択の判断基準

1. **ゲームのトーン・ジャンル**: ホラー・没入型 RPG はディエジェティック UI が向く
2. **情報密度**: 複雑な情報はノンディエジェティックの方が読み取りやすい
3. **プレイヤーの認知負荷**: ディエジェティック UI は「見つける」コストが発生する
4. **一貫性**: 混在させる場合は明確なルールが必要（何がディエジェティックかを定義）

## ナレッジベース活用メモ

- 既存の「4種類のゲーム UI」フレームワーク（reference_game-ui-4types.md）と組み合わせると UI 設計の判断基準が強化される
- ディエジェシスの概念は UI だけでなく、効果音・音楽・ナラティブ演出にも応用可能
- 「没入感を上げる」という曖昧な目標を「ディエジェティック要素を増やす」という具体的な設計指針に変換できる
