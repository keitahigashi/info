## 垂直レイヤリング (Vertical Layering)

楽曲を複数の独立したステム（ストリングス、ドラム、ボーカル等）に分解し、リアルタイムでオン/オフを切り替えて強度とムードを調整。

**動作例**: 潜入中はピアノレイヤーのみ → 発見されるとドラム・ブラスレイヤーがフェードイン

| 項目 | 内容 |
|------|------|
| **強み** | リアルタイム適応性、スムーズな遷移、感情的エンゲージメント |
| **課題** | メモリ消費が大、全レイヤー組み合わせの整合性確保が複雑 |
| **適したケース** | 戦闘、探索、ダイナミックな瞬間 |

## 水平リシーケンシング (Horizontal Resequencing)

ゲームプレイの進行に基づいて、個別の楽曲トラックやセクション間を切り替え。

**動作例**: ボス登場で緊張感のあるイントロ → バトル開始でテンポの速いアクションテーマに切り替え

| 項目 | 内容 |
|------|------|
| **強み** | リソース消費が低、明確なナラティブ進行、プラットフォーム最適化に有利 |
| **課題** | 遷移が唐突になるリスク、急変するゲーム状態への柔軟性が低い |
| **適したケース** | ボス戦、カットシーン、ナラティブビート |

## ミドルウェアツール

| ツール | 特徴 |
|--------|------|
| **FMOD** | ドラッグ&ドロップの直感的インターフェース |
| **Wwise** | 複雑なプロジェクト向けの高度な機能 |
| **Elias** | アダプティブオーディオ特化 |

ゲーム変数（プレイヤーHP、敵の近接度等）と音楽変化をリンクするリアルタイム調整が可能。

## ハイブリッドアプローチ

「最も魅力的なサウンドトラックは垂直レイヤリングと水平リシーケンシングをブレンドする」。水平リシーケンシングでセクション間を切り替え、各セクション内では垂直レイヤリングで強度を動的に調整。

## 実例

| タイトル | 手法 |
|---------|------|
| **Journey** (2012) | 垂直レイヤリング — 楽器がシームレスにブレンドし流動的な感情遷移 |
| **Celeste** (2018) | 水平リシーケンシング — 環境やチャレンジに応じたサウンドトラック適応 |
| **Hollow Knight** (2017) | ハイブリッド — エリアテーマ管理に水平、戦闘強化に垂直 |
