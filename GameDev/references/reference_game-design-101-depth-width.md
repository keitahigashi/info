## 概要

「Depth（深み）」と「Width / Complexity（幅/複雑さ）」という2軸でゲーム設計を評価するフレームワークを提示する。

## Depth（深み）の定義

- プレイヤーがゲームプレイ全体で行う**意思決定の数と頻度**
- プレイヤーがメカニクスをどれだけ発見・探索・創造的に組み合わせられるかの尺度
- **Overwatchの例**: ヒーロー選択、移動方向、アビリティタイミング、ポジショニングなど毎秒膨大な意思決定が発生
- **スーパーマリオの例**: 敵接近時にキノコを取りに行くか否か、コインを取るかなど継続的なリスク管理

## Width（幅/複雑さ）の定義

- プレイヤーが**記憶・理解しなければならないルールやメカニクスの量**
- ゲームに最初から必要な知識・システム・慣習の総体
- 「Widthはゲーム世界のルールと慣習に関するもの。プレイヤーが覚えなければならないもの」

## バランスの重要性

- **浅すぎる（Depth不足）**: 発見の機会が限られ、プレイヤーはすぐに飽きる
- **複雑すぎる（Width過多）**: プレイヤーは圧倒されてゲームを早期に諦める
- 結論: 「Depth と Width のバランスが成功するゲームプロジェクトの第一の鍵」

## 実践的含意

- メカニクスを追加する際は、Depthを増やすのかWidthを増やすのかを意識する
- Widthを増やさずにDepthを増やすことが理想的な設計方向
- プレイヤーが常に新しいことを発見できる設計がリテンションに直結する
