## アクティブラグドール (active ragdoll) とは

物理シミュレーションとアニメーションを組み合わせたキャラクター制御システム。剛体（リジッドボディ）ベースの関節に力を加えてポーズを維持・動作を生成する。

通常のアニメーション（キーフレーム主体）と異なり：
- 外部からの力・衝突に**リアルタイムで物理的に反応**する
- プロシージャル（手続き的）なモーション生成が可能
- 格闘ゲーム・物理サンドボックス・VRに適する

## 8つの実装テクニック

### 1. 基本的な歩行アニメーション

単純な力の衝撃とホバー機能だけで歩行アニメーションを実装できる：
- 胴体を一定の高さに保つ「ホバー力」を適用
- 脚部への周期的な前後力で歩幅を生成
- IK（逆運動学）なしでも基本的な二足歩行を表現可能

### 2. パンチ/打撃アニメーションの重量感

「フィストの質量を数フレーム増加させると、身体全体がその動きに追従する」：
- ヒット直前に腕の物理質量を一時的に増加
- 慣性により胴体・肩が引っ張られる連鎖動作が生まれる
- ゲームフィール (game feel) の「重さ」を物理的に演出

### 3. ドラッグと重量の一元管理

全身部位のドラッグ値・質量をマネージャースクリプトで一括管理：
```
利点：
・各部位を個別調整する代わりに全体スケールで調整可能
・プロトタイプ段階での高速イテレーション
・プレイテスト後の微調整コストを大幅削減
```

### 4. カスタム重力の実装

リジッドボディの標準重力設定ではなくカスタムスクリプトで制御：
- **質量中心**の位置を調整することで安定性・転倒挙動が変化
- **重力強度**をキャラクタータイプ別に設定（重いキャラ vs 軽いキャラ）
- 標準重力では再現しにくい「独自の重さの感覚」を実現

### 5. 垂直・水平ドラッグの分離

水平方向と垂直方向のドラッグを**別々に制御**する：
- 垂直ドラッグを低く設定 → 落下がリアルで自然に見える
- 水平ドラッグを高く設定 → 滑らない・コントロール感が向上
- この分離により「落下感はリアルで操作感は良い」という両立が可能

### 6. 落下速度の基準比較

大きな変更後は他のオブジェクト・キャラクターと比較して速度感を確認：
- 比較基準なしでは人間の知覚が「正常」を判断しにくい
- 現実物理との乖離が大きい場合はプレイヤーに違和感を与える
- イテレーションごとにビフォー/アフターを記録する

### 7. ストリングパペット (string puppet) 設計

初心者向けの簡略化アプローチ：
```
ストリングパペット構造：
  胴体（物理制御の中心）
    ├─ 腕（視覚的IKで接続）
    ├─ 脚（視覚的IKで接続）
    └─ 頭（胴体に追従）
```
- 複雑な物理計算を省いて「それっぽく見える」動きを安価に実装
- フルラグドールが不要なゲームジャンルに適する

### 8. 逆運動学 (IK) との統合

複雑なバランス維持・歩行動作には IK を物理システムと統合：
- **足の接地 IK**：不規則な地形でも足が浮かない
- **腕のリーチ IK**：物体の掴みやオブジェクトとの相互作用
- **バランス IK**：重心移動に応じた姿勢補正

## 汎用化ポイント

この記事の実装例はゲームエンジン固有の API を使用しているが、以下の概念はエンジン非依存で応用できる：

| 汎用概念 | 適用範囲 |
|---------|---------|
| 質量・ドラッグの動的変更でゲームフィールを演出 | すべての物理エンジン |
| 垂直/水平ドラッグの分離による操作感と視覚の両立 | 物理ベースキャラクター一般 |
| カスタム重力による個性的な重さ表現 | 物理シミュレーション全般 |
| IK と物理シミュレーションの組み合わせ | プロシージャルアニメーション全般 |
| マネージャーによる一括調整でイテレーション高速化 | 物理パラメータ管理全般 |

## 関連アーキテクチャ

- 通常アニメーション（キーフレーム）からのブレンド移行でアクティブラグドールを自然に組み込む
- ヒット時・死亡時に「通常アニメーション→物理ラグドール」へシームレスに切り替える
- EA Frostbiteエンジンの「ドリブンラグドール (driven ragdoll)」はアニメーションに従いながら物理反応する代表例
