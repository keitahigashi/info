## 基本概念：情報とその表現性

ゲームUIデザインには2つの軸がある：

- **情報レベル (information level)**：何を伝えるか（HP値、地図、ダメージ数値など）
- **表現性 (expressiveness)**：どう伝えるか（サイズ・色・動き・スペーシング）

「優れた情報設計」の2条件：
1. 主要情報と副次情報の適切なコントラスト
2. 情報モジュールの明確な階層構造

**情報構造が複雑になるほどユーザーの理解時間が増加する**。デザイナーは意図的に複雑さを管理する必要がある。

## 5つのデザイン戦略

### 1. 要素サイズの計画

フォントサイズ・太さを変化させて重要度を表現する。

- **主要情報**：大きく、太く
- **副次情報**：小さく、細く
- **補足情報**：最小サイズで表示

重要な指標（残りHP・スコア）は大きなフォント、説明テキストは小さく配置することで自然な視線誘導が生まれる。

### 2. グラフィック要素の活用

アイコン・背景・枠線・区切り線などでグループ化と強調を実現する。

- 関連情報をグループ化するコンテナ要素
- 重要要素を際立たせる背景色や枠線
- セクション間の明確な区切りで情報間の関係性を示す

### 3. 色彩の戦略的活用

**60-30-10の色彩配分原則**：
- **60%**：メインカラー（背景・主要エリア）
- **30%**：補助カラー（サブ要素・中間強調）
- **10%**：アクセントカラー（最重要情報・CTA）

色は階層の補強に使う。色だけで情報を伝えることは避け（アクセシビリティのため）、サイズ・形状との組み合わせで多重表現を実現する。

### 4. スペーシング（余白設計）

ホワイトスペース (white space) による情報グループの分離：

- 関連する要素は近くに配置（近接の法則）
- 独立した情報ブロックは余白で区切る
- 密集した情報は認知負荷を高める——余白は情報の一部

### 5. 動的効果（アニメーション遷移）

状態変化・フィードバック・誘導にアニメーションを活用：

- HPが減少する際の数値アニメーションで変化を強調
- 重要なイベント発生時のフラッシュ・パルス効果
- メニュー遷移のスムーズなモーションで空間関係を示す

## 情報設計と視覚表現の協調

情報設計（何を配置するか）と視覚表現（どう見せるか）は独立した作業ではなく、相互に補強し合う。

設計フロー：
```
情報の洗い出し → 優先度付け → 視覚的階層へのマッピング
     ↓                            ↓
 何が最重要か         サイズ・色・スペーシング・アニメで表現
```

## 適用例

| 情報タイプ | 推奨表現 |
|-----------|---------|
| 残りHP（最重要） | 大きなバー + アクセントカラー + 減少アニメ |
| スコア（重要） | 中サイズ + セカンダリカラー |
| コンボ数（付加情報） | 小サイズ + パルスアニメ |
| ヒント・説明 | 最小サイズ + 低コントラスト |
