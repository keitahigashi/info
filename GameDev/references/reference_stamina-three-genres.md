## スタミナの基本定義

プレイヤーが消費でき、時間とともに回復する資源。パラメータ（回復速度、最大値）の調整により異なるプレイ体験を生成する。

## 3ジャンル別分析

### 1. アクションゲーム（戦闘スタミナ）

| 項目 | 内容 |
|------|------|
| 用途 | スプリント・回避・攻撃などの高価値アクションに消費 |
| 機能 | 攻防の共有リソースによる戦術的深さの創出 |
| テンション | 「攻撃すれば防御不可」という緊張感の生成 |
| 時間スケール | 秒〜分 |
| 回復速度 | 高速（使用時のみ消費） |

**設計課題**:
- 最大値が高すぎると → スタミナが無意味化
- 回復が遅すぎると → 待機の強要
- 対象アクションが多すぎると → 選択過載

### 2. カジュアル/モバイルゲーム（ゲーティングスタミナ）

| 項目 | 内容 |
|------|------|
| 用途 | ダンジョン進入やパズル試行に消費される準有限資源 |
| 機能 | プレイ速度の制御、コンテンツ展開スケジュール調整 |
| 時間スケール | 時間〜日 |
| 回復速度 | 極低速（リアルタイム待機） |

**倫理的課題**:
- 「空のスタミナ = アクセス拒否」→ プレイヤーを排除
- 「満杯 = 進捗機会の喪失」→ ログイン圧力
- **二重心理操作**: 空でも満杯でもプレイヤーにストレス
- マネタイズと結合時に**略奪的**になるリスク

### 3. サバイバルゲーム（食料時計）

| 項目 | 内容 |
|------|------|
| 用途 | 時間経過で継続減少、消費アイテムで回復 |
| 機能 | 長期的生存ストレス管理、敗北への加速的接近 |
| 時間スケール | 十分〜時間 |
| 回復速度 | 負の回復（継続減少） |

**設計難度**:
- 敗北条件と直結 → 設定誤りが致命的
- 消費速度が速すぎる → ゲーム不可
- 消費速度が遅すぎる → 緊張感の喪失

## パラメータ設計の比較表

| パラメータ | アクション | カジュアル | サバイバル |
|----------|----------|----------|----------|
| 時間スケール | 秒〜分 | 時間〜日 | 十分〜時間 |
| 回復方向 | 正（自動回復） | 正（極低速） | 負（継続消費） |
| 最大値の感度 | 高（無効化リスク） | 低 | 高（致命的リスク） |
| 敗北との接近度 | 中 | なし | 高 |

## 成功事例: Eco

統一スタミナ「カロリー」システムが3ジャンルの機能を同時実装:

1. **食料時計**: 時間経過で消費、食料で回復
2. **全アクション統合**: スプリント〜木こり〜建設まですべて消費
3. **進捗ゲート**: 過去24時間の栄養平均がスキルポイント獲得を制御

### 創発的効果
- スタミナ消費速度 × 栄養価の選択 = **共有資源の競争**
- 「共有資源の悲劇」が自然発生 → 協力の動機づけ
- 単一メカニクスが経済・社会・進行の3層に影響

## 設計原則

- 3ジャンルの設定は**相互排他的でなく組み合わせ可能**
- パラメータ調整の影響度はジャンルごとに非対称
- **敗北条件との接近度**が設計難度を決定
- 倫理的責任とメカニクスは不可分（特にF2Pモデル）
