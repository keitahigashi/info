## 定義

ステルスエンカウンターとは「プレイヤーが敵を排除するか、敵をすり抜けることを促されるゲームの孤立したセクション」。

## 設計上の2大課題

1. **セットアップの複雑性**: ステルス空間は多層的な要素の調整が必要
2. **二重設計の要求**: ステルスシステムと戦闘システムの両方をサポート

## 安全な開始条件

プレイヤーに必要な初期条件:
- 視覚検出前に聴覚的手がかり
- 明確なしゃがみ/ステルスアニメーション状態
- 明白な初期隠れ場所
- 障害のない敵観察ポイント

## 敵配置の哲学

| 原則 | 説明 |
|------|------|
| 周辺敵は簡単 | 目標近くの敵ほど難しい配置 |
| 意図的な間隔 | 「断末魔」の連鎖反応を防止 |
| パトロール一時停止 | エンゲージメント窓を創出 |
| 対話でのテレグラフ | 移動パターンを会話で予告 |

## 空間設計原則

- **垂直性が平面レイアウトに勝る**: 高低差が戦術の幅を拡大
- カバー配置が意図された解法パスを通じてプレイヤーを誘導
- 観察と忍耐を報いる設計

## エンカウンター4分類

### 1. Outpost（前哨基地型）
- プレイヤーの自信が支配的
- 柔軟な進入点とリセット能力
- **感情**: 支配感・計画の快感

### 2. Limited Approach（限定進入型）
- 制約された進入方法
- 中程度のテンション
- **感情**: 緊張と解決の達成感

### 3. Hunted（追跡型）
- 力学の逆転 — プレイヤーが追われる側
- 脆弱性の増加がステークスを上昇
- **感情**: 恐怖・切迫感

### 4. Hide or Die（隠れるか死ぬか型）
- 最大脅威レベル
- 生存メカニクスに焦点
- **感情**: 極限の緊張・安堵

## ペーシング戦略

各タイプが異なる感情応答を生成するため、**タイプの交替**がペーシングの多様性を実現。連続する同タイプのエンカウンターは単調さを招く。

## 事例

- Tomb Raider (2013): 垂直性の活用
- Hitman: 柔軟なOutpost設計
- Far Cry 3: カバーベースナビゲーション
- Assassin's Creed: 垂直性とステルスの統合
