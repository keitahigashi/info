## 汎用化ポイント

本記事は Shattered Planet (Kitfox Games, 2014) というインディーゲームを事例とするが、プロシージャル生成を採用するすべてのゲームに適用可能なレベルデザイン設計原則を扱う。エンジン固有の実装詳細は抽象化して記載。

## 伝統的 LD 原則をプロシージャルに移植する

### 設計3本柱の策定

プロシージャル生成フロアのすべての判断基準として、事前に設計3本柱を定義:

| 柱 | 定義 | 実装への示唆 |
|----|------|------------|
| 探索への好奇心 | 探索を報酬で強化する | サイドルーム・隠し通路に価値を置く |
| 戦略的選択 | 意味ある判断を支援する | リスクとリワードの分岐ポイントを生成 |
| 即座の没入 | フィードバックで素早く参加感を与える | プレイヤー行動への即応レスポンスを設計 |

### クリティカルパス (Critical Path) の実装

プロシージャル生成でも「進行の最低保証」が必要:

```
スタート位置 → 必須ルーム群 → 出口テレポーター
```

このパスに:
- **コイントレイル**: クリティカルパスに沿って戦略的報酬を配置
- **中程度の敵難易度**: 挫折しない程度の緊張感

オプションの**サイドルーム**:
- 高リスク・高リワード（上位アイテム、難敵）
- 参加は任意 → プレイヤーの自律性と戦略選択を担保

### 部屋生成の構造設計

均一な矩形より有機的形状を生成する手法:

```
各部屋 = 
  可変サイズ（最小〜最大タイル数の範囲設定）
  + 内部障害物・「湖」（アルゴリズムで配置）
  + テーマ別タイル・装飾セット
  + パスファインディング検証（障害物が通路をブロックしていないかチェック）
```

### ロック＆キー (Lock & Key) システムの統合

生成アルゴリズムにロック＆キーを組み込む方法:

- 鍵カード（キーカード）をランダムな部屋に配置
- 対応する施錠ドアをオプション宝部屋の入口に設置
- クリティカルパスのドアには鍵を不要とし、詰まりを回避

### 多重経路と行き止まりの削減

初期実装では行き止まりが多発 → プレイヤーが迷う問題

対策: **代替接続ルートの追加**
- 部屋間に複数接続点を生成
- 行き止まりのリスクを分散
- 探索の選択肢を増やす

### 新アビリティ導入とルール更新

ゲームが進化してアビリティ（火・テレポート等）が追加されるたびに、生成ルールを更新:

```
新アビリティ追加時:
  既存の生成ルールを再評価
  → 新アビリティが解決・活用できる障害物タイプを追加
  → テストで意図しない詰まり・スキップを検証
```

## 実践的な教訓

### テストの特殊性

プロシージャル生成のバグ特定は通常開発より困難:

- 同じ状況を再現するには、複数回生成して同一パターンを待つ必要がある
- **大量生成→評価フィルタ** のアプローチが有効: 数千フロアを生成してエッジケースを自動検出

### 「完成は終わりではない」

リリース後も:
- ゲームバランスの調整が生成ルールに反映
- 新コンテンツ追加が生成システムの拡張を促す
- 小規模チームでは PCG が反復速度と多様性を同時に担保する強力なツール

## まとめ

| 伝統的 LD 原則 | プロシージャル実装 |
|---------------|-----------------|
| クリティカルパス | 必須ルームチェーンの強制生成 |
| リスクリワード | オプションサイドルームへの高価値アイテム配置 |
| ロック＆キー | アルゴリズム的な鍵・施錠ドアの分散配置 |
| 難易度曲線 | パス上の敵難易度 vs. サイドルーム高難度 |
| パスファインディング保証 | 生成後の通路疎通チェック |
