## 中心的主張

現在のゲームデザインは「錬金術」のような試行錯誤の段階にあり、**再現可能・テスト可能な科学的モデル**への転換が必要。

**プレイヤーモデルの核心**:
> 「プレイヤーは認識価値の高いスキル習得を本能的に求める存在」

## スキルアトム (Skill Atom) — ゲームの基本単位

あらゆるゲームプレイは4段階のフィードバックループで構成される:

| 段階 | 内容 |
|------|------|
| **アクション (Action)** | プレイヤーが行動を実行 |
| **シミュレーション (Simulation)** | ゲーム世界が反応 |
| **フィードバック (Feedback)** | 視覚・音声情報をプレイヤーに提供 |
| **モデリング (Modeling)** | プレイヤーが因果関係を学習・統合 |

**例：マリオのジャンプ**
1. Aボタンを押す → 2. アバターが上昇する → 3. ジャンプアニメが再生される → 4. プレイヤーが「ボタン→上昇」の因果を理解

## スキルチェーン (Skill Chain)

複数のスキルアトムを連鎖させた有向グラフ。テトリスのように「シンプルに見えるゲーム」でも、実際は多数のアトムから構成される。

**プレイヤーの習得進捗状態**:

| 状態 | 定義 |
|------|------|
| 習得済み | スキルを完全に理解している |
| 部分習得中 | 実験的に習得を進行中 |
| 未実行 | まだ試していない |
| アクティブ | 他のスキル習得の手段として利用中 |
| バーンアウト | 興味を失った（警告シグナル） |

## バーンアウト (Burnout) の診断

プレイヤーがスキルの実行を継続しなくなる現象で、設計品質の定量的指標として活用できる。

- **早期バーンアウト**: 基礎スキルの習得失敗 → 後続スキル習得不可 → 体験全体が崩壊
- **後期バーンアウト**: 新スキルの実用性が不明 → 関連スキルが衰退

## 事前スキル (Prior Skills) と赤いニシン (Red Herrings)

**事前スキル**: プレイヤーが既に保有する現実世界・他ゲームのスキル
- 不足時 → フラストレーション
- 過剰時 → 退屈
- 正確な設定がチュートリアル設計の核心

**赤いニシン**: 実用性はないが快感や没入感を与える装飾要素
- 物語・雰囲気演出
- 既存知識との連想結合
- 例：スーパーマリオのキノコ（実際の薬効は不要、「取ったら何かが起きる」という期待感を演出）

## 設計ツールとしての活用

スキルチェーン図を描くことで以下が実現できる:

1. プレイヤーに必要な基礎スキルの明確化
2. 快感・達成感が生まれるポイントの特定
3. バーンアウトが発生しやすい箇所の可視化
4. 反復的な開発プロセスへの定量的フィードバック

## 核心的洞察

ゲームデザインに「化学」をもたらすとは、直感と経験だけに頼るのではなく、**学習サイクルを分解・計測・改善できる共通言語を持つこと**。スキルアトムは、ゲームプレイの最小単位として設計の議論を明確にする。
