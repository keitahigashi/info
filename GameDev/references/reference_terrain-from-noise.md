Red Blob Games (Amit Patel): https://www.redblobgames.com/maps/terrain-from-noise/

## コアコンセプト

ノイズ関数がマップ座標に0.0〜1.0の値を割り当て、グレースケールパターンを生成。異なるノイズライブラリ（Simplex/Perlin/OpenSimplex）は類似結果を生むが出力範囲は異なる。

## 周波数とオクターブ

### 周波数 (Frequency)
振動密度を制御。高周波数 = 小さな地形特徴、低周波数 = 大きな特徴。周波数を2倍にすると特徴サイズが半分に。

### オクターブ (Octaves)
興味深い地形には複数スケールのノイズ合成が必要。典型的に3〜6オクターブを混合:

```
elevation = (1.0×noise(1×n) + 0.5×noise(2×n) + 0.25×noise(4×n)) / 1.75
```

**ゲイン (Gain) / 永続性 (Persistence):** 連続オクターブ間の振幅比率。典型的に各振幅が前の1/2。

### オクターブ間の独立性
同一座標での相関ノイズ値を避けるため、各オクターブに異なるシードまたはオフセットを使用。

## 標高再分配

生ノイズは平坦な谷のない丘を生成。べき乗関数で分布を再形成:
- `elevation = pow(e, exponent)`
- 高い指数 = 平坦な谷 + 急峻なピーク

## バイオーム (Biome) 割当

### 1パラメータ（標高のみ）
水平バンドを生成（水域→ビーチ→森林→雪）。

### 2パラメータ（標高 + 湿度）
標高と別のノイズフィールド（湿度）の組み合わせでリアルなバイオーム分布。生態学者 Robert Whittaker の実際のバイオーム研究と同じ2パラメータアプローチ。

## 島生成

マップ端を水域にするには:
1. **距離関数**: 中心=0、境界=1を割り当て
2. **形状関数**: 元の標高と距離制約をブレンド

## 高度テクニック

| テクニック | 説明 |
|-----------|------|
| **リッジノイズ** | `2×(0.5-abs(0.5-noise))` で鋭い山脈の尾根を生成 |
| **テラス** | `round(e×levels)/levels` で段階的な地形を生成 |
| **樹木配置** | 半径R内の局所最大値を検出。ポアソンディスク/ジッタードグリッドも使用可能 |
| **ラップアラウンド** | 3Dノイズで円筒、4Dノイズでトーラスマップ |
| **無限マップ** | 座標ベースの局所計算で全マップ格納不要 |

### シード管理
標高と湿度に異なるシードを使用。シード非対応なら遠い座標オフセットからサンプリング。

## 限界と利点

**限界:** 各座標を独立処理するため「すべての場所が同じ感覚」。河川・山脈のような大域的特徴には高度な手法が必要。

**利点:** シンプル（50行未満で妥当な結果）、高速（リアルタイム生成・ゲームジャム対応）、柔軟（異なるゲーム美学に調整容易）。

**進行戦略:** ノイズベース生成で高速プロトタイピング → ゲームプレイニーズが明確になったらカスタムジェネレーターに置換。
