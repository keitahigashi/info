## 基本概念

衝突判定は「デジタル空間で2つ以上のオブジェクトが接触・重なっているかを判断する処理」。ゲーム、シミュレーション、ロボット工学で不可欠。

## 二段階処理アーキテクチャ

### 広域フェーズ (Broad Phase)

遠く離れたオブジェクトを除外。オブジェクトを「箱や球のような単純な形に置き換え、その簡単な形が重なっているかだけを調べ」て効率化。

### 狭域フェーズ (Narrow Phase)

重なる可能性のあるオブジェクトを詳細検証。**分離軸定理 (SAT)** や **Gilbert-Johnson-Keerthi (GJK)** アルゴリズムで凸形状やポリゴン間の衝突を検出。

## バウンディングボックス技術

| 技術 | 特徴 |
|------|------|
| **AABB（軸平行境界ボックス）** | X/Y/Z軸に固定。重なりチェックが非常に高速 |
| **OBB（方向付き境界ボックス）** | 回転に対応。より正確だが計算コスト増 |
| **BVH（バウンディングボリューム階層）** | 複雑形状に対応する階層構造 |

## 空間分割手法

- **BSPツリー (Binary Space Partitioning)**
- **クワッドツリー (Quadtree)** — 2D空間の4分割
- **オクツリー (Octree)** — 3D空間の8分割
- **タイルベースシステム**

これらにより「複数のオブジェクト間で行う衝突チェックの回数を削減」。

## 実装最適化テクニック

1. シンプルな形状（円、球、ボックス）をプロキシとして使用
2. 近接・可視オブジェクトに限定したチェック
3. 静的/動的オブジェクトの別処理
4. コリジョンレイヤーとフィジックスマテリアルの設定活用
