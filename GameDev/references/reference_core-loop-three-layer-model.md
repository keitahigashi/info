## コアゲームプレイループ三層モデル

### 三層エンゲージメント構築モデル

```
Layer 1: コアゲームプレイ（Core Gameplay）
  ┗ 直感的で報酬的な繰り返しメカニクス
  ┗ 「基盤が弱ければ進行やビジュアルで補完できない」

Layer 2: メタゲーム（Meta-game）
  ┗ 長期的な進行システム
  ┗ 持続的な動機付け構造

Layer 3: コンテンツ戦略（Content Strategy）
  ┗ 定期的な新要素投入
  ┗ シーズン更新による鮮度維持
```

### エンゲージメント向上の三要素（優先度順）

1. コアゲームプレイループ（最重要）
2. ゲームの見た目と音声品質
3. ゲーム内機能の充実度

### 推奨実装機能

| 機能 | 目的 |
|---|---|
| 自動生成レベル | 無限のコンテンツ感の演出 |
| アチーブメントシステム | 目標の明示 |
| キャラクターカスタマイズ | プレイヤー投資感の向上 |
| 日替わり報酬 | 再訪理由の創出 |
| 難易度選択 | 包括性の確保 |
| 段階的アンロック要素 | 進行の実感 |

### ビジネス指標との連関

> 「より長いセッション時間は中盤広告エクスポージャーと全体収益に直結する」

セッション時間延長 → 広告収益最大化という連鎖において、コアループ品質が収益の根幹となる。

### 設計哲学

- **アイデア（コンセプト）よりも実行精度（エグゼキューション）** が成功を決定する
- 「コアプレイが弱ければ進行・ビジュアル・機能のいずれも補完できない」という原則はスコープ管理・プロトタイピング議論と直結する
- コアループの問題は早期プレイテストで発見し、Layer 2・3の開発前に解決する
