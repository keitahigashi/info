## 定義

プロジェクトの主要機能を示す**1枚の文書**。パブリッシャーや投資家に対してゲームのコンセプトを効果的に伝えるツール。

## 必須コンテンツ5要素

| 要素 | 内容 |
|------|------|
| **コンセプト** | ゲームの簡潔な説明文（1文） |
| **プロダクト情報** | ジャンル、プラットフォーム、ターゲットオーディエンス |
| **サマリー** | プロット・目標の詳細 |
| **アウトライン** | ゲームの概念とダイナミクス |
| **USP** | 競合との差別化要因 |

## デザイン3原則

1. **簡潔性**: 1ページで明確なアイデア
2. **整理と視認性**: 余白・段落間隔・行間を適切に配置
3. **ビジュアルアイデンティティ**: ロゴ・スクリーンショット・アートでブランド表現

## 対象者別カスタマイズ

- **パブリッシャー向け**: 詳細情報を含める（オフィスでの読了前提）
- **ピッチイベント向け**: より簡潔に（プレゼンで補完）

## 実装のヒント

- ターゲット層の研究と理解
- ラミネート加工やQRコード活用
- 著作権表記・ライセンス記載
- オリジナル素材の使用
- 反復改善（フィードバック活用）
