## コア原則

通知は「美的魅力と機能的明確性」のバランスが必要。プレイヤーの案内、ゲーム状態の伝達、報酬の通知、チャレンジの警告を担うが、直感的で非侵入的であることが条件。

### 視覚的階層
「コントラスト、色、アニメーションを通じて注意を引くが、プレイヤーを圧倒しない」設計。

### コンテキスト適応
「プレイヤーの進行、位置、ゲーム内状態を反映した現在の状況に合わせた通知」。

### タイミングと持続時間
「ゲームのペースとプレイヤーの認知負荷を考慮して、通知の表示時間を決定」。

## 5ステップ設計フレームワーク

| ステップ | 内容 |
|---------|------|
| **1. 目的定義** | ステークホルダー入力とプレイヤーリサーチで明確なゴール設定 |
| **2. コンテンツ戦略** | 簡潔で実行可能なメッセージング、ゲームナラティブとの一貫性 |
| **3. ビジュアル&インタラクション設計** | 戦略的配置とインタラクティブ要素の活用 |
| **4. データ駆動インサイト** | アナリティクスでエンゲージメントパターンを監視 |
| **5. プロトタイプ&テスト** | ユーザーフィードバックに基づく反復改善 |

## アラート疲労の回避

- 頻繁なメッセージ過多 → アラート疲労で効果低減
- 「情報が最も価値のある瞬間」にのみ表示
- より少ない通知を表示し、アイコンバッジでアクセス可能なリストに集約

## データ統合

- クリックスルー率やアラート後のエンゲージメントなどのメトリクスを活用
- 生のインタラクションデータを実行可能なレポートに変換
- 通知の「関連性とエンゲージメント」の両立を検証
