## ゲームUIデザイン基礎と設計原則

### ゲームUIの4分類

| 種別 | 特徴 | 例 |
|---|---|---|
| **ダイエジェティックUI (Diegetic UI)** | ゲーム世界内に存在（キャラクターも見える） | Dead Spaceの背中のヘルスバー |
| **ノンダイエジェティックUI (Non-diegetic UI)** | 画面外に存在する純粋なHUD | 体力バー・ミニマップ |
| **メタUI (Meta UI)** | 現実とゲーム世界の境界に存在 | 画面の血しぶき、亀裂エフェクト |
| **空間的UI (Spatial UI)** | 3D空間内に固定されたUI | キャラクター頭上のHPゲージ |

### 優れたゲームUIの5原則

1. **直感的なインタラクション設計** — 学習コストを最小化
2. **柔軟性とアクセシビリティ** — 多様なプレイヤーへの対応
3. **情報設計と視覚的ヒエラルキー (visual hierarchy)** — 優先度の明確化
4. **世界観に合ったビジュアル** — ナラティブとUIの一致
5. **シンプルなナビゲーション** — 操作の迷いを排除

### 実践プロセス（5ステップ）

```
コンセプト定義 → ワイヤーフレーム → プロトタイピング → ユーザーテスト → 実装
```

### 2026年最新トレンド

| トレンド | 内容 |
|---|---|
| **VR・AR対応UI** | 空間UIの標準化が加速。3D空間への情報配置設計が必須に |
| **AI活用パーソナライズ** | プレイヤー行動に応じたUI自動生成・調整 |
| **ミニマリズム** | 情報過多を避けた必要最小限の表示設計 |
| **アクセシビリティ標準化** | 設計段階からの組み込みが業界標準に |

### 設計のポイント

- UI種別の選択はゲームのナラティブ・没入感方針と一致させる
- ダイエジェティックUIは没入感を高めるが、情報伝達の明瞭さとのトレードオフがある
- アクセシビリティは後付けではなく設計フェーズ初期から組み込む
