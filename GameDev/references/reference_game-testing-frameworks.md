## テストピラミッド

| 層 | テスト量 | 実行速度 |
|----|---------|---------|
| ユニットテスト（底） | 多数 | 最速 |
| 統合テスト（中） | 中程度 | 中 |
| E2Eテスト（頂） | 少数 | 最遅 |

## エンジン別フレームワーク

### Unity Test Framework (UTF)
- Edit Mode（高速ユニットテスト）＋ Play Mode（統合テスト）
- NUnit ベースのアサーション
- ゲーム完全ビルド不要でテスト可能

### Unreal Engine Automation
- C++ネイティブテスト（`IMPLEMENT_SIMPLE_AUTOMATION_TEST`マクロ）
- 機能/スモーク/ストレス/スクリーンショット比較テスト対応
- Latent Commandsで複数フレームテスト
- AAAプロジェクトワークフローに最適

### Godot GUT Framework
- GDScript用xUnit式テスト
- シグナル監視＋シーンツリーテスト
- インディー開発に最適

### Playwright（Webゲーム）
- HTML5/WebGLゲームのブラウザ自動化
- スクリーンショット比較でビジュアルリグレッション
- モバイルビューポート＋ネットワーク条件シミュレーション

## CI/CD統合: GameCI

- プリインストールUnityバージョンのDockerコンテナ
- GitHub Actionsワークフローで自動ビルド＋テスト
- 全ターゲットプラットフォーム対応
- ライセンスアクティベーション＋ビルドキャッシング

パイプライン: ユニット→統合→フルビルド（並列実行で最適化）

## ベストプラクティス

### AAAパターン
Arrange（セットアップ）→ Act（実行）→ Assert（検証）

### 非決定性対策
- 乱数ジェネレータに**固定シード**
- 時間依存システムのモック化
- 決定論的物理設定
- → 環境間で再現可能な結果

### フレーキーテスト管理
- トラッキングで不安定テストを特定
- レースコンディション/タイミング依存性の調査
- 根本原因調査中は問題テストを隔離

### コードカバレッジ目標
- ゲームロジック: **70-80%**
- グラフィクス/実験的機能: 低カバレッジ許容
- **重点**: セーブシステム/ネットワーキング/セキュリティ

## パフォーマンステスト

フレーム時間/メモリアロケーション/ロード時間/ネットワーク帯域幅を計測。ベースライン確立→閾値超過で自動失敗→ダッシュボード＋アラートでトレンド追跡。

## 核心的洞察

「自動テストは手動QAを補完するが完全には置換できない」— 自動化はリグレッション検出＋機能検証に優れ、手動テストはプレイヤー体験/エッジケース/主観的品質を評価。
