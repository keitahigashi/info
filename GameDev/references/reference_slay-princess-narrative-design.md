## プレイヤー中心ナラティブデザイン (Player-centered Narrative Design)

### 核心的テーゼ

「十分に反応的な物語だけで成功するゲームを作れる」という設計思想。プレイヤーの選択・知覚・行動に対するシステムの応答が物語を生成する。

### 5つの設計フレームワーク

#### 1. 知覚ベースのキャラクター設計 (Perception-based Character Design)

プリンセスはプレイヤーの**信念と解釈**によって姿が変わる存在として機能する。同じキャラクターが「被害者」「脅威」「怪物」「神秘的存在」に変化する。

- キャラクターの本質をプレイヤーの主観として構成
- 客観的な「正解」を設けない
- 知覚の変化がキャラクターの物語変容として現れる

#### 2. 限定的な舞台による規模管理 (Scope Management via Limited Setting)

森→小屋→地下室という**意図的に狭い舞台設定**で指数関数的な分岐爆発を回避する。

| 手法 | 効果 |
|------|------|
| 空間を制限する | 分岐の数学的増加を抑制 |
| 繰り返し訪れる舞台 | 文脈の変化で意味が変容 |
| 選択肢の密度を高める | 少ない場所で深い体験 |

#### 3. 複数の声による内面表現 (Multi-Voice Inner World)

「Hero Voice（英雄的自己）」「Cold Voice（理性的・冷酷な自己）」など、**プレイヤーの潜在意識を具現化した複数の声**が選択肢を提示する。

- 意識・無意識の葛藤をゲームメカニクスとして表現
- プレイヤーは「どの自分として行動するか」を選択
- 声の構成が変化することで心理状態の変容を示す

#### 4. メタナラティブ構造 (Meta-narrative Hub Architecture)

「The Long Quiet」というハブワールドを通じて**複数の異なるルートを統合**する。

```
チャプター1（特定の知覚）
チャプター2（別の知覚）    → The Long Quiet（統合・反省空間）→ 最終章
チャプター3（第三の知覚）
```

プレイヤーは複数のプレイを経て、ハブで各知覚の意味を統合・比較できる。

#### 5. テーマ駆動型の分岐管理 (Theme-driven Branching)

各チャプターで**特定のテーマに焦点を絞る**ことで物語の複雑さを管理する。

- 1チャプター = 1テーマの原則
- テーマを軸に選択肢・結果を整理
- プレイヤーは「このチャプターは何を問いているか」を感じ取れる

### 反応的ナラティブの設計原則

1. **プレイヤーの行動履歴を記憶**し、後の物語に反映する
2. **「見られないコンテンツ」が豊富であること**が再プレイ価値を高める
3. 「正しいルート」を設けず、**すべての選択肢に等しい物語価値を与える**
4. 感情的な反応をキャラクターではなくシステムが生成する

### 実装上の課題と解決策

| 課題 | 解決策 |
|------|--------|
| 分岐の組み合わせ爆発 | 舞台・声の制限による構造的制約 |
| 長期的な一貫性 | テーマを軸としたチャプター分割 |
| プレイヤーの迷子化 | ハブワールドでの定期的な統合 |
| 全知覚の等価性維持 | 各ルートのリソース均等配分 |
