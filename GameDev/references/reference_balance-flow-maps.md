## ヒートマップの4種類

### 1. デスヒートマップ
- 被害者の位置に基づいてキルを集計
- 高い死亡集中エリアを検出

### 2. キルヒートマップ
- キラーの位置に基づいて集計
- 効果的な射撃位置を特定

### 3. バランスヒートマップ
- **デスヒートマップからキルヒートマップを減算**
- 正の値 = 効果的なキルスポット
- 負の値 = 危険エリア
- マップバランスの不均衡を視覚化

### 4. 正規化バランスヒートマップ
- セル内サンプル数で除算し[-1, 1]に正規化
- ノイズを低減し、信頼性の高い分析

## グラデーション/フローマップ

### 勾配計算
- 各セルにおける各キルの**平均方向**を算出
- ベクターフィールドで効果的な射撃方向を表現

### フローライン
- パーティクル追跡による推奨移動経路の可視化
- **「ラン&ガン」パス**: キルデータから攻撃的経路を算出
- **安全退避経路**: 被害者データから最安全ルートを算出

## 制限事項

- **大量データが必要**（セルあたり数百サンプル以上）
- 3Dデータの2D投影による歪み
- キラー・被害者の両方の座標情報が必要（テレメトリペイロード増大）

## レベルデザインへの応用

- バランス不均衡エリアの特定 → カバー配置調整
- デスクラスター → 難易度スパイク検出
- フローライン → プレイヤー動線の検証
