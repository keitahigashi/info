## 難易度曲線の目的

難易度曲線 (difficulty curve) の本質的な目的は**プレイヤーの離脱を防ぐ**こと。難易度が高すぎれば挫折し、低すぎれば退屈してやめる。この両端を避けながらプレイヤーのスキル成長と課題難度を連動させる設計が求められる。

## フロー状態 (Flow State)

チクセントミハイの「フロー理論 (flow theory)」を難易度設計に応用:

```
高難度
  ↑  [不安・欲求不満ゾーン]  ← チャレンジ >> スキル
  |
  |          ★ FLOW ZONE ★
  |     （最適な没入状態）
  |
  |  [退屈ゾーン]  ← スキル >> チャレンジ
  ↓
低難度
       低スキル ———→ 高スキル
```

**フローゾーンに留まらせるには**:
- プレイヤーのスキルをわずかに上回る課題を提供し続ける
- スキル向上に合わせて難易度を段階的に上昇させる
- 急激な難度変化で「壁」を作らない

## 保持率データに基づく調整チェックリスト

| 症状 | 意味 | 対処 |
|------|------|------|
| 特定ステージで離脱率が急増 | 難易度が高すぎる | リソース増加・複雑性低減・ヒント追加 |
| 完了率が異常に高い | 難易度が低すぎる | リソース削減・障害物追加・時間制限導入 |
| 序盤の離脱が多い | チュートリアルが不十分またはつまらない | FTUE (First Time User Experience) の改善 |

## 実践的調整手法

**難易度を上げる方法:**
- 利用可能なリソース（時間・体力・弾薬等）を削減
- 敵の数・速度・パターンの複雑性を増加
- 解法の自由度を制限

**難易度を下げる方法:**
- パワーアップ・ボーナスアイテムを追加
- チェックポイントを増やす
- 「アシストモード」的な救済メカニクムの実装

**Tall Man Runの事例:**
- モバイルランゲームで「パワーアップを難易度調整点に配置」することでフローを維持
- データ分析で離脱率が高いポイントを特定 → その直前にパワーアップを置いて自然なリカバリーを提供
- 強制的な購入誘導でなく、ゲームプレイの文脈に沿った配置が重要

## ゲームジャンル別の考え方

| ジャンル | 難易度曲線の特性 |
|----------|----------------|
| **プラットフォーマー** | 教育曲線（安全→危険の段階的導入）が基本 |
| **RPG** | レベリングで自然な曲線を形成するが、ボス前の急上昇に注意 |
| **パズル** | 「アハ体験」の山谷構造。連続失敗でフラストレーション蓄積 |
| **シューター** | ウェーブ設計で意図的に圧力→緩和のリズムを作る |
| **モバイル** | データ分析との連携が特に重要。A/Bテストで最適化 |

## 設計上の注意点

- 難易度曲線は「設計者の意図」と「プレイヤーの体験」が乖離しやすい
- 必ずプレイテストとデータ（ヒートマップ・保持率）の両方で検証する
- 最適な曲線はターゲット層のスキルレベルに依存するため、ターゲット設定が前提
