## FDDフレームワーク概要

Tracy Fullerton と Chris Swain が開発した、MDAフレームワーク (Mechanics/Dynamics/Aesthetics) を拡張した分析レンズ。ゲームデザインを **形式的・劇的・動的** の3層で分析する。

## 3つの要素

### 1. Formal Elements（形式的要素）
ゲームメカニクスの基盤。プレイヤーの行動と体験のルールセットを定義する。

| 要素 | 説明 |
|------|------|
| Players（プレイヤー） | 人数・役割・相互作用構造 |
| Objectives（目標） | 何を達成すれば勝利か |
| Procedures（手順） | 選択できる行動の種類 |
| Rules（ルール） | 行動の制限・境界 |
| Resources（リソース） | ゲーム内で管理する要素 |
| Conflict（葛藤） | 目標達成を妨げる障害 |
| Boundaries（境界） | ゲーム世界の範囲 |
| Outcomes（結果） | ゲームの終了条件と勝敗 |

### 2. Dramatic Elements（劇的要素）
プレイヤーを感情的に関与させ、ゲームへの投資を維持させる要素。

| 要素 | 説明 |
|------|------|
| Challenge（チャレンジ） | 達成可能だが困難な目標 |
| Play（プレイ） | 自由度・表現・探索の余地 |
| Story（ストーリー） | キャラクターと出来事の連鎖 |
| Character（キャラクター） | プレイヤーが関与できる人物 |
| Premise（前提） | ゲーム世界の設定・背景 |

### 3. Dynamic Elements（動的要素）
ゲームの実行時にのみ発生する要素。設計者が直接制御できない。

- **Emergence（創発）**: シンプルなルールから生まれる複雑な行動
- **Emergent Narrative（創発的ナラティブ）**: プレイヤーの行動から自然発生するストーリー
- **Playtesting（プレイテスト）**: 動的要素を観察・調整するフィードバックループ

## MDAフレームワークとの比較

| MDA | FDD |
|-----|-----|
| Mechanics | Formal Elements |
| Dynamics | Dynamic Elements |
| Aesthetics | Dramatic Elements (部分的) |

FDDの差別化点:
- **劇的要素を独立した層として明示**（MDAでは感情が Aesthetics に埋め込まれる）
- 物語・キャラクター・チャレンジをデザイン変数として扱う
- 実行時の創発を Dynamic 層で明確に位置づける

## 限界と複数フレームワーク活用

劇的要素は物語に偏重しており、**視覚・音声などの感覚的要素が明示されていない**。そのため、以下との組み合わせが推奨される:

- **MDA**: 美学・感情体験の分析に強み
- **Elemental Tetrad** (Schell): テクノロジー・美学を包含
- **Design Pillars**: チームの意思決定軸として活用

## ゲームデザインへの応用

1. **分析フェーズ**: 既存ゲームの形式的要素を一覧化し構造を把握
2. **設計フェーズ**: 劇的要素で感情目標を定義し、形式的要素で実装する
3. **評価フェーズ**: プレイテストで動的要素を観察し、設計を調整する
