## エレメンタルテトラッド（4基本要素）

| 要素 | 内容 |
|------|------|
| **メカニクス** | ルールと相互作用 |
| **ストーリー** | ナラティブ要素 |
| **美学** | ビジュアルと音響 |
| **技術** | 実装可能性 |

4要素は調和し、統一されたテーマに向かう必要がある。

## 6つのメカニクスカテゴリ

1. **空間**: 離散/連続、次元数、境界線
2. **オブジェクト・属性・状態**: 行動が同じなら見た目も同じ
3. **アクション**: オペラティブ（基本操作）vs 結果的（自然発生的戦略）
4. **ルール**: 8階層（オペラショナル〜ハウスルール）
5. **スキル**: 身体/精神/社会の3種
6. **チャンス**: 不確実性 = サプライズ = 楽しさの秘密成分

## 創発的ゲームプレイの鍵

> 複数のアクション × 複数のオブジェクト × 複数の達成方法

## 11のバランスタイプ

公正性 / チャレンジvs成功 / 有意義な選択 / スキルvsチャンス / 頭vs手 / 競争vs協力 / 短vs長 / 報酬 / 罰 / 自由vs統制 / 単純vs複雑

## フロー理論

- スキル高・チャレンジ低 → 退屈
- スキル低・チャレンジ高 → 不安
- **バランス → フロー**

条件: 明確な目標 / 注意散漫なし / 直接フィードバック / 継続的チャレンジ

## リスク軽減ループ（Boehm）

基本設計 → リスク特定 → プロトタイプ → テスト → 詳細設計 → ループ反復

## 優れたプロトタイピング原則

- 質より速度重視
- 失敗から早期撤退
- 最大リスクを優先
- デジタル以外の手法も活用
- 「おもちゃ」から開発開始

## エレガンス vs キャラクター

- **エレガンス**: 単純なシステムが複雑状況で堅牢に機能
- **キャラクター**: 愛すべき奇妙さ、独特の欠陥
- ハリウッド則: 2つ以上の目的を果たさない要素は削除
