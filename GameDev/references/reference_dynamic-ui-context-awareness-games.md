## 動的 UI (Dynamic UI) の設計原則

静的な常時表示 HUD に対して、ゲーム内の状況変化に応じて UI 要素を動的に追加・変更・削除することで**プレイヤーのパフォーマンスと没入感を向上させる**手法。

## コンテキストの 5 分類

UI を動的に変化させるトリガー（コンテキスト）を以下の 5 カテゴリで整理する：

| カテゴリ | 内容 | 例 |
|----------|------|----|
| **エンティティ** | 世界内の対象物の状態 | 敵のHP・インタラクト可能オブジェクトの有無 |
| **アクティビティ** | プレイヤーが行っている行為 | 戦闘中・会話中・探索中 |
| **時間** | ゲーム内外の時間的要素 | タイムリミット・日夜サイクル |
| **位置** | プレイヤーの空間的配置 | エリア境界・危険地帯への接近 |
| **理由** | プレイヤーの意図・目的 | 現在のクエスト目標・ミッションフェーズ |

## 視覚的キュー (Visual Cue) による注意誘導

コンテキスト変化をプレイヤーに直感的に伝えるための視覚的キュー設計：

- **色彩変化**: 状態変化に応じた色の切り替え（例：購入完了でバーが青くなる）
- **サイズ変化**: 重要度・緊急度に応じた要素の拡縮
- **位置変化**: 必要な情報を視線の自然な移動先に配置

## アニメーション原理による UI 変化の「読ませ方」

UI 要素の出現・消滅・変化を直感的に理解させるための 5 つのアニメーション原則：

| 原則 | 説明 |
|------|------|
| **モーションブラー (Motion Blur)** | 高速移動する UI 要素に残像効果を加え、動きの方向を強調 |
| **到着と出発 (Arrival & Departure)** | UI 要素の登場・退場を明確なアニメーションで表現し「変化した」ことを知覚させる |
| **先読み動作 (Anticipation)** | 大きな変化の前に小さな予備動作を加え、変化への準備を促す |
| **追従動作 (Follow Through)** | 主要な変化の後に余韻の動きを加え、フィードバックを補強する |
| **階層的タイミング (Hierarchical Timing)** | 重要な要素のアニメーションを先に完了させ、補助的要素は遅らせる |

## 実装指針

**スクリーンリーダビリティの維持**: 動的変化が多すぎると認知負荷が増大する。「変化が必要なとき以外は変化させない」原則を守る。

**プレイテストによる閾値調整**: どのコンテキスト変化がどのタイミングで UI 切り替えをトリガーするかは、実際のプレイテストで「気づきやすさ」と「邪魔にならなさ」のバランスを測定して調整する。

**アクセシビリティ配慮**: アニメーション速度の設定オプション提供、または静的表示モードへの切り替えを検討する。
