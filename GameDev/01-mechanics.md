## コアループ (Core Loop)

### コアループとは何か

コアループとは、プレイヤーが繰り返し行う主要なアクションサイクルのこと。ゲームプレイの骨格であり、プレイヤーが「もう一回」と思う根本的な理由を形成する。コアループの質がゲーム全体のエンゲージメントを左右する。

コアループは **コンパルションループ (Compulsion Loop)** とは区別される。コアループがユーザーフローの構造的記述であるのに対し、コンパルションループは心理的メカニズムによる行動条件づけを意図的に設計したものである（詳細は後述の「エンゲージメントループとコンパルションループ」セクション参照）。

### エンゲージメントループの4段階構造

プレイヤーはゲームプレイ中、以下の4段階を常に循環する。各サイクルはプレイヤーの現在のメンタルモデルから始まる。

| # | 段階 | 内容 |
|---|------|------|
| 1 | **メンタルモデル (Mental Model)** | ゲームシステム・ルール・目標に対する蓄積された理解 |
| 2 | **ゴール形成 (Goal Formation)** | 現在の理解に基づき、達成すべき目標をプレイヤーが決定 |
| 3 | **アクション (Action)** | 利用可能なゲームメカニクスを通じた入力 |
| 4 | **フィードバックと調整 (Feedback & Adjustment)** | ゲームルールが結果を返し、プレイヤーがメンタルモデルを更新 |

この4段階ループが正しく機能しているとき、プレイヤーは「理解 → 判断 → 実行 → 学習」のサイクルを自然に繰り返し、ゲームに没入する。

### ネスト構造 (Nested Loops)

エンゲージメントループは単一ではなく、**複数のネストされたスコープ**で同時に動作する。例えば Hearthstone では以下の階層が存在する。

| 階層 | スコープ | ループ時間 |
|------|---------|-----------|
| **マイクロ** | カード1枚のプレイ | 数秒 |
| **ミクロ** | 1ターン全体 | 数十秒〜数分 |
| **メソ** | 1マッチ全体 | 数分〜十数分 |
| **マクロ** | シーズン/ランク戦 | 数週間〜数ヶ月 |

各大きなループが多数の小さなループを内包し、階層的なプレイヤー進行を生む。短期的な満足と長期的な目標の両方を提供することで、プレイヤーを多層的に引きつける。

### ランダム性の配置とループ品質

ループ内のどこにランダム性を配置するかで、プレイヤー体験が大きく変わる。

| 配置 | 名称 | 効果 | 例 |
|------|------|------|-----|
| アクション**後** | 決定後ランダム性 (Output Randomness) | 興奮を生むが、結果への無力感も伴う | Hearthstone の Unstable Portal |
| アクション**前** | 決定前ランダム性 (Input Randomness) | 与えられた状況で戦略を立てるエージェンシー感を提供 | ターン開始時のカードドロー |

決定前ランダム性はプレイヤーに「この手札でどう戦うか」というパズル的な判断を促し、決定後ランダム性は「何が出るか」というギャンブル的興奮を提供する。両者の配分がゲームの性格を決定づける。

## メカニクスの分類と設計パターン

### アナログゲームから学ぶメカニクス体系

アナログゲームのメカニクスはビデオゲームに直接流用可能であり、コンピュータによる自動化で複雑性の管理、シングルプレイヤー化、リアルタイム処理の追加が可能になる。以下はメカニクスの基本分類である。

| ジャンル | 説明 | 代表例 |
|---------|------|--------|
| **アブストラクト (Abstract)** | 完全情報の抽象的なゲーム | チェス、オセロ |
| **トリックテイキング (Trick-taking)** | カード出し合いで強いカードを獲得 | ブリッジ、ハーツ |
| **ワーカープレイスメント (Worker Placement)** | 労働者配置による資源獲得 | アグリコラ |
| **拡大再生産 (Engine Building)** | 投資による段階的な成長 | カタン、テラフォーミングマーズ |
| **デッキ構築 (Deck Building)** | カードセット構築がメインメカニクス | ドミニオン |
| **交渉・競り・ドラフト (Negotiation/Auction/Draft)** | リソース獲得の相互作用パターン | — |

### ジレンマ / トレードオフ (Dilemma / Trade-off)

最適解が複数存在することで、戦術的な葛藤を生成する。メカニクスの根幹をなす要素であり、ジレンマの質がゲームの面白さを決定づける。

- **ハイリスク・ハイリターン (High-risk High-reward)**: 大きな報酬と大きな損失の可能性を天秤にかける
- **相互排他的な選択肢 (Mutually Exclusive Choices)**: 一方を選ぶと他方を失う（スキルツリーの分岐など）
- **リソース配分の最適化 (Resource Allocation Optimization)**: 限られたリソースをどこに投資するか
- **時間 vs 品質**: 速度を優先するか完成度を優先するか（レース型ゲームのリスク判断など）

### アドバンテージ生成パターン (Advantage Generation)

プレイヤーが優位性を獲得する方法は複数の次元にまたがる。

| 次元 | 手法 | 例 |
|------|------|-----|
| **リソース量** | 相手より多くのリソースを蓄積 | 拡大再生産、領地拡大 |
| **情報量** | 相手より多くの情報を得る | 偵察、スパイ |
| **確率的優位** | 有利な確率条件を構築 | デッキ圧縮、バフ蓄積 |
| **相手妨害** | 相手の行動を制限・阻害 | ブロック、デバフ |
| **情報隠蔽** | 自分の情報を相手に見せない | ブラフ、裏向きカード |

### 手番システム (Turn Systems)

- **ターン制 (Turn-based)**: 先手有利の公平化が課題。先手に制約を課す、後手にボーナスを与えるなどの対策が必要
- **早い者勝ち (First-come First-served)**: リアルタイム性が加わり、反応速度がアドバンテージに
- **順序変動メカニクス (Variable Turn Order)**: 前ターンの結果やリソース消費で手番順が変動し、戦略の幅が広がる

### ランダム性の種類 (Types of Randomness)

| 手法 | 特徴 | 設計上の注意 |
|------|------|-------------|
| **ダイスロール (Dice Roll)** | 完全なランダム。確率分布が明確 | 期待値と分散を意識した設計 |
| **山札 (Draw Deck)** | 残りカードから情報が推測可能 | カウンティングによる戦略性 |
| **抽選箱 (Lottery)** | 確率テーブルに基づく抽選 | 天井機能 (Pity System) の検討 |
| **手続き生成 (Procedural)** | アルゴリズムによる動的生成 | シード管理と再現性の確保 |

不確実性の導入は戦略の幅を拡大するが、過度なランダム性はプレイヤーのエージェンシー感を損なう。ランダム性と意思決定のバランスが重要である。

## リソース管理

### リソース管理の基本概念

リソース管理とは、ゲーム内で生成・消費されるあらゆる資源の流れを設計すること。プレイヤーに「何を、いつ、どれだけ使うか」という意思決定を迫ることがその本質であり、意味あるトレードオフの源泉となる。

### 投資リソースと非投資リソース

リソースはゲーム進行への影響度で2種類に大別できる。この区別はゲーム経済のバランス設計において極めて重要である。

| 種別 | 定義 | 特徴 | 例 |
|------|------|------|-----|
| **投資リソース (Investment)** | プレイヤーの進行速度に直接影響 | 時間制限あり・決定論的（ランダム不可） | 経験値、スキルポイント |
| **非投資リソース (Non-Investment)** | 進行に影響しないが収益を生む | ランダム取得可・コスメティック用途 | スキン、装飾アイテム |

すべてのメカニクスを監査し、見落とされた稀少性要素を特定することが、バランス崩壊を防ぐ第一歩となる。

### インベントリ制限の設計意図

プレイヤーの保管容量を制限することは、単なる不便ではなく重要な設計ツールである。

- **意味ある決断の強制**: 何を持ち歩くかの選択がゲームプレイの一部に
- **溜め込み防止**: リソースが流動し、経済が循環する
- **戦略的選択の常態化**: 常に「今何が必要か」を考えさせ、エンゲージメントを維持
- **エージェンシーの維持**: 少ないリソースが意味あるトレードオフを生む（通貨が多すぎると意思決定が減少）

### コスト体系の構築

リソースの獲得量と消費量を依存グラフでマッピングし、プレイヤーの進行スケジュールを設計する。

**設計手順**:
1. 各アクティビティの獲得量を定義（例: 1レベルで60コイン獲得）
2. 各消費ポイントのコストを定義（例: アップグレードに3,000コイン必要）
3. 必要プレイ時間を算出（例: 50レベル分のプレイが必要）
4. 「プレイヤーがいつ・どの時点で消費するか」のスケジュールを作成

### 赤字と黒字のサイクル

実体経済を模倣した周期的パターンを設計する。余剰期（リソースが潤沢）の後に稀少期（リソースが不足）を配置し、感情的なバリエーションを生む。

- **余剰期**: プレイヤーに達成感と余裕を与える。蓄積の喜び
- **稀少期**: 緊張感と意思決定の重みを増す。リソースの価値を再認識させる
- **目安**: 1セッションに2〜3の明確な感情ビート (Emotional Beats) を配置

このサイクルが、難しさと容易さの交互パターンとなり、プレイヤーのエンゲージメントを持続的に強化する。

## バランス調整

### バランス調整の基本原則

ゲームバランスとは、プレイヤーのスキル・リソース・進行度に対して、ゲームが提供するチャレンジが適切に釣り合っている状態のこと。バランスが崩れると退屈（簡単すぎ）または挫折（難しすぎ）を招き、プレイヤー離脱に直結する。

### フロー状態 (Flow State) との関係

最適なバランスはプレイヤーをフロー状態に導く。フロー状態とは、課題とスキルが均衡したときに生じる没入体験である。

```
  課題 (Challenge)
  ↑
  │    挫折ゾーン
  │   (Frustration)     ╱
  │                   ╱
  │              ╱ フロー ╲
  │            ╱  (Flow)   ╲
  │          ╱               ╲
  │        ╱    退屈ゾーン
  │      ╱     (Boredom)
  └──────────────────────→ スキル/リソース
```

| ゾーン | 状態 | 結果 |
|--------|------|------|
| **挫折ゾーン (Frustration Zone)** | 課題がスキルを大きく上回る | 進行不能感 → 離脱 |
| **フローゾーン (Flow Zone)** | 課題とスキルが均衡 | 没入 → 最高のエンゲージメント |
| **退屈ゾーン (Boredom Zone)** | スキルが課題を大きく上回る | チャレンジ不足 → 離脱 |

### バランス検証の3ステップ

Daniel Cook のバリューチェーン手法に基づく体系的な検証方法。

| ステップ | 内容 | 確認事項 |
|---------|------|---------|
| **1. 構造デバッグ** | すべてのノードが途切れのないチェーンで接続されているか確認 | 下流目的のない孤立アクティビティを特定・排除 |
| **2. ソース/シンク分類** | 各リソースの生成率を分類（定数・線形・指数） | 生成パターンの把握と意図の確認 |
| **3. パワーマッチング** | シンク消費がソース生産に適切にスケールするか確認 | **指数ソースには指数コストが必要**。不一致はバランス不能なシステムを生む |

### 分解 (Decomposition) によるバランス調整

大きなゲーム経済を一度にバランス調整するのではなく、依存プロットをセグメントに分割し、各々を独立にバランス調整する。

**時間枠別の粒度**:

| フェーズ | バランス粒度 | 注力ポイント |
|---------|------------|-------------|
| **初回ユーザー体験 (FTUE)** | 秒単位 | チュートリアルの即時フィードバック |
| **初日** | 時間単位 | 最初のセッション体験の完成度 |
| **初週** | 日単位 | リテンションとコアループの定着 |
| **長期** | 週〜月単位 | エンドコンテンツとメタゲーム |

最終的にゼロサム経済（総収入 = 総支出）の達成を目指す。

### プレイヤースキル vs キャラクタースキルのバランス

プログレッションシステムのバランスを考える上で、2種類のスキルの区別が不可欠である。

| 区分 | 定義 | 例 |
|------|------|-----|
| **プレイヤースキル (Player Skill)** | プレイヤー自身が練習と習熟で獲得する能力 | Counter-Strike の射撃精度、Tetris のブロック配置 |
| **キャラクタースキル (Character Skill)** | プレイヤーのパフォーマンスとは独立したキャラクターの能力 | RPG キャラのレベルアップによるダメージ増加 |

**整合性 (Alignment) の原則**: プログレッションシステムはゲームの主要な魅力を補完すべきである。

| ゲームの焦点 | 適したプログレッション | 不適切なプログレッション |
|------------|---------------------|----------------------|
| **プレイヤースキル重視** | スキル天井を拡張するメカニクス（Angry Birds の新バードタイプ） | 単純なステータス上昇で難易度を排除 |
| **キャラクタープログレッション重視** | ステータス増加が自然に適合 | プレイヤースキルが必要な場面での過剰なバフ |

**ハイブリッドの落とし穴**: God of War のように、武器ダメージの増加が新コンボの習熟を不要にしてしまうケース。プレイヤースキルゲームでキャラクターパワーが上がりすぎるとチャレンジが損なわれる。Uncharted MP のブースターも、プレイヤースキル基盤のゲームでスキル天井を下げてしまう問題例である。

## 経済設計 (Economy Design)

### ゲーム経済の全体像

ゲーム経済とは、ゲーム内のリソースの生成・変換・消費・交換の全体的なシステムのこと。プレイヤーエンゲージメントとマネタイゼーションの持続性を確保するために、体系的なバランス調整が必要である。理想的には専任のゲーム経済デザイナーが経済学的知見と科学的方法論を適用する。

### ゲーム経済の5つの柱

ゲーム経済を構成する5つの基本要素。これらの相互作用がゲーム経済の質を決定する。

#### 1. 蛇口 (Faucet / Source)

リソースが経済に流入する仕組み。

- 敵を倒してXPを獲得、岩から鉄鉱石を採掘、時間経過で回復するHPバー
- 蛇口はプレイヤー行動にインセンティブを与える手段
- 地域固有の素材は探索と戦闘を促進する
- コアゲームプレイから基本素材を生産し、「リソースはゲーム内で必要に応じて無から生成」される

**生成率の種類**:
- **定数 (Constant)**: 毎回同じ量を生成（デイリーボーナスなど）
- **線形 (Linear)**: 進行度に比例して増加
- **指数 (Exponential)**: 指数関数的に増加（拡大再生産系）

#### 2. インベントリ (Inventory)

プレイヤーの保管容量を制限する仕組み（前述「リソース管理」セクション参照）。

#### 3. コンバータ (Converter / Transform)

リソースを他のリソースに変換し、進行を可能にする仕組み。

- プレイヤーの意思決定を促すツールとして機能
- 限られたリソースがトレードオフを要求（防具アップグレード vs 武器強化）
- 希少な変換素材がゲームプレイの緊張を生む

**クラフトシステム設計原則**:
- **レシピの透明性**: プレイヤーがどのリソースで進行可能かを理解できること
- **任意 vs 必須入力**: 消費される素材と再利用可能なコンポーネントの区別
- **中間的複雑さ**: 複数の変換ステップをプレイヤーが圧倒されない範囲でバランス

#### 4. シンク (Sink / Drain)

リソースが経済から永久に退出する仕組み。

- Zelda の武器破壊、弾薬消費がゲームプレイへの「税」として機能
- プレイヤーに報酬を与えずに進行を遅延させ、チャレンジを増加
- 心理的アンカーに向けた進捗と引き換えにリソースを破壊

**過剰な寛大さも不足もプレイヤーモチベーション低下に直結する**。ソース（獲得手段）とシンク（消費手段）の均衡が経済の生命線。

#### 5. トレーダー (Trader)

NPC・プレイヤー間取引が市場ダイナミクスを生む。

- 投機・利益機会・経済パズルを可能にする
- マルチプレイヤーでは最も複雑な要素（CCP Games・Valve は専任エコノミストを雇用）
- バランスの取れた蛇口・コンバータ・シンクなしにトレーディング経済は成功しない

### バリューチェーン (Value Chain) による経済設計

Daniel Cook のバリューチェーン手法は、プレイヤーアクションを意味ある結果に接続する線形リソース変換シーケンスを設計するフレームワーク。「棒を拾う」のような些細な活動を、心理的欲求への接続で意味あるものにする。

```
[ソース (Faucet)] → [コンバータ (Transform)] → [シンク (Drain)]
   基本素材の生産      クラフト・合成・加工       心理的アンカーへの進捗
```

**内発的動機付けとの統合**:
- 各アクションが自律性 (Autonomy)・有能感 (Competence)・関係性 (Relatedness) を支援するよう設計
- アンカーを任意のメトリクスではなく意味ある心理的欲求に接続

### 経済バランスの5ステップ

| ステップ | 内容 | 詳細 |
|---------|------|------|
| **1. 基本的なゲーム価値を定量化** | プレイヤーアーキタイプに基づき動機を特定 | Explorers=発見、Achievers=進捗、Socialisers=交流、Killers=競争 |
| **2. 投資/非投資リソースを定義** | 進行影響度でリソースを分類 | 投資リソースはランダム不可・時間制限あり |
| **3. コスト体系を構築** | 獲得量と消費量を依存グラフでマッピング | 消費スケジュールの設計 |
| **4. 赤字/黒字サイクルを作成** | 余剰期と稀少期の周期的パターン | 1セッション2〜3の感情ビート |
| **5. 分解 (Decomposition)** | セグメント分割して独立にバランス | ゼロサム経済を達成 |

### インフレ防止の重要性

制御不能なリソース発行はゲーム経済を破壊する。

- **Axie Infinity、Pegaxy の事例**: P2E (Play-to-Earn) ゲームでトークンの過剰発行がハイパーインフレを引き起こし、経済が崩壊
- **対策**: シンクの適切な配置、蛇口の流量制御、定期的なバランスレビュー

### 忍耐の壁 (Walls of Patience)

進行を制限するポイントは、ハードペイウォール（課金しないと進めない）ではなく、以下の複数経路で乗り越えられるよう設計すべきである。

| 手段 | 説明 |
|------|------|
| **努力** | プレイ時間を投資して突破 |
| **課金** | 即時的な進行加速 |
| **広告視聴** | 報酬ビデオ等で部分的加速 |

プレイヤーに選択肢を与えることが重要であり、強制的な課金壁はリテンション低下を招く。

## 進行システム (Progression System)

### 進行システムの役割

進行システムとは、プレイヤーがゲーム内で成長・前進する仕組みの総称。長期的なモチベーションの維持とプレイヤーの投資感を生み出す。段階的なアンロックが愛着と長期的コミットメントを形成する。

### 2つのプログレッションモデル

#### 成功モデル: メカニクス拡張型

新しい要素がユニークなメカニクスを導入し、プレイヤーが習熟すべき新しいスキルを提供する。以前のメカニクスを時代遅れにするのではなく、選択肢を増やす。

- **Angry Birds**: 新しいバードタイプがそれぞれ固有のメカニクスを持ち、レベル設計が新メカニクスの習熟を自然に促す
- **特徴**: プレイヤースキルの天井を拡張し、「上達の余地」を常に提供する

#### 問題のあるモデル: 数値上昇型

プログレッションが単純な数値上昇に留まり、チャレンジを排除してしまう。

- **Uncharted MP のブースター**: 難易度変数を単に低減し、スキル基盤のゲームでスキル天井を下げる
- **God of War のダメージ増加**: 武器強化が新コンボの習熟を不要にし、戦闘の深みを失わせる
- **特徴**: 短期的な満足感はあるが、長期的にはチャレンジ不足を招く

### プログレッション設計の基本原則

1. **整合性 (Alignment)**: ゲームの主要な魅力とプログレッション方向が一致すること
2. **新規性の提供**: 単なる数値上昇ではなく、新しい遊び方の発見を促すこと
3. **スキル天井の拡張**: 上達の余地を常に残し、マスタリーへの道を示すこと
4. **適切なペーシング**: 進行速度が速すぎず遅すぎず、フローゾーンを維持すること

### 価値の定量化とプレイヤーアーキタイプ

進行システムの設計において、プレイヤーが何に価値を感じるかを理解することが出発点となる。

| アーキタイプ | 求める価値 | 進行システムでの対応 |
|------------|----------|-------------------|
| **Explorers（探索者）** | 発見とナラティブ | 新エリア・新ストーリーの段階的開放 |
| **Achievers（達成者）** | 測定可能な進捗 | レベル、装備、コレクション、リーダーボード |
| **Socialisers（社交者）** | コミュニケーション | ギルド機能、協力コンテンツのアンロック |
| **Killers（競争者）** | 対人競争 | PvP ランク、シーズン報酬 |

各価値を時間ベースの指標に変換する。例: トップリーダーボード到達は約1ヶ月のゲームプレイ。複雑さは指数的または線形成長曲線に従う。

> 📎 出典: reference_player-skill-vs-character-skill.md, reference_game-economy-balance-5steps.md, reference_engagement-loops-framework.md

## 難易度曲線 (Difficulty Curve)

### 難易度曲線とは

難易度曲線とは、プレイヤーのスキルレベルとゲーム難易度のバランスを時間軸に沿って管理するメカニズム。最適な難易度曲線はプレイヤーをフロー状態に置くことを目標とする。

### プレイヤーリテンションへの影響

難易度が不適切だと2方向の問題が発生する。

| 問題 | 症状 | 結果 |
|------|------|------|
| **難しすぎる** | 進捗が阻害される | 挫折 → 離脱。報酬ビデオや IAP へのモチベーションも低下 |
| **簡単すぎる** | チャレンジが欠如 | 退屈 → 離脱。報酬を得ても達成感がない |

どちらもマネタイゼーション（収益化）に直結する問題であり、収益面でも難易度設計は無視できない。

### 波状の難易度設計 (Wave Design)

固定的な難易度ではプレイヤーは「同じ感覚しか味わえない」。難易度を意図的に上下させることで、変化に富んだゲーム体験を創出する。

```
  難易度
  ↑     ╱╲       ╱╲       ╱╲
  │    ╱  ╲     ╱  ╲     ╱  ╲      → 全体として右肩上がり
  │   ╱    ╲   ╱    ╲   ╱    ╲
  │  ╱      ╲ ╱      ╲ ╱      ╲
  └──────────────────────────────→ 時間/進行度
```

| 局面 | 効果 | 感情 |
|------|------|------|
| **上昇局面** | チャレンジ感を提供し緊張感を高める | 集中、興奮、達成への欲求 |
| **下降局面** | 休息ポイントを作り、達成感を与える | 安堵、満足、次への準備 |
| **サイクル化** | 繰り返しにより長時間プレイを維持 | 飽きの防止、多様な体験 |

波状設計はフロー理論の実践的応用である。上昇局面でフロー状態に入り、下降局面で回復し、再び上昇局面でチャレンジに向かうサイクルが、持続的なエンゲージメントの基盤となる。

### 難易度曲線の設計手法

1. **ゲーム進行に応じた段階的な難易度調整**: 新メカニクスの導入と同期させる
2. **視覚的グラフでの曲線設計**: 意図的にデザインすることで、開発者の狙い通りにプレイヤーを導く
3. **波の振幅と周期の調整**: 波が大きすぎると不安定、小さすぎると単調に
4. **全体トレンドは右肩上がり**: 各波の谷は前回の谷より高く、全体として成長を実感させる

### 具体例: ゲート走行ゲームでの段階的難易度

Tall Man Run の事例:
- ゲート間隔の短縮と障害物追加で段階的に難度を調整
- 難易度上昇時にパワーアップが必要になる設計
- パワーアップの必要性が報酬ビデオ視聴または IAP 購入への自然な誘導に

> 📎 出典: reference_difficulty-curves-optimization.md, reference_difficulty-curve-wave-design.md

## 確率・乱数設計

### 確率設計の基本

ゲームにおける確率・乱数はプレイヤー体験の多様性を生む重要なツールだが、同時に心理学的な影響が大きく、慎重な設計が求められる。

### 強化スケジュール (Reinforcement Schedules)

B.F. スキナーのオペラント条件づけ研究に基づく報酬の提供パターン。ゲームデザインにおける最も基本的な心理学的フレームワークの一つ。

| スケジュール | 説明 | エンゲージメント効果 | ゲームでの例 |
|------------|------|-------------------|-------------|
| **固定比率 (Fixed Ratio)** | N回のアクションごとに報酬 | 中程度。予測可能で安定的 | 10体倒すごとにボーナス |
| **変動比率 (Variable Ratio)** | 予測不能な回数後に報酬 | **最も強力**。高エンゲージメント | レアドロップ、ルートボックス |
| **固定間隔 (Fixed Interval)** | 一定時間ごとに報酬 | 時間直前に行動が増加 | デイリーログインボーナス |
| **変動間隔 (Variable Interval)** | 予測不能な時間後に報酬 | 一定の行動率を維持 | ランダムイベント出現 |

**変動比率強化 (Variable Ratio Reinforcement)** は「報酬が予測不能な回数のアクション後に来る」メカニズムであり、「勝利の可能性が常にあと1プルの先にある」ため、不確実性を通じた持続的モチベーションを生む。固定スケジュールよりも消去耐性（報酬がなくなっても行動を続ける傾向）が高い。

### ニアミス効果 (Near-Miss Effect)

カジノデザインから直接借用された心理テクニック。「あと少しで当たりだった」という体験が、次の試行への強い動機づけとなる。

- **視覚的演出**: フラッシュするライト、スローモーションリビール、エキサイティングなサウンドエフェクト
- **一般的ドロップにも演出を適用**: ハズレでも「惜しかった」感を演出
- **CS:GO の事例**: レアスキンがスクロールして通過し一般アイテムに着地 → 心理的に次の購入をプライミング

### 確率テーブル設計の考慮事項

| 要素 | 設計上の注意 |
|------|-------------|
| **期待値 (Expected Value)** | プレイヤーが平均的に得られる価値を明確に把握 |
| **分散 (Variance)** | 運の良し悪しの振れ幅。高すぎると不公平感 |
| **天井機能 (Pity System)** | 一定回数ハズレ後に確定報酬。極端な不運の防止 |
| **表示確率** | 確率の明示（法規制対応含む）がプレイヤー信頼を構築 |
| **疑似乱数 vs 真乱数** | 体感の公平性のために疑似乱数で偏りを緩和する手法も |

### 決定前ランダム性 vs 決定後ランダム性

前述のコアループセクションでも触れたが、確率設計の核心的な判断軸である。

- **決定前ランダム性 (Input Randomness)**: プレイヤーが判断する前に不確実性が確定。「この手札でどう戦うか」→ **戦略性を強調**
- **決定後ランダム性 (Output Randomness)**: プレイヤーが判断した後に不確実性が確定。「攻撃が当たるかどうか」→ **興奮とリスク感を強調**

スキル重視のゲームでは決定前ランダム性を多めに、運要素を楽しむゲームでは決定後ランダム性を多めに配分することが一般的な指針となる。

> 📎 出典: reference_variable-ratio-reinforcement-gaming.md, reference_game-mechanics-catalog.md, reference_engagement-loops-framework.md

## 報酬心理学と動機づけ (Reward Psychology & Motivation)

### ドーパミンと報酬経路 (Dopamine & Reward Pathways)

ゲームはプレイヤーが目的を達成した際にドーパミン放出を引き起こすことで脳の報酬系を利用する。この神経化学的反応が行動を強化し、反復への動機付けを生み出す。「期待と報酬」の強力なエンゲージメントループが形成される。

重要なのは、ドーパミンは報酬そのものよりも**報酬の期待**によってより強く放出されること。プレイヤーが「次にもらえるかもしれない」と期待するとき、最も強い動機づけが発生する。

### コンパルションループの3フェーズ構造

スキナーのオペラント条件づけをゲームに翻訳した、行動条件づけの基本構造。

```
  ┌──────────┐     ┌──────────┐     ┌──────────┐
  │  期待     │────→│  行動     │────→│  報酬     │
  │Anticipation│    │  Action   │     │  Reward   │
  └──────────┘     └──────────┘     └──────────┘
       ↑                                  │
       └──────────────────────────────────┘
                   ループ
```

| フェーズ | 内容 | 設計変数 |
|---------|------|---------|
| **期待 (Anticipation)** | プレイヤーが望む結果を想像 | 期待の大きさ、達成可能性の知覚 |
| **行動 (Action)** | デザイナーがインセンティブを与えたい特定の行動 | 行動の面白さ、コスト |
| **報酬 (Reward)** | 行動を強化するフィードバック | 報酬比率、間隔、種類 |

**重要な変数**: 報酬比率（どれだけ与えるか）と間隔（報酬間のタイミング）。

### 報酬設計が満たすべき心理的欲求

| 欲求 | 説明 | 報酬デザインでの活用 |
|------|------|-------------------|
| **有能感 (Competence)** | スキル習熟への人間的欲求 | アチーブメント、バッジ、ランク |
| **ステータス (Status)** | 社会的比較と名声 | リーダーボード、レアアイテムの可視化 |
| **プログレッション (Progression)** | 段階的な前進の実感 | アンロック、レベルアップ、ストーリー進行 |
| **自律性 (Autonomy)** | 自分で選択している実感 | 複数の進行ルート、カスタマイズ |
| **関係性 (Relatedness)** | 他者とのつながり | 協力報酬、ギルド特典 |

### 変動報酬の心理学

ランダムまたは半ランダムな間隔での報酬提供は、予測可能なパターンよりも強い感情的反応を生む。行動条件づけに根ざしたこの原則はギャンブルメカニクスを反映しており、プレイヤーは次の予測不能な報酬を追い求めて投資を増やす。

**具体的な実装事例**:
- **Clash of Clans**: 「憧れの隣人」 — 上級プレイヤーの基地表示が進行動機を提供
- **Hay Day**: チュートリアルが高レベル農場を憧れのターゲットとして強制表示
- **FF Air Brigade**: 戦闘中の変動比率報酬（ランダム遭遇）でエンゲージメント維持

### 倫理的考慮 (Ethical Considerations)

報酬心理学の活用には倫理的責任が伴う。

| 懸念事項 | 説明 |
|---------|------|
| **ルートボックス問題** | 変動報酬メカニクスが心理的脆弱性を悪用し、強迫的な支出行動を招く可能性 |
| **若年プレイヤー** | 発達段階にある若年層への影響が特に懸念される |
| **知覚された投資** | マイクロトランザクションがエンターテイメントと強迫的消費の境界を曖昧に |
| **タイマー飽和** | 過剰なアポイントメントメカニクス（Game of War の多数タイマー）は問題のある実装 |

**エンゲージメントループ vs コンパルションループ**: 両者の混同に注意が必要。エンゲージメントループはゲーム体験の分析ツールであるのに対し、コンパルションループはフリーミアムゲームでマネタイゼーション最大化のために心理的脆弱性を悪用する捕食的デザインになりうる。ツール自体は中立であり、倫理的な設計分析と非倫理的なプレイヤー操作の両方を可能にする。

**デザイナーの責任**: プレイヤーのウェルビーイングを優先する責任ある実践と、エンゲージメントメカニクスのバランスが不可欠。タイマー飽和ではなく、よりスマートなループ設計への革新を目指すべきである。

> 📎 出典: reference_reward-psychology-games.md, reference_variable-ratio-reinforcement-gaming.md, reference_compulsion-loop-design.md, reference_engagement-loops-framework.md

## エンゲージメントループとコンパルションループ

### 2つのループの区別

| 特性 | エンゲージメントループ | コンパルションループ |
|------|---------------------|-------------------|
| **定義** | プレイヤーの行動サイクルの構造的記述 | 神経化学的報酬を得るために反復される習慣的・設計された活動の連鎖 |
| **目的** | ゲーム体験の分析・設計 | プレイヤー行動の条件づけ |
| **メカニズム** | メンタルモデル → ゴール → アクション → フィードバック | 期待 → 行動 → 報酬 |
| **倫理性** | 中立的なツール | 悪用のリスクあり |
| **適用範囲** | すべてのゲーム | 主にフリーミアム・ライブサービス型 |

### コンパルションループの3要素

コンパルションループは以下の3つの核心要素で定義される。

1. **習慣性 (Habitual)**: 長期的に常に反復される行動を生む
2. **設計された活動 (Designed Activity)**: 特に構造化された順次的アクション
3. **神経化学的動機 (Neurochemical Drive)**: ドーパミン放出を活用してプレイヤー行動を条件づけ

### 効果的なループ設計の指針

| 指針 | 説明 |
|------|------|
| **明確なゴールの提示** | プレイヤーが何を目指しているかを常に理解できること |
| **即時フィードバック** | アクションの結果が素早く返されること |
| **適切な難易度** | フローゾーンを維持するチャレンジレベル |
| **進捗の可視化** | 「どれだけ進んだか」が一目で分かること |
| **選択肢の提供** | 複数のループを選べることでエージェンシーを維持 |
| **新規性の注入** | 定期的に新要素を導入し、ループの鮮度を保つ |

### ネストされたループの設計

前述の通り、ループは階層構造を持つ。効果的なゲームは複数の時間スケールでループを設計し、短期的な満足（1アクション〜1ターン）と長期的な目標（シーズン〜キャリア）の両方を提供する。

**階層設計のポイント**:
- 各階層のループが独立して満足感を提供できること
- 下位ループの積み重ねが上位ループの進捗に自然につながること
- 上位ループのゴールが下位ループへの動機づけとなること

> 📎 出典: reference_engagement-loops-framework.md, reference_compulsion-loop-design.md

## ボス戦設計 (Boss Battle Design)

### ボス戦の2つの本質的機能

ボス戦はゲームメカニクスの集大成であり、2つの本質的機能を同時に果たす。

1. **テストである**: プレイヤーがゲーム中に学んだスキルを実証する場
2. **ストーリーである**: 伝統的ストーリーテリングに類似したナラティブビートで構成

成功するボス戦は、この2つの機能を各ビートで同時に遂行する。

### 4つの設計目標

| 目標 | 説明 |
|------|------|
| **通常ゲームプレイからの報われる休憩** | 標準的プレイとは異なる特別な体験を提供する |
| **ストーリーと感情のマイルストーン** | ゲームのペーシング内での具体的な節目となる |
| **習得メカニクスの実証機会** | これまで教えたメカニクスのマスタリーを証明する場 |
| **テンションの構築と解放** | 適切なペーシングによる満足感のある緊張の弧を描く |

### 準備フレームワーク: 4ステップ

ボス戦を設計する前に行うべき準備作業。

| ステップ | 内容 | 具体的な作業 |
|---------|------|-------------|
| **Step 1** | テストするスキルの特定 | プレイヤーが実証すべき全メカニクスを文書化（移動、ブロック、遠距離攻撃、弱点認識など） |
| **Step 2** | 各スキル用の攻撃を設計 | 文書化した各スキルを具体的にテストする個別チャレンジを作成 |
| **Step 3** | 難易度のバランス | プレイヤーのプログレッションに合わせた適切なチャレンジレベルの調整 |
| **Step 4** | テレグラフシステムの計画 | 攻撃前の明確な視覚/聴覚キューを実装し、熟練プレイヤーが適切に反応可能にする |

### ボス戦の8ビート構造 (Narrative Beats)

ボス戦は映画やドラマのようなナラティブ構造を持つ。以下の8ビートが効果的な緊張の弧を描く。

| # | ビート | 機能 | デザインの要点 |
|---|--------|------|--------------|
| 1 | **導入 (Introduction)** | ボスの存在と脅威レベルの確立 | 第一印象で恐怖/敬意を与える演出 |
| 2 | **初期交戦 (Initial Engagement)** | プレイヤーがボスメカニクスに初めて触れる | 安全に学習できる猶予期間 |
| 3 | **パターン認識フェーズ** | プレイヤーが攻撃パターンを学習 | 認識可能なシーケンスの提示 |
| 4 | **エスカレーション** | ボスが新能力を導入 or 強度を増加 | プレッシャーの段階的増大 |
| 5 | **プレイヤー適応** | 学習したパターンとカウンターの実証 | マスタリー感の提供 |
| 6 | **クライマックス** | 戦闘のピーク強度 | 最高の集中と緊張 |
| 7 | **解決 (Resolution)** | ボスの脆弱性 or 最終フェーズ開始 | 勝利への希望の提示 |
| 8 | **結末 (Conclusion)** | 満足感のある撃破と報酬 | カタルシスの提供 |

### テレグラフメカニクス (Telegraph)

効果的なボスは攻撃意図を明確に伝達する。テレグラフにより「理不尽な死」を排除し、「読み → 判断 → 回避」のスキルループを機能させる。

| 手法 | 例 | 役割 |
|------|-----|------|
| **ビジュアルキュー** | 色変化、ポジショニング、アニメーション準備動作 | 視覚的に攻撃タイプと方向を伝達 |
| **オーディオシグナル** | 効果音、音楽キューの変化 | 画面外の攻撃にも対応可能に |
| **ムーブメントパターン** | 実行前にテレグラフされる攻撃動作 | パターン認識の学習機会 |
| **タイミングウィンドウ** | 注意深いプレイヤーを報いる時間猶予 | スキル差による体験の差別化 |

### パターン認識とプレイヤースキル

ボス戦のスキル要素は以下を通じてプレイヤーの成長を促す。

- 認識可能な攻撃シーケンスを確立する
- パターン暗記を報いる（繰り返しプレイで上達する実感）
- スキル表現の機会を創出する（同じボスに対する異なるアプローチ）
- 勝利への複数の有効戦略を許容する（プレイスタイルの多様性）

### ボス撃破の4種の報酬

ボス撃破がプレイヤーに提供すべき報酬は、物質的報酬だけではない。

| 報酬の種類 | 内容 |
|-----------|------|
| **ナラティブ的満足** | ストーリーの進展、新章への到達 |
| **メカニクス的報酬** | 新能力、装備、スキルのアンロック |
| **感情的ペイオフ** | 蓄積されたテンションの解放、カタルシス |
| **達成感** | マイルストーン到達の実感、自己効力感の向上 |

### ケーススタディ

- **Zelda: OoT の Ganon**: ナラティブ構造とメカニカルテストの理想的な結合。ゲーム全体で学んだスキルの総決算
- **Portal の GLaDOS**: スキルテストとストーリーモーメントが同時に機能。物理パズルのマスタリーがナラティブの解決と一致

## クラフトシステム (Crafting Systems)

### クラフトシステムの5類型

クラフトシステムはゲーム経済のコンバータ（変換）機能の具体的実装であり、目的に応じて大きく5つの類型に分類できる。各類型はゲームデザイン上の異なるニーズに対応しており、目的なき実装は不必要な複雑さと退屈を生む。

| 類型 | メカニクス | 利点 | 欠点 | 代表作 |
|------|----------|------|------|--------|
| **通貨の別名 (Money by Another Name)** | ワークベンチ/NPCでの固定レート交換 | 即座に理解可能、既存ショップUIで実装容易 | 没入感の制限、リソース通貨が多すぎると退屈 | Dragon Age: Origins, Skyrim (鍛冶), Terraria |
| **レシピ発見 (Find the Recipe)** | 探索・報酬・購入でレシピ発見 → クラフト解禁 | ナラティブ風味、報酬多様性の拡大、コンプリート欲 | レア度とアイテム価値のバランス調整が必要 | WoW, Diablo III, Fallout 3/NV |
| **試行錯誤 (Guess and See)** | 組み合わせ実験で発見。複雑さは素材選択から配置まで多様 | 真の発見体験、メカニクスへの直接的エンゲージメント | 失敗の懲罰が探索を阻害、外部ガイドに頼りがち | Minecraft, Skyrim (錬金術), Diablo II |
| **カスタマイズオーダー (Made-to-Order)** | 素材・デザイン・美的オプションからの動的カスタマイズ | 深いカスタマイズ、戦略的ゲームプレイレイヤー | 高い実装複雑性、最適化による選択肢狭小化 | Dead Space 3, Monster Hunter Tri |
| **無限の可能性 (Anything is Possible)** | 数千の順列を持つ動的プロセス | 調査にやりがい、意味あるエージェンシー、高いリプレイ性 | 開発・バランスが極めて困難、ゲームブレイキングのリスク | Vagrant Story, Atelier シリーズ |

### クラフトが果たす設計上の役割

7つの代表的なクラフトシステムの分析から、クラフトが果たす設計上の役割は以下の4つに集約される。

| 役割 | 説明 | 事例 |
|------|------|------|
| **キャラクター投資 (Character Investment)** | 武器のカスタマイズ・名前付けがキャラクターへの感情的愛着を深める | Fire Emblem: Path of Radiance — フォージシステムで武器ステータスを調整し、武器三角形を迂回する特殊武器でユニットの役割を柔軟化 |
| **サバイバルテンション (Survival Tension)** | リソース希少性と組み合わせがプレイヤーの不安と緊張を調整する | RE4 (ハーブ組み合わせ)、The Last of Us (明快なレシピで因果関係を明確化)、Alone in the Dark (リアルタイムの環境スカベンジング) |
| **社会的役割定義 (Social Role Definition)** | 全プロフェッション習得不可の設計が特殊化と相互依存を強制し、創発的社会構造を生む | World of Warcraft — プレイヤーコミュニティの断片化が経済的相互依存を生成 |
| **世界観構築 (World Building)** | クラフトの環境的・時間的制約がナラティブアイデンティティと没入感を強化する | The Witcher — 錬金術が焚き火で実行され、ポーション効果が時間帯で変化 |

Eve Online は経済的複雑性の極致であり、プレイヤー駆動経済でクラフト（製造）が市場操作と産業競争による創発的ゲームプレイを生成する。パワープログレッションではなく経済システムを通じた持続的エンゲージメントの好例である。

### クラフトシステム設計の横断的原則

1. **目的の明確性**: キャラクター投資・テンション管理・社会構造・世界観構築のいずれかに焦点を定める。明確な目的なしの実装は避ける
2. **リソース配置**: 素材の入手可能性がプレイヤー行動と感情的ペーシングに直接影響する
3. **メカニカル透明性**: レシピの明快さ vs 複雑さはゲームのテーマ目標に合致させる。透明なレシピは感情的明瞭性を向上させ、不透明さは発見の喜びを提供する
4. **継続的リバランス**: 長寿命ゲームでは他プログレッションシステムとの共存のため定期的な調整が必須
5. **ナラティブ統合**: メカニクスがストーリーとキャラクターアイデンティティを強化するとき、クラフトの満足感は最大化される
6. **コアゲームデザインとの整合**: 不必要な複雑さは深みではなく退屈を生む。クラフトはコアループを強化すべきであり、競合してはならない

> 📎 出典: reference_crafting-systems-5approaches.md, reference_crafting-systems-7studies.md

## ステルスメカニクス (Stealth Mechanics)

### 2つのステルスデザインアプローチ

ステルスゲームの設計は、プレイヤーに与えるエージェンシーの度合いによって大きく2つに分類される。

#### リアクティブステルス (Reactive Stealth)

プレイヤーの弱さと環境回避を重視するアプローチ。プレイヤーは主に隠れることと視線を利用して検出を避けることに頼る。

| 特徴 | 説明 |
|------|------|
| **検出状態** | 二値的（追跡中 or 非追跡） |
| **インタラクション** | 限定的。隠蔽と視線メカニクスに強く依存 |
| **環境障害物** | 一時的な解決策のみ |
| **ゲームプレイ感覚** | パズル解決型 |

**代表作**: Amnesia、Haunting Ground、Outlast

**設計上の限界**: パズル的セグメントになりがちで、「実際にステルスしている感覚ではなく、小さなパズルをしている感覚」に陥りやすい。限定された AI パトロールパターンの悪用につながる。

#### アクティブステルス (Active Stealth)

検出と回避に対するプレイヤーの実質的な影響力を付与するアプローチ。

| 特徴 | 説明 |
|------|------|
| **検出操作** | プレイヤーが敵の検出を操作・変更可能 |
| **インタラクション手段** | 変装 (disguise)、陽動 (distraction)、テイクダウン (takedown) など複数 |
| **キャラクター能力** | より強い（ただし集団相手にはまだ不利） |
| **ゲームプレイ感覚** | ミッションベース |

**代表作**: Thief、Batman Arkham Asylum、Hitman、Metal Gear Solid

**検出の複雑性**: レイヤー型検出メカニズム (layered detection) を採用。Batman シリーズは段階的に照明検出、高度認識、環境破壊、ビジョンジャミングを導入し、反復戦略を防止する。

### 2類型の比較

| 側面 | リアクティブ | アクティブ |
|------|-----------|---------|
| **プレイヤーエージェンシー** | 最小限 | 高い |
| **検出システム** | 二値的 (binary) | レイヤー型 (layered) |
| **キャラクターパワー** | 弱い | 中程度 |
| **ゲームプレイ感覚** | パズル解決型 | ミッションベース |
| **エンゲージメント持続性** | 限定的 | 持続的 |

### ステルス設計の原則

#### キャラクター強度の調整 (Character Power Tuning)

「強いキャラクターが弱く感じ、弱いキャラクターに力の瞬間がある」ようにバランスを確保する。一方的な強さ/弱さではなく、状況に応じたパワーバランスの波が重要。

#### 単調さの回避策

| アプローチ | 手法 |
|-----------|------|
| **Thief 方式** | 難易度ベースの制限（高い戦利品要件、殺害制限）で同じマップに異なるプレイスタイルを促す |
| **Batman 方式** | チャレンジ固有の目標（多様なテイクダウン手法の要求）で戦術バリエーションを強制する |

#### チャレンジプログレッション (Challenge Progression)

ゲーム進行に伴い、新しい検出方法と制限を導入してエンゲージメントを維持し、戦略の停滞を防止する。これは難易度曲線設計の波状パターンとも整合する。

### ハイブリッド事例: Metro Last Light

環境照明のインタラクション（ランプ破壊、スイッチ操作、ヒューズ無効化）でガードの行動と移動を操作する。シンプルなアクティブステルスシステムがプレイヤーにコントロール感を提供しつつ、リアクティブな緊張感も維持する好例。

## 実践メモ

### 経済設計チェックリスト

- [ ] すべてのリソースのソース（獲得手段）とシンク（消費手段）を列挙したか
- [ ] 投資リソースと非投資リソースを明確に区別したか
- [ ] 指数ソースに対して指数コストを設定しているか（パワーマッチング）
- [ ] 赤字/黒字サイクルが1セッションに2〜3回含まれているか
- [ ] 忍耐の壁に複数の突破手段（努力・課金・広告）を用意したか
- [ ] インフレ防止のためのシンクが十分に機能しているか

### 難易度設計チェックリスト

- [ ] 難易度曲線を視覚的にグラフ化して確認したか
- [ ] 波状の上昇/下降サイクルが含まれているか
- [ ] FTUE（初回体験）の最初の数分は秒単位で調整したか
- [ ] プレイヤースキルとキャラクタースキルの整合性が取れているか
- [ ] フロー状態を維持できる難易度レンジ内に収まっているか

### ループ設計チェックリスト

- [ ] コアループの4段階（メンタルモデル → ゴール → アクション → フィードバック）が明確か
- [ ] ネストされた複数の時間スケールでループを設計したか
- [ ] ランダム性の配置（決定前 vs 決定後）を意図的に選択したか
- [ ] コンパルションループが倫理的境界を超えていないか確認したか
- [ ] 各ループが独立して満足感を提供できるか

### ボス戦設計チェックリスト

- [ ] テストするスキルを事前に文書化したか
- [ ] 各スキルに対応する個別チャレンジを用意したか
- [ ] テレグラフシステム（視覚/聴覚キュー）を計画したか
- [ ] 8ビート構造に沿った緊張の弧を設計したか
- [ ] 撃破報酬が4種（ナラティブ、メカニクス、感情、達成感）を含んでいるか

### 覚えておくべき原則

| 原則 | 内容 |
|------|------|
| **フロー第一** | すべての設計はプレイヤーをフロー状態に導くことを目指す |
| **整合性** | プログレッションはゲームの主要な魅力を補完すべき |
| **ゼロサム経済** | 長期的には総収入 = 総支出を目指す |
| **波状設計** | 単調な上昇ではなく、上下の波を意図的に設計する |
| **倫理的責任** | エンゲージメント設計はプレイヤーのウェルビーイングとバランスを取る |
| **多層的満足** | 短期〜長期の複数時間スケールでループを設計する |
| **テスト+ストーリー** | ボス戦はスキルテストとナラティブの両方を同時に遂行する |
