### 694. Architecture, Performance, and Games
- **URL**: https://gameprogrammingpatterns.com/architecture-performance-and-games.html
- **ソース**: Game Programming Patterns (Robert Nystrom)
- **反映先**: 06-programming
- **サマリー**: 設計・パフォーマンス・開発速度の三者トレードオフを体系化。「良い設計＝変更が最初から予期されていたかのように感じられるコード」という定義を軸に、疎結合の認知負荷削減効果、仮想ディスパッチのコスト、プロトタイプの捨て方まで実践的に解説。シンプルさは蓄積でなく蒸留という哲学も提示。

### 695. Modular Monolith: A Sane Architecture for Indie Game Devs
- **URL**: https://www.wayline.io/blog/modular-monolith-indie-game-dev
- **ソース**: Wayline Blog
- **反映先**: 06-programming
- **サマリー**: インディー向けモジュラーモノリスの6ステップ実装手順（モジュール特定→インターフェース定義→境界強制→抽象化→通信文書化→継続リファクタリング）を解説。マイクロサービスを使わずにスケーラブルな設計を実現する進化的アーキテクチャ戦略と、タイトカップリング・神クラス・循環依存の回避策を整理。

### 696. Enjoyable Game Architecture

- **反映先**: 06-programming
