### 535. Game Design as Design of Experience
- **URL**: https://www.gamedeveloper.com/design/game-design-as-design-of-experience
- **ソース**: Game Developer (Anatoly Shestov, 2015)
- **反映先**: 00-foundations
- **サマリー**: ゲームデザインの本質を「興味 (Interest)」と「興奮 (Excitement)」という2つの感情の構築として定義するフレームワーク。興味＝明確な要素と不確実な結果の並存、興奮＝持続的感情の調和状態と定義し、良いゲームはその両立を実現する。デザイナーの役割を「両親がゲストを迎える準備」に例え、ターゲットペルソナの具体化・感情の生理学的レベルへの分解・実装可能な構成要素への変換というプロセスを提示。「怒り」を例に視覚効果・QTE・選択肢削減へと感情を実装する手法を解説。

### 536. Using Player Experience to Design Games
- **URL**: https://www.gamedeveloper.com/design/using-player-experience-to-design-games
- **ソース**: Game Developer (Harm-Jan Wiechers, 2017)
- **反映先**: 00-foundations
- **サマリー**: 「何を作るか」でなく「プレイヤーにどんな体験をさせたいか」を起点にする体験中心設計 (experience-centered design) の実践手法。Laundromallard（GGJ2017）とWho Will...（2015）の2つのゲームジャム事例を通じて、ターゲット体験の言語化→逆算設計→意思決定への参照というプロセスを解説。設計停滞時の打開策・優先順位判断の基準・チーム共通目標の設定に有効。体験記述は「プレイヤーが〇〇を感じる/〇〇な状況になる」形式で、全員が同じ絵を浮かべられるレベルの具体性が必要。

### 537. Design 101: Design Goals
- **URL**: https://www.gamedeveloper.com/design/design-101-design-goals
- **ソース**: Game Developer (Dan Felder, 2015)
- **反映先**: 00-foundations
- **サマリー**: ゲームデザインの根本概念「デザインゴール (design goal)」を「プレイヤーにその時点で何を感じさせたいか」と定義し、すべてのメカニクス選択をこれに従属させるフレームワーク。ターン制ウォーゲームへのランダム性 vs 決定論的戦闘の対比で「メカニクスに絶対的優劣はなくゴール次第で正解が変わる」原則を実証。早期文書化・可視化・階層的設定・全決定への参照の4つの実践的推奨事項を提示。Timmy/Johnny/Spike理論を引用しプレイヤータイプ別ゴール設計の重要性を論じる。Design 101シリーズ第1部。
