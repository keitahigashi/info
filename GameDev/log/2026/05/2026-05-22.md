### 495. Curating your game's system design
- **URL**: https://www.gamedeveloper.com/design/curating-your-game-s-system-design
- **ソース**: Game Developer (Ben Myres, 2017)
- **反映先**: 01-mechanics
- **サマリー**: システムデザインの奥行き (depth) を最大化する3つのキュレーション手法を解説。①スキルチェーン（習熟曲線の長期化で複雑さ最小・奥行き最大）②抵抗の追加（プレイヤーの支配的戦略を防ぐ仕組み、Super Hexagon・Gonner が例）③要素の相互作用（Braid の時間免疫属性のように既存要素の交差で奥行きを生む）。「新要素追加よりキュレーション（削ぎ落とし・交差設計）が先」というシステム設計哲学。

### 496. Using the Agential Structure Model to classify fun
- **URL**: https://www.gamedeveloper.com/design/using-the-agential-structure-model-to-classify-fun
- **ソース**: Game Developer (Satoshi Ido, 2017)
- **反映先**: 00-foundations
- **サマリー**: 「行為と外部フィードバック」を軸に楽しさを8タイプ（Rewardal・Operational・Social・Considerational・Fictional・Passival・Objectual・External）・19サブタイプに分類するAgential Structure Model（ASM）を提案。MDA フレームワーク・Caillois の4分類（Agon/Alea/Mimicry/Ilinx）と対応付け。ゼルダ時のオカリナ・ポケモン・Guitar Hero を「目標の構造」で図式化し、ゲームが刺激する楽しさタイプの設計ツールとして提示。

### 497. From MDA to DDE
- **URL**: https://www.gamedeveloper.com/design/from-mda-to-dde
- **ソース**: Game Developer (Wolfgang Walk, 2015)
- **反映先**: 00-foundations
- **サマリー**: MDA フレームワークの2課題（「メカニクス」の過度な広さ・ナラティブ位置づけ不明）を解決する DDE（Design/Dynamics/Experience）フレームワークを提案。Design 層を Blueprint（コンセプト・ナラティブ統合）・Mechanics（実装）・Interface（知覚）に分離。Experience 層を感覚的旅・感情的旅・知的旅の3旅として具体化。ゲームを「価値ある対抗者」として概念化し、ルドナラティブ・ディソナンスを設計段階で検出できる実用的進化版。
