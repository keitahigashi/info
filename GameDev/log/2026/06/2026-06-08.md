### 580. Teaching Game Mechanics: A Hierarchy of Learning
- **URL**: https://www.gamedeveloper.com/design/teaching-game-mechanics-a-hierarchy-of-learning
- **ソース**: Game Developer (Josh Bycer, 2016)
- **反映先**: 01-mechanics
- **サマリー**: ゲームメカニクスを新規プレイヤーに教える3段階フレームワーク: ①What（ゲームの基本ルール・操作の把握）②How（メカニクスの機能を重要度順に提示）③Why/When（実際のプレイに適用する戦略的判断の習得）。Whyを教えない設計では「操作は知っているが活用できないプレイヤー」が生まれる。複雑なゲームではメカニクスを段階的にゲーティングすることで学習順序を制御する。

### 581. How Many Mechanics Should a Game Have?
- **URL**: https://www.gamedeveloper.com/design/how-many-mechanics-should-a-game-have-
- **ソース**: Game Developer (Bobby Lockhart, 2016)
- **反映先**: 01-mechanics
- **サマリー**: 「現在想定しているメカニクス数の半分以下にすべき」を中心命題とし、コアメカニクスのダイナミクス（相互作用から生まれる複合体験）で深さを生む設計論を提示。スーパーマリオブラザーズの「ジャンプ一つ」と Jetpack Joyride の多層的メカニクス設計を事例に解説。サポートメカニクス追加の4基準（感情的一貫性・多目的性・相互作用・連鎖性）を提唱。

### 582. An Indie-Style Experiment at a AAA Studio: Insomniac's Slow Down, Bull
- **URL**: https://www.gamedeveloper.com/business/an-indie-style-experiment-at-a-aaa-studio-insomniac-s-i-slow-down-bull-i-
- **ソース**: Game Developer (Lisa Brown / Insomniac Games, 2015)
- **反映先**: 08-production
- **サマリー**: Insomniac GamesがPC向け小規模ゲームをインディスタイルで開発した実験的ポストモーテム。成功要因: 4ヶ月の長期プロトタイピング・週次Twitchストリームによる開発透明化・AAAリソースの活用。失敗要因: 後期テレメトリー不足による難易度スパイク・プラットフォーム選択ミス（PCよりモバイル/itch.ioが適切だった）・大手スタジオ内の組織的制約。「小規模・個人的なゲームはAAA内より独立系で作る方が向いている」を結論とする。

### 583. Classic Postmortem: Harmonix's Dance Central (2010)
- **URL**: https://www.gamedeveloper.com/production/postmortem-harmonix-s-dance-central-2010-
- **ソース**: Game Developer (Marc Flury・Kasson Crooker et al. / Harmonix, 2011/2024)
- **反映先**: 08-production
- **サマリー**: Kinect向けダンスゲームのポストモーテム。成功要因: プロダンサー採用による真正性確保・ダンス教室の手法（リキャップ・スローダウン）をゲーム化したBreak it Downモード・小規模チームによる12ヶ月開発。失敗要因: 新技術（骨格追跡）の実装難易度過小評価による終盤クランチ・初心者難易度の設計ミス（習熟した内部チームによる評価の限界）・後期のキャラクターボイス追加決定による連鎖的タスク増大。

### 584. Accessibility Heuristics and Evaluation Criteria for Mobile Games
- **URL**: https://www.gamedeveloper.com/game-platforms/accessibility-heuristics-and-evaluation-criteria-for-mobile-games
- **ソース**: Game Developer (Attila F. Szenczi-Molnar / Activision Blizzard King, 2022)
- **反映先**: 04-ui-ux
- **サマリー**: W3CのWCAG4原則（知覚性・操作性・理解可能性・堅牢性）にGame Accessibility Guidelines・AbleGamers・KAISTの知見を統合した33項目モバイルゲームアクセシビリティガイドライン。タッチターゲット直径の定量基準（タブレット2.4cm以上・スマートフォン0.96cm以上）や7段階実装サイクルを提示。障害者対応はアクセシビリティ単独の問題ではなく汎用UX向上（片手操作・大きなターゲット）と重複する施策が多いことを示す。
