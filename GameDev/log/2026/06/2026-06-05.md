### 565. Hidden Costs of Extrinsic Rewards
- **URL**: https://www.gamedeveloper.com/design/hidden-costs-of-extrinsic-rewards
- **ソース**: Game Developer (Lyle Cox, 2013)
- **反映先**: 00-foundations
- **サマリー**: 外発的報酬の「隠れたコスト」を Lepper・Greene・Nisbett の古典的実験（事前告知した報酬は内発的動機付けを低下させるが予期しない報酬は影響なし）と自己決定理論で論証。条件付き報酬が自律性を阻害し最小努力での報酬獲得・易しいタスク選択・習熟離れを招く。報酬は「行動を誘導するエサ」ではなく「達成の記念」として設計し、内発的動機付けの乗数として機能させることを提言。

### 566. Design 101: Complexity vs. Depth
- **URL**: https://www.gamedeveloper.com/design/design-101-complexity-vs-depth
- **ソース**: Game Developer (Dan Felder, 2015)
- **反映先**: 01-mechanics
- **サマリー**: 複雑さを「理解複雑さ（デザイン意図の伝達困難度）」「追跡複雑さ（認知的同時把握の困難度）」「深さ（最適行動決定の困難度）」の3種類に分類。前2者は「常に有害なコスト」、深さだけが「プレイヤーの楽しさの源泉」。「複雑さはバジェット」というメタファーで、最小コストで最大深さを生む設計判断の優先順位を提示。

### 567. Postmortem: Paradox Development Studio's Stellaris
- **URL**: https://www.gamedeveloper.com/design/postmortem-paradox-development-studio-s-i-stellaris-i-
- **ソース**: Game Developer (Henrik Fåhraeus・Rikard Åslund, 2016)
- **反映先**: 08-production
- **サマリー**: 32ヶ月・8〜15名で開発した4X宇宙戦略ゲームのポストモーテム。成功要因: 早期ゲームプレイの完成度・反応型チュートリアル・テックリード×デザインリードのデュアルリード体制による「建設的緊張」・OOSを最初から設計に組み込んだマルチプレイヤー安定性。失敗要因: 4Xバランス設計の過小評価・プロトタイピング不足（戦闘問題が後期に発覚）・技術ツリーの3回のリデザイン。「ゲームのアイデンティティを早期確立し全員で共有せよ」が核心的教訓。

### 568. Postmortem: Prototyping Let's! Revolution!
- **URL**: https://www.gamedeveloper.com/production/prototyping-i-let-s-revolution-i-transforming-i-minesweeper-i-into-a-turn-based-strategy-roguelike
- **ソース**: Game Developer (Michael Highland・Ian Sundstrom, 2023)
- **反映先**: 08-production
- **サマリー**: アニメーション制作スタジオBUCKが1年でマインスイーパーをターン制ストラテジーローグライクに転換した開発記録。月別プロトタイプ進行（コアメカニクス→戦闘システム→クラス→ローグライク要素）を公開。「既知メカニクスを新コンテキストに配置すると学習コスト・設計コスト・検証コストを同時に下げる」という知見と、テーマとゲームルールの整合がゲームロジックへの自然な受容を生む事例を提示。

### 569. Hierarchy of Needs for Rewards in Games
- **URL**: https://www.gamedeveloper.com/design/hierarchy-of-needs-for-rewards-in-games
- **ソース**: Game Developer (Chelsey Webster, 2013)
- **反映先**: 00-foundations
- **サマリー**: マズローの欲求段階説をゲームリワードに応用した5層モデル: ①プレイヤー体験（基盤）②コア長期報酬③メジャー中期報酬④ショート短期報酬⑤コスメティック装飾的報酬。「下層が上層の必須条件」という依存構造が核心で、基本体験が楽しくなければいかに豊富なコスメティックを積んでも無意味。WoWの多層的報酬設計と自身の失敗事例（装飾は充実するが基本体験が弱いゲーム）を対比。
