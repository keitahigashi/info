### 625. Indie AI Programming: From Behaviour Trees to Utility AI
- **URL**: https://www.gamedeveloper.com/programming/indie-ai-programming-from-behaviour-trees-to-utility-ai
- **ソース**: Game Developer (David Huebner, 2017)
- **反映先**: 07-technical-systems
- **サマリー**: 戦術RPG「Rise of the Elders: Cthulhu」でビヘイビアツリー (behaviour tree) とユーティリティAI (utility AI) を比較実装した事例。スコアリング関数 `Evaluate(Roar) = 影響探偵数×25 - APコスト×10` の具体例を示しながら、ユーティリティAIがスケーラビリティと非予測性で優れることを解説。4種のモンスターアーキタイプ（Creeper・Deep One・Maniac・Priestess）の設計で実証した。

### 626. GDC 2024: Dial Up the Diegetics – Science and Technology
- **URL**: https://www.gamedeveloper.com/audio/gdc-2024-dial-up-the-diegetics-science-and-technology
- **ソース**: Game Developer / GDC 2024 (Winifred Phillips, 2024)
- **反映先**: 05-visual-audio
- **サマリー**: ゲームコンポーザーが衛星テレメトリー音・機械音・電子音などのダイエジェティック素材を楽曲コンポジションに統合する手法。Shattered State VR・Spyder・Fail Factory VRを事例に「サウンドデザインと音楽を一体として設計する」アプローチを解説。ゲーム世界観との音楽的コヒーレンスをルドナラティブ的に強化できる。

### 627. 10 Seminal Game Postmortems Every Developer Should Read
- **URL**: https://www.gamedeveloper.com/audio/10-seminal-game-postmortems-every-developer-should-read
- **ソース**: Game Developer (Alex Wawro, 2015)
- **反映先**: 08-production
- **サマリー**: Deus Ex・System Shock 2・Diablo II・Thief・Baldur's Gate II・Myth・Unreal Tournament・Age of Empires II・Rainbow Six・Black & Whiteの古典的ポストモーテム10本を厳選。スコープ管理・チームコミュニケーション・技術的負債・反復の価値という共通教訓パターンを抽出し、現代開発者が繰り返し参照できる形でまとめている。

### 628. Game Design: From Data Driven to Data Informed
- **URL**: https://www.gdcvault.com/play/1022825/Game-Design-From-Data-Driven
- **ソース**: GDC Vault / GDC Europe 2015 (Hugo Carr, King)
- **反映先**: 00-foundations
- **サマリー**: 純粋なデータ駆動 (data-driven) 設計は革新性を阻害するという問題提起から、デザイナーとデータサイエンティストが協働する「データインフォームド (data-informed)」設計を提唱。Farm Heroes Sagaのコアメカニクス変更事例を軸に、データは「答えを与えるもの」ではなく「問いを精錬するもの」として使うフレームワークを示す。

### 629. Artificial Intelligence: Utility Builds Character
- **URL**: https://www.gamedeveloper.com/programming/artificial-intelligence-utility-builds-character
- **ソース**: Game Developer (Miguel Nieves, 2012)
- **反映先**: 07-technical-systems
- **サマリー**: iPad戦略ゲーム「Battle of the Bulge」でユーティリティAIによる個性的アーキタイプ（Genius・Coward・Psycho・Defender・Exploiter・Troll）を実装した3層アーキテクチャの解説。Control Yoke → Strategic Brain → Tactical Nodesの構造でスコアを正規化し、重み付けのみ変えて6種類の「性格」を持つAIを実現。スクリプトファイル編集でプログラマー不要の性格追加が可能。
