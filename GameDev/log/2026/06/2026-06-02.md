### 550. GDC 2012: Applying Psychology Principles to Game Design (OCEAN Model)
- **URL**: https://www.gamedeveloper.com/design/gdc-2012-applying-psychology-principles-to-game-design
- **ソース**: Game Developer (Ben Abraham / Jason VandenBerghe講演, 2012)
- **反映先**: 00-foundations（プレイヤー心理学セクション）
- **サマリー**: 心理学のビッグ5性格特性（OCEAN）をゲームデザインのプレイヤー動機分析に適用するフレームワーク。開放性→新規性、誠実性→チャレンジ、外向性→刺激、協調性→調和、神経症傾向→脅威の5変換を提示。各特性を6側面×5特性=30軸で細分化し、「プレイヤーは〇〇を望む」という一枚岩の議論を具体的なセグメント対話に転換する手法を解説。

### 551. The Case For Game Design Patterns
- **URL**: https://www.gamedeveloper.com/design/the-case-for-game-design-patterns
- **ソース**: Game Developer (Bernd Kreimeier, 2002)
- **反映先**: 06-programming-patterns（新セクション「ゲームデザインパターンの哲学的背景」）
- **サマリー**: Christopher Alexanderのパターン言語をゲームデザイン分野に導入することを提唱した先駆的論文。名称・問題・解決策・帰結の4要素テンプレートを定義し、Proxy（代理操作）・Predictable Consequence（予測可能な結果）・Weenie Chain（目標の連鎖）の具体事例を解説。業界全体での設計知識の共有・再利用を促す「慣例（convention）」としての位置づけを主張。

### 552. What is Depth and How Do You Add It to Your Game?
- **URL**: https://www.gamedeveloper.com/game-platforms/what-is-depth-and-how-do-you-add-it-to-your-game-
- **ソース**: Game Developer (Alex Kubodera, Death's Gambitアートディレクター, 2022)
- **反映先**: 01-mechanics（深さとコアメカニクスの設計セクション）
- **サマリー**: 「複雑性（変数の数）」と「深さ（習熟を困難にする可能性空間）」を明確に区別し、深さの4カテゴリ（設定段階・投資選択・ビルド・ミクロ）を定義。Elden Ringのタリスマン+武器+Ash of War組み合わせを事例に、真の深さは「唯一最適戦略が存在しない」特性を持つと論じる。6項目の深さ評価チェックリストを提供。

### 553. Asymmetrical Overbalance in Game Design
- **URL**: https://www.gamedeveloper.com/design/asymmetrical-overbalance-in-game-design
- **ソース**: Game Developer (Josh Bycer, 2015)
- **反映先**: 01-mechanics（非対称バランスセクション）
- **サマリー**: プレイヤーとAIが異なるルールで動作する「非対称バランス」を質的非対称（特殊能力）と量的優位性（リソース量）の2タイプに分類。XCOM: Enemy Unknown（成功：透明性ある無料移動アクション）とSkyshine's Bedlam（失敗：回復余地のない3ターン連続行動）の対比から、非対称バランスを機能させる条件（透明性・対応手段・段階的圧力・テーマ整合性）を導出。

### 554. GameDev Protips: How To Properly Design Compelling Core Mechanics
- **URL**: https://www.gamedeveloper.com/business/gamedev-protips-how-to-properly-design-compelling-core-mechanics
- **ソース**: Game Developer (Daniel Doan, Black Shell Media, 2016)

- **サマリー**: コアメカニクスを「目標（Objectives）」と「技能（Skills）」の2要素に分解し、過多・過少のバランスが退屈・混乱を生む構造を解説。「[技能]を使って[目標を達成]し、[報酬]を得る」というアクティビティステートメントで設計の焦点を一文に集約する手法を提案。段階的スキル向上とプロトタイプ+プレイテストの繰り返しを重視。
