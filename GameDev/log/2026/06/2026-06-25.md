### 655. Design 101: Balancing Games
- **URL**: https://www.gamedeveloper.com/design/design-101-balancing-games
- **ソース**: Game Developer (Dan Felder, 2015)
- **反映先**: 01-mechanics
- **サマリー**: カードゲーム Faeria を事例に、ゲームバランスの本質を「意味ある選択の維持」と定義。全選択肢を等しくするのではなく多様な強みと弱みを設計することで戦略的奥行きを生む考え方を解説。パワーカーブ・フェルミ解法・クイックポインティング・ダークマジック・トリプルタッピング・セーフガードの6テクニックを体系化する実践的バランス調整ガイド。

### 656. Three Rules to Balance By
- **URL**: https://www.gamedeveloper.com/design/three-rules-to-balance-by
- **ソース**: Game Developer (Tyler Sigman, Red Hook Studios, 2010)
- **反映先**: 01-mechanics
- **サマリー**: ゲームシステムバランス調整の3原則を提唱。①シンプル化（実装→プレイテスト→必要なレイヤー追加のみ）、②独立変数の最小化（メカニクス間を依存させ連動調整）、③直線的関係の維持（剛体→線形比例→非線形の順で複雑さを抑制）。複雑さが調整コストを指数関数的に増大させるという前提に立ち、シンプルから始める設計哲学を強調する。

### 657. How Balance Can Affect Difficulty
- **URL**: https://www.gamedeveloper.com/design/how-balance-can-affect-difficulty
- **ソース**: Game Developer (Arjun Wagle, 2025)
- **反映先**: 01-mechanics
- **サマリー**: TTK（キルタイム）と武器ニッチ設計がゲーム難易度に与える影響を分析。FortnitevCounter-Strikeを比較し、高TTKはカジュアル・運依存、低TTKはスキル依存型の難易度を生むことを示す。数値増幅（スタット強化）による難易度より、バランス設計から生まれる有機的難易度の優位性を論じる最新（2025年）の実践的考察。

### 658. Reward Systems: An Excerpt From Level Design
- **URL**: https://www.gamedeveloper.com/design/reward-systems-an-excerpt-from-level-design-concept-theory-and-practice
- **ソース**: Game Developer (Rudolf Kremers, Eufloria共同制作者, 2010)
- **反映先**: 02-level-design
- **サマリー**: 書籍『Level Design: Concept, Theory, and Practice』抜粋。レベルデザインにおける報酬設計の2大アプローチを整理——ファンタビュレーション（現実制約を取り払った架空ルールと環境の創造）とシミュレーション（現実の楽しい側面を抽出した模倣）。「プレイヤーが受動的に報酬を受け取るのではなく、能動的にシステムから勝ち取る体験」の設計を最終目標として示す。

### 659. Book Excerpt: Game Engine Architecture — Real-Time Object Updates
- **URL**: https://www.gamedeveloper.com/programming/book-excerpt-game-engine-architecture
- **ソース**: Game Developer (Jason Gregory, Naughty Dog, 2009)
- **反映先**: 06-programming-patterns
- **サマリー**: 書籍『Game Engine Architecture』第14章抜粋。毎フレームのゲームオブジェクト状態更新の課題と解決パターンを体系化。バッチ更新（キャッシュ効率向上）・フェーズ更新（サブシステム計算順序制御）・バケット更新（依存関係レベルでのグループ化）・状態キャッシュ（1フレームずれバグの解消）・タイムスタンプの5パターンを詳解。単純なvirtual Update()の限界と商用エンジンが採用する高度な更新アーキテクチャの全貌を示す。
