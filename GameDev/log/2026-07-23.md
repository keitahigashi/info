### 2. Game Design 101: Depth and Width
- **URL**: https://www.gamedeveloper.com/design/game-design-101-depth-and-width
- **ソース**: Game Developer
- **反映先**: 02_mechanics.md
- **サマリー**: ゲーム設計を「Depth（意思決定の数・頻度）」と「Width（プレイヤーが覚えなければならないルール量）」の2軸で評価するフレームワークを提示。「Widthを増やさずにDepthを増やすことが理想」という命題をOverwatch・マリオを例に検証する。

### 3. Difficulty Curves Start At Their Peak
- **URL**: https://www.gamedeveloper.com/design/difficulty-curves-start-at-their-peak
- **ソース**: Game Developer
- **反映先**: 02_mechanics.md
- **サマリー**: コントロール習得によりゲーム冒頭に生じる見えない難易度スパイク「サッタースウェイトピーク（SP）」を命名・分析。コントロールの複雑さが序盤の摩擦（フリクション）を増大させるという盲点を指摘し、操作設計を難易度曲線設計の一部として扱う重要性を論じる。
