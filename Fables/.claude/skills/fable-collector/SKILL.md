# Fable Collector Skill

## このSkillを使うタイミング

以下のいずれかに該当する場合に読み込む:

- 「寓話収集」「寓話を集めて」「fable collect」等の指示があった場合
- 「続けて」「前回の続き」と言われ、文脈が寓話収集である場合
- `/loop` でデイリー自動実行する場合

---

## 収集ワークフロー（全10ステップ）

### Step 1: 収集済み寓話の読み込み

`REFERENCES.md` のテーブルを読み込み、収集済みの寓話IDリストを取得する。

### Step 2: 収集テーマの決定

以下のカテゴリから、まだ収集数が少ないものを優先して選ぶ:

| カテゴリ | 検索キーワード例 |
|---------|----------------|
| イソップ寓話 | 「イソップ寓話 一覧 あらすじ 教訓」 |
| 日本昔話 | 「日本昔話 教訓 有名」 |
| グリム童話 | 「グリム童話 寓話 教訓」 |
| 中国故事 | 「中国 故事成語 寓話 教訓」 |
| インド寓話 | 「パンチャタントラ ジャータカ 寓話」 |
| アフリカ民話 | 「アフリカ 民話 寓話 教訓」 |
| その他 | 「世界の寓話 教訓 有名」 |

### Step 3: Web検索

決定したテーマで Web検索を実行する。サブエージェントを活用して並列検索する。

### Step 4: 新規寓話の選別

検索結果から以下の基準で選別する:

- `REFERENCES.md` に同一の寓話が未登録であること（タイトル・内容で重複チェック）
- 教訓・寓意が明確であること（単なる物語ではなく「寓話」であること）
- 出典が特定できること

### Step 5: 寓話内容の取得・確認

WebFetch で参照ページの内容を取得し、あらすじ・教訓・出典情報を抽出する。
**複数のソースを突合して正確性を確保する**。

### Step 6: 寓話ファイルの作成

`_template.md` のフォーマットに従い、`fables/{origin}/` ディレクトリに寓話ファイルを作成する。

ファイル命名規則: `{origin}-{3桁連番}-{english-slug}.md`

- 連番は同一originの既存最大番号 + 1
- english-slug は英語タイトルをケバブケースにしたもの

### Step 7: INDEX.md の一覧更新

`index/{category}.md` の該当カテゴリテーブルに新規寓話を追加する。`INDEX.md` の合計数も更新する。

### Step 8: REFERENCES.md の更新

`references/{category}.md` の出典テーブルに新規寓話を追加する。参照元（Webページ等）と追加日を記載。`REFERENCES.md` の合計数も更新する。

### Step 9: 収集上限の確認

1回の実行で **最大5話** まで。超過した場合は次回に持ち越す。

### Step 10: デイリーログの作成

`log/{YYYY-MM-DD}.md` を作成（既存なら追記）する。

```markdown
# {YYYY-MM-DD} 収集ログ

## 収集結果

- 収集数: {N}話
- カテゴリ: {対象カテゴリ}

| id | タイトル | 出典体系 | 参照元URL |
|----|---------|---------|----------|
| ... | ... | ... | ... |

## メモ

（特記事項があれば）
```

## 収集戦略

### ローテーション

毎回異なるカテゴリから収集し、偏りを防ぐ。`REFERENCES.md` のカテゴリ別件数を確認して、少ないカテゴリを優先する。

### 品質基準

- あらすじは **原典に忠実** に。創作・アレンジは明記する
- 教訓は **寓意を解説** する（単なるストーリー要約ではない）
- 出典・バリエーションは可能な限り **原典まで遡る**
- 複数ソースで内容を **クロスチェック** する

### デイリー自動実行

ユーザーが希望した場合、`/loop 24h` でデイリー実行を設定する（セッション内のみ有効）。
