# Fable Collector Skill

## このSkillを使うタイミング

以下のいずれかに該当する場合に読み込む:

- 「寓話収集」「寓話を集めて」「fable collect」等の指示があった場合
- 「続けて」「前回の続き」と言われ、文脈が寓話収集である場合
- `/loop` でデイリー自動実行する場合

---

## 禁止事項

- **git commit / push は行わない**（ファイル更新まで）
- ユーザーのツール許可以外の確認ダイアログを出さない

---

## 収集ワークフロー（全8ステップ）

### Step 1: カテゴリの決定

**1回の収集は1カテゴリのみ**。以下の優先順で決定する:

1. ユーザーが指定した場合 → そのカテゴリ
2. 指定がない場合 → `log/` の直近ログから前回カテゴリを確認し、下記ローテーション順で次を選ぶ

**ローテーション順**:
`aesop` → `japanese` → `grimm` → `chinese` → `indian` → `african` → `andersen` → `other` → `aesop` …

### Step 2: 該当カテゴリの収集済み確認

**`references/{category}.md` のみ** を読み込み、収集済みタイトル一覧を取得する。
（REFERENCES.md 全体は読まない → コンテキスト節約）

### Step 3: Web検索＆内容取得（サブエージェント委譲）

サブエージェント（Explore）に以下を委譲する:

- 該当カテゴリの検索キーワードでWeb検索
- 検索結果から未収集の寓話候補を抽出
- 各候補のWebページを取得し、あらすじ・教訓・出典情報を抽出
- **有名な寓話**（検索上位に複数ヒット）→ 1ソースで可
- **マイナーな寓話** → 複数ソースでクロスチェック

サブエージェントへの指示に含めること:
- 収集済みタイトル一覧（重複回避用）
- 検索キーワード
- 抽出すべき情報（タイトル/タイトル英語/あらすじ/教訓/出典/タグ候補）

**検索キーワード例**:

| カテゴリ | キーワード例 |
|---------|------------|
| aesop | 「イソップ寓話 一覧 あらすじ 教訓」 |
| japanese | 「日本昔話 教訓 有名」 |
| grimm | 「グリム童話 寓話 教訓」 |
| chinese | 「中国 故事成語 寓話 教訓」 |
| indian | 「パンチャタントラ ジャータカ 寓話」 |
| african | 「アフリカ 民話 寓話 教訓」 |
| andersen | 「アンデルセン童話 寓話 教訓」 |
| other | 「世界の寓話 教訓 有名」 |

### Step 4: 寓話の選別

サブエージェントの結果から以下の基準で選別する:

- 収集済みタイトルと重複しないこと
- 教訓・寓意が明確であること（単なる物語ではなく「寓話」であること）
- 出典が特定できること

**1回の収集上限: 最大5話**

### Step 5: 寓話ファイルの作成

`_template.md` のフォーマットに従い、`fables/{origin}/` に寓話ファイルを作成する。

ファイル命名規則: `{origin}-{3桁連番}-{english-slug}.md`

- 連番は同一originの既存最大番号 + 1
- english-slug は英語タイトルをケバブケースにしたもの

### Step 6: index の更新

`index/{category}.md` の該当カテゴリテーブルに新規寓話を追加する。
`INDEX.md` の収録状況テーブル（合計数・カテゴリ別数）も更新する。

### Step 7: references の更新

`references/{category}.md` の出典テーブルに新規寓話を追加する。参照元URLと追加日を記載。
`REFERENCES.md` の合計数も更新する。

### Step 8: デイリーログの作成

`log/{YYYY-MM-DD}.md` を作成（既存なら追記）する。

```markdown
# {YYYY-MM-DD} 収集ログ

## Round {N}（通算 {M} 話目〜）

- 収集数: {N}話
- カテゴリ: {対象カテゴリ}

| id | タイトル | 出典体系 | 参照元URL |
|----|---------|---------|----------|
| ... | ... | ... | ... |

## メモ

（特記事項があれば）
```

---

## 品質基準

- あらすじは **原典に忠実** に。創作・アレンジは明記する
- 教訓は **寓意を解説** する（単なるストーリー要約ではない）
- 出典・バリエーションは可能な限り **原典まで遡る**

---

## デイリー自動実行

ユーザーが希望した場合、`/loop 24h` でデイリー実行を設定する（セッション内のみ有効）。
