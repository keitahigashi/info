# プロジェクト設定

## 概要

Claude Code CLIのナレッジベースリポジトリ。記事・公式情報を収集・整理し、実践的なガイドとして蓄積する。

## 起動時タスク: 記事自動収集

このディレクトリでセッションを開始した際、以下の記事収集ワークフローを実行する。

### トリガー条件

- セッション開始時に「記事収集」「情報集積」「記事探して」等の指示があった場合に実行
- 明示的な指示がなくても、ユーザーが「続けて」「前回の続き」と言った場合はこのワークフローを指す可能性が高い

### 収集ワークフロー

1. **MEMORY.mdのCollected Article URLs**を読み込み、収集済みURLリストを取得
2. **REFERENCES.md**の参照記事一覧・収集済み記事テーブルも読み込み、重複チェック対象に含める
3. 以下のキーワードでWeb検索（直近1週間）:
   - 「Claude Code 新機能」
   - 「Claude MCP エージェント」
   - 「Claude AI 最新」
4. 検索結果から**重複しない新規記事**を選別（技術的価値の高いものを優先）
5. 各記事の内容をWebFetchで取得し、詳細を抽出
6. メモリファイル（`reference_*.md`）を作成
7. MEMORY.mdのインデックスとCollected Article URLsを更新
8. REFERENCES.mdの「収集済み記事」テーブルに新規記事を追加（**収集日**列を必ず記載）
9. 1回の実行で**最大5記事**まで
10. 収集した記事を対応するナレッジベースファイル（`01-11`）に追記する
    - 各ファイルの `<!-- 日常で得た知見をここに追記 -->` コメント行の直前に追記
    - フォーマット: `### {トピック名}（自動収集 {日付}）` + 要点3〜5行 + `> 詳細: メモリ内 reference_*.md を参照`
    - REFERENCES.mdの「反映先」列を更新
11. すべての記事保存・MEMORY.md更新・ナレッジベース反映が完了したら、リポジトリ内の変更を**コミット＆プッシュ**する
    - コミットメッセージ: `docs: add N new article references ({記事タイトルの要約})`
    - 対象: REFERENCES.md, README.md, CLAUDE.md など リポジトリ内で変更があったファイル
    - メモリファイル（`reference_*.md`, `MEMORY.md`）はリポジトリ外のためコミット対象外

### メモリファイルの形式

```markdown
---
name: {記事の短いタイトル}
description: {1行の説明 — 将来の検索・関連性判定に使う}
type: reference
---

## 出典
{URL、著者、公開日}

## 概要
{記事の要約}

## 詳細
{技術的内容、コマンド例、設定例、コード例等}
```

### デイリー自動実行

ユーザーが希望した場合、`/loop 24h` でデイリー実行を設定する（セッション内のみ有効）。

## ファイル構成ルール

- `00-11` のナンバリングファイル: ナレッジベース本体（手動キュレーション）
- `README.md`: 目次・自動収集システムの説明（簡潔に保つ）
- `REFERENCES.md`: 参照記事一覧・収集済み記事テーブル（記事データの集約管理）
- メモリ内の `reference_*.md`: 自動収集した記事の詳細情報
