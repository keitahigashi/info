- **公開日**: 2026年5月20日
- **報告者**: セキュリティ研究者アオナン・グアン氏（HackerOne レポート #3646509）
- **対象**: Claude Code のネットワークサンドボックス

## 脆弱性の詳細

### 技術的な本質

**SOCKS5 ホスト名のヌルバイトインジェクション**（パーサー差異攻撃）

JavaScript の `endsWith()` と libc の `getaddrinfo()` が同じ文字列を異なる方法で解釈することを悪用。

```
attacker-host.com\x00.google.com
```

- フィルター側（JS）: `.google.com` で終わると判定 → 許可
- 実行側（libc）: ヌルバイトで終端 → `attacker-host.com` に接続

### 影響範囲

| 項目 | 詳細 |
|------|------|
| 脆弱バージョン期間 | 2025年10月20日（v2.0.24）〜 2026年4月1日（v2.1.90） |
| 影響バージョン数 | 約130バージョン |
| 継続期間 | 約5.5か月 |

### 窃取可能な情報

- AWS 認証情報
- GitHub トークン
- 環境変数
- API キー
- ソースコード

## Anthropic の対応状況

| 項目 | 状況 |
|------|------|
| HackerOne 処理 | 「重複」として処理（CVE 未発行） |
| 修正方法 | サイレントパッチ（公式アドバイザリなし） |
| 前回バイパス（CVE-2025-66479） | CVSS 1.8 と評価 |

## セキュリティ観点での課題

- 同じサンドボックスが別の手口で**再び突破**された設計・運用の体質的問題
- プロンプトインジェクションと組み合わせた複合攻撃リスクが高い
- 攻撃の検知が困難（ネットワーク異常として現れにくい）
- 5.5か月間・130バージョンにわたる長期露出

## 対策

最新バージョン（v2.1.90 以降）へのアップデートで修正済み。影響期間中に使用していたユーザーは認証情報のローテーションを推奨。
