## 停止の事実

- **停止日**: 2026年6月12日
- **原因**: 米商務省の輸出管理指令「あらゆる外国籍者によるアクセス禁止」
- **影響範囲**: 日本ユーザーを含む全世界、AWS Bedrock経由も不可
- **アカウント凍結ではない**: 他のClaudeモデル（Opus 4.8等）は稼働継続
- **復旧見通し**: 確定スケジュール未定（Anthropicは「一時停止」と表明）

## 即時対応手順

### ステップ1: 代替モデルへの切り替え

```bash
# ANTHROPIC_MODEL 環境変数またはsettings.jsonで切り替え
# 第一候補: claude-opus-4-8
```

- Claude Code / APIともに `claude-opus-4-8` が第一の代替選択肢
- 性能差はあるが、ほとんどの日常タスクでOpus 4.8で十分

### ステップ2: 現行ワークフローの確認

- Fable 5をハードコードしているパイプラインを特定
- `ANTHROPIC_MODEL` 環境変数での一元管理に移行

## 補償について

- レート制限がリセットされたという報告あり
- Anthropicは「補償」と**公式には明言していない**
- 過大な期待は禁物

## 「AIは止まりうるインフラ」という設計思想

このインシデントが示した本質的な教訓:
- AIモデルは電力・通信と同様に「突然止まる可能性があるインフラ」
- 複数モデル運用と業務設計の見直しが必要
- 単一モデルに業務クリティカルな処理を依存させる設計は廃止すべき
