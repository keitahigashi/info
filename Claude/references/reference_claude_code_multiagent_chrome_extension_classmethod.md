## マルチエージェント環境の実践投入：Chrome 拡張機能開発

### 開発環境
- Windows 11 + WSL + tmux + Claude Code
- エージェント構成：マネージャー1名、フロントエンド・バックエンド・QA 各1名（計4名）

### 開発対象
Chrome 拡張機能タブマネージャー（同じサイトのタブを自動グループ化、アルファベット・50音順整列）

### 3段階の実施内容

**1. CLAUDE.md の改善**
- ベストプラクティス参照でマルチエージェント設定を保護しつつ最適化

**2. 要件定義フェーズ**
- 一部要件を意図的に省略し「エージェントが補完できるか」を検証
- 2回の差し戻しを経て87項目の要件定義書を完成

**3. 実装フェーズ**
- 1,278行のコードを生成
- 全要件充足＋セキュリティ・アクセシビリティも対応
- Chrome 拡張機能として動作確認済み

### 得られた知見
- 要件定義と実装フェーズを分離すると進めやすい
- 複数役割によって確認観点が増え品質が向上する
- split panes の再現にはプロンプトへの明示的記述が必要
- 非機能要件は最初から明示することで実装のブレを防げる
