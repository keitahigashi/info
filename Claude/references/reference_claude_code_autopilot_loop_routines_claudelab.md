## /loop と Routines の比較

| 機能 | /loop | Routines |
|------|-------|----------|
| 動作場所 | ローカルセッション内 | クラウド環境 |
| PC電源OFF | 停止 | 継続実行 |
| トリガー | 時間間隔 | スケジュール・API・GitHubイベント |
| 有効期限 | **7日後に自動削除** | 継続 |

## /loop の使い方

```
/loop <間隔> <プロンプト>
```

- 自然言語での間隔指定も対応（例：「毎日午前7時に実行」）
- セッション内で繰り返し実行
- **重要制限**: すべての繰り返しタスクは作成から7日後に自動削除

## Routines の特徴

- ラップトップをオフにしていても実行され続ける
- 複数トリガーに対応：スケジュール・API・GitHubイベント

### 実運用のベストプラクティス

1. **明示的なトークン予算を設定**: プロンプトに「最大Nターンで完了すること」を記載
2. **/loopは短期タスク向け**: 7日制限があるため、継続的な監視にはRoutinesを使う
3. **エラー時の通知設定**: 失敗時にSlackやメール通知を設定しておく

## 使い分けの指針

- **数時間〜7日以内の反復**: `/loop`
- **週次・月次・継続的な自動化**: Routines
- **外部イベント起動（PR・API）**: Routines
