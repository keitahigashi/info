## 出典

MOSFET's blog（Hatena Blog）: https://mosfet.hatenablog.com/entry/2026/07/06/001800

## 概要

2026年6月22日〜7月3日リリースのv2.1.186〜v2.1.201変更を追跡。「実行モデルの非同期化」が最大の焦点。

## 3つの地殻変動

| 変更内容 | バージョン | 詳細 |
|---|---|---|
| Sonnet 5がデフォルトモデルに | v2.1.197 | ネイティブ1Mトークン、8/31までプロモ価格 |
| サブエージェントのバックグラウンド実行 | v2.1.198 | 非同期化・自動PR生成 |
| Fable5/Mythos5の輸出規制解除 | 6/30〜7/1 | 全プラットフォームで復帰 |

## その他の主要更新

- **デフォルト権限モードを「Manual」に改称** (v2.1.200)：曖昧性を解消
- **`/rewind`コマンド強化**：`/clear`前の会話まで巻き戻し可能
- **CPU使用を約37%削減** (v2.1.191)：ストリーミング最適化
- **`/dataviz`スキル追加** (v2.1.198)
- **Claude in Chrome が一般提供開始** (v2.1.198)
- バックグラウンドデーモンの大規模修正（Linux自死・PIDロック・SSH対応）
- Windows環境のPowerShell exitcode/CJK文字化け改善

## 実装インサイト

非同期化により「投げておける」ジョブ基盤が成熟。一方でデフォルトはより保守的になり（Manual改称、AskUserQuestion廃止）、自律性と確認の使い分けが明確化された。モデル固定が必要な環境は`/model`で明示指定を推奨。
