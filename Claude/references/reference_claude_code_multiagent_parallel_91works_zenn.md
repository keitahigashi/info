## マルチエージェント並列処理入門

### 記事概要

2026年5月2日公開。Claude Code のマルチエージェント（オーケストレーター + 複数サブエージェント）構成を初学者向けに解説。セットアップから実践例・適用判断基準まで一貫して網羅。

### セットアップ

```bash
npm install -g @anthropic-ai/claude-code  # v2.1.121 で検証
# APIキー設定（direnv による管理を推奨）
```

### 並列処理の2実践例

**例1: 並列コンテンツ生成**

「チームビルディング」「1on1ミーティング」「心理的安全性」の3テーマを並列生成。3ファイルがほぼ同時に出力され、逐次処理比で約3倍のスループットを確認。

**例2: 多視点レビュー**

同一提案書を「コスト分析専門家」「コミュニケーション専門家」「施設管理担当者」の3エージェントが並列レビュー。各視点で推奨度（★1〜5）を付与し、専門化による評価精度向上を実証。

### 適用判断マトリクス

| 向くパターン | 向かないパターン |
|---|---|
| 独立した複数テーマの生成 | 前の結果に依存するタスク |
| 同一対象への多視点レビュー | 共有リソースへの同時書き込み |
| 大量の定型処理 | 少量・単純なタスク |
| 複合評価が必要な場合 | 高度に文脈を共有するタスク |

### 注意点

- 並列実行によるコスト増（エージェント数 × 単価）
- デバッグの複雑化（どのエージェントで失敗したか追跡が必要）
- 共有リソース競合の回避設計が必須
