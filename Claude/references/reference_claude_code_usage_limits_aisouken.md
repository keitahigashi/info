## Claude Codeの利用制限と対処法

### 著者

坂本 将磨（Microsoft MVP・AIパートナー）

### 利用制限の基本前提

- Web版・Desktop・Claude Codeの利用枠は**同一フレームワークで統合管理**される
- 5時間ローリングセッション制限と週次制限の二層構造
- 2026年5月6日：5時間レート倍増、ピーク時間制限撤廃
- 週次制限は50%増（2026年7月13日まで暫定）
- **引き上げ後のボトルネックは週次枠**（5時間枠より週次枠が先に尽きるケースが増加）

### モデル選択ガイド

**Claude Opus 4.7**
- 用途：複雑な設計判断、長時間自律実行
- コスト：入力$5/MTok、出力$25/MTok
- 特性：モデル別累積枠を消費しやすい

**Claude Sonnet 4.6**
- 用途：日常コーディング（推奨デフォルト）
- コスト：入力$3/MTok、出力$15/MTok
- 特性：Opusと同等の性能で40%低コスト

**Claude Haiku 4.5**
- 用途：軽量処理、subagent、ログ解析
- コスト：入力$1/MTok、出力$5/MTok
- 特性：最低レイテンシ、コンテキスト200k

### 上限到達時の3つの対処法

1. **プランアップグレード**（Pro→Max 5x: $80/月で5倍枠）
2. **リセット待機**（5時間枠：300分後、週次枠：最大6日待機）
3. **Extra Usage**（APIレート単価で従量課金・Spending cap設定必須）

### 法人運用の選択肢

**Team Plan（5〜150名規模）**
- Standard: 開発＋非開発混在向け
- Premium: フルタイム開発者向け

**Enterprise（50名以上）**
- SSO・SCIM・監査ログ対応
- Workspace Spend Limit設定可能
- 部門別・ユーザー別の上限管理

**API・Bedrock・Vertex・Foundry経由**
- CI/CD自動実行向け
- Batch API利用で50%割引
