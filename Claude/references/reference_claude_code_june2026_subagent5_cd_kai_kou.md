## Claude Code 6月新機能：5段階エージェントと/cdコマンド実践

### 概要

公開日：2026年6月20日。Claude Code v2.1.169〜v2.1.172で追加された新機能を実践的なワークフローで解説。

### /cd コマンド（v2.1.169）

**問題点（旧来）**
- ディレクトリ変更のたびに新規セッションが必要 → 会話履歴がリセット
- モノレポ開発でコンテキストを引き継げない

**解決策**
```
/cd /path/to/other-package
```
- セッション内でディレクトリを切り替え、コンテキストを維持したまま複数リポジトリを行き来
- プロンプトキャッシュは保持されたまま移動

### ネストサブエージェント5段階（v2.1.172）

**構成例：記事執筆パイプライン**

```
Layer 1（Main）オーケストレーター
├── Layer 2a：ニュース収集エージェント
├── Layer 2b：記事執筆エージェント
│   ├── Layer 3a：ファクトチェックエージェント
│   └── Layer 3b：フォーマット検証エージェント
│       ├── Layer 4：自動修正エージェント
│       └── Layer 5：再確認エージェント
```

- **最大5段階まで**の階層構造に対応
- 並列化より「文脈分離」が設計の目的
- 複雑な自律タスクを階層化して安定動作させる

### その他の改善

| 機能 | バージョン | 内容 |
|------|-----------|------|
| レート制限時の自動リトライ | v2.1.166 | 過負荷時にfallbackModelへ自動切り替え |
| AWS Bedrock設定の自動読み込み | v2.1.169 | `.aws/credentials`を自動参照 |

### 実践ポイント

- 5段階サブエージェントは「計画・生成・評価」の3層を基本設計として拡張
- `/cd` で複数リポジトリを横断しても会話コンテキストが継続するため、モノレポ開発の生産性が大幅向上
