## 記事概要

MCPプロトコルがもたらすセキュリティリスクを整理した解説記事。Anthropicは「仕様通り」との立場で安全性は導入側の設計責任とする中、具体的なリスクと対策チェックリストを提示。

## 主要リスク3種

### 1. STDIO実行モデルの構造的問題

**概要:** 2026年4月に公表されたMCPのSTDIO実行モデルに起因する問題。

- ローカルMCPサーバーがSTDIOを通じてClaude Codeと通信する際、任意コマンドが実行可能な構造になりうる
- Anthropicは「仕様通りの動作」として公式パッチを提供しない立場

### 2. プロンプトインジェクション

**概要:** MCPサーバーのツール説明文（description）を悪用した攻撃。

- ツール説明文に隠し命令を埋め込むことで、Claudeの動作を操作可能
- MCPマーケットプレイス上の悪意あるサーバーが特に危険

### 3. サプライチェーンリスク

**概要:** 悪意あるMCPサーバーの配布による感染。

- npmパッケージや非公式配布サイト経由での悪意あるMCPサーバー
- 信頼できる発行元からのみインストールしないと感染リスク

## 安全な導入のための実務チェックリスト（6項目）

| # | チェック項目 | 内容 |
|---|-------------|------|
| 1 | **発行元の選別** | 公式・実績のあるサーバーのみを使用。npmのダウンロード数・GitHub Stars・メンテナを確認 |
| 2 | **最小権限の付与** | MCPサーバーに必要最小限のファイルアクセス・API権限のみ付与 |
| 3 | **隔離環境での実行** | 機微なシステムへのアクセス権を持つMCPサーバーはDockerやサンドボックス内で実行 |
| 4 | **サプライチェーン検証** | `npm audit` や依存関係スキャンでMCPサーバーのコードを定期的に検査 |
| 5 | **ツール説明文の審査** | インストール前にMCPサーバーのソースコードを確認し、ツール説明文に異常がないかチェック |
| 6 | **機微情報の分離** | 会計データ・個人情報を扱うMCPサーバーには特に厳格な権限設計を適用 |

## Anthropicの公式スタンス

Anthropicは「MCPの安全性は仕様通り」として、導入側がリスクを管理する責任があるとしている。安全保護機能（EMA/Enterprise-Managed Authorization）は企業向けプランで提供されるが、基本的なセキュリティ設計は各組織が担当する必要がある。

## 機微情報を扱う場合の追加推奨事項

- 会計・HR・個人情報などの機微データへのアクセスをMCPサーバーに直接付与しない
- Read-only権限で済む場合は書き込み権限を付与しない
- MCPサーバーのログを有効化し、アクセスパターンを監視する
