## ネスト型サブエージェント入門

**公開日：** 2026年6月13日（最終更新：6月16日）

### 新機能の概要

Claude Code **v2.1.172**（2026年6月10日リリース）で「サブエージェントが自身のサブエージェントを起動できるネスト型」が追加。
**最大5階層**までの構造に対応。

### エージェント定義方法

`.claude/agents/` ディレクトリにMarkdownファイルを配置：

```markdown
---
name: code-reviewer
description: コードレビューを担当する専門エージェント。ルーティング判定に使用される。
tools: [read, grep]
model: claude-sonnet-4-6
---

## 役割
コードの品質・セキュリティ・パフォーマンスをレビューする。
```

**重要**：`description`はルーティング判定に使われるため、具体的・明確に記述すること。

### トークン設計の勘所

**深い階層ほど軽いモデルを使う戦略**：

```
Opus（最上位・複雑な判断）
  └─ Sonnet（中間層・実装）
       └─ Haiku（末端・単純タスク）
```

- `CLAUDE_CODE_SUBAGENT_MODEL`環境変数でデフォルト設定を軽量化
- 必要な部分だけ格上げするアプローチを推奨

### 適用ケースの判断基準

**向いているケース**：
- 木構造タスク（親→子への分解が自然なもの）
- 再帰的な探索（ディレクトリ・データ構造の走査）
- モジュール単位の並列処理

**向いていないケース**：
- 逐次処理（前の結果に強依存）
- 横の協調が必要なタスク（エージェント間の密な通信が必要）

### 設計のポイント

- 各エージェント間では**必要最小限の情報伝達**に絞る
- 深い階層は「コンテキスト爆発」を招くため、要約を挟む
- バックグラウンドチェーンの5レベル制限を設計に組み込む
