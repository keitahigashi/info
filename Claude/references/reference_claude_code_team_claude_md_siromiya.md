## CLAUDE.md の三層構造設計

| 層 | パス | 用途 |
|----|------|------|
| グローバル | `~/.claude/CLAUDE.md` | 個人設定・コーディングスタイル |
| リポジトリ直下 | `./CLAUDE.md` | プロジェクト共通ルール |
| サブディレクトリ | `src/CLAUDE.md` など | 局所ルール（特定モジュール向け） |

**重要原則**: 「コードを読めば分かることは書かない」 — 自明なルールを書くと AI がそれに引きずられて余計な処理をするリスクがある。

## カスタムエージェントの責任分担

### 設計判断軸

「メイン文脈から切り離したいかどうか」を最初の判断基準とする。

| 用途 | 手段 | 理由 |
|------|------|------|
| プロンプトのテンプレ化 | スラッシュコマンド | コンテキストを共有したまま再利用 |
| 独立した専門責務 | カスタムエージェント | メイン文脈を汚さない |

### エージェント設計例

```markdown
# .claude/agents/code-reviewer.md
---
name: code-reviewer
description: コードレビュー専門エージェント。セキュリティ・パフォーマンス・可読性を独立した視点で評価する
---

レビュー観点:
- セキュリティリスク（SQLインジェクション、XSS等）
- パフォーマンスボトルネック
- 命名規則・コメントの適切さ
```

## MCPサーバーとフックの使い分け

### 使い分け基準

| 用途 | 適切な手段 | 理由 |
|------|-----------|------|
| DBクエリ・Issue参照 | MCPサーバー | モデルが動的に判断して呼び出す |
| lint・テスト必須化 | フック | 絶対遵守ルール、決定論的実行 |

### フック設定例（必須 lint）

```json
{
  "hooks": {
    "PreToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{ "type": "command", "command": "npm run lint --fix" }]
      }
    ]
  }
}
```

## 実運用での落とし穴と回避策

| 落とし穴 | 症状 | 回避策 |
|---------|------|--------|
| CLAUDE.md 肥大化 | AI がルールを無視し始める | 500 行以下に抑え、詳細は別ファイルへ |
| エージェント過剰分割 | 呼び出しオーバーヘッドで遅延 | 3〜5 エージェントを上限の目安に |
| フックでの過度な禁止 | 作業が詰まって人間介入が頻発 | 「禁止」より「警告」を優先 |
