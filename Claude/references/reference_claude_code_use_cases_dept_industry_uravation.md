## 部署別・業種別 Claude Code 活用事例

### 概要

Claude Code をエンジニア以外の部門で活用した100社超の支援実績から、部署別5事例・業種別5事例を具体的な削減率と共に解説。非エンジニア部門こそ最大の効果が出るという立場。

### 部署別事例（削減率）

| 部署 | 業務 | 削減率 |
|------|------|--------|
| 営業 | 提案書作成（3.5h→1.3h） | 62% |
| マーケ | コンテンツ多フォーマット展開 | 55% |
| 経理 | 月次レポートのナラティブ化 | 40% |
| 人事 | 求人票・評価資料・研修資料 | 70% |
| 情シス | 社内ツール開発（3日→1.5日） | 50% |

### 業種別事例（削減率）

| 業種 | 業務 | 削減率 |
|------|------|--------|
| 製造業 | ベテランの暗黙知をISO準拠SOPに変換 | 65% |
| 建設業 | 現場メモ→安全管理書類 | 58% |
| 小売業 | 商品スペック→EC文・SNS・LINE一括 | 72% |
| 医療 | 専門用語→患者向け説明文（医師確認必須） | 45% |
| 士業 | 顧客向けレポート・説明文書 | 60% |

### 導入の落とし穴4つ

1. 「完璧な出力」への期待（たたき台として使うべき）
2. プロンプト設計の軽視（対象・文字数・トーンを明示）
3. 数値・固有名詞の無検証使用
4. 全部門同時展開（1部門の小さな成功から始める）

### 30-60-90日ロードマップ

- Day 1-30: 最も時間がかかる業務1つに集中し成功体験を作る
- Day 31-60: プロンプトライブラリを組織共有資産化
- Day 61-90: 定量的効果測定→経営層への予算申請
