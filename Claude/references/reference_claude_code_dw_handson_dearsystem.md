## Dynamic Workflows の仕組み

**核心**: ClaudeがJavaScriptオーケストレーションスクリプトを自動生成し、独立したバックグラウンドランタイムがそれを実行する。

```
[ユーザー指示]
    ↓
[Claude: オーケストレーションスクリプト生成]
    ↓
[バックグラウンドランタイム: 並列サブエージェント実行]
    ↓
[結果の相互検証・統合]
    ↓
[最終出力]
```

## スペック

| 項目 | 値 |
|------|-----|
| 同時実行最大数 | 16エージェント |
| 累計最大数 | 1,000エージェント/実行 |
| 対象プラン | Max・Team・Enterpriseのリサーチプレビュー |
| 消費トークン | 通常セッションの大幅増（要注意） |

## 適用ケース

- 数百〜数千ファイルに関わる大規模バグ探索
- 大規模マイグレーション
- セキュリティ監査
- 複雑なアーキテクチャ分析

## 実際の大規模事例：Bun→Rust移植

- 対象: 2,188ファイル・75万行のRustコードへの移植
- 期間: 11日間で完了
- 結果: テスト通過率99.8%
- 背景: メモリレイアウト・型システム・ランタイムが根本的に異なる言語間移植をDynamic Workflowsが自律処理

## 注意点

- 現在はリサーチプレビュー段階
- トークン消費が大幅増加するため、スコープを絞った小さなタスクから始めることを推奨
- ループ暴走防止のため累計1,000エージェント上限が設定されている
