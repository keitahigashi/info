## 概要
Claude Codeで学習した内容をそのままブログ・SNSに投稿し続けるワークフローを構築する方法を解説。学習とアウトプットを別タスクにしないことが継続の鍵とし、CLAUDE.mdにコンテンツ生成ルールを定義してClaude Codeに自動化させる手法を紹介。

## 詳細

### 問題意識
- 「学習とアウトプットが別タスクになっている」ことが継続失敗の主因
- 学習中のClaude Codeとのやり取りをそのまま記事の素材にすることで解決

### CLAUDE.md によるコンテンツ生成ルール定義
CLAUDE.mdに以下を明記することでClaude Codeがコンテンツ生成をサポート:
- **ターゲット読者**: 技術レベル・興味関心の設定
- **ブログ記事の構成**: セクション・見出し・コード例の扱い
- **SNS投稿形式**: 文字数・ハッシュタグ・エンゲージメント重視の書き方
- **トーン**: 技術的正確さと読みやすさのバランス

### プロンプトテンプレート（3パターン）
1. **機能試行パターン**: 「〜を試してみたのでブログ記事化して」
2. **ドキュメント読み込みパターン**: 「このドキュメントを読んで解説記事にして」
3. **エラー解決パターン**: 「このエラーと解決策を記事にして」

### SNS投稿の自動生成
- ブログ記事から3本のツイートを自動生成
- 短文・中文・長文のバリエーション
- ハッシュタグの自動提案

### スケジュール管理
- 投稿予定をファイルで追跡（`schedule.md`など）
- 週次の投稿スケジュールをClaude Codeと一緒に管理

### 実践上のアドバイス
- 「まず動かしながら整える」アプローチを推奨
- 記事自体がこのワークフローで作成されており、実用性を実証
- 完璧なワークフローより継続できるワークフローを優先
