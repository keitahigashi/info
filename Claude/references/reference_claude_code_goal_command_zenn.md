## 概要

Claude Code v2.1.139（2026年5月12日リリース）で追加された `/goal` コマンドの技術的詳細を解説した記事。「完了条件を宣言するだけでClaudeが条件達成まで複数ターンを自律継続する」エージェント型操作を実現する。

## 仕組み

セッションスコープのプロンプトベースStop Hookのラッパーとして実装。各ターン完了後、軽量モデル（デフォルト: Haiku）が完了条件をyes/noで判定し、理由が次ターンへ渡される。

## 基本構文

```bash
/goal <完了条件>   # ゴール設定
/goal              # 状態確認
/goal clear        # ゴール削除
```

## 効果的な条件の書き方

1. **測定可能な終了状態を明示** ─ テスト結果・ビルド終了コード・ファイル数など
2. **検証手段を具体化** ─ 「`npm test` が終了コード0で終わる」のように指定
3. **副作用境界を記述** ─ 「他ファイルを変更しない」など制約を明記
4. **上限ターン数を設定** ─ `or stop after 20 turns` でループ暴走を防止

## 制限事項

- 1セッションにつき1ゴールのみ有効
- 条件テキストは最大4,000文字
- 評価モデルは会話履歴のみを根拠とするため、ツール実行なしに判定可能な条件が必要
- ワークスペース信頼ダイアログへの同意が必須
- フックシステムが無効な環境では利用不可

## CI環境での安全運用

```bash
claude -p \
  --permission-mode acceptEdits \
  --max-turns 30 \
  --max-budget-usd 10.00 \
  "/goal all tests pass"
```

トークン消費が大きくなるため、`/usage` による監視とコストガードが重要。

## 従来との比較

| 方式 | 反復コスト | 自律性 |
|------|----------|-------|
| 通常ターン制 | 毎回ユーザー指示が必要 | 低 |
| /goal 使用 | 初回宣言のみ | 高 |
