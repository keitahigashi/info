## Claude Skillsの本質

従来プロンプトとの本質的な違い:

| 比較軸 | 従来プロンプト | Claude Skills（SKILL.md） |
|-------|-------------|--------------------------|
| 永続性 | セッション毎にリセット | Gitで永続管理 |
| チーム共有 | 個人依存 | リポジトリで共有 |
| バージョン管理 | 困難 | Git履歴で追跡可能 |
| 適用範囲 | 都度指定 | 自動読み込み |

**実体**: `SKILL.md` というMarkdownファイル

**効果**: 毎回の指示を省略し「プロンプトの限界を超える永続的な業務知識」を実現

## 作成方法（3経路）

| 経路 | 方法 |
|------|------|
| Claude App版 | プロジェクト設定から直接入力 |
| Claude Code版 | `.claude/skills/{skill-name}/SKILL.md` に配置し自動読み込み |
| API版 | `system` パラメータに内容を記述 |

**利用条件**: Pro以上のプラン（月$20〜）必須

## 主要活用事例

- **Office文書自動生成**: フォーマット統一・テンプレート適用
- **ブランドガイドライン統一**: トーン・デザイン遵守の徹底
- **定型タスク標準化**: 毎回の指示を不要化
- **レビュープロセス自動化**: コードレビュー観点の標準化

## Claude Code × Skillsの組み合わせ

最強の組み合わせ: Claude Code × Skills で「完全自律型エージェント」が実現

- Skills がコンテキストとして渡されることで、エージェントが判断・実行まで完結
- ハーネス設計と組み合わせることで、人手不要の自動化ループが構築可能
- `/skill-name` のスラッシュコマンドで任意のSkillを呼び出せる

## SKILL.md の設計ポイント

```markdown
# Skill名

## トリガー
（このSkillをいつ使うか）

## 制約
（守るべきルール・上限）

## ワークフロー
（具体的な手順）

## 出力フォーマット
（返却形式の仕様）
```
