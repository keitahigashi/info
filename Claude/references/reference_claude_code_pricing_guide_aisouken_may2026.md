## 料金体系の基本構造

Claude Codeは2つの課金経路で運用される：

1. **サブスクリプション型**（Pro/Max/Team/Enterprise）
2. **API従量課金**（トークン数による支払い）

無料プランは存在しない。どのチャネルから利用しても同じ料金が適用される設計。

## プラン別料金比較表

| プラン | 月額 | デフォルトモデル | Opus利用 | 推奨用途 |
|--------|------|------------------|---------|---------|
| Pro | $17-20 | Sonnet 4.6 | Extra Usage必須 | 学習・小規模開発 |
| Max 5x | $100 | Opus 4.7 | プラン枠内 | 日常開発 |
| Max 20x | $200 | Opus 4.7 | プラン枠内 | 常用・大規模案件 |
| Team Standard | $20-25/席 | Sonnet 4.6 | Extra Usage必須 | 軽量利用 |
| Team Premium | $100-125/席 | Opus 4.7 | プラン枠内 | 開発メンバー |

## API従量課金（標準単価）

| モデル | 入力 | 出力 | 用途 |
|--------|------|------|------|
| Opus 4.7 / 4.6 | $5/MTok | $25/MTok | 設計・難易度高い修正 |
| Sonnet 4.6 | $3/MTok | $15/MTok | 日常作業（推奨） |
| Haiku 4.5 | $1/MTok | $5/MTok | ログ解析・軽量処理 |

注意：Opus 4.7は新トークナイザーで同じ内容でも約35%多くトークンを消費する可能性あり。

## 2026年6月15日の重大変更：Agent SDK月次クレジット分離

これまでプラン枠内で処理されていた以下がユーザー単位の月次クレジットに分離：

- `claude -p` 経由の実行
- Agent SDK 呼び出し
- GitHub Actions
- サードパーティエージェント

### クレジット額（消費はAPI標準レート）

| プラン | 月次クレジット |
|--------|---------------|
| Pro | $20/月 |
| Max 5x | $100/月 |
| Team Standard | $20/月 |
| Team Premium | $100/月 |
| usage-based Enterprise | $20/月 |

クレジットはユーザー単位・共有不可・繰越不可、未使用分は失効。
**重要**：Platform API key経由（Console発行）は対象外。大規模自動化はそちらに寄せるべき。

## 利用制限メカニズム

### 5時間枠
利用量に達するとリセット（約5時間待機）

### 週次枠
7日ごとにリセット。Opus系と他モデルで別管理のケースあり

**制限到達時の対応**：
- リセット待機
- 上位プランへアップグレード
- Extra Usage有効化（追加課金）
- 利用パターン見直し（会話分割、Sonnet中心に変更等）

## 支払い方法

| 項目 | 内容 |
|------|------|
| 対応手段 | クレジット/デビットカードのみ |
| 通貨 | USD建て（円換算は決済時レート＋海外事務手数料1.6-2.2%） |
| Enterprise | 新規組織作成時のみACH対応。Net termsは相談可 |
| 適格請求書 | Anthropic登録番号：T7700150134388。2026年4月以降は消費税別途課税 |

## ケース別選定ガイド

**個人開発**：Pro→Max 5xの2段階移行
- 初期：月払いProで2-3ヶ月試験
- 利用枠到達が定常化→Max 5x移行

**チーム（5-30名）**：Team混在運用
- 高頻度メンバー：Premium（$100-125）
- 軽量利用者：Standard（$20-25）

**大企業**：Max 20x または Enterprise
- 常時複数実行→Max 20x以上
- コンプライアンス・銀行振込要件→Enterprise必須

## コスト最適化の主要テクニック

| 施策 | 効果 | 実装難度 |
|------|------|----------|
| モデル三層使い分け（Opus/Sonnet/Haiku） | トークン20-30%削減 | 低 |
| プロンプトキャッシュ活用 | 読み出し単価1/10 | 中 |
| Batch API併用 | 50%割引+キャッシュで実効数十% | 中 |
| フックで入力フィルタリング | 入力トークン1桁削減可 | 中 |
| Skills/サブエージェント移行 | 定型化でトークン固定費削減 | 高 |

## よくある見積もり失敗パターン

1. エンジニア席だけで計画→展開時に予算不足
2. 全員Standardで統一→開発メンバーが枠到達多発
3. 為替変動見込まず→月額が月5-10%変動
4. Opus 4.7の増分過小評価→実コスト2-3割上振れ
5. Agent SDKを枠内と誤解→6月以降クレジット枠で停止

## 利用状況確認コマンド

- `/usage`：残枠・利用状況確認（必須）
- `/cost`：セッション別コスト表示
- `/statusline`：ステータス常時表示
- コミュニティツール `ccusage`：リポジトリ別分析
