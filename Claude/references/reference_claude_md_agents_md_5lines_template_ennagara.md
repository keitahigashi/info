## 概要

2026年5月15日投稿。開発チームが全プロジェクトのCLAUDE.md・AGENTS.mdに必ず追記する5行テンプレートを公開。Claude Code・Cursor・Windsurf等の複数AIツールに対応可能な汎用的ルール。

## 鉄板5行テンプレート

```markdown
## 鉄則

- 動かす前に「何をやろうとしているか」を1行で宣言してから着手する
- 既存のコード規約・命名・依存関係を必ず先に確認してから書く
- エラーハンドリングが書けない時は、エラー時にどうするかを聞き返す
- 完了したら「やったこと」「変更したファイル」「次に確認すべきこと」を報告する
- 不明な前提があれば、推測で進めず必ず質問する
```

## 各行の意図と防止する問題

| 行 | 防止する問題 |
|----|------------|
| 1. 着手前の宣言 | 思いつきで実装が始まり意図とズレる |
| 2. 既存規約の確認 | 新しい命名規則・ライブラリの勝手な導入 |
| 3. エラー処理の質問化 | 雑なtry/catchでエラーを握りつぶす |
| 4. 完了報告フォーマット化 | レビュー時間の増大（半分に削減できる） |
| 5. 推測の排除 | 不明な前提で勝手に進んで後戻りが発生 |

## 効果

- **AI出力品質が体感30〜50%向上**（チーム内検証）
- レビュー時間の半減
- 無駄な実装の削減
- コンテキスト汚染の防止

## 他ツールへの応用

- **Cursor**: `.cursorrules`に追記
- **Windsurf**: `AGENTS.md`に追記
- **GitHub Copilot**: `copilot-instructions.md`に追記

同じ思想で複数ツールを統一できるため、マルチツール環境でも一貫した品質管理が可能。
