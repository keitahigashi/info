## Claude Code週次利用制限50%増加（期間限定）

### 概要

2026年5月14日公開。Claude Codeの週次利用制限が2026年7月13日まで限定で50%増加。5月6日の「5時間制限2倍化」「ピーク時間廃止」に続く第3弾の緩和措置。

### 適用条件

| 項目 | 詳細 |
|------|------|
| 対象プラン | Pro・Max・Team・シートベースEnterprise（全有料プラン） |
| 対象環境 | CLI・IDE拡張・デスクトップ・ウェブ（すべての環境） |
| 期間 | 〜2026年7月13日まで（期間限定） |
| 増加率 | 週次利用制限が+50% |

### 背景

SpaceX Colossus 1データセンターとの提携による計算容量増強に伴う措置。2026年5月6日発表の以下に続く第3弾：

1. 5時間制限を2倍に（即日）
2. Pro・Maxのピーク時間帯制限廃止（即日）
3. **週次利用制限を50%増加（7月13日まで）** ← 今回

### 活用推奨ユースケース

- 大規模リファクタリング（制限内でより多くのコードを処理）
- テストコードの大量追加
- ドキュメント整備・技術文書の自動生成
- 個人開発者の生産性向上（約1.5倍）

### 注意点：6月15日の課金変更との関係

| 変更点 | 内容 |
|--------|------|
| 週次制限増加（〜7/13） | 対話的利用の枠が広がる＝プラス |
| 6/15からの課金分離 | Agent SDK・claude -pは月次クレジット別枠に＝ヘビーユーザーはコスト増 |

この2つは別の制度。週次制限緩和は対話的利用のみ適用、プログラム的利用（Agent SDK・`claude -p`）は6月15日以降に別課金体系へ移行する点に注意。

### 対策

7月13日の制限終了前に：
- 大規模タスクを前倒し処理
- Agent SDKの利用量を事前計測（6/15の課金移行に備える）
