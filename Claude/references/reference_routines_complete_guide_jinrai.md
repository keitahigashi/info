## 概要
Claude Code Routinesの包括的な実践ガイド。3つのトリガー種類、プラン別実行上限、セキュリティ設計、従来機能との比較を網羅。

## 詳細

### Routinesとは
プロンプト・リポジトリ・コネクタを一度設定するだけで、スケジュール・API・GitHubイベントに応じてAIが自律的にコードを書く仕組み。Anthropicクラウド上で実行されるためPCの電源状態に依存しない。

### 3つのトリガー
1. **スケジュール**: 毎晩2時、平日朝9時、週1回など。ローカルタイムゾーン対応
2. **API**: HTTPポストリクエストで起動。一意のエンドポイントとトークンが発行
3. **GitHubイベント**: PR作成・クローズ・同期、ラベル付与、コミットコメント、リリース、Issue。著者・タイトル・ブランチでフィルタリング可能

### 従来機能との比較
| 項目 | 従来のClaude Code | Routines |
|------|------------------|----------|
| 実行環境 | ローカルPC | Anthropicクラウド |
| PC電源オフ時 | 停止 | 稼働継続 |
| 永続性 | ターミナル閉じると消失 | 永続的に保存・再実行可能 |

### プラン別日次実行上限
- Pro: 5回/日
- Max/Team: 25回/日
- Enterprise: 25回/日（超過は従量課金）

### セキュリティ設計
- `claude/`プレフィックス付きブランチのみプッシュ可能（mainブランチ直接変更を防止）
- AIが生成したPRやコードは必ず人間がレビューしてからマージする設計

### 設定手順（3ステップ）
1. claude.ai/code/routinesにアクセスしてプロンプト入力
2. スケジュール/API/GitHubから1つ以上トリガー選択
3. GitHub、Slack、Linear等のコネクタ接続

### 実用例
- 毎晩2時にLinearのバグチケット取得→修正コード→PR作成
- PR作成時に自動セキュリティレビュー
- マージ済みPRのドキュメント更新漏れ検知
- Datadogアラートからスタックトレース照合→修正PR作成
