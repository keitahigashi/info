## 確認すべき3ステップ（再施行時に備えて）

### Step 1: 自動化処理の棚卸し

```bash
# CIパイプラインのClaude利用を確認
grep -r "claude" .github/workflows/

# cronジョブでclaudeコマンドを使っていないか確認
crontab -l | grep claude
```

### Step 2: クレジット枠の設定

- Anthropic管理画面で「Extra usage」設定を確認
- 月次クレジット上限への到達後の動作（停止 or 超過課金）を設定
- 請求サイクル開始日を把握してコスト試算を実施

### Step 3: 本番環境はAPIキー認証へ移行検討

```
サブスクリプション枠（月額定額）→ APIキー（従量課金）
・メリット：月次クレジットの上限到達リスクがない
・デメリット：利用量に応じたコスト管理が必要
```

## 移行判断フロー

```
Q. 自動化処理でClaudeを使っているか？
  → No: 対応不要
  → Yes:
    Q. 月次クレジット（Pro $20相当）で足りるか？
      → Yes: クレジット設定のみ確認
      → No: APIキー認証またはBedrock経由への移行を検討
```
