### プロンプトキャッシュの仕組み（重要度★★★）

- 5分TTLキャッシュ書込: 入力の×1.25倍（1回再利用で元が取れる）
- 1時間TTLキャッシュ書込: 入力の×2.0倍（9回以上で効果最大）
- キャッシュ読込: 入力の×0.1倍（大幅節約）
- 有効化: `ENABLE_PROMPT_CACHING_1H=1`（v2.1.108以降）

### キャッシュを壊す操作（注意）

ツール定義変更・tool_choice変更・画像追加削除・
拡張思考切替・MCP追加・モデル切替

### 推奨スラッシュコマンド

- `/usage`: セッションのトークン・推定コスト表示（/costの後継）
- `/clear`: 履歴完全リセット
- `/compact`: 履歴を要約で圧縮
- `/context`: コンテキスト内訳を可視化
- `/stats`: 使用パターン・ヒートマップ表示

### モデル使い分け

- Opus 4.7: 設計・大規模リファクタ（Sonnet比1.66倍、新トークナイザーで最大35%増）
- Sonnet 4.6: 日常コーディング全般（基準モデル）
- Haiku 4.5: ログ解析・定型処理（0.33倍）
