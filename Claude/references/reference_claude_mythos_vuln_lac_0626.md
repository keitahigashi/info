## Claude Mythosが変える脆弱性管理

**著者**: 佐宗 万祐子（2026年6月26日）

### Mythos Previewの能力と3つの新たな脅威

Claude Mythos Previewが「1,000以上のオープンソースプロジェクトから約2万件の脆弱性候補を検出し、その約9割が実際の脆弱性」だったという実績が示す通り、フロンティアAIは従来の脆弱性発見の常識を覆す。

これにより生じる3つの脅威:

| 脅威 | 内容 |
|------|------|
| 未公開脆弱性（0-day）の大量発見 | 高度スキルを持つ研究者限定だった発見が自動化される |
| 攻撃コード生成の高速化 | 発見から攻撃までのリードタイムが大幅に短縮 |
| AIエージェントによる攻撃の自動化 | MCPなどと組み合わせた継続的な自動攻撃が可能に |

### 脆弱性管理における新たな課題

- **脆弱性増加による管理負荷の増大**: AIによる大量発見で対応すべき数が激増
- **発見と修正のスピード差の拡大**: 攻撃者側の自動化が防御側を上回るリスク
- **判断負荷増大による対応遅延**: 優先度判断が追いつかなくなる可能性

### 求められる5つの対策柱

1. **資産の把握・管理**: 攻撃対象面の継続的な可視化
2. **優先順位付けの高度化**: すべてに対応するのではなく「影響の大きいリスクを見極める」
3. **対応の高速化・自動化**: パッチ適用・修正フローの自動化
4. **開発段階での検出強化（Shift Left）**: 開発初期からのセキュリティ組み込み
5. **運用ルールの明確化・体制構築**: AI時代に対応したガバナンス整備

### 実務への示唆

記事の核心は「すべてに対応する」発想から「影響の大きいリスクを見極める」アプローチへの転換。Claude CodeにはSecurity Reviewプラグインが存在し、/security-reviewコマンドでPRの差分を脆弱性チェックできる（Enterpriseでは全スキャンダッシュボードも利用可能）。
