## Claude Codeトークン消費を半減させる5フェーズ運用術（2026年4月5日）

### コアコンセプト
「コンテキストを小さく保つ」ことが削減の鍵。精度向上も同時に実現される。

### 5フェーズの概要

#### Phase 1：セッション準備
- CLAUDE.mdは **200行以下** に保つ（毎リクエストに含まれる）
- 長い内容はスキルファイルへ移動する

#### Phase 2：タスク開始

| モデル | 用途 |
|--------|------|
| Sonnet 4.6 | デフォルト（大半のコーディング） |
| Opus 4.6 | 複雑タスクのみ |
| Haiku 4.5 | サブエージェント向け |

- Extended Thinkingのレベル調整（`/effort` コマンド）
- プランモード（Shift+Tab）で実装前に方針確認

#### Phase 3：実行中
- サブエージェントへ調査を委任（最も効果大）
- `/btw` でサイドクエスチョン（会話履歴に残さない）

#### Phase 4：タスク切替（最重要フェーズ）

```bash
/rename タスク名   # セッションに名前をつけて保存
/clear             # コンテキストをリセット（最も簡単で最も効果的）
/compact           # カスタム要約でコンテキスト圧縮
```

#### Phase 5：振り返り
- `/cost`（API利用者）でセッションの消費量確認
- `/stats`（サブスクリプション利用者）

### 失敗パターン4つと対策
1. キッチンシンクセッション → `/clear` で定期リセット
2. 無限修正ループ → 2回失敗後は再スタート
3. CLAUDE.md肥大化 → 各記述の「削除可否」を自問する
4. 不要なMCPサーバー稼働 → `/mcp` で定期確認・無効化
