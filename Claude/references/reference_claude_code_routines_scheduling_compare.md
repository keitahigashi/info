## 3つの定期実行手段の比較

### 記事概要

公開日: 2026年4月15日（更新: 4月16日）
著者: アリヘイ（aria3）
Routines・/loop・Desktop タスクの3手段を軸・用途別に整理した実用比較記事。

### 機能比較表

| 比較軸 | Routines | /loop | Desktop タスク |
|--------|----------|-------|--------------|
| **実行場所** | クラウド | セッション内 | ローカル |
| **PC起動要否** | 不要 | 必要 | 必要 |
| **セッション依存** | なし | あり | なし |
| **最短間隔** | 1時間 | 任意（秒〜） | 任意 |
| **持続性** | 最大7日・自動期限切れ | セッション終了で消滅 | 設定まで有効 |
| **ローカルツール利用** | 不可 | 可 | 可 |
| **MCP利用** | 不可 | 可 | 可 |
| **タスク上限** | プラン別（5〜25回/日） | 最大50個 | 特になし |

### 各機能の詳細

**Routines**
- クラウド実行でPCを閉じても継続動作
- Schedule・API・GitHub の3トリガータイプに対応
- 複数トリガーの組み合わせも可能
- ローカルファイルやMCPサーバーへのアクセスは不可

**/loop**
- セッション内で「5分ごとにデプロイ完了確認」などの一時的ポーリング
- 間隔は自然言語で指定（例: "every 5 minutes", "hourly"）
- セッション終了で自動消滅
- ローカルツール・MCPサーバーが使える

**Desktop タスク**
- ローカル実行でセッション非依存
- ローカルファイルとMCPサーバーを活用できる
- Routinesより柔軟なスケジュール設定

### 選択基準

```
PC閉じても動かしたい → Routines
ローカルツール/MCPが必要 + 継続実行 → Desktop タスク
今のセッションで一時的に回したい → /loop
```

### 活用例マトリクス

| ユースケース | 推奨手段 | 理由 |
|------------|---------|------|
| 夜間PRレビュー | Routines | PCオフでも継続 |
| デプロイ後の動作確認 | Routines (API trigger) | CIパイプラインから呼び出し |
| ローカルDB監視 | Desktop タスク | ローカル接続が必要 |
| デバッグ中の繰り返しテスト | /loop | セッション内の一時利用 |
| Slack通知連携 | Routines (Schedule + MCP不要な場合) | クラウド実行で安定 |
