## 概要
Claude Code GitHub Actionsを使い「AIレビューだけでマージ」を実現するまでの設計・実装・段階的ロールアウト戦略を解説。カウシェ社の83%自動マージ実績を目標にした実践ガイド。

## 詳細

### 基本構成
- **2動作モード**: インタラクティブ（@claudeメンション）・オートメーション（promptパラメータでヘッドレス）
- `concurrency` で同一PR新コミット時に古いレビューをキャンセル（二重課金防止）
- `--max-turns 12` でエージェント会話ターン制限
- `--allowedTools` で許可ツール絞込

### 3ペルソナ並列レビュー
- quality-review: コード品質・テスト
- security-review: OWASP準拠
- infra-review: DB効率・トランザクション
- `dorny/paths-filter` で変更パス判定し条件付きジョブ実行

### 信頼度スコアリング
```
信頼度 = 一致度 × CI通過率 × 変更規模
```
- 一致度: 3ペルソナ揃い=1.0、分裂=0.5
- CI通過率: 全テスト=1.0、flakyあり=0.7
- 変更規模: 100行以下=1.0、500行以上=0.3
- **自動マージは信頼度0.8以上のみ**

### コスト試算
- 400行diff: 約$0.04/PR
- 10名チーム20PR/日: 月額約$24
- GitHub Actionsランナー: 月$10〜30

### 段階的ロールアウト（6ヶ月計画）
1. Step 0（2〜4週）: テストカバレッジ80%・CI整備
2. Step 1（1〜2週）: シャドーモード（ノイズ率20%以下・見逃し率10%以下）
3. Step 2（3〜4週）: 10%自動マージ（単ファイル100行未満リファクタ等）
4. Step 3（2〜3ヶ月）: 30%自動マージ+信頼度スコアリング
5. Step 4（半年後）: 83%自動マージ

### 自動マージ対象外（残り17%）
DBスキーマ・Proto定義、Terraform等インフラ、認証・決済フロー、大規模アーキテクチャ変更、複数サービス横断変更

### フォールバック設計
- キルスイッチ: `vars.CLAUDE_REVIEW_ENABLED == 'true'`
- 自動マージ率1日20%以上変動時アラート
- `/review-block PR-123` で強制人間レビュー化
