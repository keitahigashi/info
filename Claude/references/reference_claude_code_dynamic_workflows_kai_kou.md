## 概要

- 2026年5月28日にリリースされたClaude Codeの新機能
- 「JavaScriptのオーケストレーションスクリプトを自動生成し、最大1,000のサブエージェントを並列実行」

## 3つのトリガー方法

| トリガー | 方法 | 用途 |
|---------|------|------|
| `/deep-research` | ビルトインコマンド | 複数角度からの並列検索・情報クロスチェック |
| "workflow"キーワード | プロンプトに含める | 簡単な起動方法 |
| `/effort ultracode` | コマンド | 自動判断モード |

## 実行フロー（5ステップ）

1. バージョン確認（v2.1.154以降）
2. プロンプト送信
3. `Ctrl+G` でスクリプト確認
4. 並列実行監視（`/workflows`）
5. 最終レポート受信

## ワークフローの保存・再利用

- 成功したワークフローをスラッシュコマンド化して再利用可能
- プロジェクト共有版と個人用版を選択可能

## コスト管理のポイント

- トークン消費が通常使用より大幅増加
- スコープの明示で不要なスキャン範囲を絞る
- 読み取り専用フェーズと書き込みフェーズを分離
- 段階実行（まず小規模でテスト）

## 実例

- Bun開発者がDynamic Workflowsを活用し「6日間で96万行のRustコードを生成、テスト成功率99.8%」を達成

## 対応プラン

- すべての有料プラン（Pro/Max/Team/Enterprise）で利用可能
- Enterprise は管理者による Claude Code 設定での有効化が必要
