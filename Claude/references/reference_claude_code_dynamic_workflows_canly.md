## 概要

Claude Code v2.1.154 で追加されたDynamic Workflowsのリサーチプレビュー機能を実際に試した記事。技術的な概念整理と、カンリー社内での実運用体験をセットで紹介。

## Dynamic Workflowsとは

同じセッション内で数十〜数百のサブエージェントを並列起動し、大規模タスクを自動オーケストレーションする仕組み。

## 4つの特徴

| 特徴 | 内容 |
|------|------|
| 並列処理 | 複数エージェントの同時実行で処理時間を短縮 |
| 独立検証 | 各結果をチェック後に統合 |
| 進捗永続化 | 中断されたジョブを途中から再開 |
| 敵対的レビュー | 複数視点から反証し収束まで反復 |

## 起動方法

1. 直接「動的ワークフロー作成」をプロンプトで依頼
2. エフォート選択で「ultracode」（xhigh+workflows）を選択

## カンリー社内での実運用体験

社内プラグイン管理リポジトリの全体点検に活用：
- **40エージェント**が約40分で並列実行
- 監査で**71件**を発見
- 敵対的レビューで**41%を誤検知として除外**
- 結果として高品質な検証を実現

## 既存機能との違い

| 機能 | 規模感 | オーケストレーション |
|------|--------|------------------|
| Subagents | 少数 | 手動設計 |
| Dynamic Workflows | 数十〜数百 | Claude自動設計 |

## 実務上の知見

- 敵対的レビューの組み込みにより誤検知率を大幅削減できる
- 監査・点検系タスクに特に向いている
- 初回は小規模スコープで動作確認してからスケールアップが推奨
