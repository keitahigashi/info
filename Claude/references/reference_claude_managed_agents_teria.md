## Claude Managed Agents とは

2026年4月8日にAnthropicがパブリックベータをリリースした「長時間・非同期タスク実行に特化したマネージドAIエージェント基盤」。

### Messages API との違い

| 項目 | Messages API | Managed Agents |
|------|-------------|----------------|
| 位置づけ | モデルへの直接アクセス | マネージドインフラ上のエージェント |
| 最適用途 | カスタムループ・細かい制御 | 長時間タスク・非同期ワークロード |
| インフラ管理 | 自前構築 | Anthropicが管理 |

## 3つの中核コンポーネント

### ① セッション（Session）
- エージェント操作の全履歴を記録する追記専用ログ
- 長時間タスク中のコンテキスト溢れに対応
- `getEvents()` で任意位置から履歴取得可能

### ② ハーネス（Harness）
- Claudeを呼び出し、ツールをインフラへルーティング
- クラッシュ時も `wake(sessionId)` で再起動
- セキュリティ境界が明確（認証情報にアクセスしない設計）

### ③ サンドボックス（Sandbox）
- Claudeが生成したコード実行環境
- Python・Node.js・Go 対応
- コンテナ障害時は「交換可能な存在」として新規立ち上げ

## 主な機能

- 数分〜数時間のタスク継続実行
- ステートフルセッション（ファイルシステム・会話履歴の永続化）
- MCP経由でサードパーティサービス連携
- スコープ付き権限とトレーシング
- ファイル読み書き・Webブラウジング・コード実行を標準搭載
- 中断後の自動再開機能

## 料金体系

| 項目 | 料金 |
|------|------|
| モデルトークン | 標準API料金に準じる |
| 実行時間 | $0.08/セッション時間（アイドル時間除外） |
| Webサーチ | $10/1,000回 |

## 採用企業

Notion・楽天・Asana・Sentry などがパブリックベータ前から開発を進めている。

## 設計哲学

**「ペット」から「家畜」へのシフト**：従来の単一コンテナ（障害時に手動復旧が必要）から、複数コンポーネント分離（自動復旧可能）へ移行。
