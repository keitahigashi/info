## 4本柱の対策体系

### 1. データ取り扱い

- モデル学習設定の確認・統一（法人プランポリシー）
- 入力情報ルールの明文化

### 2. 権限設計（Permissions）

```json
// 推奨設定イメージ
"permissions": {
  "deny": ["sudo", "rm -rf", ".env アクセス"],
  "ask": ["rm"]
}
```

- deny: sudo・削除系コマンド・`.env`ファイルへのアクセス
- ask: `rm`コマンドなど危険度中の操作

### 3. 秘密情報管理

- APIキーを会話に直接貼り付けない
- `.env`ファイルで管理し、AIには事実のみ伝達
- リポジトリのプライベート設定を徹底

### 4. 利用ガイドライン策定（5項目）

- 利用ツール・プランの一覧
- 入力情報の区分（機密/社内/公開）
- 初期設定の規定
- 成果物取り扱いルール
- インシデント報告フロー

## 全社展開の推奨順序

業務棚卸し → セキュリティ整備 → 人事評価への組み込み → AIエージェント化

「不安だから禁止」ではなく「安心して使える仕組みを作る」という実践的なスタンスが一貫したメッセージ。
