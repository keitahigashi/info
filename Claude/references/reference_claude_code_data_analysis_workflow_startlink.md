## ChatGPT Code Interpreterとの差別化点

| 比較軸 | Claude Code | ChatGPT Code Interpreter |
|-------|------------|--------------------------|
| ファイルアクセス | ローカルFS直接アクセス | ファイルアップロード方式 |
| データ規模 | 複数GBも処理可能 | 制限あり |
| 処理統合 | 生成→実行→解釈を一体化 | 主にコード生成のみ |

## 基本ワークフロー（4ステップ）

1. **データ把握**: カラム一覧・データ型・行数・欠損値を自動分析
2. **分析実行**: 月別推移・構成比・前年比などを複数同時実行
3. **可視化**: matplotlibで日本語対応グラフを生成
4. **レポート生成**: Markdown形式のレポートを自動作成

## 実践ユースケース

| データソース | 分析内容 |
|-------------|---------|
| GA4 | チャネル別コンバージョン率分析 |
| Google Search Console | CTR改善対象キーワード抽出 |
| HubSpot CRM | 受注率ファネル分析 |
| 複数ソース統合 | マーケティングROI分析 |

## 実装例

```python
# Claude Codeに依頼するプロンプト例
"""
data.csv を読み込んで以下を実行してください：

2. 月別売上推移をグラフ化（日本語フォント対応）
3. 上位10顧客の抽出とランキング表
4. 分析結果をsummary.mdとして保存
"""
```

## 限界と使い分け

**得意**: 探索的分析（EDA）・定型分析の自動化・少〜中規模データ

**不向き**:
- リアルタイム処理
- 数十GB超の超大規模データ
- 複雑な統計モデル構築（Jupyter推奨）

## 設計上のポイント

- 分析指示は「何を知りたいか」を明示する（手法ではなく目的を伝える）
- 出力形式を最初に指定する（Markdown/CSV/グラフ等）
- 中間ファイルを残すよう指示すると再分析が容易になる
