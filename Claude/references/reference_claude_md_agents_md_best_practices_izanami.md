## CLAUDE.md / AGENTS.md 設計5原則

**5つの設計原則**

| 原則 | 内容 |
|------|------|
| WHY/WHAT/HOW を定義 | プロジェクトの目的・構造・検証方法を明文化 |
| 指示は最小限に | Claude Codeシステムが約50トークンを消費するため、残り100〜150指示に収める |
| 段階的開示 | タスク固有の詳細は別ファイルへ。CLAUDE.mdはポインタのみ記載 |
| 自動リンター活用 | コードスタイル判定をLLMに委ねず、Biome等の自動ツールに任せる |
| 自動生成を避ける | 最重要ファイルだからこそ手動で慎重に作成 |

## ファイル長・指示数の目安

- ファイル長の理想: **300行未満**（実例では60行以下）
- 指示数の上限: **150〜200件**（システム消費約50件を引いた実使用可能数）
- 情報過多になると重要な指示まで無視されやすくなる設計上の問題を指摘

## クロスプラットフォーム標準としてのAGENTS.md

AGENTS.mdはClaude Code・Cursor・Devin・Gemini CLI等でも共通参照されるクロスプラットフォーム標準として位置づけ。同一ファイルを複数AIツールで共有することで、チームのAI設定を一元管理できる。

## 段階的開示の実装パターン

- CLAUDE.md本体: プロジェクト目的・構造・禁止事項のみ記載
- 詳細ルール: `docs/ai-rules/` などサブディレクトリに分割
- CLAUDE.mdからはポインタ（ファイルパス参照）のみを記述
- タスクに応じて必要な詳細ファイルだけをコンテキストに追加
