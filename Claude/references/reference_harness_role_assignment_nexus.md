## エージェントハーネスの6要素

- **実行ループ**: エージェントが繰り返し稼働する主軸
- **ツール登録**: 使用可能な外部ツール・API一覧の定義
- **コンテキスト管理**: セッション間で必要な情報を引き継ぐ仕組み
- **状態保持**: 現在のタスク進捗・変数の永続化
- **ライフサイクルフック**: 開始・終了・エラー時のカスタム処理
- **評価インターフェース**: 実行結果の品質チェック機構

## 最小構成実装の3原則

1. **破壊的コマンドの事前ブロック**: 許可リスト外のシェルコマンドを設定レベルで禁止
2. **「何をいつ読むか」の優先度定義**: コンテキストロードのタイミングをルール化
3. **検証者の独立分離**: 実装エージェントと検証エージェントを別セッションに分離

## 役職分化フロー

```
需要調査 → 承認 → 実装 → 検証 → 課金設定 → コンテンツ作成
```

各ステップを独立したエージェントが担当し、前段の成果物を入力として次段が動作する。

## 実績・成果

- 「OK」と打つだけで5本のAPI設計・実装・セキュリティ修正30箇所を自動実行
- 258件のテスト合格（ゼロタッチ）
- Stripe課金フロー8種類を自動作成
- 役職分化により並列実行でリードタイムを大幅短縮
