## 社内プラグインマーケットプレイス設計：部署横断スキル共有の実現

### 背景と課題
- 各メンバーが作成したスキルは `~/.claude` 配下にローカル保存されるだけで、チーム間共有手段がなかった
- 類似機能の重複開発が頻発していた

### ソリューション：GitHub ベースのマーケットプレイス
必要なファイルは2種類：
- `marketplace.json`：プラグインカタログ全体の定義
- `plugin.json`：各プラグインのメタ情報

主な利点：
- Git 管理による変更履歴の透明性
- GitHub 権限体系をそのままセキュリティ設計に流用
- 開発リポジトリ単位での自動適用

### ディレクトリ設計思想
第一階層を `services`（サービス単位）と `domains`（領域単位）に分類。「変わりやすいロール・プロジェクト単位」を避け、「変化しにくいサービス・ドメイン単位」を採用することで組織変更に強い構造を実現。

### AI フレンドリーな CLAUDE.md 設計
ディレクトリ規則・必須フィールド・命名規則を AI が実行できる形で構造化。利用者が「○○のスキルをプラグインとして追加して」と指示するだけで、Claude Code がファイル配置から検証まで自動完了する。

### 成果
バックエンド・PdM・QA・インフラ・Atlassian の複数チームから Go コードレビューや仕様支援ツールなど計5プラグインが集約され、部門横断活用を実現。
