## トークン消費半減の5フェーズ運用術

### 概要

2026年4月5日公開。「コンテキストを小さく保つことがトークン削減の核心」という
実測に基づく体系的な5フェーズ運用ガイド。

### 5フェーズの詳細

#### Phase 1：セッション準備

- CLAUDE.mdを200行以下に保つ（毎リクエストに含まれるため肥大化は直結コスト増）
- デプロイ手順等はスキルに移動し必要時のみ読み込む設計

#### Phase 2：タスク開始

- Sonnet 4.6をデフォルト、複雑なタスクのみOpusを使用
- `/effort`コマンドで拡張思考レベルを調整
- Shift+Tabでプランモードに入り方針確認だけ先行

#### Phase 3：実行中

- サブエージェントに調査を委任（メインコンテキストを汚さない）
- `/btw`コマンドで履歴に残さないサイドクエスチョンを実行

#### Phase 4：タスク切替

- `/clear`でコンテキストをリセット（最重要かつ最簡単）
- `/compact`でカスタム要約・圧縮

#### Phase 5：振り返り

- `/stats`（サブスク）または`/cost`（API）でトークン消費を確認

### 削減効果早見表

| テクニック | 効果 | 難易度 |
|---------|-----|------|
| /clearでタスク切替 | 大 | 簡単 |
| サブエージェント委任 | 大 | 簡単 |
| CLAUDE.md最適化 | 中 | 簡単 |
| Sonnetデフォルト化 | 中 | 簡単 |

### 結論

「タスクが変わったら /clear を打つ」という単純ルールだけで
トークン削減と回答精度向上を同時に実現できる。
