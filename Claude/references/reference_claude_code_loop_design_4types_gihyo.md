## ループ設計の4分類と実践ガイド

### 記事概要

2026年7月9日公開。Anthropicが公開した公式ループ設計入門ドキュメントの内容を日本語で紹介・解説した記事。「エージェントに毎回プロンプトを書くのではなく、停止条件を持つ継続的作業サイクルを設計する」という思想の実践ガイド。

### ループの4分類

- **ターンベース**: ユーザーのプロンプトを起点に1サイクル完走、次は再度ユーザーが起動。短期・非定期タスク向け。
- **ゴールベース**: `/goal` コマンドで完了条件を事前定義。評価モデルが条件達成を確認し、未達なら再実行するフィードバックループ。機械的に検証可能な条件（テスト通過・Lint 0件等）が最適。
- **時間ベース**: `/loop` で実行間隔を指定し定期繰り返し。外部システムの変化への対応や定期レポート生成に適する。
- **プロアクティブ**: イベントやスケジュールをトリガーとし、人間の直接介入なしで進行。バグ報告のトリアージや Issue の自動ラベル付けなど繰り返し定型業務向け。

### 品質維持の3要件

1. コードベースの整備（CLAUDE.md 整備・スキル記述）
2. 自己検証の仕組み（テスト・Lint の自動実行）
3. 停止条件の明確化（無限ループ防止）

### トークン・コスト管理

- 機能・モデル選択を事前に絞る
- 小規模試行で単価を把握してから量産
- `/goal` でトークン消費量を随時確認

### 実践アドバイス

「まず1つのボトルネック作業を選び、段階的にループを拡充する」という漸進的アプローチを推奨。完璧なループ設計より早期の部分導入を優先。
