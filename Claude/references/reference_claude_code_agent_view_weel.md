## Agent view in Claude Code — WEEL 解説

### 概要

2026年5月14日公開。Claude Code Agent View（Research Preview、2026年5月11日リリース）の詳細解説。起動方法から内部アーキテクチャ、業界別活用シーンまで網羅。

### 起動方法

```bash
claude agents       # Agent Viewダッシュボード起動
# または
←キー              # 任意のClaude Codeセッション中にAgent Viewへ移行
```

### 主な機能

| 機能 | 説明 |
|------|------|
| セッション一覧 | 進行中・待機中・完了を1画面で表示 |
| ピーク | セッション選択でインライン確認 |
| リプライ | アタッチ不要で簡単な返答が可能 |
| アタッチ | フルインタラクションモードへ移行 |

### セッションテーブルの表示項目

- セッション名
- 状態カラーアイコン（🟢実行中 / 🟡待機 / ⚪完了）
- 最後のアクション要約
- ステータス変更からの経過時間

### アーキテクチャ

```
スーパーバイザープロセス
├── バックグラウンドセッション A（git worktree A）
├── バックグラウンドセッション B（git worktree B）
└── バックグラウンドセッション C（git worktree C）
```

- スーパーバイザーが全セッションのライフサイクルを管理
- **git worktree** でセッション間のファイル競合を防止
- 各セッションが独立した作業ディレクトリで動作

### 対応プラン

Pro、Max、Team、Enterprise、Claude API（追加料金なし）

### 業界別活用シーン

| 業界 | 活用例 |
|------|--------|
| Web開発 | フロントエンド・バックエンド・テストを並列実装 |
| 金融 | 複数レポートの同時生成・データ分析 |
| 研究 | 文献調査・コード実装・論文執筆の並行処理 |

### ピーク＆リプライの利点

わざわざセッションにアタッチしなくてもAgent View画面から直接返答可能。複数の「入力待ち」セッションを素早くさばける。
