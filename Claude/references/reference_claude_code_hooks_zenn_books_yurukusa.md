## 核心的な戦略

### 無料配布の理由
「困りごとへの直接応答の経路」として hook を公開。本の売上はその結果として捉える。
- 2025年の無料配布なし時代：売上¥0
- 配布開始後：自然な購入が発生

### 無料から有料への3つの経路

1. **README の動線**: hook の説明末尾に関連 Zenn 本への自然な導線を記載
2. **テストファイルの活用**: 実機経験を記録し、本の該当章への参照を記入
3. **取り込み依頼内での言及**: 事故予防法の説明で関連本への動線を提示

## 重要な発見：継続更新の必要性

著者が27日間、有料本への章追加を停止した期間に**新規購入が減少**することを確認。
→ 無料配布・有料コンテンツの**両方の継続更新**が「購入経路を動かす」鍵。

## 再現可能な手順

1. 自分の Claude Code 運用中の事故・失敗を毎週整理し hook として構造化
2. GitHub で自由利用条件（MIT等）で公開
3. 既存の有料本への自然な導線を README に組み込む
4. hook・本の両方を継続的に更新し続ける

## Claude Code 利用者への示唆

- 自分の運用ノウハウを hook 化することで知見が蓄積・共有される
- GitHub での無料配布はコミュニティへの貢献であり、信頼形成につながる
- 「無料 → 有料」の導線は押し付けではなく、困りごとの延長線上に置くのが有効
