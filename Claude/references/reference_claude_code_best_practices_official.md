## 概要
Claude Codeの効果的な活用パターンを網羅した公式ベストプラクティスガイド。Anthropic内部チームと多数のエンジニアの実践に基づく。最重要制約は「コンテキストウィンドウの管理」。

## 詳細

### 核心原則: コンテキストウィンドウ管理
- コンテキストウィンドウが満杯になるとパフォーマンス低下
- すべてのメッセージ、読み取ったファイル、コマンド出力がコンテキストを消費
- カスタムステータスラインでトークン使用量を継続追跡すべき

### 1. 検証方法を与える（最も高レバレッジ）
- テストケース、スクリーンショット、期待される出力を含める
- 検証なし → 正しく見えるが機能しないコードを生成するリスク
- Chrome拡張機能でUI変更を視覚的に検証
- 症状ではなく根本原因に対処する指示を出す

### 2. 探索→計画→実装→コミット（4フェーズ）
- Plan Modeで探索（ファイル読み取りのみ、変更なし）
- 計画作成 → Ctrl+Gでテキストエディタで直接編集
- Normal Modeで実装・テスト実行
- コミット＆PR作成
- 注意: 小さなタスクでは計画をスキップしてよい

### 3. プロンプトで具体的コンテキストを提供
- タスクをスコープする（どのファイル、シナリオ、テスト設定）
- ソースを指摘する（git履歴、特定ファイル）
- 既存パターンを参照する（「HotDogWidget.phpのパターンに従って」）
- 症状を説明する（エラーメッセージ貼り付け、可能性のある場所指定）
- リッチコンテンツ: @でファイル参照、画像貼り付け、データパイプ、URL指定

### 4. 環境設定のベストプラクティス
**CLAUDE.md:**
- /initで生成し時間をかけて改善
- 短く人間が読める状態に保つ
- 「削除するとClaudeが間違いを犯すか？」で各行を判定
- 膨らんだCLAUDE.mdは指示が無視される原因
- ドメイン知識はSkillsに分離（オンデマンド読み込み）

**パーミッション:**
- Auto mode: 分類器モデルがレビュー、スコープエスカレーションのみブロック
- ホワイトリスト: npm run lint, git commitなど安全なコマンド
- サンドボックス: OS レベルの分離

**CLIツール:** gh, aws, gcloud, sentry-cliなど。`--help`で未知ツールも学習可能

**MCP:** Notion, Figma, DB等の外部ツール接続

**Hooks:** 例外なしで毎回発生すべきアクションに使用（CLAUDE.mdとは異なり決定論的）

**Skills:** .claude/skills/にSKILL.md。disable-model-invocation: trueで手動トリガー専用に

**サブエージェント:** .claude/agents/に定義。独自コンテキスト・ツールセットで実行

**プラグイン:** /pluginでマーケットプレイス参照。コードインテリジェンスプラグイン推奨

### 5. 効果的なコミュニケーション
- シニアエンジニアに聞くような質問をする
- 大きな機能はClaudeにインタビューさせる（AskUserQuestionツール）
- 仕様をSPEC.mdに書き出し、新しいセッションで実装

### 6. セッション管理
- Esc: 中断、Esc+Esc or /rewind: 巻き戻し
- /clear: 関連のないタスク間でコンテキストリセット
- 2回修正失敗 → /clearして学んだことを組み込んだ新プロンプト
- /compact <instructions>: 要約のカスタマイズ
- /btw: 会話履歴に入らないサイド質問
- サブエージェントで調査を委譲（メインコンテキスト保護）
- claude --continue / --resume で再開
- /rename でセッション命名

### 7. 自動化とスケール
- claude -p "prompt": 非対話型モード（CI/パイプライン統合）
- --output-format json/stream-json: 構造化出力
- 並列セッション: デスクトップアプリ、Web、エージェントチーム
- Writer/Reviewerパターン: セッションA(実装) → セッションB(レビュー) → フィードバック反映
- ファンアウト: ファイルリスト×claude -p でバッチ処理、--allowedToolsで制限
- auto mode: --permission-mode auto で自律実行

### 8. 一般的な失敗パターン
1. **キッチンシンクセッション**: 関連のないタスクが混在 → /clear
2. **何度も修正**: 2回失敗後 → /clearして良い初期プロンプト
3. **過度なCLAUDE.md**: 長すぎて重要ルールが埋没 → 削除 or Hooks化
4. **検証ギャップ**: エッジケース未処理 → テスト必須
5. **無限探索**: 無スコープ調査でコンテキスト消費 → サブエージェント使用
