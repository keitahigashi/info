## 核心的発見

> 同じモデルでハーネスを変更するとSWE-benchスコアが**22点変動**する一方、モデルを変更しても**1点程度**しか変わらない（Morph社分析）

## Claude Code vs Codex アーキテクチャ比較

| 項目 | Claude Code | Codex |
|---|---|---|
| 設計思想 | 工房型（ローカル環境直接作用） | 密閉型（クラウドサンドボックス独立稼働） |
| エラー対応 | リアクティブ（事後対応型） | プリベンティブ（予防型、pre-commitフック） |
| SWE-bench | 80.9% | 80.0% |
| Terminal-Bench | **59.3%** | 47.6% |

一般コーディングではほぼ同等だが、DevOps/インフラタスクではClaude Codeが優位。

## ハーネスの4構成要素

### 1. コンテキストエンジニアリング
- 設定ファイルを最小化し、必要時に参照可能な形で外部化
- ステータスラインで長時間稼働時のモデルドリフトを抑制

### 2. ツールチェーン設計
- MCP: 外部システム接続の通信規格
- Skills: ツールロックインなく再利用可能なコンテキストをパッケージ化

### 3. ガードレール（3段階防衛）
- 入力: プロンプトインジェクション対策
- 処理: 権限最小化
- 出力: Human-in-the-Loop承認

### 4. エラーハンドリング・メモリ管理
- トークンコスト最適化
- マルチエージェント協調時の状態管理
- 構造化JSONログ出力

## 実践的導入手順（4ステップ）

| Step | 期間 | 内容 |
|---|---|---|
| 1 | 1-2週間 | 現状分析 |

| 4 | 継続的 | 運用・改善 |
