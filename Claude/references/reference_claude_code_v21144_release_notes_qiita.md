## Claude Code v2.1.144 の release notes を構造の視点で読む

### 分析フレーム

30件以上の修正項目の中から、**「利用者の指示や設定がcontextに存在するにもかかわらず、実際の動作がそれを参照しない」**という同型の構造的問題を示す6件を抽出。

### 6つの主要修正点

| # | 修正内容 | 問題の性質 |
|---|--------|---------|
| 1 | MCP paginated tools/list応答で最初の頁のみ読込 | "silently dropping"（沈黙で脱落） |
| 2 | IDE/applyFlagSettingsでモデル選択が未反映 | 利用者意図と実行経路の乖離 |
| 3 | BedrockとVertex用Opus 1M選択が未適用 | 選択肢表示と実際の動作の不一致 |
| 4 | forceLoginMethod設定が認証層で未参照 | 3層間の食い違い |
| 5 | MCP非対応MIME形式（SVG等）で会話崩壊 | エラーハンドリングの欠落 |
| 6 | セッション名がプラグイン出力から生成 | 局所的整合による全体的破壊 |

### Anthropicが使用した定義的用語

- **「沈黙で脱落」**（silently dropping）
- **「沈黙で失敗」**（silently fails）
- **「沈黙で上書き」**（silently overwritten）
- **「会話の崩壊」**（breaking the conversation）

著者はこれらの表現を「利用者側の長期的な観察を供給側が同じ言語で記述した証拠」と位置付けている。

### 推奨確認手順

v2.1.144更新後に以下を再検証：
1. MCPサーバーのページネーション応答が全頁読み込まれているか
2. IDEのモデル設定が実際の実行に反映されているか
3. Bedrock/Vertex環境でのモデル選択が機能しているか
4. forceLoginMethod設定が正しく動作しているか
5. SVG等の非対応MIMEタイプ処理時にエラーが適切に処理されるか
6. セッション名が想定通りに生成されているか

### 所感・背景

v2.1.144は2026年5月15日頃リリース。本記事はリリースノートの表面的な修正一覧ではなく、背後にある共通の構造問題（設定の非伝播・サイレントフェイルアー）を浮き彫りにする分析記事。
