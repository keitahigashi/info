| Explore | Haiku | 読み取り専用・高速ファイル探索 |
| Plan | - | 計画モード用リサーチ |
| General-purpose | - | 複雑なマルチステップ操作 |

## カスタムSub-Agent作成方法

```
配置場所:
  .claude/agents/          # プロジェクト固有
  ~/.claude/agents/        # グローバル共通

ファイル形式: Markdownファイル（.md）
```

**必須フィールド**:
- `name`: エージェント名（識別子）
- `description`: 起動タイミング制御の核心。ここに適切な説明を書くことで、どのタスクでこのエージェントを使うか自動判定される

**オプションフィールド**:
- `tools`: 使用可能なツールの制限
- `model`: モデルの指定（Haiku/Sonnet等）
- `memory`: セッション間の記憶設定

## 並列実行の効果と注意点

**効果**:
- 依存関係のないタスク（フロント調査・バック調査・セキュリティチェック等）を同時実行
- 作業時間を逐次比 **1/3〜1/4** に短縮可能

**注意点**:
- 同一ファイルの並列編集は競合リスクあり
- 並列化できるのは依存関係のない独立タスクのみ
- 結果の統合（merge）工程を設計に含める必要がある

## 設計パターン

- **マスター＋ワーカー**: メインエージェントが指揮し、サブが実行
- **並列探索＋統合**: 複数サブが独立探索し、メインが統合判断
- **パイプライン型**: 前段の結果を後段が受け取る逐次委譲
