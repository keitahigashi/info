## 実装フロー（3段階）

1. 現状分析: 既存フォーマット3〜5件をアップロード→Claude Codeで変動項目・固定項目を自動抽出
2. 自動生成スクリプト: 入力(CSV)→処理(テンプレート入力+計算)→出力(PDF)をPythonで実装（openpyxl/pandas/weasyprint）
3. 例外処理と手順書: エラー行のログ記録、複数フォーマット対応、非エンジニア向け引き継ぎ手順書（Markdown）を自動生成

## 重要プロンプトの型

**基本形**: 「入力：[フォーマット] → 処理：[動作] → 出力：[ファイル形式]」を明記し、「仮定した点は必ず明記してください」を付加

## よくある失敗4パターン

1. 最初から全プロセス連携を狙う（→小さく1事例から）
2. 入力・処理・出力を曖昧に指示（→3点セットを明示）
3. 例外ケースを後付けで追加（→初期設計に組み込む）
4. 担当者1人化したスクリプト（→手順書と複数人運用ルール化）
