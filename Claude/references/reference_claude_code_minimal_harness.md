## 概要
テストフレームワークやガバナンスツールを使わず、CLAUDE.md + Skills + Hooks の3要素だけでClaude Codeを制御する「最小構成」ハーネス設計。「人間が読む用に書いてません。お手持ちのClaudeに読ませてください」という前提で、AIへの直接指示を想定した構成。

## 詳細

### 3層アーキテクチャ

#### ① Hook（settings.json）
- プロンプト送信時に自動発火する初期トリガー
- 「やることリスト」を通知する役割
- ユーザー入力の前処理を担当

#### ② CLAUDE.md
- 「Claudeの行動制約。何をすべきか、何をしてはいけないかの憲法」
- 行動ガイドラインの中央集約
- 制約と原則の定義

#### ③ Skills（SKILL.md）
- 「具体的な手順書」「『どうやるか』を定義した再利用可能なプロシージャ」
- モジュール化された実行ロジック

### 設計思想
- テストフレームワーク・ガバナンスツール不要の「最小構成」
- スケーラビリティとシンプリシティのバランス
- 3要素の役割分担: Hook=いつ発火するか、CLAUDE.md=何をすべきか、Skills=どうやるか
