## 概要
エス・エム・エス社がエンジニア生産性向上のために設計した、Claude Codeのワークフロー設計知見。5段階の移譲レベル、コンテキスト管理、マルチエージェント構成を体系化。

## 詳細

### 5段階の移譲レベル
| レベル | 内容 |
|--------|------|
| Consult | 人間主導でAIに相談 |
| Agree | 人間とAIで協業 |
| Inquire | LLM主導、人間が確認 |
| Delegate | LLM完全移譲 |

### コンテキスト管理が核
「エージェントはコンテキストがすべて」。人間が自然に切り替えるワーキングメモリ状態をLLMにも持たせることが重要。

### Progressive Disclosure戦略
- CLAUDE.mdは最小限（80%以上のタスクで必要な情報のみ）
- Skills/Commandで限定的な場面の知識を段階的に供給
- 情報そのものより参照先となる起点情報を記載

### ツール選択の指標
| ツール | 用途 |
|--------|------|
| Slash Command | 人間が直接実行する手順 |
| Skill | 複数セッションで再利用できる知識 |
| Subagent | 複雑な責務の独立実行 |

### マルチエージェント・オーケストレーション
- メインセッションは調整役に専念
- 各サブエージェントが単一責任
- フロー: チケット確認→設計→API仕様決定→実装→レビュー→報告
- コードレビュー: 否定形の指摘が効果的、N並列化で質向上

### トレードオフ
- 理論的解法範囲は広がるが実装難易度が上昇
- 対話的ブラッシュアップに向かない（伝言ゲーム状態）
- 部分的な採用（設計・レビューのみ移譲）でバランスを取る
