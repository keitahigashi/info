## コア機能

動的ワークフローは「1回の会話では処理しきれない大規模な調査・コード移行・監査」を、Claudeが生成したJavaScriptスクリプトで複数のサブエージェントに分散実行する機能。

### 反証検証メカニズム
複数エージェントの主張を相互検証し、生き残った結論だけを報告する品質確保の仕組み。

## 技術仕様

| 項目 | 仕様 |
|------|------|
| 同時実行数 | 最大16体（マシンスペックに依存） |
| 1ランあたり上限 | 1,000体（暴走防止） |
| 起動方法 | プロンプトに「workflow」を含める / `/deep-research`コマンド / `/effort ultracode`で自動化 |

## 実装上の注意点（3つの失敗パターン）

1. **本番リポジトリへの直接実行**: 大規模変更が意図せず適用されるリスク
2. **コスト試算なしの常用**: トークン消費量が通常の数倍〜数十倍になる可能性
3. **許可リスト設計不足**: 長時間ランが途中で停止する原因に

## 日本企業への影響

- 人手不足の情シスが外注頼みだった大規模作業を内製化できる可能性
- コスト構造が「人月」から「トークン消費量」へシフト
- 反証検証の仕組みにより、ハルシネーションリスクを従来比で低減
