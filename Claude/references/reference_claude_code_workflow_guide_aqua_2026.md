## 出典

AQUA合同会社（AQUAテックブログ）: https://www.aquallc.jp/claude-code-workflow-guide/

## Explore→Plan→Code→Commit ワークフロー

4段階開発フロー：①コードベース理解、②変更計画立案、③実装、④Gitコミット。複雑な変更ほどPlanning段階に時間をかけることが手戻り削減の鍵。

## プランモードの活用

- 起動: `Shift+Tab` を2回押下
- 効果: ファイル読み取りのみ許可し、編集・実行を禁止して計画フェーズを強制
- 用途: 複数ファイル変更・アーキテクチャ影響時に必須

## モデル使い分け戦略

```
Opus：複雑な設計・バグ調査向け
Sonnet：日常的なコーディング向け
Haiku：単純修正・定型作業向け
```

推奨設定: `/model opusplan`（計画時自動Opus、実装時自動Sonnet）

## コンテキスト管理

- `/context` で使用量を可視化
- `/compact` で30分ごと圧縮実行
- `/clear` でタスク切り替え時にリセット

## プロンプト作成4原則

①スコープ明確化、②情報ソース指定、③既存パターン参照、④検証基準含有。

## バグ修正フロー

調査→失敗テスト作成→修正→回帰テスト確認のTDDアプローチが相性最適。

## コスト最適化

平均 $6/開発者/日。opusplan+定期compact+具体的プロンプトで効率化。

## セッション管理

`/rename` で名前付与→ `claude -c` で素早く再開。機能ブランチごとセッション分割推奨。
