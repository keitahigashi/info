## Dynamic Workflowsの核心概念

「最大1,000エージェントの稼働可能な軍隊」：
- Claudeが実行スクリプト（JavaScript）を自動生成
- 独立したランタイム（Bun等）がそのスクリプトを実行
- 従来のオーケストレーション負担をコードに移すことでスケーラビリティを大幅向上
- メインセッションのコンテキストを圧迫せずに並列処理を実現

- **規模**: 2,188ファイル、約75万行

## 技術的な仕組み

```
[Claude] → JavaScript オーケストレーションスクリプト生成
          ↓
[独立ランタイム] → スクリプト実行
          ↓
  ┌───────┬───────┬───────┐
  │Agent1 │Agent2 │Agent3 │  ... (最大1,000並列)
  └───────┴───────┴───────┘
          ↓
[結果マージ] → メインセッションに返却
```

## メインセッションへの影響なし

各エージェントが独立したコンテキストで動作するため：
- メインセッションのトークン消費は最小限
- 中間結果はスクリプト変数に保存（コンテキストウィンドウ外）
- 1,000件の処理も現実的な時間・コストで完了

## 注意事項（有料部分含む総合情報）

- 通常よりトークン消費が大幅に多い（コスト試算が必須）
