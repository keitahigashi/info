## Skills・MCP・メモリによる自己管理3ヶ月実験

### 概要

Claude CodeのSkills・MCP・メモリ機能を「人生管理」に応用した3ヶ月の実験記録。自動化の効率化よりも「言語化の強制」が最大の成果だったという逆説的な発見を報告。

### 情報管理の5軸（GitHubで管理）

| ディレクトリ | 内容 | 時制 |
|---|---|---|
| `00_self` | 価値観・性格・経歴 | 静的 |
| `01_direction` | 人生・年間目標 | 未来 |
| `02_work` | 進行中タスク | 進行中 |
| `03_journal` | 日々の記録 | 過去〜現在 |
| `04_knowledge` | 取り込んだ知識 | 蓄積 |

### 実装した技術的要素

- **Skillコマンド化**: `/life-advice`（状況を踏まえた相談）、`/new-task`（新規タスク登録）、`/monthly-review`（月次振り返り）を定義
- **MCP連携**: カレンダー・タスク管理ツールと統合し、AIが予定参照・登録を実行
- **メモリ機能**: 好みやフィードバックを記憶し、繰り返すたびに提案の解像度が向上

### 最大の発見：言語化 > 自動化

著者は「自動化でラクしたい」という動機で始めたが、実際に最も効果があったのはMDファイルに自己情報を書く「言語化の強制」だった。「自由が一番大事と思っていたのに、書く際に『自由って何からの自由?』と詰まった」という体験が、自己理解の深化につながった。

### 3つの本質的学び

1. 言語化の強制力が自動化より効果的（MDファイルへの書き出し自体が価値）
2. ゴール決定は人間が担当し、AIは実行補助に留める
3. 「自分の仕様書」を持つことで初めて戦略的な自己管理が可能になる

### 実践の4段階

- Lv1: メモアプリに箇条書き（ここから始められる）
- Lv2: MDファイルをローカル管理
- Lv3: GitHubで版管理
- Lv4: Skills・MCP・メモリを組み合わせた完全統合
