## 概要

2026年5月28日にAnthropicが導入した「Dynamic Workflows」について、公式ブログとドキュメントから整理した解説記事。Claudeが動的にJavaScriptスクリプトを生成し、1セッション内で数十〜数百のサブエージェントを並列実行する仕組みを説明する。

## Dynamic Workflowsとは

- 計画をスクリプト化し、ランタイムが実行する方式
- 従来のSubagents・Skillsと異なり、「中間結果をスクリプト変数に保持」し、会話コンテキストには最終結果のみ返す
- コンテキストウィンドウの圧迫を回避しつつ大規模タスクを実行可能

### 用途例

- コードベース監査・最適化・セキュリティ監査
- 大規模フレームワーク移行
- 複数角度からの計画検証

## 実行フロー

ユーザー依頼 → ワークフロー計画 → スクリプト生成 → バックグラウンド実行 → 並列作業 → 検証・統合 → 単一回答返却

## 制約

| 項目 | 制限 |
|------|------|
| 実行中のユーザー入力 | 不可 |
| 最大並列エージェント数 | 16 |
| 1実行あたりの上限 | 1,000エージェント |
| ファイル操作 | ワークフロー自体は直接実行しない |

## 起動方法

1. **プロンプトに「workflow」を含める** — 虹色にハイライトされトリガー
2. **`/effort ultracode`** — xhigh推論と自動オーケストレーション
3. **`/deep-research`** — 調査用組み込みワークフロー

## 実行監視

- `/workflows` コマンドで確認可能
- キーボード操作でフェーズ選択、詳細表示、一時停止/再開、停止、再起動、保存が可能

## 権限管理

- 起動時の確認プロンプト表示は権限モードに依存
- サブエージェントは常に「acceptEdits」モードで実行（ファイル編集は自動承認）
- 未許可のシェルコマンドやツールは実行時にプロンプトが出る可能性あり

## コスト管理

- 多くのトークンを消費するため、小さくスコープを切ってから開始を推奨
- 無効化方法: `/config` 設定、`~/.claude/settings.json` 編集、環境変数設定、管理者設定

## 利用可能環境

| プラン | デフォルト |
|--------|-----------|
| Max, Team, Claude Code via API | 有効 |
| Enterprise | 無効（管理者が有効化可能） |

対応ツール: CLI、Desktop、VS Code拡張、Claude API、Bedrock、Vertex AI、Microsoft Foundry

## Bun事例

- Jarred Sumner氏がZigからRustへの移植に使用
- 約750,000行のコード、既存テストの99.8%パス
- 11日間で完了（ただし「本番環境未投入」と明記）

## 実践パターン

- 独立エージェントで検出結果を検証
- バッチごとにテスト実行する移行
- 複数案検討と敵対的レビューによる計画強化

## 推奨開始方法

小規模な調査や `/deep-research` から試して、トークン消費・実行時間・権限プロンプト動作を把握してから大規模タスクに展開することを推奨。
