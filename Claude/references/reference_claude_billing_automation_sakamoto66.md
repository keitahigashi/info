## 2026年6月15日施行：Claude課金ルール変更まとめ

### 概要

公開日：2026年5月14日（施行前の事前解説）。著者（@sakamoto66）によるClaude課金変更の全容解説。Agent SDKやclaude -pコマンドなどの自動化利用がサブスク枠から分離され、別建ての「月次クレジット（USD）」に移行される変更を整理。

### 変更の概要

```
[変更前]
サブスク1本で対話も自動化も賄う（枠の奪い合い）

[変更後]
対話枠（Interactive）: チャット・Claude Code対話的利用
自動化枠（Programmatic）: 専用「月次クレジット（USD）」
```

### 自動化枠に移行される利用形態

- `claude -p`（非対話モード）
- Claude Agent SDK 経由の呼び出し
- Claude Code GitHub Actions
- 第三者アプリからのAgent SDK経由利用

### 月次クレジット額（プラン別）

| プラン | 月次クレジット |
|--------|--------------|
| Pro（$20/月） | $20 相当 |
| Max 5x（$100/月） | $100 相当 |
| Max 20x（$200/月） | $200 相当 |

### メリット

- **「自動化で対話枠が食われる」問題が解消**
- 対話と自動化の上限が独立するため、大量の自動化実行をしても翌日の対話制限に影響しない

### 注意点・落とし穴

1. **繰り越し不可**：月次クレジットは翌月に持ち越せない
2. **opt-in必須**：Anthropicからのメールでオプトイン手続きが必要
3. **Extra usage設定**：月次クレジット超過後も使い続けるか要設定
4. **モデルによる消費速度差**：Opus > Sonnet > Haiku の順に消費が早い
5. **施行後の変更**：Anthropicがこの変更を一部見送るとアナウンス（6月中旬）したため、最新状況は公式確認推奨
