## 概要
「実践 Claude Code 入門」（技術評論社）の学びをベースに、コーディングエージェントの3つの構造的課題と各解決策を体系的に整理した記事。

## 3つの課題と解決策

### 課題1: 過去の指示を忘れてしまう（コンテキスト有限）
**解決方向**: コンテキストを分離する

- **構造化ノート**: `.steering/`ディレクトリにrequirements.md、design.md、tasklist.md、blockers.md、decisions.mdを配置
- **サブエージェント**: 親エージェントから独立した専門エージェント。大量ファイル読み取り・分析に有効。エージェント間はファイルやGitHub Issueで情報受け渡し

### 課題2: 意図通りに動いてくれない（暗黙の期待が未明示）
**解決方向**: 細かく指示する

- **スラッシュコマンド**: 「何を」「どの順番で」をMarkdownで明文化
- **スキル**: 「どのツールを」「どう使うか」の方針定義。CLAUDE.mdと異なり必要時のみ読み込み→コンテキスト節約
  - 作成の3切り口: 成果物単位・タスク単位・ドメイン単位
  - ディレクトリ構造で複雑な情報管理可能
- **フック**: ライフサイクルの特定タイミングでコマンド実行。「絶対守るべきルール」に活用

### 課題3: 外部システムと連携できない（単体能力の限界）
**解決方向**: 外部システムとつなげる

- **MCP**: AI↔外部システム接続の標準プロトコル。Figma参照・ブラウザ確認等
- **CLIツール**: GitHub CLI等の既存ツールやカスタムCLIでMCP非対応システムと連携

## 機能整理表

| 機能 | 目的 | 向いている用途 |
|------|------|------------|
| サブエージェント | 親との文脈分離 | 調査・レビュー・分析 |
| スキル | ツール使用方法と方針 | コードレビュー観点・命名規則等 |
| フック | ルール強制 | フォーマット・完了通知 |
| MCP | 外部システム連携 | Figma・ブラウザ操作 |

## 実践ノウハウ
- 文脈圧縮: 95%で自動発動、重要情報喪失リスクあり
- エージェント間通信: コンテキスト非共有→レポート介在の非同期連携
- ドキュメント管理: 複数文書間の指示競合回避→一元管理
- 段階的導入: CLAUDE.md → Skills → Hooks → MCP の順
