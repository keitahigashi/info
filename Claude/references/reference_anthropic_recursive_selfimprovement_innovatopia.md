## 出典

innovaTopia (Ami): https://innovatopia.jp/ai/ai-news/107562/

## 概要

2026年6月4日、Anthropicの研究部門 The Anthropic Institute が「When AI builds itself」を公表。AIがAI開発を加速させている現状と、アライメントリスク・政策提言を包括的に示した。

## 重要な数値データ

| 指標 | 数値 |
|------|------|
| Claude が書くコード比率（2026年5月） | **社内マージコードの80%超** |
| 2025年2月以前の比率 | 1桁台前半 |
| エンジニア生産性（2024年比） | **8倍**（2026年Q2） |
| AIタスク倍増ペース | 約7か月 → **約4.3か月**に短縮 |
| Claude Mythos のコード最適化 | 約**52倍**達成 |

## 再帰的自己改善とは

> 「AIが人間の介入をほとんど受けずに後継機を設計・訓練し、性能を高めていく状態」

現行の Claude はまだ完全な再帰的自己改善段階には至っていないが、「その入り口に立っている可能性がある」と文書は示唆。

## 主要なリスクシナリオ

### アライメントのずれの蓄積
現行モデルにまれに見られるアライメントのずれが、後継機生成過程で積み重なり「より頻繁かつ理解しにくくなる」懸念。

### 安全性の競争ジレンマ
単独の企業が開発速度を落とすと、慎重でない競合が先行し全体の安全性が低下するパラドックス。

## 政策提言

- フロンティア開発を **減速・一時停止できる選択肢** を世界が持つべき
- INF全廃条約のような **多国間の検証可能な体制** を引き合いに出す
- 自発的な停止ではなく **協調的・検証可能なフレームワーク** が必要

## 独立研究による慎重な見方

METRの研究では、熟練開発者がAIツール使用時に**19%長くかかった**と報告。innovaTopia 編集部は「加速の物語と慎重さの物語が同じ文書に同居している」と指摘。

## 用語定義
- **再帰的自己改善**: AIが人間の介入なしに後継機を設計・訓練し性能向上させる状態
- **アライメントのずれ**: AIの目標・行動が人間の意図から外れること
