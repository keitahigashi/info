## 概要
Claude Codeの出力品質はプロンプト設計で決まるという前提のもと、初回正答率を大幅向上させる7つのテクニックを実装例付きで解説。

## 詳細

### 基本テクニック

**1. ゴールファースト**
- 指示の冒頭に最終状態を明記（手順先行ではなく「どうなるべきか」）
- 例: 「ESLintの設定を更新して、未使用のimport文を自動で検出・削除できる状態に」

**2. 制約の明示**
- やってはいけないことを明確化（破壊的変更防止）
- 例: 「既存ファイルは絶対に変更しない」「DROP TABLE は使用しない」
- DB操作・本番環境で特に重要

### 実践テクニック

**3. Before/Afterの提示**
- 変換パターンを具体例で示す
- APIレスポンスのエラーハンドリング統一 → 修正前後のコード例で意図を正確に伝達

**4. 段階的タスク分解**
- 大規模実装を複数フェーズに分割
- 例: 認証システム → JWT基盤→パスワードリセット→二要素認証（各フェーズ完了後に次へ）

**5. 検証手順の組み込み**
- 「実装して」で終わらず、テスト実行・lint確認・curl動作確認を指示に含める
- 例: `npm run test` → `npm run lint` → `curl http://localhost:3000/api/users`

### 環境設定テクニック

**6. CLAUDE.mdとの連携**
- プロジェクトルートに配置、セッション開始時に自動読み込み
- 記載内容: プロジェクト概要・コーディング規約・ディレクトリ構造・テストルール・禁止事項
- チーム全員がGitにコミット → 出力品質の均一性

**7. コンテキストウィンドウの最適化**
- `/compact` で会話圧縮
- `/mcp disable` で不要なMCPサーバー無効化
- 対象ファイルを明示的に絞り込み

### 使い分けマトリックス
| テクニック | 効果が高いシーン | 優先度 |
|---------|-------------|--------|
| ゴールファースト | 全ての指示 | 必須 |
| 制約の明示 | DB操作・本番環境 | 必須 |
| Before/After提示 | コード変換 | 高 |
| 段階分解 | 大規模実装 | 高 |
| 検証組み込み | API実装・バグ修正 | 高 |
| CLAUDE.md連携 | 継続的開発 | 必須 |
| コンテキスト最適化 | 長時間セッション | 中 |
