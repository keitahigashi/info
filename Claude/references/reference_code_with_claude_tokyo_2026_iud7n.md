## 主な学びポイント

### 1. エンジニアの役割転換

エンジニアの役割は**「コードを書くこと」から「AIが効果的に機能する条件を整備すること」**へシフト。

### 2. Claude Code活用の3つの技法

| 技法 | 内容 |
|------|------|
| 曖昧さの事前除去 | インタビュー形式でAIに要件を引き出させる |
| 計画書をHTMLで作成 | 可読性が向上し、AIとの認識齟齬が減少 |
| 検証の組み込み | 単体〜行動レベルの多層検証で品質担保 |

### 3. Managed Agentsのアーキテクチャ

- 実行基盤・状態管理・認証・監視をAnthropicが担当
- 「脳（思考）」と「手（実行）」の分離でスケーラブルな本番運用が可能
- 開発者はビジネスロジックに集中できる

### 4. 記憶機能の実装

- **Memory Store**: エージェント間での記憶共有
- **Dreaming（非同期記憶整理）**: オフライン中に記憶を整理・圧縮し効率化

### 5. セキュリティ設計の3層

1. **Vault**: 機密情報の安全な管理
2. **サンドボックス実行**: 環境分離
3. **ハードゲート制約**: プロンプトインジェクション耐性

### 6. ビジネス成功の必須要件

- AIエージェント投資前に**「データ基盤」「業務コンテキスト」**の整備が不可欠
- エージェントのボトルネックは思考時間ではなく**「材料集めの時間」**
- 良質なデータと業務文脈なしには、どれだけ優秀なモデルでも成果が出ない
