## Claude Code v2.1.200〜v2.1.201 主要アップデート（2026年7月4日）

### 概要
- 公開日：2026年7月3日
- 変更件数：計18件
- 主軸：バックグラウンドセッション・バックグラウンドエージェントの安定性修正

### 注目の不具合修正

| 修正内容 | 詳細 |
|---------|------|
| バックグラウンドセッション停止 | スリープ/復帰後や停止セッション再開時の無言停止を解消 |
| MCP設定クラッシュ | `.claude.json`の`disabledMcpServers`/`enabledMcpServers`に配列以外の値設定時の起動時クラッシュを修正 |
| Escキャンセル再実行バグ | 停止後の再起動でキャンセルしたターンが再実行される問題を改善 |
| daemon.lockロック問題 | クラッシュで残ったロックファイルがエージェント起動不能を招く問題を解決 |
| tmux 3.4互換性 | ターミナル出力の同期化で描画ちらつきを修正 |

### 破壊的変更（無人運用者は要確認）

**1. AskUserQuestionタイムアウトの自動継続廃止**
- 新設定項目：`askUserQuestionTimeout`
- 既定値：`"never"`（回答を無制限に待機）
- 自動継続を希望する場合は `"60s"` / `"5m"` / `"10m"` を明示的に指定が必要
- **無人バッチ運用・自動化パイプラインは設定見直し必須**

**2. パーミッションモード表示名変更**
- 「default」→「Manual」に変更
- CLI・VS Code・JetBrains 全体で統一
- 後方互換性あり（既存設定値は引き続き動作）

### その他の改善
- スクリーンリーダー対応強化
- インストールスクリプト改善（メモリ不足時の説明文表示）
- Claude Sonnet 5セッションのハーネスリマインダー改善
