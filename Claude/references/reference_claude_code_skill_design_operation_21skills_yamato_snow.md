## Claude Code Skillの作り方｜21個運用して分かった設計と育て方

## コア概念
Claude Code Skills = YAML形式フロントマターを持つマークダウンファイル。執筆者は執筆（6個）、開発（5個）、運用（5個）の3カテゴリで21個スキルを運用

## キー構造

### フロントマター必須フィールド

| フィールド | 説明 | 制限 |
|---|---|---|
| **name** | スキル識別子 | 小文字、ハイフン、最大64文字 |
| **description** | 自動起動判定の中核 | 重要キーワードは250文字内に配置 |
| **model** | 特定モデル指定 | オプション |
| **execution** | 実行制御 | オプション |
| **argument-hints** | 引数フォーマット | オプション |

## スキル配置戦略

| 場所 | パス | スコープ |
|---|---|---|
| **個人用** | `~/.claude/skills/` | 全プロジェクト対象 |
| **プロジェクト用** | `.claude/skills/` | 単一プロジェクト |
| **エンタープライズ** | 管理設定経由 | 組織全体 |

## 開発ワークフロー

### 5ステップ反復プロセス

1. **問題認識**
   - 反復的な指示パターンを特定
   - 例: 毎回同じフォーマット要件を説明

2. **プロンプト構造化**
   - 要件を明確セクションに組織
   - 優先度別に並べ替え

3. **`/skill-creator`活用**
   - インタラクティブ作成ツール利用
   - 手書き避止（テンプレートのみ）

4. **Evalモード検証**
   - A/B テストでスキル有効性比較
   - 複数バージョン同時評価

5. **改善・ベンチマーク**
   - パフォーマンス指標監視
   - 定期改善サイクル

## クリティカルベストプラクティス

### 1つのスキル = 1つの目的
複数の責務を持つスキルは自動起動失敗リスク増加

### 説明文の書き方
- **WHAT**: スキルが実行する処理を明確に
- **WHEN**: 自動起動すべき条件を明記
- **優先度**: 最重要キーワードを説明文先頭250文字に配置

### テスト必須
Evalモードで常に検証し、機械学習的な効果測定

### 定期保守
ベンチマークモードでパフォーマンス監視

### 危険スキルの制約
`disable-model-invocation: true` で副作用を持つ操作を手動呼び出し専用化

## 高度な機能

### 動的コンテキスト注入
```bash
!`<command>`
```
シェルコマンド出力を埋め込んで動的値参照

### サブエージェント実行
```yaml
context: fork
```
操作を独立させてトークン効率を保持

### パラメータ化
`$ARGUMENTS` で引数をスキルに渡す
