## 記事内容の構造化要約

### 変更の概要
- **実施日**: 2026年4月4日
- **内容**: Claude Pro / MaxサブスクリプションでのOpenClaw・OpenCode等サードパーティツール経由の利用を制限
- **対象**: APIを直接利用しているサードパーティクライアント全般

### 変更の背景（構造的な問題）
- AnthropicはPro（$20/月）・Max（$100〜$200/月）を「公式Claude Code・Claude.ai利用者」向けに設計
- 実態: 月額$200のサービスに対して、一部ユーザーが1日$1,000〜$5,000相当のコスト負担を強いる使い方をしていた
- 「サブスクリプションはサードパーティツール利用には設計されていない」と明言

### 重要な誤解の解消
> **「サードパーティでの利用そのものは禁止されていない」**
- **禁止**: Pro/Max サブスクリプション経由でのサードパーティツール利用
- **許可**: APIキー直接利用（従量課金）でのサードパーティツール接続
- つまり「APIキーを使えば引き続き完全に利用可能」

### 選択肢

| 選択肢 | 概要 | 向いているケース |
|---|---|---|
| Extra Usageオプション | 月額相当のクレジット追加付与（2026年4月17日期限） | 旧環境維持を優先する場合 |
| APIキー直接利用 | トークン従量課金（プロンプトキャッシュで最適化可能） | 使用量を細かく制御したい場合 |
| 公式Claude Codeへ移行 | サブスク内での公式利用 | 安定した予算内利用を望む場合 |

### APIキー移行時のコスト最適化ポイント
- **プロンプトキャッシュ**: 同一システムプロンプトの繰り返し呼び出しで最大90%削減
- **モデル選択**: Opus 4.7 → Sonnet 4.6 → Haiku 4.5 の段階使い分け
- **Batch API**: 非リアルタイム処理は50%割引のBatch APIを活用

### 影響範囲の整理
| 利用形態 | 影響 |
|---|---|
| Claude Code CLI（公式） | 影響なし |
| Claude.ai Web（公式） | 影響なし |
| OpenClaw + Proサブスク | 制限対象 |
| OpenCode + Maxサブスク | 制限対象 |
| OpenClaw + APIキー（従量課金） | 引き続き利用可能 |
| 自作ツール + APIキー | 引き続き利用可能 |

### 教訓・考察
- AnthropicはAPIエコシステムと消費者向けサブスクを明確に分離する方針を取り始めた
- 長期的には「公式ツールの質向上」と「API価格の維持」のトレードオフ
- サードパーティツール開発者はAPIキー課金モデルへの完全移行が推奨
