## バイブコーディングの定義と二大ツール比較

- **バイブコーディング**: 開発者がAIに意図を伝え、数千行のコード変更を一括生成させる開発スタイル
- **Cursor**: AIネイティブIDEで「Composerモード（複数ファイル同時編集）」と「非同期タスク並列実行」が強み。大規模リファクタリングを高速化
- **Claude Code**: 高度な自律修復ループとマルチエージェント機能で既存コードベースの構造的変革に優れる

## ベンチマーク結果

| ツール | SWE-bench成功率 |
|--------|----------------|
| Claude Code | 80.8% |
| Cursor | 55〜62% |

## 主要リスク：バイブコーディング・ハングオーバー

- AIが生成した保守困難なコードが蓄積するリスク
- 対策: 実装前に計画を説明させる **Plan-First** アプローチ
