## 出典

デジタルフロント株式会社ブログ: https://digital-front.jp/blog/1850/

## 概要

公開日：2026年5月22日。デジタルフロント株式会社（AIエージェント・デジタル総合支援）がCTOや情報システム担当者向けに作成した企業導入セキュリティガイド。

## 主なリスク

- 機密ソースコードの外部送信
- 権限の過剰付与
- 認証情報の不適切な管理
- 監査ログの不足

## 権限管理設計（最小権限の原則）

ロール分け：管理者 / プロジェクトオーナー / 一般メンバー / 閲覧専用

## 具体的なセキュリティ設定

- SSO連携とMFA（二要素認証）の必須化
- ネットワーク制御・IP制限
- 「個人情報・秘密鍵・APIキーは絶対に入力禁止」の徹底
- データ取り扱いポリシーの明確化

## Git連携時の注意点

- リポジトリアクセス権限は読み取り専用から開始
- セキュリティクリティカルなコードは別リポジトリ管理

## 導入前チェックリスト（5カテゴリ）

1. ポリシー整備
2. 権限管理
3. プロジェクト分離
4. ログ監査
5. 教育体制

## 教育・運用

開発者向けガイドラインの整備と「生成AIリテラシー・セキュリティ教育」の必要性を強調。
