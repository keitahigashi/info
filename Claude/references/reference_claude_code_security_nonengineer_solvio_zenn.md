## 概要

公開日：2026年4月23日。著者はSolvio株式会社のokawa2929氏。Claude Codeの急速な普及を背景に、非エンジニアが「確認を読まずYesを連打する危険性」を中心に据えた実践的ガイド。

## セキュリティ事故の実例

- 本番インフラの誤削除・消失事故
- プロンプトインジェクション攻撃による情報漏洩

## 17の設定（5段階レベル）

**レベル1（基本5項目）**
- ホームフォルダでの起動禁止
- `rm -rf`・`sudo`等の危険コマンドをdenyリスト化
- `.env`・SSH鍵等の機密ファイルへのアクセス禁止
- `curl`・`wget`等の外部ダウンロードコマンド封じ
- 間接実行（パイプ・サブシェル）による抜け道防止

**レベル2（サンドボックス4項目）**
- macOS：Seatbelt、Linux：Bubblewrapを活用した物理的アクセス制限
- ネットワーク接続先ホワイトリスト化

**レベル3〜5**
- バイパスモード（`--dangerously-skip-permissions`）封じ
- PreToolUse Hookで通知・承認フロー導入
- CLAUDE.mdによる日本語ルール記述
- 組織配布時のmanaged-settings活用

## 要点

「多層防御（禁止リスト＋サンドボックス＋フック）」を軸に、個人から組織まで段階的実装を推奨。
