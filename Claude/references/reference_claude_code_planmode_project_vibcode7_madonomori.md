## 概要

2026年5月15日公開。ウイスキー認識アプリ「AiLA」に承認制ログイン機能を実装する過程を通じて、Claude Codeを効率的に活用するための3つのテクニック（Plan Mode・プロジェクト機能・CLAUDE.md運用）を解説する実践的な連載記事。

## 主なテクニック

### 1. Plan Mode の活用

- `Shift+Tab` 2回で切り替え可能な読み取り専用モード
- 指示後、いきなりコードを書き換えず、コードベースを分析して**修正計画を表示するだけ**に留まる
- 意図しない実装を防ぐため、複雑な要件では必ずPlan Modeで計画確認してから実行に移す
- 初学者向けの推奨ワークフロー：「Plan → 確認 → 実行」

### 2. チャット版Claudeによるプロンプト練り上げ + プロジェクト機能

- 複雑な指示はClaude Codeに直接投げず、**チャット版Claudeで先に要件を整理**する
- Claudeのプロジェクト機能でCLAUDE.mdの内容を共有 → 前提情報を毎回説明しなくて済む
- 「AiLAプロジェクト」を作成し、仕様書を登録することで一貫した文脈でのサポートが得られる

### 3. CLAUDE.md の継続的メンテナンス

- 起動手順（Firebaseエミュレーター等）をCLAUDE.mdに記録し、再現性を確保
- 重要な仕様変更・設計判断の理由を都度追記
- Git運用ルール（コミット・プッシュの自動化）もCLAUDE.mdに記述して自動化

## ユースケース例：承認制ログイン実装

**課題**: Firebaseエミュレーター起動忘れ・複雑な承認ワークフロー要件のズレ

**解決フロー**:
1. チャット版Claudeで要件を整理（承認待ちステータス・管理者許可・1度のみ申請可）
2. プロジェクト機能で仕様共有
3. Plan Modeで実装計画確認
4. 実行・CLAUDE.mdへ変更内容を追記

## 実践的な教訓

- 「CLAUDE.mdにコードを書いてもらうのではなく、起動手順や仕様変更の記録場所として使う」
- 「チャット版Claudeで要件を磨いてからClaude Codeに渡す」という2段階アプローチが有効
