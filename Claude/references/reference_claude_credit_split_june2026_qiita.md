## Claude -p / Agent SDK 課金分離（2026年6月15日〜）

### 概要

2026年5月14日公開。Anthropicが2026年6月15日から実施するAgent SDK・`claude -p`の課金構造変更を詳細に分析。プラン別クレジット設定、「天井ではなく緩衝」という構造の問題点、ユーザー別の対応経路を提示。

### プラン別月次クレジット（2026年6月15日〜）

| プラン | 月額 | Agent SDK/claude -p クレジット |
|--------|------|-------------------------------|
| Pro | $20 | 別枠クレジット（上限あり） |
| Max 5x | $100 | 別枠クレジット（上限あり） |
| Max 20x | $200 | 別枠クレジット（上限あり） |

### 課金構造の変化

**変更前：** Agent SDKと`claude -p`は通常の対話枠を消費  
**変更後：** プランごとに月次クレジットを別で消費、枯渇後はAPI標準料金またはサービス停止（extra usage設定による）

### 「天井ではなく緩衝」問題

- 月100ドルのクレジットは「上限」ではなく「緩衝（バッファー）」
- 超過利用で実質「10倍の値上げ」になるケースが報告されている
- extra usageをオンにすると枯渇後もAPI標準料金で継続課金

### 最も影響を受けるユーザー

1. SDK ツール依存のユーザー（CI/CD自動化組み込み）
2. 音声利用者（Voice API経由）
3. プログラム的なバルク処理実施者（バッチ処理・定期実行）

### 対応経路（4パターン）

| 選択肢 | 内容 | 向いている人 |
|--------|------|-------------|
| ① 対話経路への移行 | claude -p → 対話型CLIへ切り替え | 軽量自動化ユーザー |
| ② API keyへの移行 | サブスクリプション外のAPI直接利用 | ヘビーユーザー |
| ③ 他システムへ乗換 | OpenAI等の代替サービスへ | コスト重視ユーザー |
| ④ プロンプトキャッシング活用 | 繰り返しプロンプトのコスト削減 | ユースケース最適化 |

### 推奨アクション（6月15日まで）

1. 現在の`claude -p`・Agent SDK利用量を計測（過去30日分）
2. 月次クレジットに収まるか試算（「32日間で計算」）
3. 超過が見込まれる場合は対応経路を選択
4. extra usage設定の意図的な有効化 or 無効化を決定
