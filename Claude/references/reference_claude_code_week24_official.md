## 記事概要

**公開日:** 2026年6月8日〜12日（v2.1.166–v2.1.176）

Claude Code Week 24の公式週次ダイジェスト。3つの主要機能と12以上の改善が含まれる。

## 主要機能

### 1. /cd でセッションを移動（v2.1.169）

プロンプトキャッシュを再構築せずに、現在のセッションを別の作業ディレクトリへ移動するコマンド。

- 新しいディレクトリの `CLAUDE.md` はシステムプロンプトを置き換えるのではなく、**メッセージとして追加**される
- セッションは新しいディレクトリのプロジェクトストレージに再配置され、`--resume`/`--continue` はそこで見つかる
- 以前そのディレクトリで作業していない場合、Claude はディレクトリを信頼するよう促す

```text
> /cd ../other-project
```

### 2. サブエージェントが独自のサブエージェントをスポーン（v2.1.172）

サブエージェントが独自のサブエージェントを生成できるようになった。

- プロンプト下のサブエージェントパネルは**完全なツリー表示**（各行に子孫カウントと `main` へのパス）
- 暴走を防ぐため**最大5レベルの深さ**に制限
- `/agents` コマンドでエージェントビューを開いてツリーを監視

```text
> /agents
```

### 3. --safe-mode でトラブルシューティング（v2.1.169）

すべてのカスタマイズを無効にした状態でClaude Codeを起動するモード。

- 無効化対象: `CLAUDE.md`・skills・plugins・hooks・MCPサーバー・カスタムコマンド・カスタムエージェント
- **引き続き機能するもの**: 認証・モデル選択・組み込みツール・アクセス許可
- セーフモードで問題が消える → いずれかのカスタマイズが原因

```bash
claude --safe-mode
# または環境変数
CLAUDE_CODE_SAFE_MODE=1 claude
```

## その他の改善（12件）

| 機能 | 説明 |
|------|------|
| `fallbackModel` | プライマリ障害時に順番に試される最大3つのフォールバックモデルを設定。`--fallback-model` もインタラクティブセッションに適用 |
| セッションタイトル言語 | 会話の言語で自動生成。`language` 設定で固定可能 |
| `claude agents --json` | `--all` フラグで完了セッションを含める。`id`・`state` フィールドを追加 |
| `/plugin` 検索バー | マーケットプレイスのプラグイン参照時に検索バーが追加 |
| `disableBundledSkills` | バンドルスキル・ワークフロー・組み込みコマンドをモデルから非表示にする設定 |
| 拒否ルールのグロブ | `"*"` ですべてのツールを拒否可能。不明なツール名は起動時に警告を表示 |
| クロスセッションメッセージング強化 | `SendMessage` 経由のメッセージはユーザー権限を持たない。auto mode はブロック |
| Amazon Bedrock AWS設定読み込み | `AWS_REGION` 未設定時に `~/.aws` 設定ファイルからリージョンを自動読み込み |
| `enforceAvailableModels` | `availableModels` 許可リストでデフォルトモデルも制限 |
| Chrome ブラウザツール | ツールごとではなく単一バッチ呼び出しで読み込み（効率化） |
| `claude update` 事前告知 | ダウンロード開始前にターゲットバージョンを表示 |
| `footerLinksRegexes` | 正規表現にマッチしたリンクバッジをフッター行に追加 |

## 活用ポイント

- `/cd` はモノレポや複数プロジェクトを1つのClaudeセッションで横断する際に有効
- ネストサブエージェント（最大5段階）により複雑な自律タスクの階層化が可能に
- `--safe-mode` は設定ファイルやプラグインの不具合切り分けに即座に使える
- `fallbackModel` でレート制限・障害時の自動フォールバックチェーンを事前設定
