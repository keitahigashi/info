## 出典

PC Watch: https://pc.watch.impress.co.jp/docs/news/2113806.html

## 概要

**日付**: 2026年6月2日（Anthropic による対応: 6月1日）

Anthropic は 6月1日、Pro および Max プランの全ユーザーを対象に「5時間ごと」および「週単位」の利用上限をリセットした。

## 不具合の詳細

### 発生していた問題

Claude Code セッションにおいて「過剰な並列サブエージェントが生成され、想定以上のペースで使用量が消費される問題」が発生。

### 根本原因

**Opus 4.8 へのリクエスト処理**にバグがあり、モデルが意図した数を超える並列ツール呼び出しを実行していた。

### 関係ない機能

- Dynamic Workflows とは**無関係** — Anthropic が明記
- 通常の対話利用には影響なし

## Anthropic の対応

| 対応 | 内容 |
|------|------|
| バグ修正 | Opus 4.8 の並列ツール呼び出し数を正常化 |
| 補償措置 | Pro・Max 全ユーザーの 5時間制限・週次制限をリセット |
| 対象プラン | Pro・Max（Team・Enterprise は別枠管理のため対象外） |

## 影響と注意点

- リセット後は通常の利用制限が復活するため、引き続き節約した利用が推奨
- Dynamic Workflows を多用しているユーザーは依然として消費量に注意
- 6月15日からの Agent SDK クレジット分離後は、プログラム的利用の消費は別枠管理となる
