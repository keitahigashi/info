## 概要
Claude Codeの実践的活用事例7つを紹介。「速くなった気がする」と「本当に速い」の違い、AIツールは単純な効率化でなく人間との協働設計であるべきという問題意識。

## 詳細

### 1. 自動取引システム構築
Polymarket向け自動取引をClaude Code単一セッションで構築。初回スキャンで182件の裁定機会発見、8取引中6勝（+9.6% ROI）。学習ループ自動化＋永続メモリに戦略保存。

### 2. 効率の幻想への警告
METR調査: 開発者は「20%高速化」と体感→客観測定では19%遅化。AI出力のレビュー・修正時間が膨らむ。AI生成コードの45%にセキュリティ脆弱性リスク。
- **処方箋**: 定型業務（テスト生成・ドキュメント作成）はAIに、文脈依存判断（アーキテクチャ・セキュリティ検証）は人間に

### 3. マルチエージェント・オーケストレーション
Oh-My-ClaudeCode: GitHub 858スター/24時間。32種の専門エージェント・5実行モード。自動モデルルーティングで30-50%トークン削減。

### 4. 広告分析の自動化
Rimo社事例: 3-5時間の広告分析→「体感7分」に短縮。BigQuery+Salesforce CLI統合。70-100項目チェックリストから改善提案を自動生成。提案はAIに、最終判断は人間に。

### 5. 非エンジニアによる日常自動化
アソビュー CPO横峯氏が11個の自動化構築。毎朝7時に自動実行。Slack+Claude Codeで実務レベルの自動化実現。

### 6. セキュリティインシデント
2026/3/31 Anthropicがソースコード約50万行を誤流出。デバッグ用マップファイルが原因。「自己修復メモリ」アーキテクチャが明らかに。

### 7. 小さな自動化の積み重ね
飛書（Feishu）ドキュメント作成を5ステップ→1クリックに短縮。派手なマルチエージェントより地味だが毎日使える仕組みが重要。

### 実装の教訓
- 学習ループの永続化: セッション間で施策を継承
- 人間の方向性指示: Rakuten事例は「人間が随時方向性を示した」ことが鍵
- デモより定着: 毎日使える仕組みが価値
