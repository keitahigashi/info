## 変更の核心

- Claude Code v2.1.200 にて、既定の権限モードが従来の自動承認「default」から**「Manual」**へ変更
- 「人の判断を前提とする設計」への転換が明示的な思想として打ち出された
- CLI・VS Code・JetBrains拡張など全配信経路に一斉適用

## 具体的な変更点

- `AskUserQuestion` ダイアログの自動継続が既定でオフになり、明示的なオプトインが必要になった
- 後方互換性は保持（既存設定を壊さないよう配慮）

## 運用への影響

- **無人運用・CI自動化を前提としていたワークフロー**は、承認待ちで処理が停止する可能性がある
- 既存の自動実行フローを持つチームには設定の再確認が必須
- 「"既定でどこまで許すか"の初期値設計が問われる」というコンテキスト

## v2.1.201 の内容

- 権限関連の大きな変更は v2.1.200 で完結
- v2.1.201 は Claude の内部挙動に関する単一修正に留まる
