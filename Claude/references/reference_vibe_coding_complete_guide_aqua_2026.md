## バイブコーディングの定義と本質

- **定義:** AIに自然言語で意図を伝え、AIがコードを生成・実行・修正する開発手法
- **命名者:** Andrej Karpathy（2025年2月）
- **コンセプト:** 「AIに完全に身を委ね、コードの存在そのものを忘れ、ただバイブ（感覚）だけで進む」

**3つの本質的特徴**
- 自然言語ファースト（日本語/英語で指示、コードではない）
- AIが主導（コード生成・修正・テスト実行を担当）
- 反復的対話（会話を通じて品質向上）

## Claude Codeでの5ステップワークフロー

### ステップ1：インストール
```bash
npm install -g @anthropic-ai/claude-code
```

### ステップ2：CLAUDE.mdファイル作成
プロジェクト固有の以下を定義：
- コーディング規約（言語別、フレームワーク別）
- アーキテクチャ方針
- テスト戦略（単体テスト、統合テスト基準）
- セキュリティポリシー

### ステップ3：自然言語で開発開始
具体的要件をプロンプトで指示
- 例：「JWTを使ったメール認証を実装」
- 背景情報と制約を明記

### ステップ4：レビューと修正ループ
- AIが生成したコードを検証
- セキュリティ・パフォーマンス目視確認
- フィードバックして品質向上

### ステップ5：品質保証とデプロイ
- 型チェック自動実施
- テスト実行
- セキュリティスキャン

## 失敗しない12のベストプラクティス（抜粋）

| 優先度 | ルール | 実装方法 |
|---|---|---|
| **必須** | CLAUDE.mdを必ず作成 | コーディング規約・テスト方針を明記 |
| **必須** | 生成コードを必ず読む | セキュリティ・パフォーマンス確認 |
| **必須** | Gitコミットを細かく | 機能単位で都度コミット |
| **重要** | 具体的に指示する | 「認証機能」→「JWT+bcryptでメール認証」 |
| **重要** | テストを先に書かせる | テスト駆動開発アプローチ |
| **重要** | アーキテクチャは人間が決める | AIは局所最適に陥りやすい |

## 3大リスクと対策

### セキュリティ脆弱性（40%のAI生成コードに脆弱性）
- CLAUDE.mdにセキュリティルールを明記
- OWASPチェックを明示的に依頼
- Snyk等のスキャンツールをCI/CDに組み込む

### 技術負債の蓄積
- 週1回のリファクタリング実施
- 依存関係を最小限に制限
- 定期的なコードメトリクス計測

### Slopsquatting（AIが幻覚するパッケージ）
- npm/PyPI公式ページで存在確認必須
- Socket.devなどのセキュリティツール導入
- パッケージ追加前に確認を強制化
