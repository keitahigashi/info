## マルチエージェント実践投入レポート

### 記事概要

2026年4月16日公開。前回の環境構築記事に続く実践編。Chrome拡張機能（タブマネージャー）を題材に、4エージェント体制で「要件定義 → 実装 → QAレビュー → 承認」まで完走した詳細レポート。

### エージェント役割設計

| エージェント | 役割 |
|---|---|
| マネージャー | 確認・差し戻し判断・全体方針 |
| フロントエンド | UI/ロジック実装 |
| バックエンド | ストレージ・API 実装 |
| QA | 全体確認・セキュリティレビュー |

### 要件定義フェーズの知見

- `plan` モード（読み取り専用、ファイル変更・コマンド実行なし）で調査・設計
- マネージャーが2回の差し戻しを実施 → 要件がより明確化
- 最終成果物: 確認済み要件・修正要件10項目・新規追加要件10項目以上を含む要件定義書

### 実装フェーズの成果

- 8ファイル、約1,278行のコードを生成
- セキュリティ検証: XSS対策・MV3 CSP準拠を確認
- 最終評価: マネージャーが「出荷可（Approve）」と判定

### Split Panes 運用の重要ポイント

`settings.json` だけでなく、**CLAUDE.md とプロンプトにも明示的に記述**することが安定運用の鍵。

### 主な学び

- フェーズ分離（要件定義と実装の明確な区切り）が心理的安心感と品質向上に貢献
- 役割分担により確認観点が増加し、見落としが減少
