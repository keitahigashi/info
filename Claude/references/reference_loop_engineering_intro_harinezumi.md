## 構造化要約

### ループエンジニアリングの定義

> エージェントにプロンプトを打つ人から自身を外し、プロンプトを打ってくれる仕組みを設計すること

### 組織管理との類似性

- マイクロマネジメントと丸投げの両方を避ける必要がある
- 目標設定、レビュー体制、引き継ぎの仕組みを整備する、という組織マネジメントの発想を援用

### 既存エンジニアリング手法との区別

| 手法 | 対象 |
|------|------|
| Prompt エンジニアリング | 単一プロンプトの設計 |
| Context エンジニアリング | コンテキスト管理 |
| Harness エンジニアリング | エージェントの動作環境 |
| **Loop エンジニアリング** | **自動発見と検証の自動化** |

### ループの構成要素

1. **トリガー**：ループ開始のきっかけ
2. **停止条件**：いつループを止めるか
3. **作業サイクル**：実際の処理内容
4. **検証（最重要）**：進捗と成功を判定する機構

### 実装が有効になる4条件

- **反復性**：同じ処理が繰り返されるパターンである
- **機械的失敗検出**：ツールが自動で失敗を検出できる
- **エージェントの自律完結**：エージェントが自動で完了できる
- **客観的完成定義**：人間判断に依存しない明確な完成条件がある
