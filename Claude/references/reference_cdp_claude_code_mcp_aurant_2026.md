## CDP × Claude Code / MCP 活用パターン 2026（2026年6月12日）

### 問題背景
データチームのキャパシティ不足で業務側からの依頼が滞留。業務要件→SQL変換の翻訳作業に多大な工数が消費されており、MCP経由の自然言語操作で解決を目指す。

### 典型実装パターン3つ

| パターン | 対象CDP/DWH | 主な効果 |
|---------|------------|---------|
| BigQuery MCP | BigQuery | 1〜2日で動作環境構築可能 |
| Treasure Data / Segment連携 | API経由 | 複合タスク自動実行対応 |
| Reverse ETL自動化 | 各社CDP | 同期定義作成工数を5〜10倍圧縮 |

### セキュリティ対策3点セット
1. テーブル構造のみ参照・サンプルデータ非開示
2. 個人情報カラムのマスキング処理
3. 出力データはDWH→宛先に直接フロー（中間コピー不可）

### 運用体制の2パターン
- **パターン1**：業務側が直接Claude使用（ガバナンス課題あり）
- **パターン2**：データチーム内部効率化（現在主流）

### 導入フェーズ（推奨4段階）
1. MCP構築：1ヶ月
2. 内部効率化：2ヶ月
3. 業務側展開検討：3ヶ月
4. 本格展開：6ヶ月以降

### 3年TCO概算と競合比較

| 費目 | 金額 |
|------|------|
| API費用 | 720万円 |
| 実装費用 | 800万円 |
| 拡張・保守 | 1,200万円 |
| 研修・監査 | 1,900万円 |
| **合計** | **4,820万円** |

ROI：2年未満での回収見込み

| 製品 | 強み | 制約 |
|------|------|------|
| Treasure AI | シームレス統合 | Treasure内限定 |
| Snowflake Cortex | SQL自動生成 | Snowflake内限定 |
| Claude Code/MCP | ベンダー横断可能 | 実装工数必要 |

### 失敗パターン3つ
- 権限設計なしでの本番データ破壊リスク
- 業務文脈不在による技術的正確性と業務的誤りの乖離
- 業務側展開の急ぎすぎ（2〜3年の段階的移行が現実的）
