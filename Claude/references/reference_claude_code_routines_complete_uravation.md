## Claude Code Routines完全解説（2026年5月3日）

### 基本概念

「プロンプト・リポジトリ・コネクターを1つのパッケージにまとめてクラウドで自動実行できる機能」（2026年4月13-17日リサーチプレビュー公開）。Linuxのcronに類似するが、コード記述不要でプロンプト入力で動作。ローカルマシンの起動状態に依存しない。

### 3種類のトリガー

| トリガー | 機能 | 最小間隔 | 用途 |
|---------|------|--------|------|
| スケジュール | 毎時・毎日・平日・週次またはcronカスタム設定 | 1時間 | 定時実行タスク |
| GitHubイベント | PR・リリース連動（オープン・クローズ・ラベル付けなど） | — | 自動レビュー・コメント付与 |
| APIトリガー | HTTPエンドポイント経由の即時起動 | — | CD連携・外部ツール統合 |

ワンショット実行: 一度限りの将来予約機能。日次実行枠にカウントされない。

### 実務活用パターン5選

1. **朝のバックログ整理**（スケジュール）: GitHub issueの優先度ラベル付け→Slackサマリー投稿

2. **デプロイ後スモークテスト**（APIトリガー）: 環境URLへのヘルスチェック→エラーログ確認→go/no-go判定

3. **PR自動コードレビュー**（GitHubイベント）: セキュリティ・パフォーマンス・規約チェック→インラインコメント

4. **週次ドキュメント鮮度確認**（スケジュール）: API変更点とドキュメント照合→更新PR自動作成

5. **SDK間並行ポート**（GitHubイベント）: Python→Node.js機能移植→テスト対応

### プラン別実行制限

| Max | 15回 | 同上 |
| Team/Enterprise | 25回 | 超過分は従量課金可能 |

### 導入失敗パターン4つ

1. **曖昧プロンプト**: 「問題を見つけて直してください」→想定外の変更リスク
2. **コネクター過剰**: 不要なサービス接続→セキュリティリスク拡大
3. **ブランチ保護無視**: `claude/`プレフィックス以外への直接pushが可能に
4. **本番依存化**: リサーチプレビュー仕様の変動に追従できない

### エンタープライズガバナンス4要素

- **アカウント管理**: 専用サービスアカウント運用を推奨
- **監査・履歴**: claude.ai/code/routines で実行ログ追跡可能
- **CLAUDE.md活用**: チーム規約・操作制限を文書化
- **コスト監視**: `/usage`コマンドで消費量定期確認
