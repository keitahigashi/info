## 概要
Claude Codeのソースコード流出事件（npmソースマップ混入）を受け、第三者がAIエージェントに「Claude CodeをPythonで再実装せよ」と指示し数時間でリライト。著作権法の「表現は保護するがアイデア・機能・アルゴリズムは保護しない」原則により、言語変換で著作権が回避される問題を提起。

## 詳細

### 経緯
1. npmパッケージv2.1.88にソースマップ混入（512,000行のTypeScript流出）
2. realsigridjin氏がGitHubにバックアップ作成
3. AIエージェントに「Pythonで再実装」を指示→数時間で完了
4. Python版として再公開→著作権的にはグレーゾーン

### 著作権法の原則
- 著作権は「表現」を保護するが「アイデア・機能・アルゴリズム」は保護しない
- TypeScript→Pythonへのリライト＝表現が変わる→著作権が適用されにくい
- Gergely Orosz氏（著作権専門家）: 「Derived worksに著作権は適用されない」

### 流出の実態
- 流出はクライアント側のオーケストレーションロジックとインターフェース部
- AIの頭脳（モデル本体）は流出していない
- 未発表機能（KAIROS、autoDream、107個のフィーチャーフラグ）が解析された

### セキュリティ上の教訓
- ソースマップはBunビルド時にデフォルト生成される→明示的に除外が必要
- プロプライエタリコードの防御にはソースコード秘匿だけでは不十分
- AI時代: リバースエンジニアリング+言語変換が容易に→ビジネスモデルの防御壁が低下
