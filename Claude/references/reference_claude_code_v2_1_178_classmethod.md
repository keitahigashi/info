## 主要な新機能

### Tool(param:value) パーミッション構文

パーミッションルールに新構文が追加され、ツール名だけでなくパラメータ単位の制御が可能に。

```yaml
# 例：特定モデルを使うサブエージェントのみ許可
allow:
  - "Agent(model:claude-opus-4-8)"
```

これにより「特定モデルを使うサブエージェント起動のみ許可」「特定ファイルパスへのWriteのみ許可」といった細粒度の制御が実現。

### ネストした .claude/skills の改善

ネストした `.claude/skills` 配下のスキルが、**その配下のファイル操作時に自動的に読み込まれる**ようになった。

- プロジェクトルートの `CLAUDE.md` に全スキルを列挙する必要がなくなる
- サブディレクトリごとに独自スキルを持つモノレポ構成が自然に機能する

## セキュリティ強化

### autoモードのサブエージェント事前評価

autoモードでサブエージェント起動が**起動前にパーミッション評価**されるようになった。

- 従来：サブエージェントがブロック対象アクションを要求してから拒否
- 改善後：ブロック対象アクションを要求するサブエージェント自体の起動を事前に防止
- 「レビューを経ずにブロック対象のアクションを要求できる抜け穴」が解消
