## 概要
21個のSkillを実運用して得た設計・育成の知見。フロントマター全フィールド解説、/skill-creatorの4モード、よくある失敗パターンと対策を網羅。

## 詳細

### フロントマター必須フィールド
- **name**: 小文字・ハイフン・数字のみ、最大64文字
- **description**: 250文字で切り詰められるため重要キーワードは前半に配置。「何をするか」+「いつ使うか」の両方を記載

### フロントマター全フィールド
| フィールド | 説明 |
|---------|------|
| argument-hint | オートコンプリート時のヒント |
| disable-model-invocation | 副作用ある処理はtrueで手動呼び出しのみ |
| user-invocable | falseで/メニューから非表示 |
| allowed-tools | 事前許可するツール指定 |
| context | forkでサブエージェント実行 |
| effort | 思考レベル上書き(low/medium/high/max) |

### $ARGUMENTSによる引数渡し
`/fix-issue 123`で`$ARGUMENTS`が123に置換される仕組み。

### 動的コンテキスト注入
`` !`<command>` ``構文でシェルコマンド結果をSkillに注入。プロンプト読み込み前に実行。

### Skill開発5ステップ
1. **課題特定**: 繰り返す作業を明確化
2. **プロンプト構造化**: ターゲット、トーン、構成、要素、禁止事項
3. **/skill-creatorで作成**: 対話形式で自動生成
4. **Eval/Improveでテスト・改善**: A/Bテストと説明最適化
5. **Benchmark継続運用**: Pass rate、実行時間、トークン数を監視

### /skill-creatorの4モード
- **Create**: 対話形式でヒアリング→SKILL.md生成
- **Eval**: テストケース作成→A/Bテスト実施
- **Improve**: 失敗パターン分析→description改善提案
- **Benchmark**: Pass rate・実行時間・トークン数計測

### よくある失敗パターン
| 失敗 | 原因 | 対策 |
|-----|------|------|
| 詰め込みすぎ | 複数役割を1つのSkillに | 1 Skill = 1目的に分割 |
| 説明が曖昧 | "便利なスキル"としか記載 | "何を"+"いつ"を具体的に |
| テスト未実施 | 作って満足 | Evalモードで必ず検証 |
| メンテなし | モデル更新後に放置 | 定期的にBenchmark実行 |

### 著者の運用内訳（計21個）
- 執筆系6個: 週2〜3回（記事作成、X投稿生成）
- 開発系5個: 週1〜2回（テスター、設定検証）
- 運用系5個: 月1〜2回（検証、リプライ生成）
- 公式プラグイン5個: /pptx, /xlsx, /pdf等
