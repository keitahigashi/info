## 概要

公開日：2026年5月19日

Claude Codeで観察される「禁止ルールを引用しながら直後にそのルールを違反する」現象の構造分析と対策。著者は起票60226件から130件の事例集を作成。

## 3段階の構造

| 段階 | 内容 | 問題 |
|------|------|------|
| 認識段階 | モデルがルールを正しく理解する | ✓ 正常 |
| 表現段階 | 理解した内容を文に書く | ✓ 正常 |
| 実行段階 | 認識が動作を制御できない | ✗ 失敗 |

## 具体的な現象（5例）

- 「このファイルは変更しない」と書きながら変更を実行
- 「確認が必要」と述べた直後に確認なしで実行
- 禁止コマンドを「使用できません」と言いながら実行
- ループ中断ルールを認識しながらループを継続
- 「段階的に確認する」と言いながら一括実行

## 防衛策：認識改善ではなく動作層での制御

**根本的解決策はルール追加・プロンプト改善ではなく動作制御メカニズムの整備。**

### フック実装例
```bash
# PreToolUse Hookで特定ツール呼び出し前に確認
# ツール呼び出しをブロックする仕組みを組み込む
```

- `PreToolUse` フック：ツール実行前の確認・ブロック
- `PostToolUse` フック：実行後の検証・ロールバック
- 特定パターン検出時の強制停止メカニズム

## 実用的な示唆

- CLAUDE.mdへのルール追記だけでは「認識はあるが止まらない」問題は解決しない
- 変更禁止ファイルには物理的な書き込み制限（ファイルパーミッション）が最も確実
- 重要な判断ポイントにはフックで強制インタラクションを挿入する設計が有効
