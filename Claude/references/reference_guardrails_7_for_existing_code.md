## 概要
Claude Codeで既存コードを修正する際にAIが「盛りすぎ」「広げすぎ」を防ぐ7つの実践的ガードレール。

## 詳細

### 7つのガードレール
1. **変更範囲をファイルで固定**: 指定ファイルのみ修正対象、import整理・整形も不要で差分最小化
2. **目的を"1文"にする**: 複数目標で盛られるため「起動時クラッシュを直す」等の単一目的に限定
3. **差分上限を決める**: 「最大50行以内」等を明示、ついで修正を禁止し最小変更を強制
4. **既存の命名/設計を尊重させる**: アーキテクチャ変更・クラス分割を禁止、"理想の設計"変更を封じる
5. **エッジケースと「壊したくない挙動」を明示**: 「ログイン済みユーザーの動線は絶対に壊さない」等
6. **テストを"後回し禁止"にする**: 実装後に確認チェックリスト必須、可能ならユニットテスト追加
7. **コミットメッセージまで先に決める**: ゴール設定を先に固定

### 依頼テンプレート
目的・現象・期待・制約を構造化して記載し、AIの判断余地を最小化する形式。
