## Anthropic公式発表の要点

### 品質低下の期間と影響範囲

- 影響期間：2026-03-04〜2026-04-20
- 影響製品：Claude Code・Claude Agent SDK・Claude Cowork
- **API経由利用（Bedrock・Vertex AI）には影響なし**

### 特定された3つの根本原因

| 原因 | 実施日 | 内容 |
|------|--------|------|
| 推論負荷の引き下げ | 2026-03-04 | `high` → `medium` に変更。コスト最適化が目的だったが品質劣化を招いた |
| キャッシュ管理バグ | 2026-03-26 | セッション終了時の思考履歴削除機能のバグにより、思考履歴が誤って消去され重複・忘却が多発 |
| システムプロンプト制約 | 2026-04-16 | 出力制限の追加により推論品質が約3%低下 |

### Anthropicの対応・修正

| 対応 | 内容 |
|------|------|
| 推論負荷設定 | **`xhigh`** に引き上げ（強化された推論能力を有効化） |
| バグ修正 | キャッシュ管理バグを修正（2026-04-10） |
| 利用制限リセット | **全サブスクライバーの使用制限枠をリセット** |

### 重要な示唆

「AIを速く動かすための調整が、深い思考を損なっていた」  
→ 推論時間（Thinking/xhigh）の重要性が実証された事例。

Anthropicは今後のユーザーフィードバックを重視し、継続的な品質モニタリングに取り組む方針を示した。

## APIユーザーへの影響

Anthropic APIを直接利用するケース（Bedrock・Vertex AI含む）は影響を受けておらず、今回の問題はClaude Codeなどの製品層のみに限定されていた。
