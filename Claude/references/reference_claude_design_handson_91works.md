## 出典

Zenn（91works Tech Blog / YoshikiAgatsuma）: https://zenn.dev/91works/articles/bb03f5adb45dea

## 記事概要

**公開日:** 2026年6月19日

Claude Designがデスクトップアプリ（claudeアプリ）に対応したことを受け、実際に操作した感想と所見をまとめたハンズオンレポート。

## 検証した2機能

### 1. Make something new（新規デザイン作成）

3つのユースケースで検証：

| ケース | 内容 | 評価 |
|--------|------|------|
| 提案資料のスライド化 | テキストから自動スライド生成 | 業務利用レベルの品質 |
| 既存システムのダッシュボードリデザイン | yesesのUIをClaude Designで再設計 | 元の設計意図を解釈した改善案が出力 |
| ウェブサイト全面リデザイン | bugmania.jpを全面リデザイン | **「AI的な見た目が消えた本格的なデザイン」と高評価** |

### 2. Design systems（デザインシステム自動生成）

2つのアプローチを検証：

| アプローチ | 説明 |
|-----------|------|
| **Create here** | Claude Design内で直接デザインシステムを構築 |
| **Create using Claude Code** | コードベースをClaudeに分析させ、そこからデザインシステムを生成 |

- コードベースから自動的にデザインドキュメントを生成する機能
- Storybook等との連携でデザインシステムの維持管理に活用可能

## 主な所見

- **デザイン品質**: 予想を上回る。特にWebリデザインでは「AI臭さ」が感じられないレベル
- **トークン消費**: 1セッション2〜6%程度（軽微）
- **位置付け**: Figmaの代替ではなく「Claudeとデザインツールの間を埋める存在」

## 推奨される活用シーン

1. **ラフデザインの高速生成** - 提案前のたたき台作成に最適
2. **コードベースからのデザインシステム抽出** - 既存プロジェクトのデザインドキュメント整備
3. **非デザイナーのUI改善提案** - エンジニアが独力でデザイン変更案を作成できる

## 注意点

- デスクトップアプリ対応はベータ段階（2026年6月時点）
- Claude Code連携（`/design-sync`）はClaudeアプリからも利用可能だが、細かい設定はClaude Code側で行う必要あり
