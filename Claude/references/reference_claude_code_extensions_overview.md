## 概要
Claude Codeの拡張機能を4層構造で体系的に整理し、各機能の関係性と実装ポイントを解説。

## 詳細

### 4層構造

| 層 | 機能 | 特徴 |
|----|------|------|
| 1 | カスタムコマンド | `.claude/commands`配下のMarkdown。短いプロンプトの定型化。v2.1.3でスキルに統合 |
| 2 | スキル | SKILL.md形式。段階的開示で必要時にロード。`disable-model-invocation`で自動実行制御 |
| 3 | サブエージェント | 独立コンテキスト。大量探索・読解を分離し要約だけ返す。Explore・plan等の組み込み型あり |
| 4 | プラグイン | 上記をパッケージ化。マーケットプレイス配布・バージョニング対応 |

### 実装ポイント
- **引数参照**: `$ARGUMENTS[0]` または `$0` を推奨
- **副作用防止**: デプロイ・メッセージ送信等は `disable-model-invocation: true` で自動発火防止
- **Agent Skills標準**: 準拠することでCodex CLIとの移植性向上

### クロスプラットフォーム互換性
- Claude Desktop: プラグインがリモートセッションで利用不可の制約あり
- Codex CLI: Agent Skills標準で高い互換性。カスタムコマンド・プラグイン形式は異なる
