## インシデント概要

2026年4月、Anthropic の Claude Code で推論基盤のリグレッションが発生し、コーディング応答精度が広範囲で低下。「同じプロンプトで全く違う結果が出る」「テストが通らないコードが量産される」といった報告がXで相次いだ。

## 根本原因（3つ）

| # | 原因 | 内容 |
|---|------|------|
| 1 | 推論最適化のバグ | 特定条件下でトークン確率分布がずれるエラーが発生 |
| 2 | 回帰テストの不足 | 評価セットが実際の劣化パターンをカバーしきれず |
| 3 | 段階展開の判断ミス | オフライン評価への過信により全量切替を急いだ |

## 対策

### 短期
- 推論経路のバグ修正
- 回帰テスト拡張

### 中長期
- 本番トラフィックでのcanary展開を必須化
- 長文・複数ターンを含むベンチマーク拡充
- オフライン評価への過度な依存を是正する組織的ガバナンス整備

## 教訓

- 「速く動かすための調整が深い思考を損なった」→ Thinking/xhigh の重要性再確認
- ベンダー透明性文化: 「黙って直す」→「説明して直す」へ
- AIツール品質変動への備え: 自社ベンチマーク構築・バージョン管理が不可欠
