## 概要
「Maxプランの5時間枠が15分で消える」現象について、独立研究者ArkNill氏のプロキシ実測分析に基づき7つのバグを体系的に整理した記事。

## 詳細

### 修正済み3件
- **B1 Sentinel（キャッシュプレフィックス破壊）**: API課金用sentinel文字列置換でキャッシュ読み率が4〜17%に低下（通常80%以上）。v2.1.86〜v2.1.88で修正
- **B2 Resume（セッション再開のキャッシュ破壊）**: `--resume`時のツール結果挿入位置不一致。100ターン超セッションで最大20倍消費。v2.1.89以降で修正
- **B3 Cache invalidation（キャッシュ無効化）**: 会話履歴がターンごとに変更されキャッシュ丸ごと無効化。220リアクション集約の最大影響バグ。4月4日CLOSED

### 未修正4件（v2.1.92時点）
- **B4 Microcompact（サイレントなツール結果消去）**: 1セッション中158回以上のclearing event。30ターン超過で新規セッション推奨
- **B5 Budget cap（200K集約上限）**: 261回のbudget event。古いツール結果が1〜41文字に切り詰め
- **B6 False rate limit（偽レート制限）**: 65セッション中151個のsyntheticエントリ（API未呼出しなのにカウント）。並行サブエージェントが原因推定
- **B7 Log inflation（ログのトークン水増し）**: Extended Thinking重複記録で最大2.87倍膨張。`/cost`表示が約2倍に水増し（サーバー課金には無影響）

### 推奨対処法
1. 最新版へのアップデート（B1〜B3解消）
2. `--resume`フラグ回避
3. セッション30ターン上限でリセット
4. ツール呼び出しデータ量抑制（offset/limit活用）
5. `/cost`表示を疑う（表示値÷2が実測値）
6. JSONLログで自己診断
7. プロキシ監視でキャッシュ比率検証
