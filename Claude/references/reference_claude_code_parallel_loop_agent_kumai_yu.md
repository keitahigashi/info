## 並列ループエージェント実践ガイド

### 概要

公開日：2026年6月16日。著者が実際に構築した並列ループエージェントにより、**約10人月の受託開発が2週間で完了**するようになった事例とその設計原則を公開。

### 4つの核心教訓

| 教訓 | 内容 |
|------|------|
| **無限ループで止めない** | 必ず停止条件と最大反復回数を設定 |
| **テスト駆動で完了を定義** | 「何ができたら完了か」をテストで先に定義 |
| **並列化** | 独立タスクは並列実行でスループット向上 |
| **出力最小化** | コンテキストウィンドウを守るため出力を絞る |

### 実装フロー（5フェーズ）

```
[調査フェーズ（並列）]
  └→ [仕様書作成]
      └→ [人間レビュー（停止①）]
          └→ [実装フェーズ（TDD・並列）]
              └→ [統合ゲート]
                  └→ [検証フェーズ（並列）]
                      └→ [構造化レビュー（停止②）]
```

**重要な停止ポイントは2箇所のみ**：
1. 仕様レビュー（人間が要件を確認）
2. 最終確認（成果物の品質チェック）

それ以外は自律的に進行。

### 並列化の設計指針

- **調査フェーズ**：複数情報源を独立タスクとして並列収集
- **実装フェーズ**：モジュール単位でTDDと並列実装を組み合わせ
- **検証フェーズ**：テストスイートを並列実行で高速化

### コンテキスト管理の要点

- ループ1周ごとに結果をファイルに書き出し、コンテキストから削除
- 「前の実行結果のサマリー」だけをコンテキストに保持
- 並列エージェント同士は直接通信せず、共有ファイルを介して協調

### 経済効果

- 10人月（約800時間相当）→ 2週間（約80時間）に圧縮
- コスト削減率：推定90%
- クオリティは同等以上（テスト通過率で担保）
