Mozilla Hacks: https://hacks.mozilla.org/2026/05/behind-the-scenes-hardening-firefox/

## 概要

2026年5月7日公開。MozillaのFirefoxセキュリティチームがClaude Mythos Previewを活用したエージェント型AIハーネスを構築し、Firefox 150リリースで271個のバグを特定した舞台裏を解説した公式ブログ。

## 背景

- 2026年4月のFirefoxセキュリティバグ修正数：**423個**（月平均21個の約20倍）
- そのうち271個（約64%）をClaude Mythos Previewが特定
- Claude Opus 4.6で約6,000件のC++ファイルをスキャンし22件を確認（Firefox 148で修正）

## 発見された主な脆弱性

| カテゴリ | 具体例 |
|---------|------|
| JITバグ | WebAssembly GC構造体の初期化最適化バグ |
| レガシー欠陥 | 15年前の`<legend>`要素のバグ |
| メモリ安全性 | 20年前のXSLTエンジンのUAF（解放後使用）バグ |
| サンドボックス回避 | IPCレース条件を悪用するサンドボックスエスケープ |
| 型安全性 | NaN型ポインタ逆シリアル化 |

## 脆弱性の深刻度内訳（271件）

| 評価 | 件数 |
|------|------|
| sec-high（高） | 180件 |
| sec-moderate（中） | 80件 |
| sec-low（低） | 11件 |

## エージェント型ハーネスの進化

**初期アプローチ（静的解析）:**
- シンプルなプロンプト：「このコードにバグがあります。見つけてテストケースを作成してください」
- 問題：推測と実際のバグを区別できない

**改善後（動的検証）:**
- AI生成の仮説を「実行可能なテストケース作成による動的検証」で評価
- 再現可能なPoC（概念実証）テストケースを自動生成
- 誤検知を大幅削減 → 大規模展開が現実的に

## 実践的意義

- AIによるセキュリティ脆弱性発見が**工業スケール**で実現可能と実証
- 静的解析ではなく動的エージェント型ハーネスが突破口
- 「ハーネスエンジニアリング」の具体的なセキュリティ適用事例
