## Claude Code 6月版アップデートまとめ（v2.1.153〜v2.1.178）

### 概要

公開日：2026年6月17日。本番運用を支える足回りの強化に重心が移った時期として位置づけ。「新機能よりも、信頼性と安全性の底上げ」がテーマ。

### 5つの主要新機能

#### 1. /cd コマンド（v2.1.169）

```
/cd /path/to/dir
```

- セッション内でディレクトリを移動しつつ**プロンプトキャッシュを壊さずに**移動できる
- モノレポで複数パッケージ間を行き来する際に会話履歴を保持
- 従来：ディレクトリ変更のたびに新規セッション（会話履歴リセット）が必要

#### 2. 入れ子サブエージェント（v2.1.172）

- サブエージェントが子を起動でき、**最大5段階まで**可能
- 設計思想は「並列化」ではなく「文脈分離」
- 例：オーケストレーター → 収集 → 執筆 → ファクトチェック → 自動修正 → 再検証

#### 3. --safe-mode フラグ（v2.1.169）

```bash
claude --safe-mode
```

- CLAUDE.md・プラグイン・スキル・MCPを全て無効化して起動
- トラブル時の原因切り分けに使用

#### 4. fallbackModel 設定（v2.1.166）

```json
{
  "fallbackModel": ["claude-sonnet-4-6", "claude-haiku-4-5"]
}
```

- 最大3つのモデルを連鎖的に試すフェイルオーバー機能
- 過負荷時のみ自動切り替え（通常時は指定モデルを使用）

#### 5. Tool(param:value) 権限構文（v2.1.178）

```
Allow: Bash(command:git*)
Deny: Bash(command:rm -rf*)
```

- ツール名だけでなく、**パラメータ値で許可・拒否を判定**
- 細粒度の権限管理が可能に

### 業界動向

Claude Codeが個人開発から本番・組織運用を支える段階へ移行。安全性・フェイルオーバー・権限管理の強化がその証左。
