## 記事の概要

2026年4月21日公開。プロンプトエンジニアリングの次のパラダイムとして「ハーネスエンジニアリング」を解説。AIエージェントが自律的に動作する環境全体を設計するアプローチを、4設計要素と3ステップ実装で体系化。

## ハーネスエンジニアリングとは

- 「馬具（harness）」の概念をAI開発に転用
- 「優秀なAIモデルも適切な環境設計がなければ成果を生み出せない」
- Mitchell Hashimoto提唱（2026年2月）、OpenAI・Anthropicが公式記事を公開
- SWE-benchでハーネス設計の差によるスコア変動が最大22ポイント（モデル差は1ポイント）

## 4つの設計要素

| 要素 | 内容 | 実装方法 |
|---|---|---|
| ①コンテキスト設計 | AIに見せる情報の選定 | CLAUDE.md・Rules |
| ②行動設計 | 許可範囲と制約の定義 | settings.json・Hooks |
| ③フィードバック設計 | 評価と修正ループ | PostToolUse Hook・Stop Gate |
| ④運用設計 | 継続的な運用・改善 | Skills・定期メンテ |

## Claude Code向け3ステップ実装

```
1. .claude/ ディレクトリを作成
2. CLAUDE.md でルール・コンテキスト・制約を定義
3. skills/ にワークフロー手順書（SKILL.md）を保存
```

## プロンプトエンジニアリングとの違い

| 観点 | プロンプトエンジニアリング | ハーネスエンジニアリング |
|---|---|---|
| 対象 | 単一プロンプトの質 | エージェント動作環境全体 |
| 範囲 | 入力の最適化 | 実行・検証・フィードバックループ |
| スケール | 1回の応答 | 長時間・複数ステップの自律実行 |

## Meta-Harness（最先端事例）

- ハーネス自体を自動生成・自己改善させる概念
- ルール自動更新・スキル自動生成への応用
- AutoHarness Skillパターンの実装例がZennで公開済み

## Claude Code / Cursor共通の実践ポイント

- 「何をするか」より「どんな環境でするか」を設計する
- 最小構成から始めてフィードバックループで段階的に育てる
- deny設定で十分な段階ではHooksを導入しない
