## 概要

連載「プロと実践！ ゼロから始めるバイブコーディング」第7回（2026年5月15日公開）。執筆者: 柳谷智宣氏。ウイスキー認識アプリ「AiLA」の開発過程で、承認制ログイン機能を実装する際の高度なAI活用テクニックを扱う。非エンジニアが長期プロジェクトをAIと協業する上での持続可能な運用方法を実例で示す。

## 実践フロー（段階的開発の具体例）

### 1. 前提情報の永続化（CLAUDE.md）

Firebaseエミュレーターの起動手順を CLAUDE.md に記載することで、新しいチャットでも毎回同じ説明を繰り返す手間を排除。CLAUDE.md を「プロジェクトの長期記憶」として運用する実例を示す。

### 2. 複雑な要件の事前整理（プロジェクト機能）

承認制ログイン（ユーザー申請→管理者承認→ログイン許可）のような複雑なワークフローは、チャット版Claudeで要件を整理してから Claude Code に渡す二段階アプローチを採用。プロジェクト機能で前提情報を共有することで「毎回前提を説明する必要がなく、精度が安定する」。

### 3. Plan Mode による設計確認

```
実装手順の例:
1. Plan Mode を起動 → 修正方針の提示を要求
2. 提示された計画を確認・修正承認
3. 承認後に実装実行
```

認証周辺のような慎重な変更では「意図しない実装が進むリスクを大きく減らせる」として Plan Mode の使用を強く推奨。Plan Mode を使わずにいきなり複雑な指示を投げることを「アンチパターン」として明示。

### 4. ドキュメント・Git管理の最適化

- 重要な仕様・判断理由は CLAUDE.md に追記する
- 開発の区切りごとに Git コミットを自動促すルールを CLAUDE.md に追記
- ユーザーがコミットメッセージを確認してからプッシュする体制で「セーブポイント」を確保

## 実装成果

承認制ログインの完成ワークフロー:

- 新規ユーザー: 登録時に承認待ちステータス
- 管理者: 承認待ちユーザー一覧から許可・拒否操作
- 申請: 1ユーザーにつき1回のみ可能

## バイブコーディング手法の要点

要件整理 → プロンプト練成 → Plan Mode での確認 → ドキュメント更新 → Git管理という段階的・反復的プロセスが、開発精度と持続可能性を高める鍵として強調されている。
