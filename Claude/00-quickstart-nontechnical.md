## Claude Code とは

ひとことで言うと、**パソコンの中のファイルを読んだり書いたり、調べものをしたりできるAIアシスタント**です。

普通のChatGPTやClaudeとの違いは、**あなたのパソコン上のファイルを直接扱える**こと。チャットで「この資料を要約して」「表にまとめて」と頼むだけで、実際にファイルを作ったり書き換えたりしてくれます。

```
あなた（日本語で指示）→ Claude Code → ファイルの読み書き・調べもの・整理
```

### ChatGPT / Claude（通常版）との違い

| | ChatGPT / Claude（通常版） | Claude Code |
|---|---|---|
| ファイル操作 | チャット画面にコピペが必要 | パソコン内のファイルを直接読み書き |
| 作業の継続 | 会話が途切れるとリセット | 前回の続きから再開できる |
| まとめて作業 | 1つずつ手動でやりとり | 複数ファイルをまとめて処理 |
| プロジェクト理解 | 毎回説明が必要 | フォルダ全体を把握して動く |

## 準備：デスクトップアプリのインストール

### ステップ1：ダウンロード

1. **claude.ai/download** にアクセスする
2. 「Windows版をダウンロード」をクリック
3. ダウンロードされたインストーラーを実行する

### ステップ2：ログイン

1. アプリを起動すると、ログイン画面が表示される
2. Anthropicアカウント（またはGoogle/GitHubアカウント）でログインする
   - アカウントがない場合は新規作成する

### ステップ3：作業フォルダを開く

1. アプリが起動したら、**作業したいフォルダを開く**
   - 例：企画書が入っているフォルダ、プロジェクトの共有フォルダなど
2. フォルダを開くと、Claudeがそのフォルダの中身を把握できるようになる

> **ポイント**: Claude Codeは「開いたフォルダの中」が作業範囲です。関係するファイルは同じフォルダにまとめておくとスムーズです。

## 基本の使い方 — 5つだけ覚える

### 1. 日本語で話しかけるだけ

チャット欄に日本語で指示を入力するだけです。特別な書き方は不要です。

```
> このフォルダにある議事録をすべて読んで、決定事項だけ一覧にして
> 先週の週報をもとに、今週のタスクリストを作って
> この企画書の要点を3行にまとめて
```

### 2. ファイルを指定して渡す（`@`マーク）

`@` のあとにファイル名を書くと、「このファイルを見て」と伝えられます。

```
> @企画書_v2.docx この企画書のターゲット層を整理して表にして
> @売上データ.csv 先月と今月を比較して、変化が大きい項目を教えて
> @会議メモ/ このフォルダの議事録をまとめて
```

> **ポイント**: `@` を入力すると候補が表示されるので、選ぶだけでOKです。

### 3. 複雑な作業は「計画して」と頼む

やりたいことが大きいときは、いきなり作業させず**まず計画を出してもらう**のがコツです。

```
> 来月の新商品ローンチに向けて、マーケティング施策の整理をしたい。まず計画して
```

Claudeが「こういう手順で進めます」と計画を出してくれます。内容に納得したら「それで進めて」と伝えます。

### 4. 前回の続きから再開する

作業の途中でアプリを閉じても大丈夫です。次に開いたとき、**前回の会話の続きから再開できます**。

> **ポイント**: 長い作業を何日かに分けて進めることもできます。

### 5. 分からないことは質問させる

あいまいな指示でも、Claudeに質問してもらえば具体的になります。

```
> 新しい企画書を作りたい。足りない情報があれば質問して
```

Claudeが「ターゲットは？」「予算規模は？」「納期は？」のように聞いてくれるので、答えていくだけで企画書の骨子ができあがります。

## 覚えておくと便利な操作

| やりたいこと | 操作 | どんなとき |
|------------|------|----------|
| 会話をすっきりさせる | チャット欄に `/compact` と入力 | やりとりが長くなって回答がぼんやりしてきたとき |
| 安全モードの切り替え | `Shift+Tab` キー | ファイル変更の許可方法を変えたいとき |
| 直前の操作を取り消す | `Esc` キーを2回押す | Claudeが見当違いの作業を始めたとき |
| 変更の確認 | チャット欄に `/diff` と入力 | Claudeがどこを変えたか見たいとき |
| 使用量の確認 | チャット欄に `/cost` と入力 | どれくらい使ったか知りたいとき |

## 安全のしくみ — 権限モード

Claude Codeはファイルを書き換えられるため、**勝手に変更されないようにする安全装置**があります。

| モード | 動き | おすすめ |
|--------|------|---------|
| **default** | 何をするにも毎回「OK？」と聞いてくる | **最初はこれ**。Claudeが何をしているか観察できる |
| **acceptEdits** | ファイルの変更は自動でOK。それ以外は確認される | 慣れてきたら。作業がサクサク進む |
| **plan** | 読むだけ。ファイルは変更しない | 「まず見てもらうだけ」のとき |

`Shift+Tab` で切り替えられます。

> **最初のおすすめ**: `default` のまま使い始めてください。Claudeが「ファイルを変更します。よろしいですか？」と聞いてくるので、内容を確認して「OK」を押す流れに慣れましょう。

## CLAUDE.md — 毎回伝えることの「メモ帳」

毎回同じことを説明するのが面倒になったら、**CLAUDE.md** というファイルを作りましょう。作業フォルダに置いておくと、Claudeが毎回自動で読んでくれます。

### 書き方の例

```markdown
# プロジェクト概要
新商品「〇〇」のマーケティングプロジェクト

# いつも使うルール
- 資料は「です・ます調」で統一
- 表やグラフを使って視覚的にまとめる
- ファイル名は「日付_タイトル」の形式にする（例: 0521_企画書.md）
- 社外秘情報は扱わない

# チームメンバー
- 田中: プロジェクトリーダー
- 鈴木: デザイン担当
- 佐藤: 営業担当
```

> **ポイント**: 最初は3〜5行で十分です。「あ、また同じこと言ってるな」と思ったタイミングで書き足していけばOKです。

## 上手な指示の出し方

### コツ1：「何をしてほしいか」を最初に書く

```
○ 「競合3社の価格比較表を作って。対象はA社・B社・C社。フォーマットはMarkdownの表」
× 「最近競合が気になっていて、価格とか調べたいんですけど、どうしたらいいですかね...」
```

**最初の1文で「やってほしいこと」**を伝え、次に詳細や条件を添えます。

### コツ2：1回の指示は1〜2個まで

たくさん頼みたいときは、先に全体の計画を作ってから1つずつ進めましょう。

```
> 以下の3つをやりたい。まず計画を立てて
> 1. 先月の売上データの集計
> 2. 前年同月との比較
> 3. レポート作成
```

### コツ3：参考になるファイルを渡す

```
> @前回のレポート.md これと同じ形式で、今月のレポートを作って
> @テンプレート.md このテンプレートに沿ってまとめて
```

「こんな感じで」という見本があると、仕上がりの精度がぐっと上がります。

### コツ4：制約条件を伝える

```
> ただし以下の条件で：
> - A4で2ページ以内
> - 図表を最低2つ入れる
> - 専門用語は使わず、役員にも伝わる表現で
```

## よくある困りごとと対処法

| 困りごと | 原因 | 対処法 |
|---------|------|--------|
| 回答がざっくりしてきた | 会話が長くなりすぎた | `/compact` と入力して会話を整理する |
| ファイルを変更してくれない | 安全モードで止まっている | 確認メッセージが出ていたら「OK」を押す |
| 見当違いな作業を始めた | 指示があいまいだった | `Esc` を2回押して取り消し、指示を具体的に書き直す |
| 毎回同じ説明をしている | CLAUDE.mdがない | CLAUDE.md を作って繰り返す情報を書いておく |
| 「そのファイルが見つかりません」と言われる | フォルダが違う | 作業フォルダを正しく開き直す |
| 思った形式と違うファイルができた | 出力形式を指定していなかった | 「Markdown形式で」「CSV形式で」など形式を指定する |

## 1週間の慣れ方ロードマップ

| 日 | やること | ゴール |
|----|---------|--------|
| 1日目 | アプリを入れて、簡単な質問をしてみる | 「話しかけると答えてくれる」を体感する |
| 2日目 | `@` でファイルを渡して、要約や整理を頼む | ファイルを扱えることを体感する |
| 3日目 | 「計画して」と頼んでから作業させる | 計画→実行の流れに慣れる |
| 4日目 | CLAUDE.md を作ってみる | 毎回の説明が不要になることを体験する |
| 5日目 | `/compact` や `Esc×2` を試す | 会話の管理方法を覚える |
| 6日目 | 実際の業務タスクを1つ任せてみる | 仕事に使える手応えを得る |
| 7日目 | 1週間のふりかえり。CLAUDE.md に追記する | 自分なりの使い方が定まる |

## 大事な心構え：「なぜやるか」は自分で持つ

Claude Codeは「作業を素早くこなす」のが得意です。でも、**「なぜこの企画をやるのか」「そもそもやるべきか」を考えるのは自分自身の役割**です。

1. **指示する前に**「これは何のためにやるのか」を一行書いてみる
2. **仕上がりを見て**「これで目的を達成できるか」を自分で判断する
3. **全部任せない** — 判断と最終チェックは必ず自分でやる

AIは優秀なアシスタントですが、プロジェクトの「Why（なぜやるか）」を握っているのはあなたです。
